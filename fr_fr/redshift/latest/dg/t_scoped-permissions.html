Autorisations étendues

Utilisez des autorisations étendues pour adapter les autorisations accordées à un utilisateur ou à un rôle à l’étendue d’une base de données ou d’un schéma.

Les autorisations étendues s’appliquent aux objets de l’étendue sélectionnée lorsque vous accordez ou révoquez l’autorisation, ainsi qu’aux nouveaux objets créés après avoir accordé ou révoqué l’autorisation. Par exemple, l’octroi d’une autorisation étendue à un rôle pour effectuer une sélection parmi les tables d’une base de données permet aux utilisateurs disposant de ce rôle de sélectionner les tables créées dans le futur, ainsi que les tables qui existent actuellement dans la base de données.

Vous pouvez utiliser des autorisations étendues pour définir les autorisations par défaut sur une base de données partagée créée à partir d’une unité de partage des données. Un utilisateur d’unité de partage des données côté consommateur disposant d’autorisations étendues sur une base de données partagée obtiendra automatiquement ces autorisations pour tout nouvel objet ajouté à l’unité de partage des données côté producteur.

Pour plus d’informations sur l’application d’autorisations étendues, consultez GRANT et REVOKE.

Considérations relatives à l’utilisation d’autorisations étendues

Tenez compte des éléments suivants lorsque vous utilisez des autorisations étendues :

  • Vous pouvez utiliser des autorisations étendues pour accorder des autorisations sur une étendue d’une base de données ou d’un schéma à un utilisateur ou à un rôle choisi, ou les révoquer. Vous ne pouvez pas accorder d’autorisations étendues à des groupes d’utilisateurs, vous ne pouvez donc pas les révoquer.

  • L’octroi ou la révocation d’autorisations étendues modifie les autorisations pour tous les objets actuels et futurs du champ d’application.

  • Les autorisations étendues s’appliquent indépendamment des autorisations de niveau objet. Elles ne s’excluent pas mutuellement. Par exemple, considérez un utilisateur qui dispose d’une autorisation de niveau objet sur une table, ainsi que d’une autorisation étendue pour le schéma dans lequel se trouve la table. Dans les deux scénarios suivants, l’utilisateur conserve l’accès à la table.

    • L’autorisation étendue de l’utilisateur pour le schéma dans lequel se trouve la table est révoquée, mais il a toujours accès à la table via l’autorisation de niveau objet pour la table.

    • L’autorisation de niveau objet de l’utilisateur pour la table est révoquée, mais il a toujours accès à la table via l’autorisation étendue pour le schéma.

  • Pour accorder ou révoquer des autorisations étendues, vous devez répondre à l’un des critères suivants :

    • Super-utilisateurs

    • Utilisateurs disposant de l’option d’octroi de cette autorisation Pour plus d’informations sur les options d’octroi, consultez le paramètre WITH GRANT OPTION dans GRANT.

  • Les autorisations étendues ne peuvent être accordées ou révoquées que pour des objets de la base de données connectée, ou à partir de bases de données importées depuis une unité de partage des données.

  • Vous pouvez accorder des autorisations étendues qui s’appliquent aux bases de données créées à partir d’unités de partage des données.
