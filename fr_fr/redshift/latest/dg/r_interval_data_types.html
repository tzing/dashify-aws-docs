Types de données d'intervalle

Vous pouvez utiliser un type de données d'intervalle pour spécifier des périodes ou des plages de temps12 hours, telles que6 weeks, ou1 month. Les types de données INTERVAL vous permettent de spécifier des périodes ou des plages de temps12 hours, telles que6 weeks, ou1 month. Vous pouvez utiliser des types de données d'intervalle dans les calculs, tels que le calcul des différences entre des dates/heures ou des intervalles, des sommes d'intervalles ou des sommes d'une date/heure et d'un intervalle.

Les intervalles suivants sont pris en charge :

  • ANNÉE PAR MOIS

  • JOUR PAR SECONDE

Syntaxe

Pour créer un intervalle de type YEAR TO MONTH :

INTERVAL <year_to_month_qualifier>

Pour créer un littéral d'intervalle de type YEAR TO MONTH :

INTERVAL <interval-string> <qualifier>

Utilisez la syntaxe suivante pour créer un intervalle de type DAY TO SECOND. Le fractional_precision paramètre n'est nécessaire que si votre intervalle contient SECOND.

INTERVAL <qualifier> [ <fractional_precision> ]

Utilisez la syntaxe suivante pour créer un littéral d'intervalle de type DAY TO SECOND.

INTERVAL <interval-string> <qualifier> [ <fractional_precision> ]

Arguments

Chaîne d'intervalles

Spécifie la valeur de l'intervalle. Obligatoire uniquement en cas de création d'une constante d'intervalle

Qualificateur

Spécifie la plage de l'intervalle. Si vous utilisez un qualificatif et créez un intervalle dont les unités de temps sont inférieures au qualificatif, Amazon Redshift tronque et supprime les plus petites parties de l'intervalle. L'exemple suivant illustre l'utilisation d'un qualificatif MINUTE, et Amazon Redshift supprime les unités de temps inférieures à MINUTE.

INTERVAL '1 day 1 hour 1 minute 1.123 seconds' MINUTE

est tronqué en 

'1 day 01:01:00'
Précision fractionnée

Paramètre facultatif qui spécifie le nombre de chiffres fractionnaires autorisés dans l'intervalle. Par exemple, SECOND (3) crée un intervalle qui n'autorise que trois chiffres fractionnaires, tels que 1,234 seconde. Applicable uniquement aux intervalles de jour à seconde contenant des secondes. Le nombre maximum de chiffres pris en charge est de six.

Le type de données YEAR TO MONTH prend en charge les qualificatifs suivants :

  • YEAR

  • MONTH

  • ANNÉE PAR MOIS

Le type de données DAY TO SECOND prend en charge les qualificatifs suivants :

  • DAY

  • HOUR

  • MINUTE

  • SECOND

  • JOUR PAR HEURE

  • JOUR PAR MINUTE

  • JOUR PAR SECONDE

  • HEURE PAR MINUTE

  • HEURE PAR SECONDE

  • MINUTE PAR SECONDE

Exemples

L'exemple suivant montre comment créer une constante INTERVAL de 1 an.

INTERVAL '1' YEAR

Si vous spécifiez une chaîne d'intervalle qui dépasse le qualificatif, les unités de temps restantes sont tronquées par rapport à l'intervalle. Dans l'exemple suivant, un intervalle de 13 mois devient 1 an et 1 mois, mais le mois restant est omis en raison du qualificatif YEAR.

INTERVAL '13 months' YEAR
'1 year'

Si vous utilisez un qualificatif inférieur à votre chaîne d'intervalle, les unités restantes sont incluses.

INTERVAL '13 months' MONTH
'1 year 1 month'

La spécification d'une précision dans votre intervalle tronque le nombre de chiffres fractionnaires à la précision spécifiée.

INTERVAL '1.234567' SECOND (3)
'00:00:01.234'

Si vous ne spécifiez aucune précision, Amazon Redshift utilise la précision maximale de 6.

INTERVAL '1.23456789' SECOND
'00:00:01.234567'

L'exemple suivant montre comment créer un intervalle échelonné.

INTERVAL '2:2' MINUTE TO SECOND
'00:02:02'

Les qualificatifs dictent les unités que vous spécifiez. Par exemple, même si l'exemple suivant utilise la même chaîne d'intervalle « 2:2 » que l'exemple précédent, Amazon Redshift reconnaît qu'il utilise des unités de temps différentes en raison du qualificatif

INTERVAL '2:2' HOUR TO MINUTE
'02:02:00'

Les abréviations et les pluriels de chaque unité sont également pris en charge ; par exemple : 5s, 5 second et 5 seconds sont des intervalles équivalents. Les unités prises en charge sont les années, les mois, les heures, les minutes et les secondes.

INTERVAL '5s' SECOND
'00:00:05'

INTERVAL '5 second' SECOND
'00:00:05'
                     
INTERVAL '5 HOURS' HOUR
'05:00:00'
                     
INTERVAL '5 h' HOUR
'05:00:00

Les exemples suivants montrent comment utiliser les types de données INTERVAL avec des tables

create table sample_intervals (y2m interval month, m2s interval hour to minute);
insert into sample_intervals values (interval '20' month, interval '2 days 1:1:1.123456' day to second);
select y2m::text, m2s::text from sample_intervals;

      y2m      |       m2s       
---------------+-----------------
 1 year 8 mons | 2 days 01:01:00

update sample_intervals set y2m = interval '2' year where y2m = interval '1-8' year to month;
select * from sample_intervals;

   y2m   |       m2s       
---------+-----------------
 2 years | 2 days 01:01:00

delete from sample_intervals where m2s = interval '2 1:1:0' day to second;
select * from sample_intervals;

 y2m | m2s 
-----+-----

Styles d'intervalle

Vous pouvez utiliser la SET commande pour modifier le format de sortie de vos chaînes d'intervalle. Les options disponibles sont les suivantes : postgressql_standard, etpostgres_verbose. L’argument par défaut est postgres. Des exemples de chaque format de sortie sont présentés ci-dessous.

format de sortie sql_standard

Les valeurs d'intervalle d'une année à l'autre sont formatées comme suit. Si vous spécifiez une valeur négative avant l'intervalle, cela indique que l'intervalle est une valeur négative et qu'il s'applique à l'ensemble de l'intervalle.

'[-]yy-mm'

Les valeurs de l'intervalle entre le jour et la seconde sont formatées comme suit.

'[-]dd hh:mm:ss.ffffff'
INTERVAL '1-2' YEAR TO MONTH outputs as '1-2'
                     
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 2:03:04.5678'

format de sortie postgres

Voici le format de sortie pour postgres. Chaque valeur numérique peut être négative.

[@] '<numeric> <unit> [, <numeric> <unit> ...]'
INTERVAL '1-2' YEAR TO MONTH outputs as '1 year 2 mons'
                  
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 day 02:03:04.5678'

format de sortie postgres_verbose

La syntaxe de postgres_verbose est similaire à celle de postgres, mais les sorties postgres_verbose contiennent également l'unité de temps.

'[@] <numeric> <unit> [, <numeric> <unit> ...] [direction]'
INTERVAL '1-2' YEAR TO MONTH outputs as '@ 1 year 2 mons'

INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '@ 1 day 2 hours 3 mins 4.5678 secs'

La commande suivante montre comment définir le style d'intervalle sursql_standard.

SET intervalstyle to 'sql_standard';

Limites

Cette fonctionnalité INTERVAL de version préliminaire présente les limites suivantes :

  • Si vous utilisez le pilote JDBC, vous ne pouvez pas utiliser les intervalles d'une année à l'autre et de jour à seconde comme paramètres pour les instructions PREPARE.

  • Le pilote JDBC interprète toutes les valeurs d'intervalle sous forme de chaînes. Vous ne pouvez donc obtenir les intervalles renvoyés par une requête SELECT qu'à l'aide de cette méthode. getString

  • Les procédures stockées n'acceptent pas les valeurs d'intervalle comme paramètres de fonction.

  • Vous ne pouvez pas convertir un chiffre en intervalle.
