Formats de modèle AWS CloudFormation

Vous pouvez créer des modèles AWS CloudFormation au format JSON ou YAML. Nous prenons en charge toutes les fonctionnalités et fonctions d'AWS CloudFormation pour les deux formats, y compris dans AWS CloudFormation Designer.

S'agissant du format à privilégier, optez pour celui qui vous est le plus familier. De même, tenez compte du fait que YAML offre par essence des fonctionnalités qui ne sont pas disponibles en JSON (par exemple, les commentaires).

Important

Nous vous conseillons de ne pas ajouter de commentaires YAML # à vos modèles dans Designer. Si votre modèle YAML comporte des commentaires #, Designer ne les conserve pas lors de la conversion du modèle en JSON. En outre, si vous modifiez votre modèle dans Designer (par exemple, si vous déplacez une ressource sur la zone de dessin), vos commentaires sont perdus.

Vous pouvez ajouter des commentaires aux modèles AWS CloudFormation que vous créez, en dehors de Designer. L'exemple suivant illustre un modèle YAML avec des commentaires en ligne.


AWSTemplateFormatVersion: "2010-09-09"
Description: A sample template
Resources:
  MyEC2Instance: #An inline comment
    Type: "AWS::EC2::Instance"
    Properties: 
      ImageId: "ami-0ff8a91507f77f867" #Another comment -- This is a Linux AMI
      InstanceType: t2.micro
      KeyName: testkey
      BlockDeviceMappings:
        -
          DeviceName: /dev/sdm
          Ebs:
            VolumeType: io1
            Iops: 200
            DeleteOnTermination: false
            VolumeSize: 20

Pour plus d'informations sur la syntaxe des modèles pour chaque format, consultez Anatomie du modèle.

AWS CloudFormation prend en charge les spécifications JSON et YAML suivantes :

JSON

AWS CloudFormation se conforme à la norme JSON ECMA-404. Pour plus d'informations sur le format JSON, consultez http://www.json.org.

YAML

AWS CloudFormation prend en charge la spécification YAML version 1.1 à quelques exceptions près. AWS CloudFormation ne prend pas en charge les fonctionnalités suivantes :

  • les balises binary, omap, pairs, set et timestamp ;

  • Alias

  • les fusions de hachage.

Pour plus d'informations sur YAML, consultez https://yaml.org/.
