Livraison continue avec CodePipeline

La livraison continue est une pratique de publication dans laquelle les modifications du code sont automatiquement générées, testées et préparées en vue d'une publication en production. Avec AWS CloudFormation et CodePipeline, vous pouvez utiliser la livraison continue pour créer et tester automatiquement les modifications de vos modèles AWS CloudFormation avant de les implémenter dans les piles de production. Ce processus de publication vous permet d'apporter des modifications de manière rapide et fiable à votre infrastructure AWS.

Par exemple, vous pouvez créer un flux de travail qui crée automatiquement une pile de test lorsque vous soumettez un modèle mis à jour à un référentiel de code. Lorsqu'AWS CloudFormation crée la pile de test, vous pouvez la tester puis décider de transmettre les modifications vers la pile de production. Pour plus d'informations sur les avantages de la livraison continue, consultez Qu'est-ce que la livraison continue ?.

Utilisez CodePipeline pour créer un flux de livraison continue en créant un pipeline pour les piles AWS CloudFormation. CodePipeline a une intégration intégrée avec AWS CloudFormation, afin que vous puissiez spécifier des actions spécifiques à AWS CloudFormation, comme la création, la mise à jour ou la suppression d'une pile dans un pipeline. Pour plus d'informations sur CodePipeline, veuillez consulter le Guide de l'utilisateur AWS CodePipeline.
