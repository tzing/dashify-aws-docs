Transformation AWS::SecretsManager

Utilisez la transformation AWS::SecretsManager, qui est une macro hébergée par AWS CloudFormation, pour spécifier une fonction AWS Lambda afin d'effectuer la rotation des secrets. Avec Création d'un jeu de modifications ou Mise à jour des piles à l'aide de jeux de modifications, et les références de modèles AWS::SecretsManager, AWS CloudFormation génère une fonction AWS Lambda pour effectuer la rotation des secrets. Utilisez le type de propriété HostedRotationLambda de la ressource AWS::SecretsManager::RotationSchedule pour spécifier les attributs de la fonction AWS Lambda souhaitée.

La fonction AWS Lambda se trouve dans une pile imbriquée (une ressource AWS::CloudFormation::Stack) dans le modèle traité. Cette ressource se connecte ensuite au modèle de fonction approprié dans le référentiel AWS Secrets Manager Rotation Lambda Functions en fonction du RotationType spécifié dans la ressource AWS::SecretsManager::RotationSchedule.

Utilisation

Utilisez la transformation AWS::SecretsManager au niveau supérieur du modèle. Vous ne pouvez pas utiliser AWS::SecretsManager comme transformation incorporée dans une autre section de modèle.

La valeur de la déclaration de transformation doit être une chaîne littérale. Vous ne pouvez pas utiliser de paramètre ou de fonction pour spécifier la valeur de transformation.

Syntaxe au niveau supérieur d'un modèle

Pour inclure la transformation AWS::SecretsManager au niveau supérieur d'un modèle, dans la section Transform, utilisez la syntaxe suivante.

JSON

{
  "Transform": "AWS::SecretsManager-2020-07-23",
    . . .
}

YAML

Transform: AWS::SecretsManager-2020-07-23

Paramètres

La transformation AWS::SecretsManager n'accepte aucun paramètre. Au lieu de cela, spécifiez les propriétés de la fonction AWS Lambda de rotation des secrets que vous voulez créer en utilisant le type de propriété HostedRotationLambda pour ressources AWS::SecretsManager::RotationSchedule du modèle de pile.

Remarques

Pour obtenir les considérations générales sur l'utilisation des macros, consultez Considérations relatives à la création de définitions de AWS CloudFormation macros.

Exemple

L'exemple de modèle partiel suivant montre comment utiliser la transformation AWS::SecretsManager afin de spécifier une fonction AWS Lambda pour la rotation des secrets sur une base de données MySQL pour un seul utilisateur, en fonction des propriétés spécifiées dans le type de propriété HostedRotationLambda de la ressource AWS::SecretsManager::RotationSchedule.

Pour obtenir des exemples complets de modèles illustrant des rotations secrètes pour les bases de données RDS, les clusters Amazon Redshift et les clusters DocumentDB, consultez la section Exemples de AWS::SecretsManager::RotationSchedule.

JSON

{
        "AWSTemplateFormatVersion": "2010-09-09",
        "Transform": "AWS::SecretsManager-2020-07-23",
        "Resources": {
            
            . . . 
            
            "MySecretRotationSchedule": {
                "Type": "AWS::SecretsManager::RotationSchedule",
                "DependsOn": "SecretRDSInstanceAttachment",
                "Properties": {
                    "SecretId": {
                        "Ref": "MyRDSInstanceRotationSecret"
                    },
                    "HostedRotationLambda": {
                        "RotationType": "MySQLSingleUser",
                        "RotationLambdaName": "SecretsManagerRotation",
                        "VpcSecurityGroupIds": {
                            "Fn::GetAtt": [
                                "TestVPC",
                                "DefaultSecurityGroup"
                            ]
                        },
                        "VpcSubnetIds": {
                            "Fn::Join": [
                                ",",
                                [
                                    {
                                        "Ref": "TestSubnet01"
                                    },
                                    {
                                        "Ref": "TestSubnet02"
                                    }
                                ]
                            ]
                        }
                    },
                    "RotationRules": {
                        "AutomaticallyAfterDays": 30
                    }
                }
            }
        }
    }

YAML

 AWSTemplateFormatVersion: 2010-09-09
 Transform: AWS::SecretsManager-2020-07-23
 Resources:

      . . . 
           
      MySecretRotationSchedule:
        Type: AWS::SecretsManager::RotationSchedule
        DependsOn: SecretRDSInstanceAttachment 
        Properties:
          SecretId: !Ref MyRDSInstanceRotationSecret
          HostedRotationLambda:
            RotationType: MySQLSingleUser
            RotationLambdaName: SecretsManagerRotation
            VpcSecurityGroupIds: !GetAtt TestVPC.DefaultSecurityGroup
            VpcSubnetIds:
              Fn::Join:
                - ","
                - - Ref: TestSubnet01
                  - Ref: TestSubnet02
          RotationRules:
            AutomaticallyAfterDays: 30
