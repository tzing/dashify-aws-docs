Utilisation d'extensions publiques dans CloudFormation

Les extensions publiques sont des CloudFormation extensions, telles que des types de ressources ou des modules, publiées publiquement dans le registre pour être utilisées par tous les CloudFormation utilisateurs. Cela inclut toutes les extensions publiées par AWS. Les tiers peuvent également publier des extensions publiques.

Il existe deux types d'extension publique :

  • Extensions publiques AWS : les extensions publiées par AWS sont toujours publiques et activées par défaut, de sorte que vous n'avez pas à effectuer d'actions avant de les utiliser dans votre compte. En outre, AWS contrôle la gestion des versions de l'extension, afin que vous utilisiez toujours la dernière version disponible.

  • Extensions publiques tierces : il s'agit d'extensions mises à disposition pour une utilisation générale par d'autres éditeurs que AWS. Pour utiliser une extension publique, vous devez d'abord l'activer dans votre compte et votre région.

    Vous pouvez publier vos propres extensions tierces pour les mettre à la disposition des CloudFormation utilisateurs généraux. Pour plus d'informations, consultez la section Publication d'extensions dans le Guide de l'utilisateur de l'interface de ligne de CloudFormation commande.

Note

Comme pour les extensions privées, les extensions publiques d'éditeurs tiers peuvent implémenter des gestionnaires d'événements qui s'exécutent pendant les opérations de création de ressource, de lecture, de mise à jour, de liste et de suppression de pile. De ce fait, l'utilisation de ces extensions dans vos CloudFormation piles peut entraîner des frais sur votre compte. Ils s'ajoutent aux frais encourus pour les ressources créées. Pour en savoir plus, consultez AWS CloudFormation Tarification.

Activation d'extensions publiques à utiliser dans votre Compte AWS

Afin d'utiliser une extension tierce publique dans votre modèle, vous devez tout d'abord activer l'extension du compte et de la région dans lesquels vous souhaitez l'utiliser. L'activation d'une extension la rend utilisable dans les opérations de pile dans le compte et la région dans lesquels elle est activée. Lorsque vous activez une extension tierce publique, CloudFormation crée une entrée dans le registre des extensions de votre compte pour l'extension activée en tant qu'extension privée. Cela vous permet de personnaliser l'extension telle qu'elle est activée dans votre compte comme suit :

  • Spécifiez un alias à utiliser plutôt que le nom de l'extension tierce publique. Cela permet d'éviter les collisions de noms entre les extensions tierces.

  • Spécifiez si l'extension est automatiquement mise à jour lorsqu'une nouvelle version mineure ou corrective est disponible.

  • Spécifiez le rôle d'exécution CloudFormation utilisé pour activer l'extension, en plus de configurer la journalisation de l'extension.

Lorsque vous activez une extension tierce publique, vous pouvez également définir les propriétés de configuration incluses dans l'extension. Les propriétés de configuration définissent la façon dont l'extension est configurée pour un compte et une région donnés. Pour plus d'informations sur la configuration, veuillez consulter ce qui suit :

  • Définition de la configuration d'une extension au niveau du compte

  • Schéma de configuration de hooks

Paramètre CloudFormation pour utiliser automatiquement les nouvelles versions des extensions

Lorsque vous activez une extension, vous pouvez également spécifier le type d'extension pour utiliser la dernière version mineure. Votre type d'extension met à jour la version mineure chaque fois que l'éditeur publie une nouvelle version de votre extension activée.

Par exemple, la prochaine fois que vous effectuez une opération de pile, telle que la création ou la mise à jour d'une pile, à l'aide d'un modèle incluant cette extension, vous CloudFormation utiliserez la nouvelle version mineure.

La mise à jour vers une nouvelle version d'extension, soit automatiquement soit manuellement, n'affecte pas les instances d'extension déjà allouées dans les piles.

CloudFormation considère les mises à jour des versions majeures des extensions comme pouvant contenir des modifications majeures, et vous oblige donc à effectuer une mise à jour manuelle vers une nouvelle version majeure d'une extension.

Les extensions publiées par AWS sont activées par défaut pour tous les comptes et régions dans lesquels ils sont disponibles, et utilisent toujours la dernière version disponible dans chaque Région AWS.

Important

Étant donné que vous contrôlez si et quand les extensions sont mises à jour vers la dernière version dans votre compte, vous pourriez vous retrouver avec différentes versions de la même extension déployée dans différents comptes et régions.

Cela peut entraîner des résultats inattendus lors de l'utilisation du même modèle contenant cette extension, pour ces comptes et régions.

Spécification d'alias pour faire référence aux extensions

Vous ne pouvez pas activer plus d'une extension avec un nom donné dans un compte et une région donnés. Étant donné que différents éditeurs peuvent proposer des extensions publiques portant le même nom d'extension CloudFormation , vous pouvez spécifier un alias pour toute extension publique tierce que vous activez.

Si vous spécifiez un alias pour l'extension, CloudFormation traitez-le comme le nom du type d'extension dans le compte et la région. Vous devez utiliser l'alias pour faire référence à l'extension dans vos modèles, vos appels d'API et votre CloudFormation console.

Des alias d'extension doivent être uniques au sein d'un compte et d'une région donnés. Vous pouvez activer la même ressource publique plusieurs fois dans le même compte et la même région, en utilisant différents alias de nom de type.

Important

Bien que les alias d'extension ne doivent être uniques que dans un compte et une région donnés, nous recommandons vivement aux utilisateurs de ne pas attribuer le même alias à différentes extensions publiques tierces sur l'ensemble des comptes et des régions. Cela pourrait entraîner des résultats inattendus lors de l'utilisation d'un modèle contenant l'alias d'extension pour plusieurs comptes ou régions.

Spécification d'un rôle d'exécution lors de l'activation d'extensions

Lorsque vous activez une ressource, vous pouvez spécifier le rôle d'exécution IAM CloudFormation à assumer lorsque vous invoquez cette extension dans votre compte et dans votre région.

CloudFormation Pour pouvoir assumer le rôle d'exécution, le rôle doit avoir une politique de confiance définie avec CloudFormation. En outre, l'autorisation d'effectuer une opération est accordée en créant une politique IAM et en l'attachant au rôle d'exécution. Les autorisations requises sont définies dans la section du gestionnaire du schéma d'extension.

Voici un exemple de politique d'approbation de rôle IAM pour le type de ressource et l'extension :

{
    "Version": "2012-10-17",
    "Statement":[
        {
            "Effect": "Allow",
            "Principal":{
                "Service": "resources.cloudformation.amazonaws.com"
            },
            "Action":"sts:AssumeRole",
            "Condition":{
                "StringEquals":{
                    "aws:SourceAccount":"123456789012"
                },
                "StringLike":{
                    "aws:SourceArn":"arn:aws:cloudformation:us-east-1:123456789012:type/resource/Organization-Service-Resource/*"
                }
            }
        }
    ]
}

Voici un exemple de politique d'approbation de rôle IAM pour l'extension de hook :

{
    "Version":"2012-10-17",
    "Statement":[
        {
            "Effect":"Allow",
            "Principal": {
                "Service": [
                    "resources.cloudformation.amazonaws.com",
                    "hooks.cloudformation.amazonaws.com"
                ]
            },
            "Action":"sts:AssumeRole",
            "Condition":{
                "StringEquals":{
                    "aws:SourceAccount":"123456789012"
                },
                "StringLike":{
                     "aws:SourceArn":"arn:aws:cloudformation:us-east-1:123456789012:type/hook/Organization-Service-Hook/*"
                }
            }
        }
    ]
}

Pour plus d'informations, veuillez consulter la rubrique Modification d'une politique d'approbation de rôle dans le Guide de l'utilisateur AWS Identity and Access Management.

Pour activer une extension publique dans le registre

Activez votre extension dans le registre AWS CloudFormation afin qu'elle soit disponible pour utilisation.

Pour activer une extension publique à utiliser dans votre compte à l'aide de la CloudFormation console

  1. Connectez-vous à la AWS Management Console et ouvrez la console AWS CloudFormation à l'adresse https://console.aws.amazon.com/cloudformation.

  2. Dans le volet CloudFormationde navigation, sous CloudFormation Registre, sélectionnez Extensions publiques.

  3. Utilisez Filter (Filtrer) pour sélectionner le type d'extension, puis sélectionnez Third party (Tiers). (Les extensions publiées par AWS sont activées par défaut.)

  4. Sélectionnez l'extension, puis Activate (Activer).

    Note

    Si plusieurs versions d'une extension sont disponibles, vous pouvez utiliser le menu Version pour sélectionner la version de l'extension que vous souhaitez activer. La version par défaut est la plus actuelle.

  5. Utilisez les options de la page Activer l'extension pour personnaliser la façon dont l'extension est activée dans ce compte.

    • Activation de l'extension à l'aide d'un alias de type d'extension :

      1. Sélectionnez Override default (Remplacer la valeur par défaut).

      2. Saisissez l'alias du type d'extension que vous souhaitez utiliser avec cette extension. L'alias doit respecter le format recommandé pour le type d'extension.

      Pour plus d'informations, veuillez consulter la rubrique Spécifier des alias pour faire référence aux extensions.

    • Si vous activez une ressource privée, vous pouvez spécifier le rôle IAM CloudFormation à assumer lors de l'appel de l'extension. Pour plus d'informations, veuillez consulter la rubrique Spécifier un rôle d'exécution lors de l'activation des extensions.

    • Sous Logging config (Configuration de la journalisation), spécifiez les informations de configuration de la journalisation pour une extension, si vous le souhaitez. Par exemple :

      {
    "logRoleArn": "arn:aws:iam::account:role/rolename",
    "logGroupName": "log-group-name"
}

      Les informations de configuration de la journalisation ne sont pas requises, mais recommandées à des fins de débogage. Pour utiliser la configuration de la journalisation avec des hooks, ajoutez la même politique d'approbation que le rôle d'exécution spécifié, afin que le rôle du journal puisse écrire des journaux dans votre groupe de journaux.

      logRoleArnet les noms de logGroupNameclés distinguent les majuscules et minuscules.

    • Sous Versioning (Gestion des versions), spécifiez la manière dont vous recevez les mises à jour.

      • On (Activé) : met automatiquement à jour vers la dernière version mineure. Les versions majeures sont mises à jour manuellement.

      • On (Désactivé) : ne met jamais automatiquement à jour vers la dernière version. Toutes les versions sont mises à jour manuellement.

      Pour plus d'informations, consultez la section Configuration CloudFormation pour utiliser automatiquement les nouvelles versions des extensions.

  6. Spécifiez les données de configuration.

    Note

    Cette étape est requise pour les hooks. Vous devez spécifier ALL pour la propriété TargetStack. Cette opération active les propriétés du hook définies dans la section propriétés de schéma du hook.

    Si l'extension nécessite une configuration supplémentaire au niveau du compte et de la région, CloudFormation inclut la section Configuration de la page Activer l'extension. Vous avez la possibilité de spécifier les données de configuration maintenant ou après l'activation de l'extension. Si l'extension ne nécessite pas de configuration, vous ne verrez pas la section Configuration.

    • Pour spécifier les données de configuration lors de l'activation de l'extension :

      1. Sélectionnez Configure now (Configurer maintenant), puis Activate extension (Activer l'extension).

        CloudFormation affiche la page Configurer l'extension. Pour afficher le schéma de configuration actuel de l'extension, assurez-vous d'activer View configuration schema (Afficher le schéma de configuration).

      2. Dans la zone de texte Configuration JSON (Fichier JSON de configuration), saisissez une chaîne JSON qui représente le schéma de configuration que vous souhaitez spécifier pour cette extension.

      3. Sélectionnez Configure extension (Configurer l'extension).

    • Pour spécifier les données de configuration ultérieurement :

      • Sélectionnez Configure later (Configurer plus tard), puis Activate extension (Activer l'extension).

      Pour spécifier les données de configuration d'une extension déjà activée, rendez-vous sur la page de détails de l'extension activée.

    Une fois l'extension activée avec succès, CloudFormation affiche la page de détails de cette extension.

Pour mettre à jour une extension publique activée dans votre compte à l'aide de la CloudFormation console

Mettez à jour votre extension dans le registre AWS CloudFormation afin qu'elle soit disponible pour utilisation.

Après avoir activé une extension dans votre compte et votre région, vous pouvez mettre à jour les propriétés suivantes de cette extension lorsqu'elle est activée, notamment :

  • Le schéma de configuration.

  • S'il faut activer les mises à jour automatiques des versions

  • Pour les extensions enregistrées privées, le rôle d'exécution à utiliser.

  • Les informations de configuration de journalisation.

  1. Connectez-vous à la AWS Management Console et ouvrez la console AWS CloudFormation à l'adresse https://console.aws.amazon.com/cloudformation.

  2. Dans le volet CloudFormationde navigation, sous CloudFormation Registre, sélectionnez Extensions publiques.

  3. À partir du menu Actions, sélectionnez Edit (Modifier), puis l'option d'édition appropriée :

    • Pour mettre à jour le schéma de configuration, voir Pour spécifier les propriétés de configuration d'une extension dans la CloudFormation console

    • Pour activer ou désactiver les mises à jour automatiques :

      1. Sélectionnez Edit automatic updates (Modifier les mises à jour automatiques).

      2. Sélectionnez On (Activé) ou Off (Désactivé), puis Save (Enregistrer).

    • Pour mettre à jour le rôle d'exécution :

      1. Sélectionnez Edit execution role (Modifier le rôle d'exécution).

      2. Spécifiez l'ARN du rôle IAM que vous CloudFormation souhaitez utiliser lors de l'appel de cette extension, puis sélectionnez Enregistrer.

    • Pour mettre à jour la configuration de la journalisation :

      1. Sélectionnez Edit logging config (Modifier la configuration de la journalisation).

      2. Modifiez le fichier JSON de configuration de la journalisation, puis sélectionnez Save (Enregistrer).

Pour activer une extension publique à utiliser dans votre compte à l'aide de la AWS CLI

Activez votre extension dans le registre AWS CloudFormation via l'AWS CLI.

Pour activer des hooks

Une fois que vous avez développé et enregistré votre hook, vous pouvez l'activer dans votre Compte AWS en le publiant dans le registre.

  • Pour activer un hook dans votre compte, utilisez l'opération SetTypeConfiguration. Cette opération active les propriétés du hook définies dans la section properties de schéma du hook.

    Note

    En activant les hooks dans votre compte, vous autorisez un hook à utiliser les autorisations définies dans votre Compte AWS. CloudFormation supprime les autorisations non requises avant de transmettre vos autorisations au hook. CloudFormation recommande aux clients ou aux utilisateurs du hook de vérifier les autorisations du hook et de connaître les autorisations autorisées par les hooks avant de les activer dans votre compte.

    Spécifiez les données de configuration de votre extension hook enregistrée dans le même compte et la même Région AWS.

    aws cloudformation --region us-west-2 set-type-configuration \
  --configuration "{"CloudFormationConfiguration":{"HookConfiguration":{"TargetStacks": "ALL", "FailureMode": "FAIL", "Properties":{}}}}" \
  --type-arn $HOOK_TYPE_ARN
    Important

    Pour permettre à votre hook d'inspecter de manière proactive la configuration de votre pile, vous devez définir les TargetStacks sur ALL dans la section HookConfiguration.

Pour activer les types de ressource et les modules
Pour activer une extension publique à utiliser dans votre compte à l'aide de la AWS CLI

  • Utilisez activate-type pour activer l'extension et spécifier s'il faut la mettre à jour chaque fois qu'une nouvelle version mineure de l'extension est publiée.

    L'exemple ci-dessous spécifie l'Amazon Resource Name (ARN) public d'une extension publique à activer pour ce compte. En outre, il précise que l'extension est CloudFormation mise à jour chaque fois qu'une nouvelle version mineure est publiée.

    aws cloudformation activate-type \
  --public-type-arn public_extension_ARN \
  --auto-update true

    Cette commande renvoie un ARN de l'extension activée, spécifique à ce compte et à cette région.

    {
    "Arn": "624af370-311a-11e8-b6b7-500cexample"
}
Pour mettre à jour la version d'une extension publique à l'aide de l'AWS CLI

Utilisez activate-type pour réactiver l'extension.

Utilisez le paramètre --version-bump pour spécifier si l'extension doit être mise à jour vers la version MAJOR ou la version MINOR la plus récente.

aws cloudformation activate-type \
    --region us-west-2 \
    --type RESOURCE \
    --type-name Example::Test::1234567890abcdef0 \
    --type-name-alias Example::Test::Alias \
    --version-bump MAJOR

Pour désactiver les extensions publiques dans votre compte

Les sections suivantes montrent comment désactiver les extensions publiques dans votre compte.

Pour désactiver une extension publique de votre compte à l'aide de la console CloudFormation

  1. Utilisez le CloudFormation registre pour trouver l'extension.

  2. À partir du menu Actions, sélectionnez Deactivate (Désactiver).

  3. Sélectionnez Deactivate (Désactiver).

Pour désactiver une extension publique dans votre compte à l'aide de l'AWS CLI

Pour désactiver une extension publique dans votre compte, spécifiez deactivate-type. Voici un exemple de commande de désactivation de la AWS CLI.

aws cloudformation deactivate-type \
    --region us-west-2 \
    --type MODULE \
    --type-name Example::Test::Type::MODULE

Pour désactiver un hook dans votre compte à l'aide de l'AWS CLI

La désactivation d'un hook l'empêche d'exécuter votre Compte AWS.

TargetStacks défini sur NONE désactive le hook dans votre compte, de sorte qu'il ne s'applique pas aux opérations de pile.

Utilisez l'opération set-type-configuration et spécifiez TargetStacks en tant que NONE pour désactiver un hook.

L'exemple suivant spécifie la Région AWS et l'Amazon Resource Name (ARN) du hook qui est désactivé.

aws cloudformation set-type-configuration \
  --region us-west-2 \
  --configuration "{"CloudFormationConfiguration":{"HookConfiguration":{"TargetStacks": "NONE", "FailureMode": "FAIL", "Properties":{}}}}" \
  --type-arn HOOK_TYPE_ARN
