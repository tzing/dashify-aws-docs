Exportation de valeurs de sortie de pile

Pour partager des informations entre plusieurs piles, il convient d'exporter les valeurs de sortie d'une pile. Les autres piles qui se trouvent dans le même compte et la même région AWS peuvent importer les valeurs exportées. Par exemple, une même pile de réseau peut exporter les ID d'un sous-réseau et d'un groupe de sécurité pour des serveurs web publics. Les piles dotées d'un serveur web public peuvent facilement importer ces ressources de mise en réseau. Vous n'avez pas besoin de coder en dur les ID de ressources dans le modèle de la pile ou transmettre les ID en tant que paramètres d'entrée.

Pour exporter la valeur de sortie d'une pile, utilisez le champ Export de la section Sorties du modèle de la pile. Pour importer ces valeurs, utilisez la fonction Fn::ImportValue dans le modèle pour les autres piles. Pour obtenir une procédure pas à pas et des exemple de modèles, consultez Procédure pas à pas : création de références à des sorties de ressources dans une autre pile AWS CloudFormation.

Note

Lorsqu'une autre pile importe une valeur de sortie, vous ne peut pas supprimer la pile qui exporte la valeur de sortie ou modifier la valeur de sortie exportée. Toutes les importations doivent être supprimées avant de pouvoir supprimer la pile d'exportation ou modifier la valeur de sortie.

Exportation de valeurs de sortie de pile comparée à l'utilisation de piles imbriquées

Une pile imbriquée est une pile que vous créez dans une autre pile à l'aide de la ressource AWS::CloudFormation::Stack. Avec les piles imbriquées, vous déployez et gérez toutes les ressources à partir d'une même pile. Vous pouvez utiliser les sorties d'une pile du groupe de piles imbriquées comme entrées dans une autre pile du groupe. Cela est différent de l'exportation de valeurs.

Si vous voulez isoler le partage d'informations au sein d'un groupe de piles imbriquées, nous vous conseillons d'utiliser des piles imbriquées. Pour partager des informations avec d'autres piles (et pas seulement dans le groupe de piles imbriquées), il convient d'exporter les valeurs. Par exemple, vous pouvez créer une pile unique avec un sous-réseau, puis exporter son ID. Les autres piles peuvent utiliser ce sous-réseau en important son ID ; il n'est pas nécessaire de créer un sous-réseau pour chaque pile. Tant que les piles importent l'ID du sous-réseau, vous ne pouvez pas le modifier ni le supprimer.

Affichage des valeurs de sortie exportées

Pour voir les valeurs que vous pouvez importer, affichez toutes les valeurs de sortie exportées à l'aide de la console AWS CloudFormation, de la AWS CLI ou de l'API AWS CloudFormation. AWS CloudFormation présente les noms et les valeurs des sorties exportées pour la région actuelle, ainsi que la pile à partir de laquelle les sorties sont exportées. Pour référencer une valeur de sortie exportée dans le modèle d'une pile, utilisez le nom de l'exportation et la fonction Fn::ImportValue.

Pour afficher les valeurs de sortie exportées (console)

  • Dans le volet de CloudFormationnavigation de la AWS CloudFormationconsole, sélectionnez Exports.

Pour afficher les valeurs de sortie exportées (AWS CLI)

  • Exécutez la commande aws cloudformation list-exports.

Pour afficher les valeurs de sortie exportées (API)

  • Exécutez l'opération ListExportsAPI.
