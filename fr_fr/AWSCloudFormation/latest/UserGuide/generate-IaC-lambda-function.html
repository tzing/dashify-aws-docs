Mise à jour AWS::Lambda::Function des ressources

Lorsqu'un modèle généré contient des AWS::Lambda::Functionressources, des avertissements sont générés indiquant que Code/S3Bucket les Code/S3Key propriétés sont identifiées comme tellesMUTUALLY_EXCLUSIVE_PROPERTIES. En outre, l'Code/S3ObjectVersionétablissement reçoit un UNSUPPORTED_PROPERTIES avertissement. Vous remarquerez peut-être que les Code/ZipFile propriétés Code/ImageUri et sont absentes du modèle généré. Cela est dû au fait qu'il s'agit de propriétés en écriture seule qui s'excluent mutuellement. Toute AWS::Lambda::Function ressource ne peut avoir qu'un seul de ces ensembles de propriétés exclusifs :

  • Code/S3Bucketet les Code/S3Key propriétés, et éventuellement la Code/S3ObjectVersion propriété

  • propriété Code/ImageUri

  • propriété Code/ZipFile

Le générateur IaC ne peut pas déterminer quel ensemble de propriétés exclusives en écriture seule a été utilisé pour créer ou mettre à jour la ressource. Par conséquent, le générateur IaC inclut uniquement le premier ensemble de propriétés dans le modèle généré. Pour inclure AWS::Lambda::Function des ressources dans un modèle généré, celui-ci doit être téléchargé et mis à jour avec les informations correctes pour le code Lambda.

Si vous stockez votre code Lambda dans un compartiment Amazon S3 et que vous n'utilisez pas cette S3ObjectVersion propriété, vous pouvez importer le modèle généré sans aucune modification. Le générateur iAC vous demandera le compartiment et la clé Amazon S3 comme paramètres de modèle lors de l'opération d'importation.

Si vous stockez votre code Lambda dans un référentiel Amazon ECR, vous pouvez mettre à jour votre modèle en suivant les instructions suivantes :

  1. Téléchargez le modèle généré.

  2. Supprimez les propriétés et les paramètres correspondants pour les Code/S3Key propriétés Code/S3Bucket et du modèle généré.

  3. Remplacez les propriétés supprimées dans le modèle généré par la Code/ImageUri propriété, en spécifiant l'URL du référentiel Amazon ECR.

  4. Ouvrez le modèle généré dans la console du générateur iAC et cliquez sur le bouton Importer le modèle modifié.

Si vous stockez votre code Lambda sous forme de fichier zip, vous pouvez mettre à jour votre modèle en suivant les instructions suivantes :

  1. Téléchargez le modèle généré.

  2. Supprimez les propriétés et les paramètres correspondants pour les Code/S3Key propriétés Code/S3Bucket et du modèle généré.

  3. Remplacez les propriétés supprimées dans le modèle généré par la Code/ZipFile propriété.

  4. Ouvrez le modèle généré dans la console du générateur iAC et cliquez sur le bouton Importer le modèle modifié.

Si vous n'avez pas de copie de votre code Lambda, vous pouvez mettre à jour votre modèle en suivant les instructions suivantes :

  1. Utilisez l'action AWS Lambda GetFunctionAPI (par exemple, à l'aide de la aws lambda get-functionAWS CLIcommande).

  2. Dans la réponse, le RepositoryType paramètre est de S3 savoir si le code se trouve dans un compartiment Amazon S3 ou ECR s'il se trouve dans un référentiel Amazon ECR.

  3. Dans la réponse, le Location paramètre contient une URL pré-signée que vous pouvez utiliser pour télécharger le package de déploiement pendant 10 minutes. Téléchargez le code.

  4. Téléchargez le code dans un compartiment Amazon S3.

  5. Exécutez une opération d'importation avec le modèle généré et fournissez le nom du compartiment et la clé sous forme de valeurs de paramètre.
