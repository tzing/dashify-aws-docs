Suppression d'un ensemble de piles

Lorsque vous avez terminé la procédure Mise en route avec AWS CloudFormation StackSets , vous pouvez suivre les procédures indiquées dans cette section pour supprimer des ensembles de piles et d'autres ressources que vous avez créés dans cette procédure. Pour supprimer un ensemble de piles, vous devez commencer par supprimer toutes les instances de piles dans cet ensemble de piles. Pour plus d'informations sur la suppression de toutes les instances de piles, consultez Supprimer les instances de pile de votre ensemble de piles.

Supprimer un ensemble de piles à l'aide de l’AWS Management Console

  1. Sur la page StackSets, sélectionnez l'ensemble de piles que vous avez créé dans Créer un ensemble de piles. Dans cette procédure, nous avons créé un ensemble de piles nommé my-awsconfig-stackset.

  2. Une fois l'ensemble de piles sélectionné, choisissez Supprimer dans le StackSet menu Actions.

  3. Lorsque vous êtes invité à confirmer que vous souhaitez supprimer l'ensemble de piles, choisissez Supprimer StackSet.

Supprimer un ensemble de piles à l'aide de l’AWS CLI

Lorsque vous agissez en tant qu'administrateur délégué, vous devez définir le paramètre --call-as sur DELEGATED_ADMIN chaque fois que vous exécutez une commande StackSets.

--call-as DELEGATED_ADMIN

  1. Exécutez la commande suivante. Lorsque vous êtes invité à confirmer, tapez y, puis appuyez sur Entrée.

    aws cloudformation delete-stack-set --stack-set-name my-awsconfig-stackset

  2. Exécutez la commande list-stack-sets pour vérifier que l'ensemble de piles a bien été supprimé. Les résultats de la list-stack-sets commande devraient afficher votre pile avec un statut deDELETED.

    aws cloudformation list-stack-sets

Supprimer les rôles de service (facultatif)

Supprimez les rôles de service StackSets nécessaires à la création d'un ensemble de piles.

Pour les ensembles de piles autogérés, les rôles que vous avez créés dans le cadre de Prérequis pour les opérations sur les ensembles de piles pour la procédure pas à pas de ce guide sont nommés AWSCloudFormationStackSetAdministrationRole dans le compte administrateur et AwsCloudFormationStackSetExecutionRole dans chaque compte cible.

Pour les ensembles de piles gérés par des services, les rôles créés automatiquement dans le StackSets cadre de la procédure pas à pas Prérequis pour les opérations sur les ensembles de piles décrite dans ce guide ont le suffixe CloudFormationStackSetsOrgAdmin dans le compte de gestion de l'organisation et CloudFormationStackSetsOrgMember dans chaque compte cible.

Pour plus d'informations sur la suppression de rôles, consultez Suppression de rôles ou de profils d'instance dans le Guide de l'utilisateur IAM.

Pour supprimer un rôle de service à l'aide d'AWS Management Console

  1. Connectez-vous à l’AWS Management Console et ouvrez la console IAM à l’adresse https://console.aws.amazon.com/iam/.

  2. Dans le panneau de navigation, choisissez Rôles, puis cochez la case en regard du rôle à supprimer.

  3. Dans le menu Actions du rôle en haut de la page, choisissez Supprimer le rôle.

  4. Dans la boîte de dialogue de confirmation, choisissez Yes, Delete (Oui, supprimer). Si vous êtes sûr, procédez à la suppression même si les dernières données consultées dans le service sont toujours en cours de chargement.

Pour supprimer un rôle de service à l'aide d'AWS CLI

  • Exécutez la commande suivante. Lorsque vous êtes invité à confirmer, tapez y, puis appuyez sur Entrée.

    aws iam delete-role --role-name role name
