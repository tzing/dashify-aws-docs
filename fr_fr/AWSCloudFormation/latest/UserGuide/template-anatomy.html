Anatomie du modèle

Un modèle est un fichier texte au format JSON ou YAML qui décrit votre infrastructure AWS. Les exemples suivants illustrent une structure de modèle AWS CloudFormation et ses sections.

JSON

L'exemple suivant illustre un fragment de modèle au format JSON.

{
  "AWSTemplateFormatVersion" : "version date",

  "Description" : "JSON string",

  "Metadata" : {
    template metadata
  },

  "Parameters" : {
    set of parameters
  },
  
  "Rules" : {
    set of rules
  },

  "Mappings" : {
    set of mappings
  },

  "Conditions" : {
    set of conditions
  },

  "Transform" : {
    set of transforms
  },

  "Resources" : {
    set of resources
  },
  
  "Outputs" : {
    set of outputs
  }
}

YAML

L'exemple suivant illustre un fragment de modèle au format YAML.

---
AWSTemplateFormatVersion: "version date"

Description:
  String

Metadata:
  template metadata

Parameters:
  set of parameters

Rules:
  set of rules

Mappings:
  set of mappings

Conditions:
  set of conditions

Transform:
  set of transforms

Resources:
  set of resources

Outputs:
  set of outputs

Sections de modèle

Les modèles incluent plusieurs sections importantes. La section Resources est la seule section obligatoire. Certaines sections d'un modèle peuvent se trouver dans n'importe quel ordre. Toutefois, lorsque vous créez un modèle, il peut être utile de vous baser sur l'ordre logique indiqué dans la liste suivante, car les valeurs d'une section peuvent faire référence à celles d'une section précédente.

Version de format (facultative)

La version du modèle AWS CloudFormation que le modèle respecte. La version du format d'un modèle diffère de la version de l'API ou de la version WSDL. La version du format d'un modèle peut changer indépendamment des versions d'API et des versions WSDL.

Description (facultative)

Chaîne de texte qui décrit le modèle. Cette section doit toujours suivre la section de version du format du modèle.

Métadonnées (facultatives)

Objets qui fournissent des informations supplémentaires sur le modèle.

Paramètres (facultatifs)

Les valeurs à transmettre au modèle lors de l'exécution (lorsque vous créez ou mettez à jour une pile). Vous pouvez faire référence aux paramètres dans les sections Resources et Outputs du modèle.

Règles (facultatif)

Valide un paramètre ou une combinaison de paramètres transmis à un modèle lors d'une création ou d’une mise à jour de pile.

Mappages (facultatifs)

Mappage de clés et de valeurs associées que vous pouvez utiliser pour spécifier des valeurs de paramètres conditionnelles, comme pour une table de recherche. Pour associer une clé à une valeur correspondante, utilisez la fonction intrinsèque Fn::FindInMap dans les sections Resources et Outputs.

Conditions (facultatives)

Conditions déterminant si certaines ressources sont créées ou si une valeur est attribuée à certaines propriétés de ressources pendant la création ou la mise à jour de la pile. Par exemple, vous pouvez créer une ressource de manière conditionnelle, laquelle varie selon que la pile est destinée à un environnement de production ou de test.

Transformation (facultative)

Pour des applications sans serveur (également appelées applications basées sur Lambda), elle spécifie la version d'AWS Serverless Application Model (AWS SAM) à utiliser. Lorsque vous spécifiez une transformation, vous pouvez utiliser la syntaxe AWS SAM afin de déclarer des ressources dans votre modèle. Le modèle définit la syntaxe que vous pouvez utiliser et comment elle est traitée.

Vous pouvez également choisir les transformations AWS::Include pour utiliser les extraits de modèle stockés séparément du modèle AWS CloudFormation principal. Vous pouvez stocker vos fichiers d'extrait dans un compartiment Amazon S3, puis réutiliser les fonctions entre plusieurs modèles.

Ressources (obligatoires)

Cette section spécifie les ressources de la pile et leurs propriétés, telles qu'une instance Amazon Elastic Compute Cloud ou un compartiment Amazon Simple Storage Service. Vous pouvez faire référence à des ressources dans les sections Resources et Outputs du modèle.

Sorties (facultatives)

Décrit les valeurs qui sont renvoyées chaque fois que vous affichez les propriétés de la pile. Par exemple, vous pouvez déclarer une sortie pour un nom de compartiment S3, puis appeler la commande aws cloudformation describe-stacks de l'AWS CLI pour afficher le nom.

Note

Lorsque vous créez des modèles, n'utilisez pas de sections principales dupliquées, par exemple en utilisant plusieurs sections de ressources. Bien que CloudFormation puisse accepter le modèle, son comportement n'est pas défini lors du traitement du modèle et peut incorrectement allouer les ressources ou renvoyer des erreurs inexplicables.
