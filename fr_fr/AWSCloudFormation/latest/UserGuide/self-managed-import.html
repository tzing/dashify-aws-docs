Importation de pile autogérée pour AWS CloudFormation StackSets

L'opération d'importation de pile AWS CloudFormation peut importer des piles existantes dans des ensembles de piles nouveaux ou existants, de sorte que vous puissiez migrer des piles existantes vers un ensemble de piles en une seule opération. StackSets étend les fonctionnalités des piles en vous permettant de créer, de mettre à jour ou de supprimer des piles dans plusieurs comptes et régions en une seule opération.

Par exemple, si vous avez une pile qui spécifie un rôle d'administrateur AWS Identity and Access Management (IAM) sur plusieurs comptes, vous pouvez importer cette pile dans un ensemble de piles. En utilisant l'importation de pile, vous évitez les temps d'arrêt et les pannes sans supprimer et recréer ces ressources. Une fois que la pile est importée dans un ensemble de piles, la pile d'origine devient une instance de pile de l'ensemble de piles nouvellement généré.

Exigences autogérées pour l'importation de pile

Outre la section Exigences pour l'importation de pile, les importations de pile autogérée nécessitent les éléments suivants.

  • L'opération d'importation de pile prend en charge la création d'un ensemble de piles avec des autorisations autogérées.

  • L'opération d'importation de pile nécessite un compte administrateur dans lequel vous créez un ensemble de piles et un compte cible contenant une pile.

  • Le compte de destination doit être autorisé à utiliser l'opération GetTemplate avec l'entrée de l'ID de pile ou de l'ARN. Pour cette raison, votre compte administrateur doit recevoir les autorisations AWSCloudFormationStackSetsAdminstration ou AWSCloudFormationStackSetsExectionRole.

Importer une pile dans un nouvel ensemble de piles

Importer une pile dans un nouvel ensemble de piles à l'aide de la AWS Management Console

Pour importer une pile dans un ensemble de piles, identifiez une pile qui contient la ressource que vous souhaitez importer.

  1. Connectez-vous à la AWS Management Console et ouvrez la console AWS CloudFormation à l'adresse https://console.aws.amazon.com/cloudformation.

  2. Choisissez StackSets dans le volet de navigation.

  3. En haut de la page StackSets, choisissez Create StackSet (Créer un ensemble de piles).

  4. Sur la page Choisir un modèle, spécifiez un modèle de pile en utilisant l'une des options suivantes et sélectionnez Next (Suivant).

    • Choisissez Amazon S3 URL (URL Amazon S3), puis spécifiez l'URL de votre modèle dans la zone de texte.

    • Choisissez Upload a template file (Charger un fichier de modèle), puis recherchez votre modèle.

    • Choisissez From stack ID (Depuis l'ID de la pile) et saisissez l'ID de votre pile.

  5. Sur la page Spécification des détails du StackSet, saisissez le nom d'un ensemble de piles que vous souhaitez créer et sélectionnez Next (Suivant).

    (Facultatif) Saisissez une description de l'ensemble de piles.

  6. Sur la page Configurer les options du StackSet, examinez vos choix et sélectionnez Next (Suivant).

  7. Sur la page Définir les options de déploiement, sélectionnez Import stacks to stack set (Importer des piles dans un ensemble de piles).

  8. Saisissez l'ID de la pile que vous souhaitez importer dans le champ Stacks to import (Piles à importer). Par exemple, arn:123456789101:us-east-1:StackToImport.

    (Facultatif) Sélectionnez Add another stack ID (Ajouter un autre ID de pile) et saisissez l'ID de la pile que vous souhaitez importer. Vous pouvez ajouter jusqu'à 10 piles par opération d'importation de pile.

  9. Passez en revue vos options de déploiement et sélectionnez Next (Suivant).

  10. Sur la page Review, vérifiez vos choix et les propriétés de votre ensemble de piles. Lorsque vous êtes prêt à importer votre pile dans votre ensemble de piles, sélectionnez Submit (Soumettre).

Résultats : la pile importée est désormais une instance de pile de l'ensemble de piles spécifié. Pour en savoir plus sur l'état d'importation de piles, veuillez consulter la rubrique Codes d'état des ensembles de piles et des instances de piles.

Importer une pile autogérée dans un ensemble de piles existant

Importer une pile dans un ensemble de piles existant à l'aide de la AWS Management Console

Pour importer une pile existante dans un ensemble de piles, identifiez une pile qui contient la ressource que vous souhaitez importer.

  1. Connectez-vous à la AWS Management Console et ouvrez la console AWS CloudFormation à l'adresse https://console.aws.amazon.com/cloudformation.

  2. Choisissez StackSets dans le volet de navigation.

  3. Dans la page StackSets, sélectionnez l'ensemble de piles dans lequel vous souhaitez importer une pile.

  4. L'ensemble de piles étant sélectionné, choisissez Add stacks to StackSet (Ajouter des piles dans l'ensemble de piles) dans le menu Actions.

  5. Dans la page Définir les options de déploiement, sélectionnez Import stacks to stack set (Importer des piles dans un ensemble de piles) et saisissez l'ID de la pile que vous souhaitez importer dans le champ Stacks to import (Piles à importer). Par exemple, arn:123456789101:us-east-1:StackToImport.

    (Facultatif) Sélectionnez Add another stack ID (Ajouter un autre ID de pile) et saisissez l'ID de la pile que vous souhaitez importer. Vous pouvez ajouter jusqu'à 10 piles par opération d'importation de pile.

  6. Choisissez Next (Suivant).

  7. Sur la page Specify Overrides (Spécifier des remplacements), vérifiez vos choix et sélectionnez Next (Suivant).

  8. Sur la page Review, vérifiez vos choix et les propriétés de votre ensemble de piles. Lorsque vous êtes prêt à créer votre ensemble de piles, choisissez Submit (Soumettre).

Résultats : la pile importée est désormais une instance de pile de l'ensemble de piles spécifié. Pour en savoir plus sur l'état d'importation de piles, veuillez consulter la rubrique Codes d'état des ensembles de piles et des instances de piles.

Importation d'une pile dans un ensemble de piles (AWS CLI)

Importer une pile dans un ensemble de piles

Pour importer une pile existante dans un ensemble de piles, identifiez une pile qui contient les ressources que vous souhaitez importer. Dans cet exemple, la pile à importer est arn:123456789101:us-east-1:StackToImport.

  1. Créez un ensemble de piles à partir d'un ID de pile en spécifiant l'ARN complet de la pile CloudFormation à importer.

    aws cloudformation create-stack-set \
  --stack-id "arn:aws:cloudformation:us-east-1:123456789012:stack/StackToImport/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set-name "SingleStackSetName" \
  --permission-model "SELF_MANAGED" \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  2. Importez une pile spécifiée dans votre ensemble de piles.

    aws cloudformation import-stacks-to-stack-set \
  --stack-ids "arn:aws:cloudformation:us-east-1:123456789012:stack/StackToImport/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set "SingleStackSetName" \
  --permission-model SELF_MANAGED \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  3. Clonez la pile importée dans d'autres régions et comptes.

    aws cloudformation create-stack-instances \
  --stack-set-name "StackSetToWhichStackimported" \
  --accounts "123556789101" \
  --regions "us-east-1"

Importer des piles dans un ensemble de piles

Pour importer des piles existantes dans un ensemble de piles, identifiez les piles qui contiennent les ressources que vous souhaitez importer. Dans cet exemple, la pile à importer est arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport1/f449b250-b969-11e0-a185-5081d0136786 et arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport2/f449b250-b969-11e0-a185-5081d0136786.

  1. Créez un jeu de piles à partir d'un ID de pile.

    aws cloudformation create-stack-set \
  --stack-id "arn:123456789101:us-east-1:StackToImport" \
  --stack-set-name "StackSetName" \
  --permission-model "SELF_MANAGED" \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  2. Importez des piles spécifiées dans votre ensemble de piles.

    aws cloudformation import-stacks-to-stackset 
  --stack-ids "arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport1/f449b250-b969-11e0-a185-5081d0136786, arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport2/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set "StackSetName" 
  --permission-model SELF_MANAGED \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  3. Clonez la pile importée dans d'autres régions et comptes.

    aws cloudformation create-stack-instances \
  --stack-set-name "StackSetName" \
  --accounts "123456789012" \
  --regions "us-east-1"
