Configuration d'un portail de compte cible dans AWS CloudFormation StackSets

Un portail de compte est une fonctionnalité facultative qui vous permet de spécifier une fonction AWS Lambda pour vérifier qu'un compte cible répond à certaines exigences avant que AWS CloudFormation StackSets ne commence des opérations de pile dans ce compte. Cette fonctionnalité permet, par exemple, de vérifier que le compte cible n'a aucune alarme CloudWatch active ou ouverte. StackSets appelle la fonction chaque fois que vous lancez des opérations de pile dans le compte cible, et ne continue que si la fonction renvoie un code SUCCEEDED. Si la fonction Lambda renvoie un état FAILED, StackSets ne poursuit pas l'opération que vous avez demandée. Si aucune fonction Lambda de consignation de compte n'est configurée, StackSets ignore la vérification et poursuit l'opération.

Si votre compte de destination échoue à une vérification du portail de compte, l'opération qui a échoué est prise en compte dans le nombre ou le pourcentage de piles spécifié dans la tolérance aux pannes. Pour plus d'informations sur la tolérance aux pannes, consultez Options d'opération d'ensemble de piles.

La fonctionnalité de consignation de compte n'est disponible que pour les opérations StackSets. Cette fonctionnalité n'est pas disponible pour les opérations AWS CloudFormation autres que StackSets.

Configurations d'installation requises

La liste suivante décrit la configuration requise pour la consignation de compte.

  • Pour utiliser la fonctionnalité de consignation de compte StackSets, votre fonction Lambda doit être nommée AWSCloudFormationStackSetAccountGate.

  • Le rôle AWSCloudFormationStackSetExecutionRole doit disposer des autorisations nécessaires pour appeler votre fonction Lambda. Sans ces autorisations, StackSets ne vérifie pas le compte et continue d'effectuer des opérations de pile.

  • Pour que la consignation de compte fonctionne, l'autorisation Lambda InvokeFunction doit être ajoutée aux comptes de destination. La politique d'approbation du compte de destination doit avoir une relation d'approbation avec le compte d'administrateur. Voici un exemple de déclaration de politique qui accorde des autorisations Lambda invokefunction.

    {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "lambda:InvokeFunction"
            ],
            "Resource": "*"
        }
    ]
}

Exemples de fonctions de consignation de compte Lambda

Vous disposez des exemples de modèles AWS CloudFormation suivants pour créer des fonctions Lambda AWSCloudFormationStackSetAccountGate. Pour plus d'informations sur la façon de créer une pile à l'aide de l'un de ces modèles, consultez Création d'une pile dans ce guide.

Emplacement du modèle

Description

https://s3.amazonaws.com/cloudformation-stackset-templates-us-east-1/cloudformation-stack-set-accountgate-succeeded.template

Crée une pile qui implémente une fonction de consignation de compte Lambda qui renvoie le statut SUCCEEDED.

https://s3.amazonaws.com/cloudformation-stackset-templates-us-east-1/cloudformation-stack-set-accountgate-failed.template

Crée une pile qui implémente une fonction de consignation de compte Lambda qui renvoie le statut FAILED.
