Configuration des points de terminaison d'un VPC pour AWS CloudFormation

Vous pouvez améliorer le niveau de sécurité de votre VPC en configurant AWS CloudFormation pour utiliser un point de terminaison de VPC d'interface. Les points de terminaison d'interface reposent sur PrivateLink, une technologie qui vous permet d'accéder de façon privée aux API CloudFormation via des adresses IP privées. PrivateLink limite l'ensemble du trafic réseau entre votre VPC et CloudFormation sur le réseau Amazon. De même, vous n'avez pas besoin d'une passerelle Internet, d'un périphérique NAT ou d'une passerelle privée virtuelle.

Il n'est pas obligatoire de configurer PrivateLink, mais cela est conseillé. Pour plus d'informations sur PrivateLink et les points de terminaison d'un VPC, veuillez consulter la rubrique Accès aux services AWS via PrivateLink.

Avant de commencer

Avant de configurer des points de terminaison de VPC pour CloudFormation, tenez compte des considérations suivantes.

  • Lorsque vous utilisez la fonctionnalité de point de terminaison d'un VPC, accordez l'accès aux compartiments S3 spécifiques à CloudFormation pour les ressources d'un VPC qui doivent répondre à une requête de ressource personnalisée ou à une condition d'attente.

    Si vous utilisez CloudFormation pour créer des ressources dans un VPC comprenant un point de terminaison, vous devez modifier votre politique de point de terminaison IAM afin qu'elle autorise l'accès à certains compartiments S3.

    CloudFormation utilise des compartiments S3 dans chaque région pour surveiller les réponses à une requête de ressource personnalisée ou à une condition d'attente. Si un modèle inclut des ressources personnalisées ou des conditions d'attente dans un VPC, la politique de point de terminaison du VPC doit permettre aux utilisateurs d'envoyer des réponses aux compartiments suivants :

    • Pour les ressources personnalisées, autorisez le trafic vers le compartiment cloudformation-custom-resource-response-region. Lors de l'utilisation de ressources personnalisées, les noms de région ne contiennent pas de tirets. Par exemple, uswest2.

    • Pour les conditions d'attente, autorisez le trafic vers le compartiment cloudformation-waitcondition-region. Lors de l'utilisation de conditions d'attente, les noms de région contiennent des tirets. Par exemple, us-west-2.

    Si la politique de point de terminaison bloque le trafic vers ces compartiments, CloudFormation ne recevra pas de réponses et l'opération de la pile échouera. Par exemple, si une ressource d'un VPC situé dans la région us-west-2 doit répondre à une condition d'attente, celle-ci doit être en mesure d'envoyer une réponse au compartiment cloudformation-waitcondition-us-west-2.

    Pour obtenir la liste des régions prises en charge par CloudFormation, consultez la page Régions et points de terminaison dans Référence générale d'Amazon Web Services.

  • Les points de terminaison d'un VPC ne prennent pas en charge de demandes interrégionales pour le moment. Assurez-vous d'avoir créé votre point de terminaison dans la région où vous envisagez d'effectuer vos appels d'API à CloudFormation.

  • Les points de terminaison d'un VPC prennent uniquement en charge le DNS fourni par Amazon via Route 53. Si vous souhaitez utiliser votre propre DNS, vous pouvez utiliser le transfert DNS conditionnel. Pour plus d'informations, consultez Jeux d'options DHCP dans le Amazon VPC Guide de l'utilisateur.

  • Le groupe de sécurité attaché au point de terminaison d'un VPC doit autoriser les connexions entrantes sur le port 443 à partir du sous-réseau privé du VPC.

Création du point de terminaison de VPC pour AWS CloudFormation

Pour créer le point de terminaison de VPC pour le service CloudFormation, utilisez la procédure Création d'un point de terminaison d'interface dans le Amazon VPC Guide de l'utilisateur afin de créer le point de terminaison suivant :

com.amazonaws.région.cloudformation

region représente l'identifiant de région d'une Région AWS prise en charge par CloudFormation, telle que us-east-2 pour la région USA Est (Ohio).
