Spécification du nom de la pile et des paramètres

Après avoir sélectionné un modèle de pile, spécifiez le nom de la pile et les valeurs des paramètres qui ont été définis dans le modèle.

Avec les paramètres, vous pouvez personnaliser votre pile au moment de la création. Les valeurs de paramètres peuvent être utilisées dans le modèle de pile pour modifier la configuration des ressources. De cette façon, vous n'avez pas besoin de coder en dur les valeurs dans plusieurs modèles pour spécifier différents paramètres. Pour plus d'informations sur les paramètres d'un modèle AWS CloudFormation, consultez Paramètres.

Pour spécifier un nom de pile et les valeurs des paramètres

  1. Sur la page Specify stack details (Spécifier les détails de la pile), entrez un nom pour la pile dans la zone Nom de la pile.

    Ce nom est un identifiant qui vous aidera à retrouver une pile particulière dans une liste de piles. Un nom de pile ne peut contenir que des caractères alphanumériques (sensibles à la casse) et des traits d'union. Il doit commencer par un caractère alphabétique et ne doit pas dépasser 128 caractères.

  2. Dans la section Paramètres, spécifiez les paramètres définis dans le modèle de la pile.

    Vous pouvez utiliser ou modifier n'importe quel paramètre contenant des valeurs par défaut.

  3. Lorsque vous êtes satisfait de vos valeurs de paramètres, choisissez Next afin de définir les options de la pile.

Types de paramètres spécifiques à AWS

Lorsque vous créez des piles qui contiennent des types de paramètres spécifiques à AWS, la console CloudFormation fournit des listes déroulantes de valeurs valides pour ces paramètres. En fonction du type de paramètre, vous pouvez rechercher les valeurs par ID, nom ou valeur de balise (Name). Par exemple, avec le type de paramètre AWS::EC2::VPC::Id, vous pouvez rechercher un ID de VPC spécifique, tel que vpc-b47658d1. Si le VPC contient une balise de nom, telle que Name:TestVPC, vous pouvez également rechercher TestVPC. Actuellement, vous pouvez uniquement rechercher les valeurs de balise qui utilisent la clé Name.

Note

La console ne fournit pas de liste déroulante et ne vous permet pas de rechercher des valeurs avec le type de paramètre AWS::EC2::Image::Id. CloudFormation vérifie uniquement si les valeurs d'entrée sont des ID d'image Amazon Elastic Compute Cloud valides.

Regroupement et tri des paramètres

La console classe les paramètres d'entrée par ordre alphabétique en fonction de leur ID logique. Lorsque vous créez un modèle, vous pouvez utiliser la clé de métadonnées AWS::CloudFormation::Interface pour remplacer le classement par défaut. Pour plus d'informations et pour voir un exemple de la clé de métadonnées AWS::CloudFormation::Interface, consultez AWS::CloudFormation::Interface.
