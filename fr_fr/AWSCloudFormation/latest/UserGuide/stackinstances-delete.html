Supprimer les instances de pile de votre ensemble de piles

Vous pouvez supprimer les instances de piles d'un ensemble dans AWS Management Console ou à l'aide des commandes AWS CloudFormation dans l'AWS CLI. Dans cette procédure, nous allons supprimer toutes les piles.

Pour un ensemble de piles avec des autorisations gérées par le service, si vous supprimez des instances de pile d'une UO de niveau supérieur, l'UO est supprimée en tant que cible de l'ensemble de piles.

Rubriques
  • Supprimer les instances de pile à l'aide de la AWS Management Console
  • Supprimer les instances de pile à l'aide de la AWS CLI

Supprimer les instances de pile à l'aide de la AWS Management Console

  1. Ouvrez la console AWS CloudFormation à l'adresse https://console.aws.amazon.com/cloudformation.

  2. Choisissez StackSets dans le volet de navigation. Sur la page StackSets, sélectionnez l'ensemble de piles que vous avez créé dans Créer un ensemble de piles.

  3. L'ensemble de piles étant sélectionné, choisissez Delete stacks from StackSet (Supprimer des piles de l'ensemble de piles) dans le menu Actions.

  4. Dans la page Définir les options de déploiement, choisissez les comptes à partir desquels supprimer des instances de pile.

    1. [Autorisations autogérées] Pour Comptes, choisissez Déployer les piles dans les comptes. Collez vos numéros de compte de destination dans la zone de texte, en séparant les numéros par des virgules.

      [Autorisations gérées par le service] Pour Comptes, choisissez Déployer les piles dans les unités organisationnelles. Collez les ID des UO que votre ensemble de piles cible.

      Note

      StackSets supprime également les instances de pile de toutes les UO enfants des UO cibles spécifiées.

    2. Pour Régions de déploiement, choisissez les régions à partir desquelles vous souhaitez supprimer des instances de piles. Dans ce cas, Région USA Est (Virginie du N.) et Région USA Ouest (Oregon).

    3. Pour Deployment options (Options de déploiement) :

      • Pour Maximum concurrent accounts (Nombre maximal de comptes en parallèle), conservez les valeurs par défaut Number (Nombre) et 1.

      • Pour Failure tolerance (Tolérance aux pannes), conservez les valeurs par défaut Number (Nombre) et 0.

      Dans la zone Retain stacks (Conserver les piles), conservez la valeur par défaut Disabled (Désactivé).

      Lorsque vous supprimez des piles d'un ensemble de piles, l'option Retain stacks (Conserver les piles) vous permet de choisir de supprimer les instances de piles de votre ensemble de piles, mais d'enregistrer les piles et les ressources qui leur sont associées. Lorsque vous enregistrez des piles d'un ensemble de piles en choisissant l'option Retain stacks (Conserver les piles), les ressources de la pile restent dans leur état actuel, mais la pile n'appartient plus à l'ensemble de piles. Pour réassocier une pile ou ajouter une pile existante à un ensemble de piles, veuillez consulter la rubrique Importation d'une pile dans AWS CloudFormation StackSets.

      Choisissez Next (Suivant).

  5. Sur la page Vérification, vérifiez vos choix et choisissez Submit (Soumettre).

  6. Lorsque la suppression des piles est terminée, vous pouvez vérifier que les instances de piles ont été supprimées de votre ensemble de piles dans la page des détails de l'ensemble de piles, dans l'onglet Stack instances (Instances de piles).

Supprimer les instances de pile à l'aide de la AWS CLI

Lorsque vous agissez en tant qu'administrateur délégué, vous devez définir le paramètre --call-as sur DELEGATED_ADMIN chaque fois que vous exécutez une commande StackSets.

--call-as DELEGATED_ADMIN

  1. Exécutez la commande delete-stack-instances. Pour --stack-set-name, spécifiez le nom de l'ensemble de piles my-awsconfig-stackset.

    Spécifiez la tolérance aux pannes et le nombre maximal de comptes en parallèle en définissant FailureToleranceCount sur 0 et MaxConcurrentCount sur 1 dans le paramètre --operation-preferences, comme indiqué dans l'exemple suivant. Pour appliquer des pourcentages au lieu de nombres, utilisez FailureTolerancePercentage ou MaxConcurrentPercentage. Dans le cadre de la présente procédure, nous n'utilisons pas des pourcentages mais des nombres.

    Note

    La valeur de MaxConcurrentCount dépend de la valeur de FailureToleranceCount. FailureToleranceCount est au maximum supérieur à MaxConcurrentCount.

    Étant donné que --retain-stacks est un paramètre obligatoire de delete-stack-instances, si vous ne souhaitez pas conserver (enregistrer) de piles, ajoutez --no-retain-stacks. Dans cette démonstration, nous ajoutons le paramètre --no-retain-stacks, car nous ne conservons pas de piles.

    [Autorisations autogérées] Remplacez Account_ID par les comptes que vous avez utilisés pour créer votre ensemble de piles dans Créer un ensemble de piles.

    aws cloudformation delete-stack-instances --stack-set-name my-awsconfig-stackset --accounts '["0123456789012"]' --regions '["eu-west-1"]' --operation-preferences FailureToleranceCount=0,MaxConcurrentCount=1 --no-retain-stacks

    [Autorisations gérées par le service] Pour --deployment-targets, spécifiez l'ID d'organisation (racine) ou les ID d'UO dans lesquels vous avez créé des instances de pile.

    Note

    StackSets supprime également les instances de pile de toutes les UO enfants des UO cibles spécifiées.

    aws cloudformation delete-stack-instances --stack-set-name my-awsconfig-stackset --deployment-targets OrganizationalUnitIds='["ou-rcuk-1x5jlwo", "ou-rcuk-slr5lh0a"]' --regions '["eu-west-1"]' --no-retain-stacks

  2. Facultativement, une fois la suppression des piles terminée, vous pouvez vérifier que les instances de piles ont été supprimées de l'ensemble de piles en exécutant la commande describe-stack-set-operation pour afficher l'état et les résultats de l'opération de suppression des piles. Pour --operation-id, utilisez l'ID d'opération qui a été renvoyé par la commande delete-stack-instances.

    aws cloudformation describe-stack-set-operation --stack-set-name stackSetName --operation-id ddf16f54-ad62-4d9b-b0ab-3ed8e9example
