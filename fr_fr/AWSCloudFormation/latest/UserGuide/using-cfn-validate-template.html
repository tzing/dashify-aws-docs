Validation d'un modèle

Pour vérifier la présence d'erreurs de syntaxe dans votre modèle de fichier, vous pouvez utiliser la commande aws cloudformation validate-template.

Note

La commande aws cloudformation validate-template est conçue pour vérifier uniquement la syntaxe de votre modèle. Elle ne garantit pas que les valeurs de propriété que vous avez spécifiées pour une ressource sont valides pour cette ressource. Elle ne détermine pas non plus le nombre de ressources qui existeront lorsque la pile sera créée.

Pour vérifier la validité opérationnelle, vous devez essayer de créer la pile. Il n'y a pas d'environnement de test (sandbox) ni de zone de test pour les piles AWS CloudFormation. Les ressources que vous créez pendant les tests vous sont donc facturées.

Pendant la validation, AWS CloudFormation vérifie d'abord si le modèle est dans un format JSON valide. Si ce n'est pas le cas, CloudFormation vérifie si le modèle est dans un format YAML valide. Si les deux vérifications échouent, CloudFormation retourne une erreur de validation de modèle. Vous pouvez valider les modèles localement à l'aide du paramètre --template-body ou à distance avec le paramètre --template-url.

Modèle valide

L'exemple suivant transmet le paramètre --template-url pour valider un modèle dans un emplacement à distance :

aws cloudformation validate-template --template-url https://s3.amazonaws.com/cloudformation-templates-us-east-1/S3_Bucket.template

Le résultat de la commande précédente est le suivant.

{
    "Description": "AWS CloudFormation Sample Template S3_Bucket: Sample template showing how to create a publicly accessible S3 bucket. **WARNING** This template creates an S3 bucket.
You will be billed for the AWS resources used if you create a stack from this template.",
    "Parameters": [],
    "Capabilities": []
}

Aucun message d'erreur est le résultat attendu, avec des informations sur tous les paramètres répertoriés.

Erreur de validation

Cette section génère une erreur de validation lors de l'exécution de la commande aws cloudformation validate-template.

L'exemple suivant transmet le paramètre --template-body pour valider localement un modèle.

aws cloudformation validate-template --template-body file:///home/local/test/sampletemplate.json

Le résultat de la commande précédente est le suivant.

{
    "ResponseMetadata": {
        "RequestId": "4ae33ec0-1988-11e3-818b-e15a6df955cd"
    },
    "Errors": [
        {
            "Message": "Template format error: JSON not well-formed. (line 11, column 8)",
            "Code": "ValidationError",
            "Type": "Sender"
        }
    ],
    "Capabilities": [],
    "Parameters": []
}
A client error (ValidationError) occurred: Template format error: JSON not well-formed. (line 11, column 8)

Le résultat attendu est un message d'erreur contenant des informations sur l'erreur.
