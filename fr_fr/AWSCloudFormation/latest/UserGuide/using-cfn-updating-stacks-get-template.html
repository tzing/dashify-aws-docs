Modification d'un modèle de pile

Si vous voulez modifier des ressources et les propriétés qui sont déclarées dans un modèle de pile, vous devez modifier le modèle de la pile elle-même. Pour vous assurer que vous mettez uniquement à jour les ressources prévues, utilisez le modèle de la pile existante comme point de départ et apportez-y vos modifications. Si vous gérez le modèle dans un système de contrôle de code source, utilisez une copie de ce modèle comme point de départ. Vous pouvez également obtenir une copie d'un modèle de pile à partir d' AWS CloudFormation.

Si vous voulez uniquement modifier les paramètres d'une pile (tels que sa rubrique Amazon SNS), vous pouvez réutiliser le modèle de pile existant. Vous n'avez pas besoin de vous procurer une copie du modèle ni d'apporter de modifications au modèle de la pile.

Note

Si votre modèle inclut une modification non prise en charge, AWS CloudFormation renvoie un message indiquant que la modification n'est pas autorisée. Ce message peut toutefois s'afficher de manière asynchrone, car les ressources sont créées et mises à jour par défaut AWS CloudFormation dans un ordre non déterministe.

Mettre à jour le modèle d'une pile (Console)

  1. Sur la page Piles de la consoleAWS CloudFormation, cliquez sur le nom de la pile que vous souhaitez mettre à jour.

  2. Dans le volet des détails de la pile sélectionnée, sélectionnez le volet Modèle, puis cliquez sur View in Designer (Afficher dans Designer).

    AWS CloudFormation ouvre une copie du modèle de la pile dans AWS CloudFormation Designer.

  3. Modifiez le modèle.

    Vous pouvez utiliser l' drag-and-drop interface AWS CloudFormation Designer ou les éditeurs JSON et YAML intégrés pour modifier le modèle. Pour plus d'informations sur l'utilisation de AWS CloudFormation Designer, consultezQu'est-ce qu'AWS CloudFormation Designer ?.

    Modifiez uniquement les ressources que vous souhaitez mettre à jour. Utilisez les mêmes valeurs que celles de la configuration actuelle de la pile pour les ressources et les propriétés que vous ne mettez pas à jour. Pour modifier le modèle, vous pouvez effectuer l'une des opérations suivantes :

    • Ajoutez ou supprimez des ressources.

      Pour la plupart des ressources, la modification de leur nom logique revient à supprimer les ressources correspondantes et à les remplacer par une nouvelle. Toutes les autres ressources qui dépendent de la ressource renommée doivent également être mises à jour et peuvent entraîner leur remplacement. D'autres ressources nécessitent que vous mettiez à jour une propriété (non seulement le nom logique) pour lancer une mise à jour.

    • Ajoutez, modifiez ou supprimez des propriétés de ressources existantes.

      Veuillez consulter la Référence des types de ressourcesAWS pour plus d'informations sur l'impact de la mise à jour de propriétés particulières d'une ressource. Pour chaque propriété, une mise à jour peut avoir l'un des impacts suivants :

      • Requis pour la mise à jour : aucune interruption

      • Requis pour la mise à jour : quelques interruptions

      • Requis pour la mise à jour : un remplacement

    • Ajoutez, modifiez ou supprimez des attributs pour des ressources (Metadata, DependsOn, CreationPolicy, UpdatePolicy et DeletionPolicy).

      Important

      Vous ne pouvez pas mettre à jour l'attribut CreationPolicy ou UpdatePolicy de façon autonome. Vous pouvez uniquement les mettre à jour lorsque vous intégrez des changements qui ajoutent, modifient ou suppriment des ressources. Par exemple, vous pouvez ajouter ou modifier l'attribut de métadonnées d'une ressource.

    • Ajoutez, modifiez ou supprimez des déclarations de paramètre. Cependant, vous ne pouvez pas ajouter, modifier ou supprimer un paramètre utilisé par une ressource qui ne prend pas en charge les mises à jour.

    • Ajoutez, modifiez ou supprimez des déclarations de mappage.

      Important

      Si les valeurs d'un mappage ne sont pas utilisées par votre pile, vous ne pouvez pas mettre à jour le mappage seul. Dans ce cas, vous devez inclure des modifications qui ajoutent, modifient ou suppriment des ressources. Par exemple, vous pouvez ajouter ou modifier l'attribut de métadonnées d'une ressource. Si vous mettez à jour une valeur de mappage utilisée par votre pile, vous n'avez pas besoin d'effectuer d'autres modifications pour lancer une mise à jour.

    • Ajoutez, modifiez ou supprimez des déclarations de condition.

      Important

      Vous ne pouvez pas mettre à jour les conditions par elles-mêmes. Vous pouvez uniquement les mettre à jour lorsque vous intégrez des changements qui ajoutent, modifient ou suppriment des ressources. Par exemple, vous pouvez ajouter ou modifier l'attribut de métadonnées d'une ressource.

    • Ajoutez, modifiez ou supprimez des déclarations de valeur de sortie.

    Certaines ressources ou propriétés sont parfois associées à des contraintes de valeurs ou à des contraintes liées à la modification de ces valeurs. Par exemple, les modifications apportées à la propriété AllocatedStorage d'une ressource AWS::RDS::DBInstance doivent être supérieures au paramètre actuel. Si la valeur spécifiée pour la mise à jour ne respecte pas ces contraintes, la mise à jour de cette ressource échoue. Pour voir les contraintes spécifiques liées aux modifications de la propriété AllocatedStorage, consultez ModifyDBInstance.

    Les mises à jour d'une ressource peuvent avoir une incidence sur les propriétés d'autres ressources. Si vous avez utilisé la fonction Ref ou la Fn::GetAtt fonction pour spécifier un attribut d'une ressource mise à jour dans le cadre d'une valeur de propriété dans une autre ressource du modèle, met AWS CloudFormation également à jour la ressource contenant la référence à la propriété modifiée. Par exemple, si vous avez mis à jour la MasterUsername propriété d'une AWS::RDS::DBInstance ressource et que vous disposiez d'une AWS::AutoScaling::LaunchConfiguration ressource dont la UserData propriété contenait une référence au nom de l'instance de base de données à l'aide de la Ref fonction, vous AWS CloudFormation recréeriez l'instance de base de données sous un nouveau nom et mettriez également à jour la LaunchConfiguration ressource.

  4. Pour vérifier les erreurs de syntaxe dans votre modèle, dans la barre d'outils du AWS CloudFormation concepteur, choisissez Valider le modèle ( ).

    Affichez et corrigez toutes les erreurs dans le volet Messages, puis validez le modèle à nouveau. Si vous ne voyez pas d'erreur, le modèle est valide du point de vue syntaxique.

  5. Dans la barre d'outils de AWS CloudFormation Designer, choisissez le menu Fichier ( ) puis Enregistrer pour enregistrer le modèle dans un compartiment S3 ou localement.

  6. Utilisez votre modèle modifié pour mettre à jour votre pile directement ou mettez à jour votre pile à l'aide d'un jeu de modifications.

Obtenir et mettre à jour un modèle pour une pile (interface de ligne de commande)

  1. Pour récupérer le modèle de la pile que vous souhaitez mettre à jour, utilisez la commande aws cloudformation get-template.

  2. Copiez le modèle, collez-le dans un fichier texte, modifiez-le et enregistrez-le. Copiez uniquement le modèle. La commande délimite le modèle par des guillemets. Ne copiez pas ces guillemets. Le modèle lui-même commence par une accolade d'ouverture et se termine par une accolade de fermeture. Spécifiez les modifications que vous souhaitez apporter aux ressources de la pile dans ce fichier.

  3. Utilisez votre modèle modifié pour mettre à jour votre pile directement ou mettez à jour votre pile à l'aide d'un jeu de modifications.
