Détails de l'événement de modification de l'état des ressources

Vous trouverez ci-dessous les champs détaillés relatifs aux événements de changement de statut des ressources.

Les detail-type champs source et sont inclus car ils contiennent des valeurs spécifiques pour les AWS CloudFormation événements.

{
  . . .,
  "detail-type": "CloudFormation Resource Status Change",
  "source": "aws.cloudformation",
  . . .,
  "detail": {
    "stack-id" : "string",
    "logical-resource-id" : "string",
    "physical-resource-id": "string",
    "status-details": {
        "status": "string",
        "status-reason": "string"
    },
     "resource-type": "string",
     "client-request-token": "string"
  }
}
detail-type

Identifie le type d'événement.

Pour les événements d'état des ressources, cette valeur est CloudFormation Resource Status Change.

source

Identifie le service qui a généré l'événement. Pour les CloudFormation événements, cette valeur estaws.cloudformation.

detail

Un objet JSON qui contient des informations sur l'événement. Le service qui génère l'événement détermine le contenu de ce champ.

Pour les événements d'état des ressources, ces données incluent :

stack-id

L'ID unique de la pile auquel la pile est associée.

logical-resource-id

Le nom logique de la ressource, tel que spécifié dans le modèle.

physical-resource-id

Le nom ou l'identifiant unique qui correspond à l'ID d'instance physique d'une ressource prise en charge par CloudFormation.

status-details
status

État de la ressource.

status-reason

Raison de l'état de la ressource.

resource-type

Type de ressource. Par exemple, AWS::S3::Bucket.

client-request-token

Un jeton d'accès utilisé pour appeler l'API. Tous les événements lancés par une opération de pile donnée se voient attribuer le même jeton de demande client, que vous pouvez utiliser pour suivre les opérations. Les opérations de pile lancées depuis la console utilisent le format de jeton Console- StackOperation -ID, qui vous permet d'identifier facilement l'opération de pile. Par exemple, si vous créez une pile à l'aide de la console, chaque événement de pile résultant reçoit le même jeton au format suivant : Console-CreateStack-7f59c3cf-00d2-40c7-b2ff-e75db0987002.

Exemple : événement de modification du statut des ressources

Voici un exemple d'événement d'état des ressources. Cet événement indique qui CloudFormation a réussi à créer la ressource demandée, un compartiment Amazon S3, dans la pile spécifiée.

{
    "version":"0",
    "id":"6a7e8feb-b491-4cf7-a9f1-bf3703467718",
    "detail-type":"CloudFormation Resource Status Change",
    "source":"aws.cloudformation",
    "account":"111122223333",
    "time":"2017-12-22T18:43:48Z",
    "region":"us-west-1",
    "resources":[
        "arn:aws:cloudformation:us-west-1:111122223333:stack/teststack"
    ],
    "detail":{
        "stack-id":"arn:aws:cloudformation:us-west-1:111122223333:stack/teststack",
        "logical-resource-id":"my-s3-bucket",
        "physical-resource-id":"arn:aws:s3:::my-s3-bucket-us-east-1",
        "status-details":{
            "status":"CREATE_COMPLETE",
            "status-reason":""
        },
        "resource-type":"AWS::S3::Bucket",
        "client-request-token":""
    }
}
