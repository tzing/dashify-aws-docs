Limitazioni per la condivisione di dati (anteprima)

Questa è una documentazione di pre-rilascio per le operazioni di scrittura in più data warehouse tramite la funzionalità di condivisione dei dati per Amazon Redshift, disponibile in anteprima pubblica nella traccia PREVIEW_2023. La documentazione e la funzionalità sono soggette a modifiche. Consigliamo di utilizzare questa funzione solo con cluster di test e non in ambienti di produzione. Per i termini e le condizioni di anteprima, vedere Partecipazione al servizio beta in Termini del servizio AWS.

Se non hai ancora creato un unità di condivisione dati nella traccia di anteprima, vai a Condivisione di dati di lettura e scrittura in uno o più account AWS per iniziare.

Di seguito sono riportate le limitazioni relative all'utilizzo delle unità di condivisione dati in Redshift:

  • Le operazioni di scrittura in più warehouse tramite condivisione dei dati non sono attualmente disponibili nei cluster ra3.xlplus. Per utilizzare questa funzionalità, crea i cluster con i seguenti tipi di nodi:

    • ra3.4xl

    • ra3.16xl

    Funziona anche con i gruppi di lavoro Amazon Redshift serverless con la traccia di anteprima.

  • Per la condivisione dei dati tra account e regioni, sia il cluster producer che quello consumer e gli spazi dei nomi serverless devono essere crittografati. Questo è per motivi di sicurezza. Tuttavia, non è necessario che condividano la stessa chiave di crittografia.

  • È possibile condividere le UDF SQL solo tramite le unità di condivisione dati. Le funzioni Python e Lambda definite dall'utente non sono supportate.

  • Se il database del produttore dispone di regole di confronto specifiche, utilizzare le stesse impostazioni di confronto per il database del consumatore.

  • Redshift non supporta l'aggiunta di schemi esterni, tabelle o viste ad associazione tardiva delle tabelle esterne alle unità di condivisione dati.

  • Redshift non supporta funzioni SQL definite dall'utente nidificate sui cluster producer.

  • Redshift non supporta la condivisione di tabelle con chiavi di ordinamento interleaved e viste che fanno riferimento a tabelle con chiavi di ordinamento interleaved.

  • I consumer non possono aggiungere oggetti di unità di condivisione dati a un'altra unità di condivisione dati. I consumer, inoltre, non possono aggiungere viste che fanno riferimento a oggetti di unità di condivisione dati a un'altra unità di condivisione dati.

  • Redshift non supporta l'accesso a un oggetto unità di condivisione dati in cui si è verificato un DDL simultaneo tra le fasi di preparazione ed esecuzione dell'accesso.

  • Redshift non supporta la condivisione di stored procedure tramite unità di condivisione dati.

Per ulteriori informazioni sulle limitazioni della condivisione dei dati, consulta Requisiti e limitazioni per l'unità di condivisione dati in anteprima.
