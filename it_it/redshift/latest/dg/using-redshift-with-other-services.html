Utilizzo di Amazon Redshift con altri servizi

Amazon Redshift si integra con altri servizi AWS per permetterti di spostare, trasformare e caricare i dati in modo rapido e affidabile, tramite caratteristiche di sicurezza dei dati.

Spostamento di dati tra Amazon Redshift e Amazon S3

Amazon Simple Storage Service (Amazon S3) è un servizio Web che consente di memorizzare dati nel cloud. Amazon Redshift sfrutta l'elaborazione parallela per leggere e caricare dati da più file di dati archiviati nei bucket Amazon S3. Per ulteriori informazioni, consultare Caricamento di dati da Amazon S3.

È possibile usare l'elaborazione parallela anche per esportare dati dal data warehouse di Amazon Redshift in più file di dati in Amazon S3. Per ulteriori informazioni, consultare Scaricamento dei dati.

Utilizzo di Amazon Redshift con Amazon DynamoDB

Amazon DynamoDB è un servizio di database NoSQL completamente gestito. È possibile usare il comando COPY per caricare in una tabella di Amazon Redshift dati provenienti da una singola tabella Amazon DynamoDB. Per ulteriori informazioni, consultare Caricamento di dati da una tabella Amazon DynamoDB.

Importazione di dati da host remoti tramite SSH

È possibile usare il comando COPY in Amazon Redshift per caricare dati da uno o più host remoti, come cluster Amazon EMR, istanze Amazon EC2 o altri computer. COPY si connette agli host remoti utilizzando SSH ed esegue comandi sugli host remoti per generare dati. Amazon Redshift supporta più connessioni simultanee. Il comando COPY legge e carica l'output da più origini host in parallelo. Per ulteriori informazioni, consultare Caricamento di dati da host remoti.

Automazione del caricamento dei dati tramite AWS Data Pipeline

È possibile usare AWS Data Pipeline per automatizzare lo spostamento e la trasformazione di dati da e verso Amazon Redshift. Usando le funzionalità di pianificazione integrate di AWS Data Pipeline è possibile pianificare ed eseguire processi ricorrenti senza dover scrivere logica personalizzata complessa per il trasferimento o la trasformazione dei dati. Ad esempio, è possibile configurare un processo ricorrente per la copia automatica di dati da Amazon DynamoDB ad Amazon Redshift. Per un tutorial che descrive dettagliatamente il processo di creazione di una pipeline per lo spostamento periodico di dati da Amazon S3 ad Amazon Redshift, consultare la sezione relativa alla copia di dati in Amazon Redshift tramite AWS Data Pipeline nella Guida per lo sviluppatore AWS Data Pipeline.

Migrazione di dati tramite AWS Database Migration Service (AWS DMS)

È possibile eseguire la migrazione di dati in Amazon Redshift usando AWS Database Migration Service. AWS DMS è in grado di eseguire la migrazione dei dati da e verso i più comuni database commerciali e open source, ad esempio Oracle, PostgreSQL, Microsoft SQL Server, Amazon Redshift, Aurora, DynamoDB, Amazon S3, MariaDB e MySQL. Per ulteriori informazioni, consultare Utilizzo di un database Amazon Redshift come destinazione per AWS Database Migration Service.
