Tag risorsa

Puoi utilizzare la proprietà Resource Tags per applicare tag alle risorse a scopo di identificazione e classificazione. Puoi taggare solo le risorse per le quali AWS CloudFormation supporta l'etichettatura. Per informazioni sulle risorse con cui puoi taggare CloudFormation, consulta le singole risorse inAWS riferimento ai tipi di risorse e proprietà.

Nota

Le implementazioni del tagging potrebbero variare in base alla risorsa. Ad esempio, AWS::AutoScaling::AutoScalingGroup fornisce un'ulteriore proprietà PropagateAtLaunch obbligatoria come parte del proprio sistema di tagging.

Oltre ai tag che definisci, crea CloudFormation automaticamente i seguenti tag a livello di pila con il prefisso: aws:

  • aws:cloudformation:logical-id

  • aws:cloudformation:stack-id

  • aws:cloudformation:stack-name

Il aws: prefisso è riservato all'uso. AWS Questo prefisso non fa distinzione tra maiuscole e minuscole. Se si utilizza questo prefisso nella proprietà Key o Value, non è possibile aggiornare o eliminare il tag. I tag con questo prefisso non contano ai fini del numero di tag per risorsa.

La propagazione dei tag a livello di stack alle risorse, inclusi i tag creati automaticamente, può variare in base alla risorsa. Ad esempio, i tag non sono propagati ai volumi Amazon EBS creati da una mappatura dei dispositivi a blocchi.

Sintassi

JSON

{
  "Key" : String,
  "Value" : String
}

YAML

Key: String
Value: String

Proprietà

Key

Nome chiave del tag. È possibile specificare un valore composto da 1 a 128 caratteri Unicode, che non può avere il prefisso aws:. Puoi utilizzare uno qualsiasi dei seguenti caratteri: l'insieme delle lettere Unicode, cifre, spazi, _, ., :, /, =, +, @, - e ".

Campo obbligatorio: sì

Tipo: stringa

Value

Valore per il tag. È possibile specificare un valore di lunghezza compreso tra 1 e 256 caratteri. Puoi utilizzare uno qualsiasi dei seguenti caratteri: l'insieme delle lettere Unicode, cifre, spazi, _, ., /, =, + e -.

Campo obbligatorio: sì

Tipo: stringa

Esempio

Questo esempio illustra una proprietà Tags. Devi specificare questa proprietà all'interno della sezione Properties di una risorsa che la supporta. Quando la risorsa viene creata, viene contrassegnata con i tag dichiarati.

JSON

"Tags" : [
   {
      "Key" : "keyname1",
      "Value" : "value1"
   },
   {
      "Key" : "keyname2",
      "Value" : "value2"
   }
]

YAML

Tags: 
  - Key: "keyname1"
    Value: "value1"
  - Key: "keyname2"
    Value: "value2"
