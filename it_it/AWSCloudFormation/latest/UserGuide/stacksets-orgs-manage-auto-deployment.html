Gestione delle distribuzioni automatiche per i set di stack con autorizzazioni gestite dal servizio

Con l'implementazione automatica abilitata, StackSets esegue automaticamente l'implementazione negli account che in futuro verranno aggiunti all'organizzazione o alle unitÃ  organizzative (UO) di destinazione. Con la conservazione degli stack abilitata, quando un account viene rimosso da una UO di destinazione, le risorse dello stack nell'account vengono mantenute. Le impostazioni di distribuzione automatica specificate al momento della creazione del set di stack possono essere modificate in qualsiasi momento.

Nota

I valori dei parametri sostituiti si applicano solo agli account attualmente presenti nelle UO di destinazione e nelle relative UO figlio. Gli account aggiunti alle UO di destinazione e alle relative UO figlio in futuro utilizzeranno i valori predefiniti del set di stack e non i valori sostituiti.

Gestire le distribuzioni automatiche utilizzando la console AWS CloudFormation

  1. Aprire la console AWS CloudFormation all'indirizzo https://console.aws.amazon.com/cloudformation..

  2. Nel riquadro di navigazione scegliere StackSets.

  3. Nella pagina StackSets (StackSet), selezionare il set di stack creato in Creazione di un set di stack con autorizzazioni gestite dal servizio.

  4. Con il set di stack selezionato, scegliere Edit automatic deployment (Modifica distribuzione automatica) dal menu Actions (Operazioni). La distribuzione automatica viene impostata a livello di set di stack. Non Ã¨ possibile regolare le distribuzioni automatiche in modo selettivo per unitÃ  organizzative, account o Regioni.

  5. Nella finestra modale Edit automatic deployment (Modifica distribuzione automatica) gestire le impostazioni Automatic deployment (Distribuzione automatica) e (Account removal behavior (Comportamento rimozione account).

    Nota

    Con l'opzione Retain stacks (Mantieni stack) selezionata, le istanze dello stack vengono rimosse dal set di stack, ma gli stack e le relative risorse associate vengono mantenuti. Le risorse rimangono nello stato corrente, ma non faranno piÃ¹ parte del set di stack.

  6. Seleziona Salva.

Gestire le distribuzioni automatiche utilizzando AWS CLI

  1. Apri la AWS CLI.

  2. Esegui il comando update-stack-set, specificando il set di stack creato in Creazione di un set di stack con autorizzazioni gestite dal servizio. La distribuzione automatica viene impostata a livello di set di stack. Se si specifica --auto-deployment nell'aggiornamento del set di stack, non Ã¨ possibile specificare --deployment-targets o --regions.

    aws cloudformation update-stack-set --stack-set-name StackSet_myApp --auto-deployment Enabled=false

  3. Utilizzando l'operation-id restituito come parte dell'output update-stack-set nel passaggio 2, eseguire describe-stack-set-operation per verificare che lo stack sia stato aggiornato correttamente.

    aws cloudformation describe-stack-set-operation --operation-id operation_ID

Esempio di distribuzione automatica

Quando abiliti le implementazioni automatiche, queste vengono attivate in caso di aggiunta di account a un'organizzazione o UO di destinazione, rimozione di account da un'organizzazione o UO di destinazione o spostamento di account tra UO di destinazione.

Ad esempio, un set di stack StackSet1 Ã¨ destinato a un'unitÃ  organizzativa OU1 nella Regione us-east-1. Un set di stack StackSet2 Ã¨ destinato a un'unitÃ  organizzativa OU2 nella Regione us-east-1. OU1 contiene un account denominato AccountA.

Se spostiamo AccountA da OU1 a OU2 con le implementazioni automatiche abilitate, StackSets esegue automaticamente un'operazione di eliminazione per rimuovere l'istanza StackSet1 da AccountA e mette in coda un'operazione di creazione che aggiunge l'istanza StackSet2 a AccountA.
