cfn-get-metadata

Descrizione

È possibile utilizzare lo script helper cfn-get-metadata per recuperare un blocco di metadati da CloudFormation e stamparlo in standard out. Puoi anche stampare una sottostruttura del blocco di metadati se specifichi una chiave. Tuttavia, solo le chiavi di primo livello sono supportate.

Nota

cfn-get-metadata non richiede credenziali, perciò non è necessario utilizzare le opzioni --access-key, --secret-key,--role o --credential-file. Tuttavia, se non viene specificata alcuna credenziale, CloudFormation verifica la sottoscrizione allo stack e limita l'ambito della chiamata allo stack a cui l'istanza appartiene.

Sintassi

cfn-get-metadata --access-key access.key \
                 --secret-key secret.key \
                 --credential-file|f credential.file \
                 --key|k key \
                 --stack|-s stack.name.or.id \
                 --resource|-r logical.resource.id \
                 --role IAM.role.name \
                 --url|-u service.url \
                 --region region

Opzioni

Nome Descrizione Obbligatorio

-k, --key

Per una coppia chiave-valore, restituisce il nome della chiave per il valore specificato.

Tipo: stringa

Esempio: per { "Key1": "SampleKey1", "Key2": "SampleKey2" }, cfn-get-metadata -k Key2 restituisce SampleKey2.

No

-s, --stack

Nome dello stack.

Tipo: stringa

Default: nessuna

Esempio: -s { "Ref" : "AWS::StackName" },

Sì

-r, --resource

L'ID di risorsa logica della risorsa che contiene i metadati.

Tipo: stringa

Esempio: -r WebServerHost

Sì

--role (solo per la segnalazione della risorsa)

Il nome di un ruolo IAM associato all'istanza.

Tipo: stringa

Condizione: il parametro del file delle credenziali sostituisce questo parametro.

No

--region

La Regione da cui derivare l'URL CloudFormation.

Tipo: stringa

Default: nessuna

Esempio: --region ", { "Ref" : "AWS::Region" },

No

--access-key

La chiave di accesso AWS per un account con l'autorizzazione per chiamare DescribeStackResource su CloudFormation.

Tipo: stringa

Condizione: il parametro del file delle credenziali sostituisce questo parametro.

Condizionale

--secret-key

La chiave segreta AWS corrispondente alla chiave di accesso AWS specificata.

Tipo: stringa

Condizione: il parametro del file delle credenziali sostituisce questo parametro.

Condizionale

-f, --credential-file

Un file che contiene sia una chiave segreta sia una chiave di accesso.

Tipo: stringa

Condizione: il parametro del file delle credenziali sostituisce i parametri di chiave di accesso e chiave segreta.
