Selezione di un modello di stack

Dopo l'avvio della creazione guidata di stack, è necessario specificare il modello che AWS CloudFormation dovrà utilizzare per creare lo stack.

I modelli CloudFormation sono file in formato JSON o YAML che specificano le risorse AWS che compongono lo stack. Per ulteriori informazioni sui modelli CloudFormation, consulta Utilizzo dei modelli AWS CloudFormation.

Per selezionare un modello di stack:

  1. Nella pagina Specify template (Specifica modello), scegliere un modello di stack utilizzando una delle opzioni seguenti:

    Template is ready (Il modello è pronto)

    Specificare un modello completato pronto per la creazione di uno stack.

    Nella sezione Specify template (Specifica modello), selezionare l'opzione appropriata in base alla posizione del modello:

    • Amazon S3 URL

      Specifica un URL su un modello in un bucket S3.

      Immettere l'URL nel campo URL Amazon S3.

      Importante

      Se il modello include stack nidificati (per esempio, stack descritti in altri documenti modello ubicati in sottodirectory), verifica che il bucket S3 contenga i file e le directory necessari.

      Se disponi di un modello in un bucket con la funzione Versioni multiple abilitata, puoi indicare una versione specifica di modello, ad esempio https://s3.amazonaws.com/templates/myTemplate.template?versionId=123ab1cdeKdOW5IH4GAcYbEngcpTJTDW. Per ulteriori informazioni, consulta Utilizzo di oggetti in un bucket che supporta la funzione Controllo delle versioni nella Guida per l'utente di Amazon Simple Storage Service.

      L'URL deve puntare a un modello con dimensioni massime di 1 MB, archiviato in un bucket S3 per cui disponi delle autorizzazioni in lettura. La lunghezza massima dell'URL è 1024 caratteri. Alcune risorse potrebbero richiedere che il bucket si trovi nella stessa Regione dello stack.

    • Upload a template file (Carica un file modello)

      Selezionare un modello CloudFormation sul computer locale.

      Scegliere Choose File (Scegli file) per selezionare il file di modello da caricare. Il modello caricato può avere una dimensione massima di 1 MB. Una volta scelto il modello, CloudFormation carica il file e visualizza l'URL S3.

      Se utilizzi la AWS CLI o l'API per creare uno stack, puoi caricare un modello con una dimensione massima di 51.200 byte.

      Nota

      Se carichi un file del modello locale, CloudFormation lo carica su un bucket Amazon Simple Storage Service (Amazon S3) nell'account AWS. Se non si dispone già di un bucket S3 creato da CloudFormation, questo crea un bucket univoco per ciascuna Regione in cui è possibile caricare un file di modello. Se disponi già di un bucket S3 creato da AWS CloudFormation nell'account AWS, CloudFormation aggiunge il modello a quel bucket.

      Considerazioni da tenere a mente sui bucket S3 creati da CloudFormation

      • I bucket sono accessibili a tutti gli utenti dotati di autorizzazioni Amazon S3 nell'account AWS.

      • CloudFormation crea il bucket con crittografia lato server abilitata per impostazione predefinita e di conseguenza esegue la crittografia di tutti gli oggetti memorizzati nel bucket.

        Puoi gestire direttamente le opzioni di crittografia per i bucket creati da CloudFormation. Ad esempio, puoi utilizzare la console Amazon S3 in https://console.aws.amazon.com/s3/ oppure la AWS CLI. Per ulteriori informazioni, consulta Crittografia predefinita Amazon S3 per i bucket S3 nella Guida per l'utente di Amazon Simple Storage Service.

      • Puoi utilizzare il tuo bucket e gestirne le autorizzazioni caricando manualmente i modelli su Amazon S3. Quando crei o aggiorni uno stack, specifica l'URL Amazon S3 di un file del modello.

    Use a sample template (Utilizza un modello di esempio)

    • Per visualizzare più modelli di esempio e snippet, organizzati in base a un servizio AWS, fare clic su View more sample templates (Visualizza altri modelli di esempio).

    Create template in Designer (Crea modello in Designer)

    Crea o modifica un modello utilizzando AWS CloudFormation Designer, l'interfaccia basata sul trascinamento per creare diagrammi grafici dei modelli. Per ulteriori informazioni, consulta Cos'è AWS CloudFormation Designer?.

  2. Per accettare le impostazioni, scegliere Next (Avanti) e procedere specificando il nome dello stack e i parametri.

    Prima di creare le risorse, CloudFormation convalida il modello per rilevare errori di sintassi e alcuni errori semantici, ad esempio dipendenze circolari. Durante la convalida, CloudFormation verifica innanzitutto se il modello è in un formato JSON valido. Se non lo è, CloudFormation verifica se il modello è in un formato YAML valido. Se entrambi i controlli hanno esito negativo, CloudFormation restituisce un errore di convalida del modello.
