Metadati

Puoi utilizzare la sezione opzionale Metadata per includere oggetti YAML o JSON arbitrari che forniscono dettagli sul modello. Ad esempio, puoi includere i dettagli di implementazione del modello relativi a risorse specifiche, come illustrato nel frammento di codice seguente:

Importante

Durante l'aggiornamento di uno stack non puoi aggiornare solo la sezione Metadata. Puoi aggiornarla solo quando includi modifiche che prevedono l'aggiunta, la modifica o l'eliminazione di risorse.

Importante

CloudFormation non trasforma, modifica oppure omette le informazioni incluse nella sezione Metadata. Per questo motivo, si consiglia vivamente di non utilizzare questa sezione per archiviare informazioni riservate, ad esempio password o segreti.

JSON

"Metadata" : {
  "Instances" : {"Description" : "Information about the instances"},
  "Databases" : {"Description" : "Information about the databases"}
}

YAML

Metadata:
  Instances:
    Description: "Information about the instances"
  Databases: 
    Description: "Information about the databases"

Chiavi di metadati

Alcune caratteristiche di AWS CloudFormation consentono di recuperare le impostazioni o le informazioni di configurazione definite nella sezione Metadata. Puoi definire queste informazioni nelle seguenti chiavi di metadati specifiche di AWS CloudFormation:

AWS::CloudFormation::Init

Definisce le attività di configurazione per lo script di supporto cfn-init. Questo script è utile per configurare e installare le applicazioni su istanze EC2. Per ulteriori informazioni, consulta AWS::CloudFormation::Init.

AWS::CloudFormation::Interface

Definisce il raggruppamento e l'ordine dei parametri di input quando sono visualizzati nella console AWS CloudFormation. Per impostazione di default, la console AWS CloudFormation ordina alfabeticamente i parametri in base al relativo ID logico. Per ulteriori informazioni, consulta AWS::CloudFormation::Interface.

AWS::CloudFormation::Designer

Descrive il layout delle risorse in AWS CloudFormation Designer (Designer). Designer aggiunge automaticamente queste informazioni quando lo utilizzi per creare e aggiornare i modelli. Per ulteriori informazioni, consulta Cos'è AWS CloudFormation Designer?.
