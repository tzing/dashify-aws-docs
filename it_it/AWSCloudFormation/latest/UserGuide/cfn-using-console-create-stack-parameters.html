Specificazione del nome e dei parametri di uno stack

Dopo aver selezionato un modello di stack, specifica il nome dello stack e i valori dei parametri che sono stati definiti nel modello.

I parametri ti consentono di personalizzare lo stack al momento della creazione. I valori dei parametri possono essere utilizzati nel modello dello stack per modificare il modo in cui le risorse vengono configurate. In questo modo non è necessario avere valori hardcoded in più modelli per specificare impostazioni diverse. Per ulteriori informazioni sui parametri in un modello AWS CloudFormation, consulta Parametri.

Per specificare i valori dei parametri e il nome dello stack

  1. Nella pagina Specify stack details (Specifica dettagli stack), digitare il nome dello stack in Stack name (Nome stack).

    Il nome dello stack è un identificatore che ti aiuta a trovare un determinato stack da un elenco di stack. Il nome di uno stack può contenere solo caratteri alfanumerici (con distinzione tra lettere maiuscole e minuscole) e trattini. Deve iniziare con un carattere alfabetico e non può avere una lunghezza superiore a 128 caratteri.

  2. Nella sezione Parameters (Parametri), specificare i parametri definiti nel modello di stack.

    È possibile utilizzare o modificare qualsiasi parametro con i valori predefiniti.

  3. Una volta specificati i valori dei parametri desiderati, selezionare Next (Avanti) per procedere con l'impostazione delle opzioni per lo stack.

Tipi di parametri specifici per AWS

Quando crei stack che includono tipi di parametri specifici per AWS, la console CloudFormation fornisce elenchi a discesa di valori validi per i parametri. A seconda del tipo di parametro, puoi cercare i valori per ID, nome o valore del tag Name. Ad esempio, con il tipo di parametro AWS::EC2::VPC::Id puoi cercare un ID VPC specifico, come vpc-b47658d1. Se al VPC è stato assegnato un tag con il nome, ad esempio Name:TestVPC, puoi anche eseguire una ricerca per TestVPC. Al momento, puoi cercare solo valori di tag con la chiave Name.

Nota

La console non fornisce un elenco a discesa né consente di cercare valori con il tipo di parametro AWS::EC2::Image::Id. CloudFormation verifica soltanto se i valori di input sono ID immagine Amazon Elastic Compute Cloud validi.

Raggruppamento e ordinamento di parametri

La console elenca in ordine alfabetico i parametri di input in base al loro ID logico. Quando crei un modello, puoi utilizzare la chiave di metadati AWS::CloudFormation::Interface per sostituire l'ordine predefinito. Per ulteriori informazioni e un esempio della chiave di metadati AWS::CloudFormation::Interface, consulta AWS::CloudFormation::Interface.
