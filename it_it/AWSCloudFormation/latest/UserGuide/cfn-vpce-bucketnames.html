Configurazione degli endpoint VPC per AWS CloudFormation

Puoi migliorare l'assetto di sicurezza del VPC configurando AWS CloudFormation in modo che utilizzi un endpoint VPC di interfaccia. Gli endpoint di interfaccia sono basati su PrivateLink, una tecnologia che consente l'accesso privato alle API di CloudFormation utilizzando indirizzi IP privati. PrivateLink limita il traffico di rete tra il VPC e CloudFormation alla rete Amazon. Inoltre, non hai bisogno di un gateway Internet, di un dispositivo NAT o di un gateway virtuale privato.

Non è obbligatorio configurare PrivateLink, ma è consigliato. Per ulteriori informazioni sugli endpoint PrivateLink e VPC consulta Accesso ai servizi AWS tramite PrivateLink.

Prima di iniziare

Prima di configurare gli endpoint VPC per CloudFormation, tenere presente le considerazioni riportate di seguito:

  • Quando si utilizza la caratteristica endpoint VPC, concedere l'accesso a bucket S3 specifici di CloudFormation per le risorse in un VPC che deve rispondere a una richiesta di risorse personalizzate oppure a una condizione di attesa.

    Se si utilizza CloudFormation per creare le risorse in un VPC con un endpoint VPC, potrebbe essere necessario modificare la policy di endpoint IAM in modo che consenta l'accesso a determinati bucket S3.

    CloudFormation include bucket S3 in ciascuna Regione per monitorare le risposte a una richiesta di risorse personalizzate oppure a una condizione di attesa. Se un modello include risorse personalizzate o condizioni di attesa in un VPC, la policy di endpoint VPC deve consentire agli utenti di inviare risposte ai seguenti bucket:

    • Per le risorse personalizzate, consenti il traffico verso il bucket cloudformation-custom-resource-response-region. Quando si utilizzano risorse personalizzate, i nomi delle Regioni non contengono trattini. Ad esempio, uswest2.

    • Per le condizioni di attesa, consenti il traffico verso il bucket cloudformation-waitcondition-region. Quando si utilizzano condizioni di attesa, i nomi delle Regioni contengono trattini. Ad esempio, us-west-2.

    Se la policy di endpoint blocca il traffico a questi bucket, CloudFormation non riceverà risposte e l'operazione dello stack avrà esito negativo. Ad esempio, se disponi di una risorsa in un VPC nella Regione us-west-2 che deve rispondere a una condizione di attesa, la risorsa deve essere in grado di inviare una risposta al bucket cloudformation-waitcondition-us-west-2.

    Per un elenco delle Regioni supportate da CloudFormation, consulta la pagina Regioni ed endpoint nella Riferimenti generali di Amazon Web Services.

  • Al momento, gli endpoint VPC non supportano richieste tra più Regioni: assicurati di creare l'endpoint nella stessa Regione in cui intendi effettuare le chiamate API a CloudFormation.

  • Gli endpoint VPC supportano solo il DNS fornito da Amazon tramite Route 53. Se si desidera utilizzare il proprio DNS, è possibile usare l'inoltro condizionale sul DNS. Per ulteriori informazioni, consulta Set opzioni DHCP nella Guida per l'utente di Amazon VPC.

  • Il gruppo di sicurezza collegato all'endpoint VPC deve consentire le connessioni in entrata sulla porta 443 dalla sottorete privata del VPC.

Creazione dell'endpoint VPC per AWS CloudFormation

Utilizzare la procedura Creazione di un endpoint di interfaccia nella Guida per l'utente di Amazon VPC per creare il seguente endpoint VPC per il servizio CloudFormation:

com.amazonaws.region.cloudformation

region rappresenta l'identificatore di Regione per una Regione AWS supportata da CloudFormation, ad esempio us-east-2 per la Regione Stati Uniti orientali (Ohio).
