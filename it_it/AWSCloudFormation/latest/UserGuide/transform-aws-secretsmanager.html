Trasformazione AWS::SecretsManager

Utilizza l'operazione transform di AWS::SecretsManager, che è una macro ospitata da AWS CloudFormation, per specificare una funzione AWS Lambda al fine di eseguire la rotazione dei segreti. Quando Creazione di un set di modifiche o Aggiornamento di stack utilizzando i set di modifiche e i modelli fanno riferimento a AWS::SecretsManager, AWS CloudFormation genera una funzione AWS Lambda per eseguire la rotazione dei segreti. Utilizza il tipo di proprietà HostedRotationLambda della risorsa AWS::SecretsManager::RotationSchedule per specificare gli attributi della funzione AWS Lambda desiderata.

La funzione AWS Lambda è inclusa come stack nidificato (ossia una risorsa AWS::CloudFormation::Stack) nel modello elaborato. Questa risorsa si collega quindi al modello di funzione appropriato nel repository delle funzioni Lambda per la rotazione di AWS Secrets Manager in base alla proprietà RotationType specificata nella risorsa AWS::SecretsManager::RotationSchedule.

Utilizzo

Utilizzare la trasformazione AWS::SecretsManager al livello superiore del modello. Non è possibile utilizzare AWS::SecretsManager come trasformazione incorporata in qualsiasi altra sezione del modello.

Il valore della dichiarazione della trasformazione deve essere una stringa letterale. Non è possibile utilizzare un parametro o una funzione per specificare un valore di trasformazione.

Sintassi al livello superiore di un modello

Per includere AWS::SecretsManager al livello superiore di un modello, nella sezione Transform, utilizza la sintassi riportata di seguito.

JSON

{
  "Transform": "AWS::SecretsManager-2020-07-23",
    . . .
}

YAML

Transform: AWS::SecretsManager-2020-07-23

Parametri

La trasformazione AWS::SecretsManager non accetta alcun parametro. Specifica invece le proprietà della funzione AWS Lambda per la rotazione dei segreti che desideri creare utilizzando il tipo di proprietà HostedRotationLambda delle risorse AWS::SecretsManager::RotationSchedule nel modello dello stack.

Remarks

Per considerazioni generali sull'uso di macro, consulta Considerazioni sulla creazione di definizioni di macro AWS CloudFormation.

Esempio

Nell'esempio di modello parziale seguente viene illustrato come utilizzare l'operazione transform di AWS::SecretsManager per specificare una funzione AWS Lambda per la rotazione dei segreti in un database MySQL per un singolo utente, in base alle proprietà specificate nel tipo di proprietà HostedRotationLambda della risorsa AWS::SecretsManager::RotationSchedule.

Per esempi completi di modelli che illustrano le rotazioni dei segreti per i database RDS, i cluster Amazon Redshift e i cluster Document DB, consulta la sezione Esempi di AWS::SecretsManager::RotationSchedule.

JSON

{
        "AWSTemplateFormatVersion": "2010-09-09",
        "Transform": "AWS::SecretsManager-2020-07-23",
        "Resources": {
            
            . . . 
            
            "MySecretRotationSchedule": {
                "Type": "AWS::SecretsManager::RotationSchedule",
                "DependsOn": "SecretRDSInstanceAttachment",
                "Properties": {
                    "SecretId": {
                        "Ref": "MyRDSInstanceRotationSecret"
                    },
                    "HostedRotationLambda": {
                        "RotationType": "MySQLSingleUser",
                        "RotationLambdaName": "SecretsManagerRotation",
                        "VpcSecurityGroupIds": {
                            "Fn::GetAtt": [
                                "TestVPC",
                                "DefaultSecurityGroup"
                            ]
                        },
                        "VpcSubnetIds": {
                            "Fn::Join": [
                                ",",
                                [
                                    {
                                        "Ref": "TestSubnet01"
                                    },
                                    {
                                        "Ref": "TestSubnet02"
                                    }
                                ]
                            ]
                        }
                    },
                    "RotationRules": {
                        "AutomaticallyAfterDays": 30
                    }
                }
            }
        }
    }

YAML

 AWSTemplateFormatVersion: 2010-09-09
 Transform: AWS::SecretsManager-2020-07-23
 Resources:

      . . . 
           
      MySecretRotationSchedule:
        Type: AWS::SecretsManager::RotationSchedule
        DependsOn: SecretRDSInstanceAttachment 
        Properties:
          SecretId: !Ref MyRDSInstanceRotationSecret
          HostedRotationLambda:
            RotationType: MySQLSingleUser
            RotationLambdaName: SecretsManagerRotation
            VpcSecurityGroupIds: !GetAtt TestVPC.DefaultSecurityGroup
            VpcSubnetIds:
              Fn::Join:
                - ","
                - - Ref: TestSubnet01
                  - Ref: TestSubnet02
          RotationRules:
            AutomaticallyAfterDays: 30
