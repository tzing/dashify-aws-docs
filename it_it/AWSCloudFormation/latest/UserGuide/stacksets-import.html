Importazione di uno stack in AWS CloudFormation StackSets

L'operazione di importazione di stack di AWS CloudFormation può importare stack esistenti in set di stack nuovi o esistenti, in modo da poter migrare gli stack esistenti in un set di stack in un'unica operazione. StackSets estende le funzionalità di stack consentendoti di creare, aggiornare o eliminare stack in più account e Regioni in un'unica operazione.

Utilizza le operazioni di importazione per set di stack gestiti dal cliente o gestiti dal servizio. Per i set di stack gestiti dal cliente, l'operazione di importazione può importare stack nell'account amministratore o in diversi account di destinazione e Regioni AWS. Per i set di stack gestiti dal servizio, l'operazione di importazione può importare qualsiasi stack nello stesso AWS Organizations dell'account di gestione. L'operazione di importazione può importare fino a 10 stack utilizzando gli ID stack inline o fino a 200 stack utilizzando un oggetto Amazon S3.

L'importazione di stack è disponibile ovunque siano supportati i set di stack. Per informazioni sul supporto della Regione StackSets, consulta StackSets regional support (Supporto regionale di StackSets).

Requisiti per l'importazione di stack

Poiché i set di stack eseguono operazioni di stack in più account, prima di creare il primo set di stack è necessario che nel tuo Account AWS siano definite le autorizzazioni necessarie.

  • Per configurare le autorizzazioni necessarie per creare un set di stack con autorizzazioni gestite dal cliente, vedere Esecuzione di operazioni dei set di stack che coinvolgono Regioni disabilitate per impostazione predefinita e Concedere autorizzazioni gestite dal cliente.

  • Per configurare le autorizzazioni necessarie per creare un set di stack con autorizzazioni gestite dal servizio, vedere Esecuzione di operazioni dei set di stack che coinvolgono Regioni disabilitate per impostazione predefinita e Attiva l'accesso attendibile con AWS Organizations.

Importante

La proprietà NoEcho non è supportata dall'importazione di StackSet. Gli stack che contengono NoEcho non verranno importati in nuovi set di stack tramite l'importazione di StackSet.

Per ulteriori informazioni sui requisiti di StackSets, consulta Prerequisiti per le operazioni dei set di stack.

Prima di importare uno stack in un set di stack, assicurati di aver compreso i seguenti requisiti:

  • Gli stack possono appartenere solo a un set di stack.

  • Puoi implementare i tag degli stack nel set di stack specificando i tag in modo esplicito come parametri nell'operazione di importazione di stack.

  • Le sostituzioni dei parametri personalizzati di uno stack non sono interessate durante l'operazione di importazione.

  • Le quote e le istanze stack di StackSets si applicano al momento dell'importazione degli stack. Per ulteriori informazioni su queste quote, consulta Quote di AWS CloudFormation.
