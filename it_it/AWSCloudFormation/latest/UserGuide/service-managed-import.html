Importazione di stack gestita dal servizio per AWS CloudFormation StackSets

L'operazione di importazione di stack di AWS CloudFormation può importare stack esistenti in set di stack nuovi o esistenti, in modo da poter migrare gli stack esistenti in un set di stack in un'unica operazione. StackSets estende la funzionalità degli stack, in modo da poter creare, aggiornare o eliminare gli stack su più account e regioni con un'unica operazione.

Requisiti gestiti dal servizio per l'importazione di stack

Oltre alla sezione Requisiti per l'importazione di stack, per le importazioni di stack gestiti dal servizio sono necessari i requisiti seguenti.

  • L'operazione di importazione degli stack richiede un account di gestione o un account amministratore delegato in cui è possibile gestire gli elementi associati, ad AWS Organizations esempio abilitando l'accesso fiduciario con. StackSets

  • Gli account di destinazione devono essere membri dell'AWS Organizations gestito dall'account di gestione o dall'account amministratore delegato.

  • Lo stack di destinazione esiste in una delle unità organizzative di destinazione.

  • L'account di destinazione deve essere un membro di AWS Organizations.

  • L'accesso di AWS Organizations dovrebbe essere nello stato ACTIVATED per Organizations.

  • Gli stack importati devono essere presenti in uno degli account membri e non nell'account di gestione.

Importazione di uno stack gestito dal servizio in un nuovo set di stack (console)

Importazione di uno stack in un nuovo set di stack tramite la AWS Management Console

Per importare uno nuovo stack in un set di stack, identifica uno stack che contiene la risorsa che desideri importare.

  1. Accedere alla AWS Management Console e aprire la console di AWS CloudFormation all'indirizzo https://console.aws.amazon.com/cloudformation.

  2. Dal pannello di navigazione, scegli. StackSets

  3. Nella parte superiore della StackSetspagina, scegli Crea StackSet.

  4. Aggiungi le informazioni seguenti nella pagina Choose a template (Scegli un modello).

    1. Per StackSet il modello di autorizzazione, seleziona Autorizzazioni gestite dal servizio.

    2. Per Prerequisite - Prepare template (Prerequisito - Prepara modello), seleziona Template is ready (Il modello è pronto).

      1. Per URL Amazon S3, immetti l'URL di Amazon S3 nel campo URL Amazon S3.

      2. Per Upload a template file (Carica un file modello), seleziona un modello CloudFormation memorizzato localmente nel computer.

    Accetta le impostazioni desiderate e scegli Next (Avanti).

  5. Aggiungi le seguenti informazioni alla pagina StackSet Specificare i dettagli.

    1. Immettete il nome di un set di stack nella casella del StackSet nome.

    2. (Facoltativo) Inserite una descrizione nella sezione StackSet descrizione.

    Nella pagina Configura le StackSet opzioni, rivedi le tue scelte e seleziona Avanti.

  6. Aggiungi le informazioni seguenti nella pagina Set deployment options (Imposta opzioni di implementazione).

    1. Per Add stacks to stack set (Aggiungi stack al set di stack), seleziona Import stacks to stack set (Importa stack nel set di stack).

    2. Per Stacks to import (Stack da importare), scegli il metodo di importazione dello stack.

      1. Per Stack ID (ID stack), immetti l'ID stack.

      2. Per Stack URL (URL stack), immetti l'URL Amazon S3.

  7. Aggiungi le informazioni seguenti nella sezione Associate organizational units (Associa unità organizzative).

    1. Seleziona Associate with organization (Associa all'organizzazione) per utilizzare l'unità organizzativa radice.

    2. Seleziona Associate with organizational units (OUs) Associa alle unità organizzative [UO]) per immettere gli ID dell'unità organizzativa padre per gli stack da importare. Ad esempio, se Stack 1 e Stack 2 sono sotto OU1 e Stack 3 è sotto OU2, immetti OU1 e OU2.

    Accetta le impostazioni desiderate e scegli Next (Avanti).

  8. Rivedi le impostazioni nella pagina Review (Rivedi) e scegli Submit (Invia).

Importazione di uno stack gestito dal servizio in un set di stack esistente (console)

Seleziona il set di stack e identifica lo stack da importare.

Per importare una pila in un set di stack esistente

  1. Accedere alla AWS Management Console e aprire la console di AWS CloudFormation all'indirizzo https://console.aws.amazon.com/cloudformation.

  2. Dal pannello di navigazione, scegli StackSets.

  3. Seleziona il set di stack in cui desideri importare uno stack, quindi scegli Aggiungi pile StackSet dal menu a discesa Azioni.

  4. Aggiungi le informazioni seguenti nella pagina Set deployment options (Imposta opzioni di implementazione).

    1. Per Add stacks to stack set (Aggiungi stack al set di stack), seleziona Import stacks to stack set (Importa stack nel set di stack).

    2. Aggiungi le informazioni seguenti nella sezione Stacks to import (Stack da importare).

      1. Per Stack ID (ID stack), immetti l'ID stack.

      2. Per Stack URL (URL stack), immetti l'URL Amazon S3.

    3. Aggiungi le informazioni seguenti nella sezione Associate organizational units (Associa unità organizzative).

      1. Seleziona Associate with organization (Associa all'organizzazione) per utilizzare l'unità organizzativa radice.

      2. Seleziona Associate with organizational units (OUs) Associa alle unità organizzative [UO]) per immettere gli ID dell'unità organizzativa padre per gli stack da importare. Ad esempio, se Stack 1 e Stack 2 sono sotto OU1 e Stack 3 è sotto OU2, immetti OU1 e OU2.

      Accetta le impostazioni desiderate e scegli Next (Avanti).

  5. Rivedi la pagina Specify stack details (Specifica dettagli stack) e scegli Next (Avanti).

  6. Conferma e rivedi la pagina Review (Rivedi) e scegli Submit (Invia).

Importazione di uno stack gestito dal servizio in un set di stack (AWS CLI)

Una volta creato un set di stack, puoi importare gli stack passando gli ID stack degli stack che vengono importati. Puoi inoltre passare l'elenco degli ID delle unità organizzative a cui desideri mapparlo.

StackSets importerà gli stack forniti dall'utente all'interno di tali unità organizzative e utilizzerà tali unità organizzative come obiettivi di distribuzione per i set di stack. Gli ID stack presentati nell'input verranno mappati internamente all'unità organizzativa più vicina nell'input dell'elenco di ID delle unità organizzative. Se uno stack non appartiene a un ID dell'unità organizzativa esistente nell'elenco di input, AWS CLI restituirà l'errore StackNotFoundException.

L'operazione import-stacks-to-stack-set istanze di stack per gli stack nell'input dell'ID dell'unità organizzativa. I seguenti esempi di AWS CLI utilizzano l'operazione import-stacks-to-stack-set per importare uno stack in un set di stack.

  • Per utilizzare l'operazione import-stacks-to-stack-sets, specifica stack-ids o stack-ids-url che desideri importare nel set di stack.

    aws cloudformation import-stacks-to-stack-set \
  --stack-set-name ServiceMangedStackSet \
  --stack-ids "arn:123456789012:us-east-1:Stack1" \
  --organizational-unit-ids ou-examplerootid111-exampleouid111
    aws cloudformation import-stacks-to-stack-set \
  --stack-set-name ServiceMangedStackSet \
  --stack-ids-url https://DOC-EXAMPLE-BUCKET \
  --organizational-unit-ids ou-examplerootid111-exampleouid111
Nota

L'operazione import-stacks-to-stack-sets richiede di specificare almeno un ID unità organizzativa (ID UO) in modo che possa associare lo stack importato a quella particolare unità organizzativa. Questa operazione non crea istanze di stack per altri account membri nelle unità organizzative associate. Per aggiornare gli account membri per le unità organizzative associate, utilizza create-stack-instances o update-stack-instances.

create-stack-set crea istanze di stack per tutti gli account nelle unità organizzative con un modello fornito dall'utente tramite caricamento diretto o Amazon S3. I seguenti esempi di AWS CLI utilizzano l'operazione create-stack-set per importare uno stack in un nuovo set di stack.

  • Per utilizzare l'operazione create-stack-set, specifica il nome del set di stack e importa uno stack in un set di stack appena creato.

    aws cloudformation create-stack-set \
  --template-url https://DOC-EXAMPLE-BUCKET \
  --permission-model SERVICE_MANAGED \
  --auto-deployment Enabled=true
