Prerequisiti per le operazioni dei set di stack

Poiché gli stack set eseguono operazioni di stack su più account, prima di poter creare il primo set di stack sono necessarie le autorizzazioni necessarie definite negli account. AWS

Per configurare le autorizzazioni necessarie per creare un set di stack con autorizzazioni gestite dal cliente, vedere Esecuzione di operazioni dei set di stack che coinvolgono Regioni disabilitate per impostazione predefinita e Concedere autorizzazioni gestite dal cliente.

Per configurare le autorizzazioni necessarie per creare un set di stack con autorizzazioni gestite dal servizio, vedere Esecuzione di operazioni dei set di stack che coinvolgono Regioni disabilitate per impostazione predefinita e Attiva l'accesso attendibile con AWS Organizations.

Nota

L'attivazione dell'accesso affidabile con AWS Organizations for AWS CloudFormation StackSets non è attualmente supportata nelle regioni di Cina, Pechino e Ningxia.

Esecuzione di operazioni dei set di stack che coinvolgono Regioni disabilitate per impostazione predefinita

Regioni AWS introdotte dopo il 20 marzo 2019, ad esempio Asia Pacifico (Hong Kong), sono disabilitate per impostazione predefinita. Occorre abilitare queste Regioni per gli account prima di poterli utilizzare. Per questo motivo, tenere presente quanto segue prima di eseguire operazioni di set di stack che coinvolgono account nelle Regioni disattivate per impostazione predefinita.

  • Per creare un set di stack dall'account amministratore di un set di stack (se si utilizzano autorizzazioni gestite dal cliente) o dall'account di gestione dell'organizzazione (se si utilizzano autorizzazioni gestite dai servizi) in una Regione disabilitata per impostazione predefinita, è innanzitutto necessario abilitare tale Regione per l'account dell'amministratore o di gestione.

  • Per AWS CloudFormation creare o aggiornare correttamente un'istanza dello stack:

    • L'account di destinazione deve risiedere in una Regione attualmente abilitata per tale account di destinazione.

    • L'account amministratore o l'account di gestione dell'organizzazione del set di stack deve avere la stessa Regione abilitata dell'account di destinazione.

Importante

Tenere presente che durante le operazioni di set di stack, gli account amministratore e di destinazione si scambiano metadati relativi agli account stessi, oltre alle istanze del set di stack e del set di stack coinvolte.

Inoltre, se disattivi una Regione che contiene un account in cui risiedono le istanze del set di stack, non sei responsabile dell'eliminazione di tali istanze o risorse, se desideri farlo. Inoltre, tenere presente che i metadati relativi all'account di destinazione nella Regione disabilitata verranno conservati nell'account amministratore.

Per ulteriori informazioni su come abilitare e disabilitare le Regioni, vedere Gestione delle Regioni AWS nella Guida di riferimento generale diAWS .

Argomenti
  • Concedere autorizzazioni gestite dal cliente
  • Attiva l'accesso attendibile con AWS Organizations
  • Registrazione di un amministratore delegato
