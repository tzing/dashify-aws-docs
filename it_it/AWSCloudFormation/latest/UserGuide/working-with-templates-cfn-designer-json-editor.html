Editor JSON e YAML integrato

È possibile utilizzare l'editor JSON e YAML integrato di Designer per visualizzare e modificare i dettagli dei modelli. Ad esempio, è possibile utilizzare l'editor integrato per definire le proprietà di una risorsa o per modificare un parametro di modello. Nell'editor integrato sono disponibili due visualizzazioni: una visualizzazione Components (Componenti) e una visualizzazione Template (Modello).

Per apportare modifiche minori a una sezione specifica di un modello, utilizzare la visualizzazione Components (Componenti). Nella visualizzazione Components (Componenti), i componenti che è possibile modificare sono divisi in schede. Queste schede cambiano a seconda che sia selezionata una risorsa.

Ad esempio, se si seleziona una risorsa, Designer fornisce schede per modificare le proprietà e gli attributi della risorsa, ad esempio una policy di aggiornamento o una policy di creazione. Se non è stato selezionato nulla, Designer fornisce schede per modificare parametri, mappature, condizioni, metadati e output del modello. Le modifiche apportate nella visualizzazione Components (Componenti) deve essere un markup JSON o YAML valido. Se si introduce sintassi YAML o JSON non valida, Designer ripristina il markup valido quando si lascia la visualizzazione Components (Componenti).

Per apportare modifiche importanti al modello, utilizzare la visualizzazione Template (Modello). Nella visualizzazione Template (Modello) l'editor JSON e YAML integrato mostra il linguaggio JSON o YAML non elaborato dell'intero modello. Per modificare una risorsa, selezionala nel riquadro canvas. Designer evidenzia automaticamente la risorsa nell'editor JSON e YAML integrato.

Conversione di modelli in YAML o JSON

Puoi convertire il linguaggio di un modello valido da JSON a YAML e viceversa selezionando il pulsante di scelta appropriato in Choose template language (Scegli linguaggio modello). Designer può convertire solo modelli JSON o YAML validi. Se la conversione riesce, nel riquadro Messages (Messaggi) viene visualizzato un messaggio del tipo: Conversione del modello in YAML riuscita.

Importante

È consigliabile non aggiungere commenti YAML # ai modelli in Designer. Se il modello YAML presenta commenti #, Designer non li conserva durante la modifica del modello YAML o la conversione in formato JSON. Se si modifica il modello in Designer (ad esempio, se si trascina una risorsa sul canvas), i commenti andranno persi.

Una volta scelto un linguaggio per il modello, le nuove risorse che si trascinano sul canvas vengono create nella lingua selezionata. Per passare a un altro linguaggio, accertarsi che il modello sia valido e selezionare YAML o JSON in Choose template language (Scegli linguaggio modello).

Nota

Quando si converte un modello in YAML, Designer usa la notazione in formato breve per le funzioni. Ad esempio, - !GetAtt. Inoltre, tutti i collegamenti visivi tracciati utilizzeranno la notazione in formato breve in modalità YAML. Per ulteriori informazioni sulle funzioni intrinseche, vedi Ref.

Tasti di scelta rapida

Nell'editor JSON e YAML integrato di Designer sono disponibili le seguenti scelte rapide da tastiera:

CTRL+Barra spaziatrice

All'interno della chiave Properties di una risorsa, elenca tutte le proprietà disponibili per la risorsa.

Ctrl+F

Cerca un valore specificato.

Per evidenziare tutto quello che corrisponde al valore specificato, premere Alt+Enter.
