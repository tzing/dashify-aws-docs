Attributo CreationPolicy

Associa l'attributo CreationPolicy con una risorsa per evitare che il suo stato passi a creazione completa fin quando AWS CloudFormation non riceve un determinato numero di segnali o il periodo di timeout viene superato. Per segnalare una risorsa, puoi utilizzare lo script o l'API di supporto cfn-signal. SignalResource CloudFormation pubblica segnali validi negli eventi dello stack in modo da tenere traccia del numero di segnali inviati.

La politica di creazione viene richiamata solo quando viene CloudFormation creata la risorsa associata. Attualmente, le uniche CloudFormation risorse che supportano le politiche di creazione sono:

  • AWS::AppStream::Fleet

  • AWS::AutoScaling::AutoScalingGruppo

  • AWS::EC2::Instance

  • AWS::CloudFormation::WaitCondition

Utilizza l'attributo CreationPolicy per restare in attesa durante le operazioni di configurazione della risorsa prima che la creazione dello stack continui. Ad esempio, se esegui l'installazione e la configurazione di applicazioni software su un'istanza EC2, potresti volere che tali applicazioni siano in esecuzione prima di continuare. In questi casi, puoi aggiungere un attributo CreationPolicy all'istanza e quindi inviare un segnale di successo all'istanza dopo l'installazione e la configurazione delle applicazioni. Per un esempio dettagliato, consulta Distribuzione di applicazioni su Amazon EC2 con AWS CloudFormation.

Proprietà di creazione di Dimensionamento automatico Amazon EC2

Configurazione di Dimensionamento automatico Amazon EC2 per una policy di creazione.

AutoScalingCreationPolicy

Per un nuovo gruppo Dimensionamento automatico Amazon EC2, specifica il numero di istanze che devono riferire un esito positivo prima di impostare lo stato del gruppo su CREATE_COMPLETE.

MinSuccessfulInstancesPercent

Specifica la percentuale di istanze in Dimensionamento automatico Amazon EC2 che devono riferire un esito positivo prima di impostare lo stato del gruppo su CREATE_COMPLETE. Puoi specificare un valore compreso tra 0 a100. CloudFormation arrotonda al decimo di percentuale più vicino. Ad esempio, se crei cinque istanze con una percentuale di riuscita minima pari a 50, tre istanze devono inviare segnali di esito positivo. Se un'istanza non invia un segnale entro il tempo specificato dalla Timeout proprietà, CloudFormation presuppone che l'istanza non sia stata creata.

Default: 100

Tipo: integer

Required: No

ResourceSignal

Quando CloudFormation crea la risorsa associata, configura il numero di segnali di successo richiesti e il periodo di CloudFormation attesa di tali segnali.

Count

Il numero di segnali di successo che CloudFormation deve ricevere prima di impostare lo stato della risorsa su. CREATE_COMPLETE Se la risorsa riceve un segnale di errore o non riceve il numero di segnali specificato prima della scadenza del periodo di timeout, la creazione della risorsa fallisce e ripristina lo CloudFormation stack.

Default: 1

Tipo: integer

Required: No

Timeout

Il periodo di CloudFormation attesa del numero di segnali specificato nella proprietà. Count Il periodo di timeout inizia dopo che CloudFormation avvia la creazione della risorsa e scade non prima di quanto specificato, ma può farlo poco dopo. Il tempo massimo che puoi specificare è 12 ore.

Il valore deve essere in formato di durata ISO8601, nel formato: PT#H#M#S, dove ogni # è il numero rispettivamente di ore, minuti e secondi. Per ottenere risultati ottimali, specifica un valore che garantisca alle istanze molto tempo per attivarsi, perché un timeout più breve può causare un rollback.

Predefinito: PT5M (5 minuti)

Tipo: stringa

Required: No

Esempi

Gruppo con scalabilità automatica

Nell'esempio seguente viene mostrato come aggiungere una policy di creazione a un gruppo Dimensionamento automatico Amazon EC2. Le policy di creazione richiede tre segnali di successo e il timeout è dopo 15 minuti.

Per fare in modo che le istanze attendano un controllo dello stato Elastic Load Balancing prima di inviare il segnale di operazione riuscita, è possibile aggiungere una verifica del controllo dello stato utilizzando lo script assistente cfn-init. Per un esempio, consulta il comando verify_instance_health nel modello di esempio degli Aggiornamenti in sequenza di Dimensionamento automatico Amazon EC2.

JSON
"AutoScalingGroup": {
  "Type": "AWS::AutoScaling::AutoScalingGroup",
  "Properties": {
    "AvailabilityZones": { "Fn::GetAZs": "" },
    "LaunchConfigurationName": { "Ref": "LaunchConfig" },
    "DesiredCapacity": "3",
    "MinSize": "1",
    "MaxSize": "4"
  },
  "CreationPolicy": {
    "ResourceSignal": {
      "Count": "3",
      "Timeout": "PT15M"
    }
  },
  "UpdatePolicy" : {
    "AutoScalingScheduledAction" : {
      "IgnoreUnmodifiedGroupSizeProperties" : "true"
    },
    "AutoScalingRollingUpdate" : {
      "MinInstancesInService" : "1",
      "MaxBatchSize" : "2",
      "PauseTime" : "PT1M",
      "WaitOnResourceSignals" : "true"
    }
  }
},
"LaunchConfig": {
  "Type": "AWS::AutoScaling::LaunchConfiguration",
  "Properties": {
    "ImageId": "ami-16d18a7e",
    "InstanceType": "t2.micro",
    "UserData": {
      "Fn::Base64": {
        "Fn::Join" : [ "", [
          "#!/bin/bash -xe\n",
          "yum install -y aws-cfn-bootstrap\n",
          "/opt/aws/bin/cfn-signal -e 0 --stack ", { "Ref": "AWS::StackName" },
          " --resource AutoScalingGroup ",
          " --region ", { "Ref" : "AWS::Region" }, "\n"
        ] ]
      }
    }
  }
}
YAML
AutoScalingGroup:
  Type: AWS::AutoScaling::AutoScalingGroup
  Properties:
    AvailabilityZones:
      Fn::GetAZs: ''
    LaunchConfigurationName:
      Ref: LaunchConfig
    DesiredCapacity: '3'
    MinSize: '1'
    MaxSize: '4'
  CreationPolicy:
    ResourceSignal:
      Count: '3'
      Timeout: PT15M
  UpdatePolicy:
    AutoScalingScheduledAction:
      IgnoreUnmodifiedGroupSizeProperties: 'true'
    AutoScalingRollingUpdate:
      MinInstancesInService: '1'
      MaxBatchSize: '2'
      PauseTime: PT1M
      WaitOnResourceSignals: 'true'
LaunchConfig:
  Type: AWS::AutoScaling::LaunchConfiguration
  Properties:
    ImageId: ami-16d18a7e
    InstanceType: t2.micro
    UserData:
      "Fn::Base64":
        !Sub |
          #!/bin/bash -xe
          yum update -y aws-cfn-bootstrap
          /opt/aws/bin/cfn-signal -e $? --stack ${AWS::StackName} --resource AutoScalingGroup --region ${AWS::Region}

WaitCondition

Nell'esempio seguente viene mostrato come aggiungere una policy di creazione a una condizione di attesa.

JSON
"WaitCondition" : {
    "Type" : "AWS::CloudFormation::WaitCondition",
    "CreationPolicy" : {
        "ResourceSignal" : {
            "Timeout" : "PT15M",
            "Count" : "5"
        }
    }
}
YAML
WaitCondition:
  Type: AWS::CloudFormation::WaitCondition
  CreationPolicy:
    ResourceSignal:
      Timeout: PT15M
      Count: 5
