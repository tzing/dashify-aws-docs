偵測堆疊和資源的未受管組態變更

即使您透過管理資源 CloudFormation，使用者也可以在之外變更這些資源 CloudFormation。使用者可以使用建立資源的基礎服務來直接編輯資源。例如，您可以使用 Amazon EC2 主控台更新建立為 CloudFormation 堆疊一部分的伺服器執行個體。有些變更可能是意外，有些則是為了回應時間急迫性運作事件而刻意為之。無論如何，在 CloudFormation 以外進行的變更都會使堆疊更新或刪除作業複雜化。您可以使用漂移偵測來識別已在 CloudFormation 管理之外進行組態變更的堆疊資源。然後，您可以採取矯正動作，讓堆疊資源與堆疊範本中的定義再次同步，例如直接更新偏離的資源，以符合其範本定義。解決偏離有助於確保組態一致性和成功的堆疊操作。

什麼是偏離？

漂移偵測可讓您偵測堆疊的實際組態是否不同於 (或已漂移) 其預期組態。用 CloudFormation 於偵測整個堆疊或堆疊中個別資源上的漂移。如果資源的任何實際屬性值不同於預期的屬性值，資源就視為已偏離。這包括是否已刪除屬性或資源。如果堆疊的一個或多個資源已偏離，堆疊就視為已偏離。

要確定資源是否已漂移，請確 CloudFormation 定預期的資源屬性值（如堆棧模板中所定義）以及任何指定為模板參數的值。 CloudFormation 然後將這些預期值與堆疊中目前存在的資源屬性的實際值進行比較。如果資源的一個或多個屬性已遭刪除，或值已變更，資源就視為已偏離。

CloudFormation 生成堆棧中已漂移的每個資源的詳細信息。

CloudFormation 檢測那些支持漂移檢測的 AWS 資源上的漂移。不支援資源漂移偵測的資源會指派為 NOT_CHECKED 的漂移狀態。如需支援漂移偵測的 AWS 資源清單，請參閱資源類型支援。

此外，還 CloudFormation 支援可預備的私有資源類型上的漂移偵測；也就是說，其佈建類型為或FULLY_MUTABLE。IMMUTABLE若要對私人資源類型的資源執行漂移偵測，您在帳號中註冊之資源類型的預設版本必須是可佈建的資源。如需有關資源佈建類型的詳細資訊，請ProvisioningType參閱 AWS CloudFormation API 參考中的DescribeType動作參數和命DescribeType令參考中的 AWS CLI 命令。如需私人資源的詳細資訊，請參閱使用 AWS CloudFormation 登錄。

您可以在處於下列狀態的堆疊上執行漂移偵測：CREATE_COMPLETE、UPDATE_COMPLETE、UPDATE_ROLLBACK_COMPLETE 和 UPDATE_ROLLBACK_FAILED。

偵測堆疊上的漂移時， CloudFormation 不會偵測屬於該堆疊的任何巢狀堆疊上的漂移。如需詳細資訊，請參閱 使用巢狀堆疊。反而是，您可以直接在巢狀堆疊上啟動漂移偵測操作。

注意

CloudFormation 僅確定透過堆疊範本或指定範本參數來明確設定的屬性值的漂移。這不包含資源屬性的預設值。若要 CloudFormation 追蹤資源屬性以確定漂移，請明確設定屬性值，即使您將屬性值設定為預設值也一樣。

漂移偵測狀態碼

本節中的表格描述用於漂移偵測的各種狀態類型：

  • 漂移偵測操作狀態描述漂移操作的目前狀態。

  • Drift status (偏離狀態)

    針對「堆疊集合」，這會描述整個堆疊集合的偏離狀態，而此狀態是以屬於該堆疊集合堆疊執行個體的偏離狀態為基礎。

    針對「堆疊執行個體」，這會描述堆疊執行個體的偏離狀態，而此狀態是以其相關聯堆疊的偏離狀態為基礎。

    針對「堆疊」，這會描述整個堆疊的偏離狀態，而此狀態是以其資源的偏離狀態為基礎。

  • 資源偏離狀態描述個別資源的偏離狀態。

下表列出 CloudFormation 指定給堆疊漂移偵測操作的狀態碼。

漂移偵測操作狀態 描述

DETECTION_COMPLETE

對於堆疊中支援漂移偵測的所有資源，堆疊漂移偵測操作已成功完成。

DETECTION_FAILED

堆疊中至少一個資源的堆疊漂移偵測操作已失敗。 CloudFormation 成功完成漂移檢測的資源將可獲得結果。

DETECTION_IN_PROGRESS

堆疊漂移偵測操作目前進行中。

下表列出 CloudFormation 指定給堆疊的漂移狀態碼。

Drift status (偏離狀態) 描述

DRIFTED

針對堆疊：堆疊不同於或已「偏離」其預期範本組態。如果堆疊的一個或多個資源已偏離，堆疊就視為已偏離。

針對堆疊執行個體：如果與其相關聯的堆疊已發生偏離，則會將堆疊執行個體視為已偏離。

針對堆疊集合：如果有一或多個堆疊執行個體已發生偏離，則會將堆疊集合視為已偏離。

NOT_CHECKED

CloudFormation 尚未檢查堆棧，堆棧集或堆棧實例是否與其預期的模板配置不同。

IN_SYNC

每個受支援資源的目前組態符合其預期範本組態。沒有支援漂移偵測資源的堆疊、堆疊集合，或堆疊執行個體，其狀態也會是 IN_SYNC。

下表列出了 CloudFormation 分配給堆棧資源的漂移狀態代碼。

資源偏離狀態 描述

DELETED

資源不同於其預期範本組態，因為已刪除資源。

MODIFIED

資源不同於其預期範本組態。

NOT_CHECKED

CloudFormation 尚未檢查資源是否與預期的模板配置不同。

IN_SYNC

資源的目前組態符合其預期範本組態。

下表列出 CloudFormation 指派給資源屬性的差異類型狀態碼，這些資源屬性與預期的範本配置不同。

屬性差異類型 描述

ADD

作為陣列或清單資料類型的資源屬性已新增一個值。

REMOVE

已從目前的資源組態中移除屬性。

NOT_EQUAL

目前的屬性值不同於堆疊範本中定義的預期值。

偵測偏離時的考量

為了在堆疊上成功執行漂移偵測，使用者必須有以下許可：

  • 對於堆疊包含的每個支援漂移偵測的資源，需要有讀取許可。例如，如果堆疊包含 AWS::EC2::Instance 資源，則您必須有 ec2:DescribeInstances 許可，才能在堆疊上執行漂移偵測。

  • cloudformation:DetectStackDrift

  • cloudformation:DetectStackResourceDrift

  • cloudformation:BatchDescribeTypeConfigurations

如需有關在中設定權限的詳細資訊 CloudFormation，請參閱控制存取 AWS Identity and Access Management。

在某些邊緣情況下， CloudFormation 可能無法始終返回準確的漂移結果。您應該注意這些極端案例，以適當解讀您的漂移偵測結果。

  • 在某些情況下，屬性陣列包含的物件會回報為偏離，而事實上，它們是由負責資源的基礎服務提供給屬性的預設值。

  • 某些資源與相關資源之間存在附加關係，以致於一個資源實際上可能連接或移除其他資源 (在同一個或另一個範本中定義) 的屬性值。例如，AWS::EC2::SecurityGroupIngress 和 AWS::EC2::SecurityGroupEgress 資源可用來連接和移除 AWS::EC2::SecurityGroup 資源的值。在這些情況下，請先 CloudFormation 分析堆疊範本中的附件，然後再執行漂移比較。但是， CloudFormation 無法跨堆棧執行此分析，因此可能無法返回附加資源駐留在不同堆棧中的準確漂移結果。

    支援漂移偵測且允許或需要從其他資源附加的資源包括：

    資源類型 附加資源類型

    AWS::SNS::Topic

    AWS::SNS::Subscription

    AWS::IAM::User

    AWS::IAM::UserToGroupAddition

    AWS::IAM::Group

    AWS::IAM::Role

    AWS::IAM::User

    AWS::IAM::Policy

    AWS::IAM::ManagedPolicy

    AWS::ElasticLoadBalancingV2::Listener

    AWS::ElasticLoadBalancingV2::ListenerCertificate

    AWS::EC2::SecurityGroup

    AWS::EC2::SecurityGroupEgress

    AWS::EC2::SecurityGroupIngress

  • CloudFormation 不會對任何資源的KMSKeyId屬性執行漂移偵測。由於索引 AWS KMS 鍵可由多個別名參考，因此 CloudFormation 無法保證此屬性的一致精確漂移結果。

  • 您可以在堆棧模板中指定某些資源屬性，根據其本質而言 CloudFormation ，無法與生成的堆棧資源中的屬性進行比較。因此，這些屬性無法納入漂移偵測結果中。這些屬性分為兩大類：

    • CloudFormation 無法對應回堆疊範本中初始資源屬性值的屬性值。

      例如， CloudFormation 無法將 Lambda 函數的原始程式碼對應回AWS::Lambda::Function資源的Code屬性類型，因此 CloudFormation 無法將其包含在漂移偵測結果中。

    • 負責資源的服務不會傳回的屬性值。

      某些屬性值原本就不可能由資源所屬的服務傳回。這些通常包含機密資訊，例如，不應公開的密碼或其他敏感資料。例如，IAM 服務永遠不會傳回 IAM User Password 屬性類型的屬 LoginProfile性值，因此 CloudFormation 無法將其包含在漂移偵測結果中。

    • 陣列中的物件實際上可能是服務預設，而不是手動新增的漂移。

  • 如果您遇到任何誤報，請使用 CloudFormation 控制台中的反饋鏈接向我們發送您的評論，或通過以下方式與我們聯繫AWS re:Post。

  • 部分屬性的輸入值可能會相等但不相同。為避免誤報，您應確保預期組態與實際組態相符。

    • 例如，資源屬性的預期組態可能是 1024 MB，而相同資源屬性的實際組態可能是 1 GB。1024 MB 和 1 GB 相等，但不相同。

      如果在此資源屬性上執行漂移偵測，漂移偵測會發出漂移結果訊號。

      若要避免此誤報，請將資源屬性的預期組態變更為 1024 MB，然後執行漂移偵測。
