自訂資源

注意

與自訂資源相比， CloudFormation 登錄提供了許多優點，這些優點包括：

  • 支援建模、佈建和管理第三方應用程式資源

  • 支援 Create、Read、Update Delete、和 List (CRUDL) 操作

  • 支援私有和第三方資源類型的偏移偵測

與自訂資源不同，以登錄為基礎的資源不需要關聯 Amazon SNS 主題或 Lambda 函數來執行 CRUDL 操作。如需有關 CloudFormation 登錄的詳細資訊，請參閱使用AWS CloudFormation 登錄。

自訂資源可讓您在範本中撰寫自訂佈建邏輯，讓 AWS CloudFormation 可在您建立、更新 (若您變更自訂資源) 或刪除堆疊時隨時執行。例如，您可能會希望包含無法作為 AWS CloudFormation 資源類型使用的資源。您可以使用自訂資源包含那些資源。以此方式，您仍然可以在單一堆疊中管理所有相關資源。

使用 AWS::CloudFormation::CustomResource 或 Custom::MyCustomResourceTypeName 資源類型在您的範本中定義自訂資源。自訂資源需要一個屬性：服務權杖，其指定 AWS CloudFormation 傳送請求的目的地，例如 Amazon SNS 主題。

注意

若您使用 VPC 端點功能，VPC 中的自訂資源必須要能存取 AWS CloudFormation 限定的 S3 儲存貯體。自訂資源必須傳送回應至預先簽章的 Amazon S3 URL。如果資源無法將回應傳送至 Amazon S3，AWS CloudFormation 不會收到回應，而堆疊操作會失敗。如需詳細資訊，請參閱 設定 AWS CloudFormation 適用的 VPC 端點。

自訂資源的運作方式

任務對自訂資源採取的動作都會涉及三個對象。

範本開發人員

建立包含自訂資源類型的範本。範本開發人員會在範本中指定服務字符和任何輸入資料。

custom resource provider

擁有自訂資源，並決定如何處理和回應來自 AWS CloudFormation 的請求。自訂資源提供者必須提供範本開發人員使用的服務字符。

AWS CloudFormation

在堆疊操作期間，傳送請求到在範本中定義的服務字符，然後等待回應，再繼續堆疊操作。

範本開發人員和自訂資源提供者可為相同的個人或實體，但程序是相同的。下列步驟說明一般程序：

  1. 範本開發人員在其範本中定義自訂資源，其中包含服務字符和任何輸入資料參數。根據自訂資源，輸入資料可能為必要項目。但是，服務字符永遠都是必要項目。

    服務權杖指定 AWS CloudFormation 傳送請求的目的地，例如 Amazon SNS 主題 ARN 或 AWS Lambda 函數 ARN。如需詳細資訊，請參閱 AWS::CloudFormation::CustomResource。輸入資料的服務字符和結構由 custom resource provider 定義。

  2. 每當任何人使用範本建立、更新或刪除自訂資源時，AWS CloudFormation 都會傳送請求至指定的服務字符。服務字符必須位於堆疊建立的相同區域。

    在請求中，AWS CloudFormation 會包含請求類型和預先簽章的 Amazon Simple Storage Service URL (為自訂資源傳送回應的目的地) 等資訊。如需請求中包含項目的詳細資訊，請參閱 自訂資源請求物件。

    以下範例資料顯示 AWS CloudFormation 在請求中包含的項目：

    {
   "RequestType" : "Create",
   "ResponseURL" : "http://pre-signed-S3-url-for-response",
   "StackId" : "arn:aws:cloudformation:us-west-2:123456789012:stack/stack-name/guid",
   "RequestId" : "unique id for this create request",
   "ResourceType" : "Custom::TestResource",
   "LogicalResourceId" : "MyTestResource",
   "ResourceProperties" : {
      "Name" : "Value",
      "List" : [ "1", "2", "3" ]
   }
}
    注意

    在此範例中，ResourceProperties 可讓 AWS CloudFormation 建立自訂承載，以傳送至 Lambda 函數。

  3. 自訂資源提供者會處理 AWS CloudFormation 請求，並將 SUCCESS 或 FAILED 的回應傳回至預先簽章的 URL。custom resource provider 會以 JSON 格式檔案提供回應，並上傳至預先簽章的 S3 URL。如需詳細資訊，請參閱《Amazon Simple Storage Service 使用者指南》中的使用預先簽章的 URL 上傳物件。

    在回應中，自訂資源提供者也可包含範本開發人員可存取的名稱值對。例如，若請求成功，回應可包含輸出資料，或是若請求失敗，則包含錯誤訊息。如需回應的詳細資訊，請參閱 自訂資源回應物件。

    重要

    如果名稱值對包含敏感資訊，您應該使用 NoEcho 欄位來遮罩自訂資源的輸出。否則，這些值可透過表示屬性值 (例如 DescribeStackEvents) 的 API 可見。

    如需使用 NoEcho 遮罩敏感資訊的詳細資訊，請參閱請勿在您的範本中內嵌憑證最佳實務。

    custom resource provider 負責接聽和回應請求。例如，針對 Amazon SNS 通知，自訂資源提供者必須接聽和回應傳送到特定主題 ARN 的通知。AWS CloudFormation 會在預先簽章的 URL 位置等待並接聽回應。

    以下範例資料顯示回應中可能包含的自訂資源：

    {
   "Status" : "SUCCESS",
   "PhysicalResourceId" : "TestResource1",
   "StackId" : "arn:aws:cloudformation:us-west-2:123456789012:stack/stack-name/guid",
   "RequestId" : "unique id for this create request",
   "LogicalResourceId" : "MyTestResource",
   "Data" : {
      "OutputName1" : "Value1",
      "OutputName2" : "Value2",
   }
}

  4. 在取得 SUCCESS 回應後，AWS CloudFormation 便會繼續堆疊操作。若傳回 FAILED 回應或沒有任何回應，則操作便會失敗。任何來自自訂資源的輸出資料都會存放在預先簽章的 URL 位置。範本開發人員可使用 Fn::GetAtt 函數擷取該資料。
