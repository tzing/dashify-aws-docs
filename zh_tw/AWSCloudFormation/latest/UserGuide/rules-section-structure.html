規則

選用 Rules 區段會驗證在堆疊建立或堆疊更新期間傳遞給範本的參數或參數組合。若要使用範本規則，請在範本中明確宣告 Rules，後面接著宣告內容。在建立或更新資源之前，請使用規則區段來驗證參數值。

使用規則

每個範本規則由兩個屬性組成：

  • 規則條件 (選用) – 決定規則何時生效。

  • 宣告 (必要) – 描述使用者可針對特定參數指定的值。

規則可包含 RuleCondition 屬性，而且必須包含 Assertions 屬性。對於每個規則，您只能定義一個規則條件。您可以在 Assertions 屬性中定義一或多個宣告。如果您未定義規則條件，則該規則的宣告一律生效。

規則特定的內部函數

若要定義規則條件和宣告，請使用規則特定的內部函數，這些函式只能在範本的 Rules 區塊中使用。您可以建立巢狀函式，但規則條件或宣告的最終結果必須為 true 或 false。

您可以使用下列的規則特定的內部函數，來定義規則條件與宣告：

  • Fn::And

  • Fn::Contains

  • Fn::EachMemberEquals

  • Fn::EachMemberIn

  • Fn::Equals

  • Fn::If

  • Fn::Not

  • Fn::Or

  • Fn::RefAll

  • Fn::ValueOf

  • Fn::ValueOfAll

規則特定的內部函數用於規則的條件或宣告中。條件屬性確定是否 AWS CloudFormation 應用斷言。如果條件評估為true，則 AWS CloudFormation 會評估宣告，以在建立或更新佈建產品時驗證參數值是否有效。如果參數值無效，則 AWS CloudFormation 不會建立或更新堆疊。如果條件評估為false，則 AWS CloudFormation 不檢查參數值並繼續堆棧操作。

語法

JSON

範本的 Rules 區塊包含了金鑰名稱 Rules，後面接著單一冒號。您必須使用大括號來括住所有規則宣告。如果宣告多項規則，這些規則會以逗號分隔。對於每項規則，您會在引號中宣告其邏輯名稱，後面依序接著冒號和括號，括號之中是規則條件與宣告。

{
    "Rules": {
        "Rule01": {
            "RuleCondition": {
                "rule-specific intrinsic function": "Value01"
            },
            "Assertions": [
                {
                    "Assert": {
                        "rule-specific intrinsic function": "Value02"
                    },
                    "AssertDescription": "Information about this assert"
                },
                {
                    "Assert": {
                        "rule-specific intrinsic function": "Value03"
                    },
                    "AssertDescription": "Information about this assert"
                }
            ]
        },
        "Rule02": {
            "Assertions": [
                {
                    "Assert": {
                        "rule-specific intrinsic function": "Value04"
                    },
                    "AssertDescription": "Information about this assert"
                }
            ]
        }
    }
}

YAML

Rules:
  Rule01:
    RuleCondition:
      rule-specific intrinsic function: Value01
    Assertions:
      - Assert:
          rule-specific intrinsic function: Value02
        AssertDescription: Information about this assert
      - Assert:
          rule-specific intrinsic function: Value03
        AssertDescription: Information about this assert
  Rule02:
    Assertions:
      - Assert:
          rule-specific intrinsic function: Value04
        AssertDescription: Information about this assert

範例

有條件地驗證參數值

在下列範例中，兩個規則會檢查 InstanceType 參數的值。視環境參數 (test 或 prod) 的值而定，使用者必須針對 a1.medium 參數指定 a1.large 或 InstanceType。InstanceType 與 Environment 參數必須在同一個範本的 Parameters 區塊中宣告。

範例 JSON

{
    "Rules": {
        "testInstanceType": {
            "RuleCondition": {
                "Fn::Equals": [
                    {
                        "Ref": "Environment"
                    },
                    "test"
                ]
            },
            "Assertions": [
                {
                    "Assert": {
                        "Fn::Contains": [
                            [
                                "a1.medium"
                            ],
                            {
                                "Ref": "InstanceType"
                            }
                        ]
                    },
                    "AssertDescription": "For a test environment, the instance type must be a1.medium"
                }
            ]
        },
        "prodInstanceType": {
            "RuleCondition": {
                "Fn::Equals": [
                    {
                        "Ref": "Environment"
                    },
                    "prod"
                ]
            },
            "Assertions": [
                {
                    "Assert": {
                        "Fn::Contains": [
                            [
                                "a1.large"
                            ],
                            {
                                "Ref": "InstanceType"
                            }
                        ]
                    },
                    "AssertDescription": "For a production environment, the instance type must be a1.large"
                }
            ]
        }
    }
}

範例 YAML

Rules:
  testInstanceType:
    RuleCondition: !Equals 
      - !Ref Environment
      - test
    Assertions:
      - Assert:
          'Fn::Contains':
            - - a1.medium
            - !Ref InstanceType
        AssertDescription: 'For a test environment, the instance type must be a1.medium'
  prodInstanceType:
    RuleCondition: !Equals 
      - !Ref Environment
      - prod
    Assertions:
      - Assert:
          'Fn::Contains':
            - - a1.large
            - !Ref InstanceType
        AssertDescription: 'For a production environment, the instance type must be a1.large'
