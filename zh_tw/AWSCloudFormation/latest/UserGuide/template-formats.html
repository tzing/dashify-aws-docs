AWS CloudFormation 範本格式

您可用 JSON 或 YAML 格式編寫 AWS CloudFormation 範本。我們支援這兩種格式的所有 AWS CloudFormation 功能和函數，包括在 AWS CloudFormation 設計工具中。

決定要使用的格式時，請挑選您最順手的格式。亦請考慮 YAML 有而 JSON 沒有的一些功能，例如注釋。

重要

我們建議您不要在 設計工具 中將 # YAML 註解新增至您的範本。若您的 YAML 範本具有 # 註解，設計工具在將範本轉換成 JSON 時不會保留那些註解。此外，如果您在設計工具中修改您的範本 (例如，若您在畫布上移動資源)，您會遺失評論。

您可以在設計工具外建立的 AWS CloudFormation 範本中新增註解。以下範例示範有內嵌評論的 YAML 範本。


AWSTemplateFormatVersion: "2010-09-09"
Description: A sample template
Resources:
  MyEC2Instance: #An inline comment
    Type: "AWS::EC2::Instance"
    Properties: 
      ImageId: "ami-0ff8a91507f77f867" #Another comment -- This is a Linux AMI
      InstanceType: t2.micro
      KeyName: testkey
      BlockDeviceMappings:
        -
          DeviceName: /dev/sdm
          Ebs:
            VolumeType: io1
            Iops: 200
            DeleteOnTermination: false
            VolumeSize: 20

如需每種格式之範本語法的詳細資訊，請參閱範本剖析。

AWS CloudFormation 支援以下 JSON 和 YAML 規格：

JSON

AWS CloudFormation 遵循 ECMA-404 JSON 標準。如需 JSON 格式的詳細資訊，請參閱 http://www.json.org。

YAML

AWS CloudFormation 支援 YAML 1.1 版規格，但有幾個例外。AWS CloudFormation 不支援下列功能：

  • binary、omap、pairs、set 和 timestamp 標籤

  • 別名

  • 雜湊合併

如需 YAML 的詳細資訊，請參閱 https://yaml.org/。
