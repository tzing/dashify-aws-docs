演練：使用 AWS CloudFormation 設計工具來建立基本 Web 伺服器

AWS CloudFormation 設計工具會以圖形方式代表您的範本，協助您了解範本中的資源及其連接方式。整合式 JSON 及 YAML 編輯器可讓您直接在 AWS CloudFormation 主控台中輕鬆修改範本。為了示範如何使用這兩種元件，我們會使用 AWS CloudFormation 設計工具在 VPC 中建置基本 Web 伺服器。然後，我們將保存模板並使用它來創建一個 CloudFormation 堆棧。

在演練期間，您將完成下列步驟：

  1. 新增並連線資源。

    第一次開啟 AWS CloudFormation 設計工具時，您需由空白範本開始進行。我們會先使用 AWS CloudFormation 設計工具拖曳資源以填入範本；例如，您可將 VPC 和 EC2 執行個體拖放到您的範本中。我們也會建立資源之間的連結。例如，我們將使用 AWS CloudFormation 設計工具建立網際網路閘道和 VPC 之間的連線。

  2. 新增範本參數、映射和輸出。

    我們會使用 AWS CloudFormation 設計工具整合式編輯器新增其他範本元件，讓範本更有用。例如，我們會將參數新增至範本，以便您在建立堆疊時可以指定輸入值。如此一來，您就不用為了可能會經常變更的屬性值而持續編輯範本。

  3. 指定資源屬性。

    我們要再次使用 整合式編輯器 來為資源指定組態設定。

  4. 佈建資源

    在您建立堆疊之後，才會啟動與執行範本資源。我們將使用您剛建立的範本來啟動 CloudFormation 堆疊，並由該堆疊來佈建範本中定義的所有資源。

    注意

    CloudFormation 這是一項免費服務；但是，您需要按照每個服務的當前費率向您收取包含在堆棧中的AWS資源費用。如需有關 AWS 定價的詳細資訊，請參閱 http://aws.amazon.com 上各項產品的詳細資訊頁面。

必要條件

本逐步解說假設您具備 Amazon Virtual Private Cloud (Amazon VPC)、亞馬遜彈性運算雲 (Amazon EC2) 和 CloudFormation. 在內容方面，每個程序都提供各項資源的一些基本資訊。

此外，在開始之前，請先確認您要建立堆疊的區域中已具備 Amazon EC2 金鑰對。如需詳細資訊，請參閱《Amazon EC2 Linux 執行個體使用者指南》中的 Amazon EC2 金鑰對。

步驟 1：新增並連線資源

我們將使用AWS CloudFormation設計師 drag-and-drop 界面新增 Amazon EC2 執行個體和網路資源，例如 VPC、子網路、路由表和網際網路閘道。新增所有資源之後，我們會建立資源之間的連線。例如，我們會將網際網路閘道與 VPC 建立關聯。

將資源新增至範本

  1. 於 https://console.aws.amazon.com/cloudformation/designer 開啟 AWS CloudFormation 設計工具。

  2. 在整合式編輯器頁面下半部中，選擇 Edit (編輯) ( )。

  3. 將範本名稱變更為 BasicWebServerInVPC，然後按 Enter。

    目前，我們有一個無效的空白範本。在接下來的步驟中，我們會新增資源以讓該範本有效。

  4. 在 Resource types (資源類型) 窗格的 EC2 類別中，將 VPC 資源類型拖曳至 Canvas (畫布) 窗格中。

    資源會依資源類別來組織。我們新增的所有資源都位於 EC2 類別中。

    AWS CloudFormation 設計工具會立即修改您的範本以包含 VPC 資源，其結果看起來如下列 JSON 程式碼片段。

      "Resources": {
    "VPC431KO": {
      "Type": "AWS::EC2::VPC",
      "Properties": {},
      "Metadata": {
        "AWS::CloudFormation::Designer": {
          "id": "445730ea-0d11-45ba-b6ac-12345EXAMPLE"
        }
      }
    }
  }

    YAML 程式碼片段類似如下。

    Resources:
  VPC431KO:
    Type: 'AWS::EC2::VPC'
    Properties: {}
    Metadata:
      'AWS::CloudFormation::Designer':
        id: 9430b008-7a03-41ed-b63e-12345EXAMPLE

    請注意，我們仍需要指定 VPC 屬性，例如 VPC 的 CIDR 區塊。我們稍後會執行此作業。針對我們新增的所有資源，都要這樣做。

  5. 重新命名 VPC。

    注意

    重新命名資源時，您會重新命名其邏輯 ID，也就是範本中參照的名稱 (而非 CloudFormation 建立資源時指定的名稱)。如需詳細資訊，請參閱 資源。

    1. 選擇 VPC 資源。

    2. 在整合式編輯器中，選擇 Edit (編輯) 圖示 ( )。

    3. 將名稱變更為 VPC，然後選擇 Enter。

    接著，我們要將資源新增至 VPC。

  6. 拖曳 VPC 資源的邊角將其展開，讓它的大小足以容納更多資源。

    由於您無法將 EC2 執行個體 (用於託管網站) 直接新增至 VPC，因此我們需要新增一個子網路，且執行個體必須位於子網路中。

  7. 在 VPC 內部新增一個 Subnet (子網路) 資源類型，並將其重新命名為 PublicSubnet。

    我們會使用子網路來配置 VPC 中的 IP 地址範圍，您可以將此子網路與其他 AWS 資源 (例如 Amazon EC2 執行個體) 建立關聯。

    當您在 VPC 內部新增子網路時，AWS CloudFormation 設計工具會自動將子網路與 VPC 建立關聯。此關聯是一種容器模型，在此模型容器內的資源會自動與容器資源建立關聯。

  8. 在 PublicSubnet 資源內部新增一個 Instance (執行個體) 資源類型，並將其重新命名為 WebServerInstance。

    執行個體是一種虛擬運算環境，您可在其中託管基本網站。這類似於子網路和 VPC 的運作方式；當您將執行個體新增至子網路時，該執行個體會自動與子網路建立關聯。

  9. 在 VPC 內新增SecurityGroup資源類型並重新命名。WebServerSecurityGroup

    安全群組是一種虛擬防火牆，可控制 Web 伺服器執行個體的傳入和傳出流量。VPC 中的執行個體也需要安全群組。我們需要將 Web 伺服器執行個體與此安全群組建立關聯，並於稍後指定執行個體的屬性時進行此作業。

  10. 在 VPC 之外的任何位置新增InternetGateway資源類型並重新命名。InternetGateway

    網際網路閘道可讓 VPC 內部的執行個體和網際網路進行通訊。如果沒有網際網路閘道，就沒有人可以存取您的網站。

    雖然您可以將網際網路閘道拖曳到 VPC 內部，但這並不會與 VPC 建立關聯。網際網路閘道不會遵循容器模型；反之，您必須將連線從網際網路閘道拖曳到 VPC，如下一個步驟中所述。

  11. 建立 InternetGateway 資源和 VPC 資源之間的連線。

    1. 在 InternetGateway 資源上，將滑鼠的游標移至網際網路閘道附件 (AWS::EC2::VPCGatewayAttachment) 上。

    2. 將連線拖曳到 VPC。

      有效目標資源的邊界會變更顏色。在此案例中，VPC 是唯一有效的目標資源。此連線會建立附件資源，以建立網際網路閘道與 VPC 的關聯。

  12. 接著，我們需要新增路由表和路由來指定如何引導子網路的網路流量。在 VPC RouteTable內添加一個並重命名。PublicRouteTable

    這會將新的路由表與 VPC 建立關聯。

  13. 若要將路由規則新增至路由表，請在 PublicRouteTable 資源內部新增一個 Route (路由) 資源類型，並將其重新命名為 PublicRoute。

    我們會使用該路由來指定要將流量引導到何處。

  14. 針對公有路由，我們要讓網際網路閘道成為目的地目標。使用 GatewayId 建立從 PublicRoute 資源到網際網路閘道的連線，其方式與建立網際網路閘道和 VPC 之間的連線類似。

    CloudFormation 在將網際網路閘道與虛擬私人 VPC 建立關聯之前，無法將路由與網際網路閘道相關聯。這表示我們需要為網際網路閘道 VPC 附件建立明確的相依性，如下一個步驟中所述。如需詳細資訊，請參閱 DependsOn 屬性。

  15. 建立 PublicRoute 資源和網際網路閘道 VPC 附件之間的明確相依性。

    1. 在PublicRoute資源上，將游標暫留在DependsOn點上。

    2. 將連線拖曳到網際網路閘道 VPC 附件 (AWS::EC2::VPCGatewayAttachment)。

      AWS CloudFormation 設計工具會使用 DependsOn 連線來建立相依性 (DependsOn 屬性)，在相依性中，原始資源需仰賴目標資源。在此案例中，AWS CloudFormation 設計工具會新增 DependsOn 屬性至 PublicRoute 資源，並將閘道 VPC 附件指定為相依性。

  16. 建立 WebServerInstance 資源到 PublicRoute 資源的另一個相依性。

    WebServerInstance 資源會仰賴公有路由以將流量路由到網際網路。如果沒有公開路由，執行個體就無法傳送訊號 (使用 cfn-signal 協助程式指令碼)，以便在執行個體設定和應用程式部署完成 CloudFormation 時通知。

  17. 將連線從 PublicRouteTable 資源拖曳至 PublicSubnet 資源，以建立路由表和子網路之間的關聯。

    現在，公有子網路會使用公有路由表來引導流量。

  18. 若要本機儲存範本，請使用 AWS CloudFormation 設計工具工具列中的 File (檔案) 選單 (檔案圖示)

    AWS CloudFormation 設計工具會將您的範本儲存到硬碟中。您可以稍後使用範本來建立堆疊。建議您定期儲存範本，以免遺失變更。

在此步驟中，我們會將七種資源新增到您的範本，並以易記的名稱重新命名它們的邏輯 ID。此外，我們已為大部分的資源建立視覺化連線，以建立關聯性和相依性。不過，我們還必須多建立幾個連線 (例如將執行個體與安全群組建立關聯)，並指定每個資源的屬性，之後才可以使用此範本來建立堆疊。在接下來的步驟中，我們將逐步解說如何使用 AWS CloudFormation 設計工具整合式編輯器來修改範本的其他元件，例如輸入參數。

步驟 2：新增範本參數、映射和輸出

在指定資源屬性之前，我們需要新增其他範本元件以在多個環境中重複使用範本。在此步驟中，我們會使用 AWS CloudFormation 設計工具整合式編輯器新增參數、映射和輸出。然後，我們可以在指定資源屬性時參閱這些參數和映射。此演練提供範例 JSON 和 YAML；您可以將其複製並貼入 整合式編輯器 中。

新增參數

您在建立堆疊時指定的輸入值即為參數。參數對傳遞值來說非常實用，可讓您不需要將值寫死在範本中。例如，您不需要在範本中寫死您的 Web 伺服器執行個體類型；反之，您可以在建立堆疊時使用參數來指定執行個體類型。這樣一來，您即可使用相同的範本，建立多個含不同執行個體類型的 Web 伺服器。如需詳細資訊，請參閱 參數。

  1. 按一下 AWS CloudFormation 設計工具畫布上的開放區域。

    整合式編輯器 會根據您所選取的項目顯示範本層級或資源層級的元件，以供您編輯。若是範本層級，您可以編輯範本的 Resources (資源) 區段以外的所有其他區段，例如範本參數、映射和輸出。在資源層級，您可以編輯資源屬性 (resource property) 和屬性 (attribute)。

    按一下畫布的開放區域可讓您編輯範本層級的元件。若要編輯資源層級的元件，請選取資源。

  2. 在整合式編輯器窗格中，選擇 Components (元件) 檢視中的 Parameters (參數) 索引標籤。

新增映射

映射是指與名稱值對相關聯的金鑰組。若要依據輸入參數值來指定值，這就非常實用。在這個演練中，我們會依據執行個體類型和您建立堆疊的所在區域，使用映射來指定 EC2 執行個體的 AMI ID。如需詳細資訊，請參閱 映射項目。

  1. 在整合式編輯器窗格中，選擇 Mappings (映射) 標籤。

輸出宣告您希望可用於 describe stacks API 呼叫或透過 CloudFormation 主控台堆疊 [輸出] 索引標籤的值。在這個演練中，我們會輸出網站的 URL，以讓您在網站建立之後輕鬆檢視。如需詳細資訊，請參閱 輸出。

  1. 在整合式編輯器窗格中，選取 Outputs (輸出) 標籤。

  2. 複製下列 JSON 輸出並貼入 整合式編輯器。

    輸出會使用 Fn::GetAtt 內部函數來取得 Web 伺服器執行個體的公有 IP。

    {
  "Outputs": {
    "URL": {
      "Value": {
        "Fn::Join": [
          "",
          [
            "http://",
            {
              "Fn::GetAtt": [
                "WebServerInstance",
                "PublicIp"
              ]
            }
          ]
        ]
      },
      "Description": "Newly created application URL"
    }
  }
}

    Outputs:
  URL:
    Value: !Join 
      - ''
      - - 'http://'

          - WebServerInstance
          - PublicIp
    Description: Newly created application URL

步驟 3：指定資源屬性

許多資源都具備必要屬性以定義組態或設定，例如 Web 伺服器要使用哪種執行個體類型。和上一個步驟類似，我們會使用 AWS CloudFormation 設計工具整合式編輯器來指定資源屬性。我們會提供範例 JSON 和 YAML；您可以將其複製並貼入 整合式編輯器 中。

指定資源屬性

  1. 在 AWS CloudFormation 設計工具畫布中，選擇 VPC 資源。

    整合式編輯器 會顯示資源層級的元件 (例如資源屬性)，以供您編輯。

  2. 在整合式編輯器窗格中，選擇 Properties (屬性) 標籤。

  3. 複製下列 JSON 程式碼片段並貼入整合式編輯器的 Properties (屬性) 大括號 ({}) 之間。

    此程式碼片段會指定 DNS 設定和 VPC 的 CIDR 區塊。

            "EnableDnsSupport": "true",
        "EnableDnsHostnames": "true",
        "CidrBlock": "10.0.0.0/16"

    若是 YAML，請在 Properties: 之後輸入新的一行，並貼上下列程式碼片段。

          EnableDnsSupport: 'true'
      EnableDnsHostnames: 'true'
      CidrBlock: 10.0.0.0/16
    注意

    為了確保效率，我們提供 JSON 和 YAML 程式碼片段，供您複製貼上。不過，請注意，編輯器都有一個自動完成功能，可讓您手動指定每個屬性。如需詳細資訊，請參閱 整合式 JSON 及 YAML 編輯器。

  4. 針對下列資源重複這個程序：

    PublicSubnet

    在 VPC ID 屬性之後，新增下列 CIDR 區塊屬性。AWS CloudFormation當您將子網路拖曳至 VPC 內部時，設計工具即會自動新增 VPC ID 屬性。

    注意

    您會看到其他幾個由 AWS CloudFormation 設計工具自動為您建立的關聯。您只要新增新的屬性 (粗體) 即可。

    JSON

            "VpcId": {
          "Ref": "VPC"
        },
        "CidrBlock": "10.0.0.0/24"

    YAML

          VpcId: !Ref VPC
      CidrBlock: 10.0.0.0/24
    PublicRoute

    新增下列目的地 CIDR 區塊屬性，其會將所有流量引導至網際網路閘道：

    JSON

            "DestinationCidrBlock": "0.0.0.0/0",
        "RouteTableId": {
          "Ref": "PublicRouteTable"
        },        
        "GatewayId": {
          "Ref": "InternetGateway"
        }

    YAML

          DestinationCidrBlock: 0.0.0.0/0
      RouteTableId: !Ref PublicRouteTable
      GatewayId: !Ref InternetGateway
    WebServerSecurityGroup

    新增下列傳入規則，判斷哪些流量可以到達 Web 伺服器執行個體。這些規則允許所有 HTTP 和特定 SSH 流量；您可在建立堆疊時以參數值的形式來指定這些規則。

    JSON

            "VpcId": {
          "Ref": "VPC"
        },
        "GroupDescription" : "Allow access from HTTP and SSH traffic",
        "SecurityGroupIngress": [
          {
            "IpProtocol": "tcp",
            "FromPort": 80,
            "ToPort": 80,
            "CidrIp": "0.0.0.0/0"
          },
          {
            "IpProtocol": "tcp",
            "FromPort": 22,
            "ToPort": 22,
            "CidrIp": {
              "Ref": "SSHLocation"
            }
          }
        ]

    YAML

          VpcId: !Ref VPC
      GroupDescription: Allow access from HTTP and SSH traffic
      SecurityGroupIngress:
        - IpProtocol: tcp
          FromPort: 80
          ToPort: 80
          CidrIp: 0.0.0.0/0
        - IpProtocol: tcp
          FromPort: 22
          ToPort: 22
          CidrIp: !Ref SSHLocation
    WebServerInstance

    您需要為 Web 伺服器執行個體指定多個屬性，因此我們僅反白幾個項目以作示範用途。InstanceType 和 ImageId 屬性會使用我們在上一節中指定的參數和映射值。在建立堆疊時，您可以將執行個體類型指定為參數值。ImageId 值為對應項目，其會以您的堆疊區域及您指定的執行個體類型為依據。

    NetworkInterfaces 屬性可指定 Web 伺服器執行個體的網路設定。此屬性可讓我們為安全群組、子網路與執行個體建立關聯。雖然 AWS CloudFormation 設計工具是使用 SubnetId 屬性來為子網路與執行個體建立關聯，但我們需要使用 NetworkInterfaces 屬性，因為這是將公有 IP 提供給 Web 伺服器的唯一方式。此外，當您指定 NetworkInterfaces 屬性時，您必須在該屬性內指定子網路和安全群組。

    在 UserData 屬性中，我們會指定要在執行個體啟動與執行之後執行的組態指令碼。執行個體的中繼資料定義所有組態資訊；我們會在下一個步驟中新增此中繼資料。

    在 Messages (訊息) 窗格中檢視並修復錯誤，接著再次驗證範本。如果您沒有看到錯誤，表示該範本的語法有效。

  7. 儲存完成的範本，以保留您做的所有變更。

您現在有了一個完整的 CloudFormation 範本，可用來在 VPC 中建立基本 Web 伺服器。為了建立範本，我們已先使用 AWS CloudFormation 設計工具畫布窗格來新增並連線範本資源。接著，我們使用 整合式編輯器 來新增其他範本元件，並指定資源屬性。在接下來的步驟中，我們將使用此範本來建立堆疊。

步驟 4：佈建資源

若要建立堆疊，您可以從 AWS CloudFormation Designer 啟動「 CloudFormation 建立堆疊精靈」。我們將使用在先前步驟中建立的範本來建立 CloudFormation 堆疊。 CloudFormation 佈建所有資源後，您將擁有一個基本的網站啟動並運行。

建立堆疊

  1. 在 AWS CloudFormation 設計工具工具列上，選擇 Create Stack (建立堆疊) (雲端圖示)。

    AWS CloudFormation設計師將開啟的範本儲存在 S3 儲存貯體中，然後啟動 CloudFormation 建立堆疊精靈。 CloudFormation 每當您上傳範本時，都會使用所建立的相同 S3 儲存貯體。

  2. CloudFormation 會自動填入範本 URL；選擇「下一步」。

  3. 在 Specify Details (指定詳細資訊) 區段的 Stack name (堆疊名稱) 欄位中，輸入堆疊名稱。在此範例中，使用 BasicWebServerStack。

  4. 在「參數」區段的KeyName欄位中，輸入您要建立堆疊的相同區域中有效 Amazon EC2 key pair 的名稱。

  5. 保留其他預設參數值，然後選擇 Next (下一步)。

  6. 在此逐步解說中，您不需要新增標籤或指定進階設定，因此請選擇 Next (下一步)。

  7. 確認堆疊名稱和 Amazon EC2 金鑰對名稱是否正確，然後選擇 Create (建立)。

AWS CloudFormation 可能需要幾分鐘的時間來建立您的堆疊。若要監控進度，請檢視堆疊事件。如需如何檢視堆疊事件的詳細資訊，請參閱 檢視 AWS CloudFormation 堆疊資料和資源 AWS Management Console。建立堆疊之後，請檢視堆疊輸出並前往範例網站的 URL，以驗證網站是否正在執行。如需詳細資訊，請參閱 檢視 AWS CloudFormation 堆疊資料和資源 AWS Management Console。
