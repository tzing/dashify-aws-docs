開始使用

使用正確的範本，您可以立即部署應用程式所需的所有 AWS 資源。在本節中，您將檢查一個模板，該模板將為 WordPress 博客聲明資源，將 WordPress 博客創建為堆棧，監視堆棧創建過程，檢查堆棧上的資源，然後刪除堆棧。您可以使用 AWS Management Console 來完成這些工作。

步驟 1：選擇範本

首先，您需要一個範本，以指定堆疊中所需的資源。在此步驟中，您會使用已就緒的範例範本。範例範本會建立一個基本 WordPress 部落格，該部落格使用單一 Amazon EC2 執行個體搭配本機 MySQL 資料庫進行儲存。範本也會建立 Amazon EC2 安全群組，以控制 Amazon EC2 執行個體的防火牆設定。

重要

AWS CloudFormation 是免費的，但 CloudFormation 創建的 AWS 資源是活的（而不是在沙箱中運行）。您將需要針對這些資源支付標準使用費，直到您在此教學的最後一項任務中終止這些資源為止。總計費用會很少。如需如何將任何費用降到最少的資訊，請前往 http://aws.amazon.com/free/。

檢視範本

  • 您可以檢視 JSON 或 YAML WordPress 範例範本。您不需要將它下載，因為您將在本指南稍後使用範本 URL。如需範本格式的詳細資訊，請參閱AWS CloudFormation 範本格式。

範本是 JSON 或 YAML 文字檔案，其中包含您要在堆疊中建立之 AWS 資源的相關設定資訊。在本演練中，範例範本包含六個最上層區段：AWSTemplateFormatVersion、Description、Parameters、Mappings、Resources 和 Outputs；不過，只需要 Resources 區段。

[資源] 區段包含您要使用範本建立之 AWS 資源的定義。每個資源會分別列出，並指定建立該特定資源所需的屬性。下列資源宣告是 EC2 執行個體的組態，在此範例中具有邏輯名稱 WebServer：

範例 JSON
"Resources" : {
  ...

    "Type" : "AWS::EC2::Instance",
    "Properties": {
      "ImageId" : { "Fn::FindInMap" : [ "AWSRegionArch2AMI", { "Ref" : "AWS::Region" },

      "InstanceType"   : { "Ref" : "InstanceType" },
      "SecurityGroups" : [ {"Ref" : "WebServerSecurityGroup"} ],
      "KeyName"        : { "Ref" : "KeyName" },
      "UserData" : { "Fn::Base64" : { "Fn::Join" : ["", [
                     "#!/bin/bash -xe\n",
                     "yum update -y aws-cfn-bootstrap\n",

                     "/opt/aws/bin/cfn-init -v ",
                     "         --stack ", { "Ref" : "AWS::StackName" },
                     "         --resource WebServer ",
                     "         --configsets wordpress_install ",
                     "         --region ", { "Ref" : "AWS::Region" }, "\n",

                     "/opt/aws/bin/cfn-signal -e $? ",
                     "         --stack ", { "Ref" : "AWS::StackName" },
                     "         --resource WebServer ",
                     "         --region ", { "Ref" : "AWS::Region" }, "\n"
      ]]}}
    },
    ...
  },
  ...

    "Properties" : {

Resources: 
  ...    
  WebServer:
    Type: AWS::EC2::Instance
    Properties:
      ImageId: !FindInMap [AWSRegionArch2AMI, !Ref 'AWS::Region', !FindInMap [AWSInstanceType2Arch, !Ref InstanceType, Arch]]      
      InstanceType:
        Ref: InstanceType
      KeyName:
        Ref: KeyName
      SecurityGroups:
      - Ref: WebServerSecurityGroup
      UserData:

           #!/bin/bash -xe

           /opt/aws/bin/cfn-signal -e $? --stack ${AWS::StackId} --resource WebServer --region ${AWS::Region}

  ...

      SecurityGroupIngress:

        FromPort: 80
        IpProtocol: tcp
        ToPort: 80
      - CidrIp: !Ref SSHLocation
        FromPort: 22
        IpProtocol: tcp
        ToPort: 22

如果您之前已建立 EC2 執行個體，您可以辨識決定執行個體組態的屬性，例如 ImageId、InstanceType 和 KeyName。資源宣告是一次指定所有組態設定的有效方式。當您將資源宣告放在範本中時，您可以透過使用範本建立堆疊，來建立及設定所有宣告的資源。建立使用相同範本的新堆疊，以啟動資源的相同組態。

資源宣告是以指定資源邏輯名稱的字串開頭。如您所見，邏輯名稱可用來參考範本內的資源。

您可以使用 Parameters 區段，宣告可在建立堆疊時傳遞至範本的值。參數是指定敏感資訊 (例如使用者名稱和密碼) 的有效方式，您不想要將此資訊存放在範本本身中。它也可讓您指定可能對您要部署之特定應用程式或組態唯一的資訊，例如網域名稱或執行個體類型。當您在本節稍後建立 WordPress 堆疊時，您會看到範本中宣告的參數集會出現在 [建立堆疊] 精靈的 [指定詳細資料] 頁面上，您可以在其中指定參數，然後再建立堆疊。

在 WebServer 資源宣告中，您會看到使用 KeyName 參數指定的 KeyName 屬性：

範例 JSON
"WebServer" : {
  "Type": "AWS::EC2::Instance",
  "Properties": {
    "KeyName" : { "Ref" : "KeyName" },
    ...
  }
},
範例 YAML
WebServer:
  Type: AWS::EC2::Instance
  Properties:
    KeyName:
      Ref: KeyName
    ...

大括弧包含 Ref 函數的呼叫，並以 KeyName 作為其輸入。Ref 函數會傳回其所參考的物件值。在本例中，Ref 函數會將 KeyName 屬性設為建立堆疊時為 KeyName 指定的值。

Ref 函數也可以將資源的屬性設為另一個資源的值。例如，資源宣告 WebServer 包含下列屬性宣告：

範例 JSON
"WebServer" : {
  "Type": "AWS::EC2::Instance",
  "Properties": {
    ...
    "SecurityGroups" : [ {"Ref" : "WebServerSecurityGroup"} ],
    ...
  }
},
範例 YAML
WebServer:
  Type: AWS::EC2::Instance
  Properties:
    SecurityGroups:
    - Ref: WebServerSecurityGroup
    ...

SecurityGroups 屬性接受 EC2 安全群組清單。Ref 函數具有 WebServerSecurityGroup 的輸入 (也就是範本中安全群組的邏輯名稱)，並會將 WebServerSecurityGroup 的名稱新增至 SecurityGroups 屬性。

在範本中，您也會找到 Mappings 區段。您可以使用映射來宣告條件值，其評估方式類似於查詢表陳述式。範本使用映射為區域選取正確的 Amazon Machine Image (AMI)，並為執行個體類型選取架構類型。輸出定義在堆疊建立後由aws cloudformation describe-stacks命令和 CloudFormation 主控台「輸出」索引標籤中傳回的自訂值。您可以使用輸出值從堆疊中的資源傳回資訊，例如在範本中建立之網站的 URL。我們將在了解範本的基本知識中進一步說明映射、輸出及其他範本的相關資訊。

步驟 2：確定堆疊所需的任何項目都已就緒

從範本建立堆疊之前，您必須確定範本所需的所有相依資源可供使用。模板可以使用或引用模板本身中聲明的現有資 AWS 源和資源。 CloudFormation 負責檢查模板中資源的引用，並檢查對現有資源的引用，以確保它們存在於您正在創建堆棧的區域中。如果您的範本參考了不存在的相依資源，堆疊建立會失敗。

範例 WordPress 範本包含一個輸入參數KeyName，該參數指定用於範本中宣告之 Amazon EC2 執行個體的 key pair。範本需要從範本建立堆疊的使用者為 KeyName 參數提供有效的 Amazon EC2 金鑰對。如果您提供有效的金鑰對名稱，堆疊會成功建立。如果您未提供有效的金鑰對名稱，堆疊會復原。

確定您擁有有效的 Amazon EC2 金鑰對並記錄金鑰對名稱，再建立堆疊。

若要查看您的金鑰對，請開啟 Amazon EC2 主控台，然後選擇導覽窗格中的 Key Pairs (金鑰對)。

注意

如果您沒有 Amazon EC2 金鑰對，您必須在建立堆疊的相同區域中建立金鑰對。如需有關建立金鑰對的資訊，請參閱《Amazon EC2 Linux 執行個體使用者指南》中的取得 SSH 金鑰對。

現在您有一個有效的 key pair，讓我們使用 WordPress 模板創建一個堆棧。

步驟 3：建立堆疊

您將根據前面討論的 WordPress-1.0.0 文件創建堆棧。範本包含數個 AWS 資源，例如 EC2 執行個體。

建立 WordPress堆疊的步驟

  1. 請登入 AWS Management Console 並開啟 AWS CloudFormation 主控台，網址為 https://console.aws.amazon.com/cloudformation。

  2. 選擇 Create Stack (建立堆疊)。

  3. 在 [指定範本] 區段中，選取 Amazon S3 範本 URL 以輸入或貼上範例 WordPress範本的 URL，然後選擇 [下一步]：

    https://s3.us-west-2.amazonaws.com/cloudformation-templates-us-west-2/WordPress_Single_Instance.template

    注意

    AWS CloudFormation 建立堆疊的使用者必須可存取儲存在 S3 儲存貯體中的範本，而且必須位於與正在建立的堆疊位於相同的區域。

  4. 在指定堆疊詳細資訊區段的堆疊名稱 欄位中，輸入一個名稱。在此範例中，使用 MyWPTestStack。堆疊不可含有空格。

  5. 在「參數」段落中，您必須為所有沒有預設值的參數提供值，包括 DBuser、DBPassword、DB RootPassword 和. KeyName 在KeyName欄位中，輸入您要建立堆疊的相同區域中有效 Amazon EC2 配對的名稱。

  6. 選擇下一步。

  7. 在此案例中，我們不會新增任何標籤。選擇下一步。標籤 (也就是鍵/值對) 可以協助您識別堆疊。如需詳細資訊，請參閱將標籤新增至您的 CloudFormation 堆疊。

  8. 檢閱堆疊的資訊。當您滿意設定時，請選擇 Submit (提交)。

您的堆疊可能需要幾分鐘時間來建立，但您可能並不想只是坐著枯等。如果您和我們一樣，您會想要知道堆疊建立的進度。

步驟 4：監控堆疊建立的進度

完成 [建立堆疊] 精靈之後，就會 CloudFormation 開始建立範本中指定的資源。您的新堆疊會出現在CloudFormation主機上方的清單中。MyWPTestStack其狀態應該是 CREATE_IN_PROGRESS。您可以透過檢視其事件來查看堆疊的詳細狀態。

檢視堆疊的事件

  1. 在主 CloudFormation 控台上，選取清單MyWPTestStack中的堆疊。

  2. 在堆疊詳細資訊窗格中，選擇 Events (事件) 標籤。

    主控台會每隔 60 秒自動重新整理事件清單，以包含最新的事件。

Events (事件) 標籤會顯示堆疊建立過程中的每個主要步驟，並依每個事件的時間排序，最新的事件放在最上方。

第一個事件 (位於事件清單的底部) 是開始堆疊建立程序：

2013-04-24 18:54 UTC-7 CREATE_IN_PROGRESS AWS::CloudFormation::Stack MyWPTestStack User initiated

接下來是標記每個資源建立開始和完成的事件。例如，建立 EC2 執行個體會產生下列項目：

2013-04-24 18:59 UTC-7 CREATE_COMPLETE AWS::EC2::Instance...

2013-04-24 18:54 UTC-7 CREATE_IN_PROGRESS AWS::EC2::Instance...

CloudFormation 報告CREATE_IN_PROGRESS事件已開始建立資源時，便會記錄事件。當資源成功建立時，會記錄 CREATE_COMPLETE 事件。

CloudFormation 成功創建堆棧後，您將在事件選項卡的頂部看到以下事件：

2013-04-24 19:17 UTC-7 CREATE_COMPLETE AWS::CloudFormation::Stack MyWPTestStack

如果 CloudFormation 無法建立資源，它會報告CREATE_FAILED事件，並依預設回復堆疊，並刪除已建立的任何資源。Status Reason (狀態原因) 欄會顯示導致失敗的問題。

步驟 5：使用您的堆疊資源

當堆疊 MyWPTestStack 的狀態為 CREATE_COMPLETE 時，即表示 CloudFormation 已完成建立堆疊，而且您可以開始使用其資源。

範例 WordPress 堆疊會建立 WordPress 網站。您可以執行安裝指令碼來繼續 WordPress 安裝程 WordPress 式。

完成安 WordPress裝

  1. 在 Outputs (輸出) 標籤的 WebsiteURL 列中，選擇 Value (值) 欄中的連結。

    WebsiteURL輸出值是您使用堆疊建立之 WordPress 網站之安裝指令碼的 URL。

  2. 在 WordPress 安裝的網頁上，遵循螢幕上的指示完成 WordPress 安裝。如需有關安裝的詳細資訊 WordPress，請參閱 https://wordpress.org/support/article/how-to-install-wordpress/。

    完成安裝並登錄後，您將被定向到儀表板，您可以在其中為 WordPress 博客設置其他選項。然後，您可以開始為使用 CloudFormation 範本成功建立的部落格撰寫文章。

步驟 6：清除

您已完成 CloudFormation 入門工作。若要確定您不會針對任何不想要的服務支付費用，您可以透過刪除堆疊及其資源來清除這些服務。

刪除堆疊及其資源

  1. 從主 CloudFormation 控台選取MyWPTestStack堆疊。

  2. 選擇 Delete Stack (刪除堆疊)。

  3. 在出現的確認訊息中，選擇 Yes, Delete (是，刪除)。

MyWPTestStack 的狀態會變更為 DELETE_IN_PROGRESS。如同監控堆疊的建立，您可以透過使用 Event (事件) 標籤來監控其刪除。當 CloudFormation 完成堆棧的刪除，它從列表中刪除堆棧。

恭喜您！您已成功選擇範本、建立堆疊、檢視和使用其資源，並刪除堆疊及其資源。不僅如此，您還可以使用 CloudFormation 模板設置 WordPress 博客。您可以在 AWS CloudFormation 範例範本程式庫中尋找其他範本。
