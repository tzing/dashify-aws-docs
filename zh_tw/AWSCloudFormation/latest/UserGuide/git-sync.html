使用 AWS CloudFormation Git 同步

使用 Git 同步，您可以使用原始檔控制來管理您的 CloudFormation 堆疊。您可以設定 AWS CloudFormation 監視 Git 儲存庫來執行此操作。系統會監視儲存庫是否有對兩個檔案的變更：

  • 定義堆疊的 CloudFormation 範本檔

  • 包含設定堆疊之參數的堆疊部署檔案

當您將變更交付至範本或部署檔案時， CloudFormation 會自動更新堆疊。如此一來，您就可以使用提取要求和版本追蹤，從集中位置設定、部署和更新 CloudFormation 堆疊。

Git sync 提供了一個控制台界面，您可以使用它來鏈接到存儲庫，生成堆棧部署文件，更新 CloudFormation 模板以及提交提取請求到您的存儲庫。Git 同步也提供了一個狀態儀表板，您可以用來監視、編輯和疑難排解作用中的 Git 同步堆疊部署。Git 同步是通過CloudFormation 控制台，當你創建一個堆棧訪問。您也可 Git 使用AWS CodeStar 連線. 如需詳細資訊，請參閱開發人員工具主控台使用者指南中的使用連結儲存庫的同步設定。

Git 同步支持 GitHub，GitHub GitLab企業和比特桶存儲庫。

注意

Git 同步可在下列區域使用：美國東部 (維吉尼亞北部)、美國東部 (俄亥俄)、美國西部 (加利佛尼亞北部)、美國西部 (奧勒岡)、加拿大 (中部)、亞太區域 (孟買)、亞太區域 (東京)、亞太區域 (首爾)、亞太區域 (新加坡)、亞太區域 (雪梨)、歐洲 (愛爾蘭)、、歐洲 (倫敦)、歐洲 (巴黎)、歐洲 (斯德哥爾摩)、歐洲 (法蘭克福)、歐洲 (米蘭) 和南美洲 (聖保羅)。
