建立變更集

若要建立執行中堆疊的變更集，則可以提供修改後的範本、新的輸入參數值，或同時提供兩者，藉此提交要進行的變更。CloudFormation 會比較堆疊與提交的變更，進而產生變更集。

您可以在建立變更集之前或在建立變更集期間修改範本。

Create a change set for nested stacks (console)
建立巢狀堆疊的變更集 (主控台)

  1. 在 AWS CloudFormation 主控台的 Stacks (堆疊) 中，選擇您要建立變更集的執行中堆疊。

  2. 在 stack details (堆疊詳細資訊) 窗格中，選擇 Stack actions (堆疊動作)，然後選擇 Create change set for current stack (建立目前堆疊的變更集)。

  3. 在 Create change set for stack-name (建立堆疊名稱的變更集) 頁面上，執行以下其中一項操作，修改輸入參數值、指定更新範本的位置或修改範本：

    任務 動作
    修改輸入參數值 選擇 Use current template (使用目前的範本)，然後選擇 Next (下一步) 繼續輸入或修改輸入參數值。
    指定更新的範本位置

    如已修改範本，請選擇 Replace current template (取代目前的範本)，然後執行以下其中一項操作：

    • 如需存放在 Amazon S3 儲存貯體的範本，請選擇 Amazon S3 URL。輸入或貼上該範本的 URL，然後選擇 Next (下一步)。

      如果您的範本位於已啟用版本控制的儲存貯體中，則您可以指定範本的特定版本，例如 https://s3.amazonaws.com/templates/myTemplate.template?versionId=123ab1cdeKdOW5IH4GAcYbEngcpTJTDW。如需詳細資訊，請參閱《Amazon Simple Storage Service 使用者指南》中的管理已啟用版本控制之儲存貯體中的物件。

    • 針對存放在本機電腦上的範本，選擇 Upload a template file (上傳範本檔案)。選擇 Choose File (選擇檔案) 導覽至該檔案並選取它，然後選擇 Next (下一步)。
    修改範本 如尚未修改範本，請選擇 Edit template in designer (使用設計工具編輯範本)，然後選擇 View in designer (使用設計工具檢視)。您會重新導向到 AWS CloudFormation 設計工具。範本修改完成後，請選擇 返回 Create change set for stack-name (建立堆疊名稱的變更集) 頁面，然後選擇 Next (下一步)。

  4. 如果您的範本包含參數，請在 Specify stack details (指定堆疊詳細資訊) 頁面上，輸入或修改合適的輸入參數值，然後選擇 Next (下一步)。

    若重複使用堆疊的範本，CloudFormation 會將堆疊中目前的值填入各參數 (不包含 NoEcho 屬性所宣告的參數)。選取 Use existing value (使用現有的值)，即可使用現有的參數值。

    如需使用 NoEcho 遮罩敏感資訊，以及使用動態參數管理密碼的詳細資訊，請參閱請勿在您的範本中內嵌憑證最佳實務。

  5. 在 Configure stack options (設定堆疊選項) 頁面上，更新堆疊標籤、IAM 服務角色、堆疊原則、復原組態、Amazon SNS 通知主題 (如果適用) 或變更集，然後選擇 Next (下一步)。

    注意

    巢狀堆疊的變更集預設為 Enabled (啟用)，這將會為範本中指定的所有巢狀堆疊建立變更集。若要取得有關巢狀堆疊變更集的更多資訊，請參閱巢狀堆疊變更集。

  6. 在 Review stack-name (檢閱堆疊名稱) 頁面上，檢閱此變更集的變更。

    如果範本包括 AWS Identity and Access Management (IAM) 資源，選取 I acknowledge that CloudFormation might create IAM resources (我確認 CloudFormation 可能會建立 IAM 資源)。IAM 資源可修改 AWS 帳戶中的許可；請檢閱這些資源，確保系統僅能執行所需的動作。如需更多詳細資訊，請參閱 控制存取 AWS Identity and Access Management。

  7. 選擇 Create change set (建立變更集)。指定變更集名稱以及選擇性指定變更集描述以識別其用途。然後，選擇 Create change set (建立變更集)。

    您會被重新導向到變更集詳細資訊頁面的 Changes (變更) 標籤。CloudFormation 產生變更集時，該變更集會顯示 CREATE_IN_PROGRESS 狀態。變更集建立完成後，CloudFormation 即會將狀態設為 CREATE_COMPLETE。在 Changes (變更) 區段中，CloudFormation 會列出將對堆疊進行的所有變更。如需更多詳細資訊，請參閱 檢視變更集。

    若 CloudFormation 無法建立變更集 (回報 FAILED 狀態)，請修復 Status (狀態) 欄位中顯示的錯誤，接著重新建立變更集。

建立變更集 (AWS CLI)

  • 執行 aws cloudformation create-change-set 命令。

    您能夠以命令選項的形式來提交變更，亦能指定新的參數值、修改後的範本，或同時指定兩者。例如，以下命令將為 SampleChangeSet 堆疊建立 SampleStack 變更集。該變更集會使用目前的堆疊範本，但 Purpose 參數將具備不同值：

    aws cloudformation create-change-set --stack-name arn:aws:cloudformation:us-east-1:123456789012:stack/SampleStack/1a2345b6-0000-00a0-a123-00abc0abc000
--change-set-name SampleChangeSet --use-previous-template --parameters ParameterKey="InstanceType",UsePreviousValue=true ParameterKey="KeyPairName",UsePreviousValue=true ParameterKey="Purpose",ParameterValue="production"
