資源標籤

您可以使用 Resource Tags 屬性將標籤套用至資源，協助您識別和分類那些資源。您只能標記 AWS CloudFormation 支援標記的資源。有關可以使用哪些資源進行標記的詳細資訊 CloudFormation，請參閱中的個別資源AWS 資源和屬性類型參考。

注意

標記實作可能會根據資源而有所不同。例如，作為其標記配置的一部分，AWS::AutoScaling::AutoScalingGroup 提供額外、必要的 PropagateAtLaunch 屬性。

除了您定義的任何標籤之外，還 CloudFormation 會自動建立下列具有前置詞aws:的堆疊層級標籤：

  • aws:cloudformation:logical-id

  • aws:cloudformation:stack-id

  • aws:cloudformation:stack-name

前aws:綴保留供 AWS 使用。此字首不區分大小寫。如果您在 Key 或 Value 屬性中使用此字首，則無法更新或刪除標籤。具有此字首的標籤不會計入每個資源的標籤數。

堆疊層級標籤到資源的傳播 (包含自動建立的標籤) 可能因資源而異。例如，標籤不會傳播到從區塊型裝置映射建立的 Amazon EBS 磁碟區。

語法

JSON

{
  "Key" : String,
  "Value" : String
}

YAML

Key: String
Value: String

屬性

Key

標籤的鍵名稱。您可以指定長度為 1 到 128 個 Unicode 字元的值，並且不可在前面加上 aws:。您可以使用以下任何字元：Unicode 字母、數字、空格、_、.、:、/、=、+、@、- 和 " 的組合。

必要：是

類型：字串

Value

標籤的值。您可以指定長度為 1 到 256 個字元的值。您可以使用以下任何字元：Unicode 字母、數字、空格、_、.、/、=、+ 和 - 的組合。

必要：是

類型：字串

範例

此範例顯示 Tags 屬性。您可以在支援此屬性資源內的 Properties 區段指定此屬性。當資源建立時，便會加上您宣告的標籤。

JSON

"Tags" : [
   {
      "Key" : "keyname1",
      "Value" : "value1"
   },
   {
      "Key" : "keyname2",
      "Value" : "value2"
   }
]

YAML

Tags: 
  - Key: "keyname1"
    Value: "value1"
  - Key: "keyname2"
    Value: "value2"
