Amazon CloudWatch 日誌模板片段

Amazon CloudWatch 日誌可以監控來自 Amazon EC2 執行個體或其他來源的系統、應用程式和自訂日誌檔。您可以使用 AWS CloudFormation 佈建和管理日誌群組與指標篩選條件。如需開始使用 Amazon CloudWatch 日誌的詳細資訊，請參閱 Amazon CloudWatch 使用者指南中的監控系統、應用程式和自訂日誌檔。

將記錄檔從 Linux 執行個體傳送至 CloudWatch 記錄檔

以下範本描述 Web 伺服器及其自訂指標。來自 Web 伺服器日誌的日誌事件提供自訂指標的資料。若要將日誌事件傳送到自訂指標，該UserData欄位會在 Amazon EC2 執行個體上安裝 CloudWatch 日誌代理程式。代理程式的組態資訊，例如伺服器日誌檔的位置、日誌群組名稱和日誌串流名稱，皆定義於 /tmp/cwlogs/apacheaccess.conf 檔案。日誌串流是在 Web 伺服器開始將日誌事件傳送至 /var/log/httpd/access_log 檔案之後建立的。

注意

有關許可的說明：WebServerHost 執行個體會參考 LogRoleInstanceProfile 執行個體設定檔，而後者又參考 LogRole 角色。LogRole 指定 arn:aws:s3:::* 的 s3:GetObject 許可。

需要此權限，因為從本UserData節中的 Amazon S3 WebServerHost 下載 CloudWatch 日誌代理程式 (awslogs-agent-setup.py)。

這兩個指標篩選條件描述日誌資訊如何轉換成 CloudWatch 指標。404 指標會計算 404 事件的數量。大小指標會追蹤請求的大小。如果 2 分鐘內有兩個 404s 以上，或者平均請求大小超過 3500 KB 超過 10 分鐘，則兩個 CloudWatch 警報將發送通知。
