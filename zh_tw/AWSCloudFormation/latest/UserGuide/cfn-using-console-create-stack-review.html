在 AWS CloudFormation 主控台上檢閱您的堆疊

堆疊啟動前的最後一步，就是檢視堆疊建立期間輸入的值。

  1. 請詳閱 Review (檢閱) 頁面上的堆疊詳細資訊。

    啟動堆疊前如需要變更任何值，請選擇合適區段的 Edit (編輯) 返回有想要變更設定的頁面。

  2. 檢閱堆疊建立設定後，請選擇建立堆疊來啟動您的堆疊。

    注意

    此時，您也可以選擇建立新的變更集，而不是建立新的堆疊。若要這樣做，請按一下 Create change set (建立變更集) 而不是 Create stack (建立堆疊)。如需詳細資訊，請參閱 透過變更集建立堆疊

    CloudFormation 會顯示新堆疊 Stack details (堆疊詳細資訊) 頁面的 Events (事件) 窗格。您可以在這裡檢視堆疊的事件、資料或資源。CloudFormation 每分鐘會自動重新整理堆疊事件。此外，CloudFormation 會在新的堆疊事件發生時顯示 New events available (有新事件可用) 徽章，選擇重新整理圖示將這些事件載入清單。透過檢視堆疊建立事件，您即可了解建立堆疊前的事件順序，或者了解對堆疊進行除錯失敗前的事件順序。

    正在建立的堆疊會列在 Stacks (堆疊) 頁面中，狀態為 CREATE_IN_PROGRESS (正在建立)。

    堆疊建立成功後，其狀態將變更為 CREATE_COMPLETE。接著，您可選擇 Outputs (輸出) 標籤，檢視您的堆疊輸出 (如已在範本中定義)。

確定失敗的根本原因

如果您的堆疊無法建立，則 AWS CloudFormation 可協助您確定可能是堆疊故障之根本原因的事件。若要執行此操作，請在堆疊清單中選取故障堆疊，然後選取偵測根本原因。

CloudFormation 將對故障進行分析，並指出可能是故障根本原因的事件。
