使用 AWS CloudFormation 登錄

CloudFormation 登錄可讓您管理公用和私有擴充功能，例如資源、模組和掛接，可在 AWS 帳戶. 目前，您可以在 AWS 登錄中使用下列擴充功能類型：資源類型、模組和掛接。登錄可讓您更輕鬆地在 AWS CloudFormation 範本中探索和佈建擴充功能，方式與您使用 AWS-all 資源相同。

公有和私有擴充功能

擴充功能類型註冊為公有或私有。目前，登錄檔提供下列擴充功能類型：

  • 資源類型 — 使用中的堆疊來建模和佈建自訂邏輯作為資源 CloudFormation。

  • Modules (模組) – 採用透明、可管理且可重複的方式封裝包含在堆疊範本之間的資源組態。

  • 掛鉤 — 在佈建之前主動檢查資 AWS 源的配置。

公有擴充功能類型

公用擴充功能是公開發佈在登錄中供所有使用 CloudFormation 者使用的擴充功能。這包括由 AWS 和第三方擴充功能發佈者發佈的擴充功能。

有兩種公有擴充功能：

  • AWS 公開擴充功能 — 由發佈的擴充功能永遠 AWS 是公開的，且預設為啟用，因此您不需要採取任何動作，才能在帳戶中使用這些擴充功能。此外， AWS 控制擴充功能的版本控制，因此您將始終使用最新的可用版本。

  • 第三方公有擴充功能 – 這些是可供 AWS以外的發佈者普遍使用的擴充功能。

如需詳細資訊，請參閱使用公有擴充功能。

私有擴充功能類型

私有擴充功能是您明確啟用以用於 AWS 帳戶中的第三方擴充功能。

有兩種私有擴充功能：

  • 已啟用的私有擴充功能 – 您為帳戶和區域啟用的第三方擴充功能之本機複本。當您啟用第三方公開擴充功能時， CloudFormation會在帳戶登錄中建立該擴充功能的本機副本。

  • 已註冊的私有擴充功能 — 也可以啟用公用登 CloudFormation錄中未列出的私人擴充功能。這些可以是您自行建立的擴充功能，也可以是組織或其他第三方與您共用的擴充功能。若要在帳戶中使用這種私有擴充功能，您必須先註冊該擴充功能。註冊擴充功能會將其副本上傳到您帳戶中的註冊 CloudFormation 表，並啟用該擴充功能。

如需詳細資訊，請參閱使用私有擴充功能。

透過 CloudFormation登錄管理擴充功能

使用 CloudFormation 登錄管理您帳戶中的擴充功能，包括：

  • 檢視可用和已啟用擴充功能。

  • 註冊私有擴充功能。

  • 啟用公有擴充功能。

在 CloudFormation主控台中檢視擴充功能

  1. 請登入 AWS Management Console 並開啟 AWS CloudFormation 主控台，網址為 https://console.aws.amazon.com/cloudformation。

  2. 從功能CloudFormation窗格的 [登錄] 底下，選取您要檢視的擴充功能類別：

    • 公有擴充功能顯示帳戶中可用的公有擴充功能。

      1. 選擇擴充功能類型：Resource types (資源類型)、Modules (模組) 或 Hooks (掛鉤)。

      2. 選擇發佈者：AWS 或 Third party (第三方)。

      3. 使用 Filter (篩選條件) 選項進一步選取要檢視的擴充功能。

    • 已啟用的擴充功能顯示帳戶中已啟用的公有和私有擴充功能。

      1. 選擇擴充功能類型：Resource types (資源類型)、Modules (模組) 或 Hooks (掛鉤)。

      2. 使用 Filter (篩選條件) 下拉式選單進一步選取要檢視的擴充功能：

        • AWS— 列出由發行的副檔名 AWS。 AWS 發佈的擴充功能預設為已啟用。

        • 第三方 — 列出發佈商所提供的任何公開擴充功能 AWS ，而非您在此帳戶中啟用的擴充功能

        • Registered (已註冊) – 列出此帳戶中已啟用的任何私有擴充功能。

    • Publisher (發佈者) 顯示您使用此帳戶發佈的任何公有擴充功能。如需詳細資訊，請參閱 User Guide for Extension Development (《擴充功能開發使用者指南》) 中的 Publishing extensions (發佈擴充功能)。

  3. 搜尋或選取擴充功能名稱以檢視擴充功能詳細資訊。

記錄資源類型 AWS Config

您可以指定 AWS Config 自動追蹤您的私有資源類型，並將這些資源的變更記錄為組態項目。這可讓您檢視這些私有資源類型的組態歷史記錄，以及撰寫 AWS Config 規則 規則以驗證組態最佳作法。 AWS Config 鉤延伸是必需的。

若要 AWS Config 自動追蹤您的私有資源類型：

  • 通過管理資源 CloudFormation。這包括透過執行所有資源建立、更新和刪除作業 CloudFormation。

    注意

    如果您使用 IAM 角色執行堆疊操作，則該 IAM 角色必須具有呼叫下列 AWS Config 動作的權限：

    • PutResourceConfig

    • DeleteResourceConfig

  • 設定 AWS Config 以記錄所有資源類型。如需詳細資訊，請參閱《AWS Config 開發人員指南》中的記錄第三方資源的組態。

    注意

    AWS Config 不支持記錄包含定義為必需和只寫屬性的私有資源。

    根據設計，定義為唯寫的資源屬性不會在用來建立 AWS Config 組態項目的結構描述中傳回。因此，如果包含定義為唯寫和必要的屬性，則會導致建立組態項目失敗，因為將不存在必要屬性。若要檢視將用來建立組態項目的結構描述，您可以檢視DescribeType動作的schema內容。

如需有關組態項目的詳細資訊，請參閱《AWS Config 開發人員指南》中的組態項目。

防止在組態項目中記錄感應性屬性

您的資源類型可能包含您認為機密資訊 (例如密碼、秘密或其他機密資料) 的內容，意即您不想記錄為組態項目的一部分。若要防止將內容記錄在組態項目中，您可以在資源類型結構描述的 writeOnlyproperties 清單中包含該內容。使用者可指定列為 writeOnlyproperties 的資源屬性，但不會由 read 或 list 請求傳回。

若要取得更多資訊，請參閱《指CloudFormation 令行介面使用指南》中的〈資源提供者結構

預防混淆代理人

混淆代理人問題屬於安全性議題，其中沒有執行動作許可的實體可以強制具有更多許可的實體執行該動作。在中 AWS，跨服務模擬可能會導致混淆的副問題。在某個服務 (呼叫服務) 呼叫另一個服務 (被呼叫服務) 時，可能會發生跨服務模擬。可以操縱呼叫服務來使用其許可，以其不應有存取許可的方式對其他客戶的資源採取動作。為了預防這種情況， AWS 提供的工具可協助您保護所有服務的資料，而這些服務主體已獲得您帳戶中資源的存取權。

我們建議在資源策略中使用aws:SourceArn和aws:SourceAccount全域條件內容索引鍵，以限制將其他服務 AWS CloudFormation 提供給擴充功能的權限。如果 aws:SourceArn 值不包含帳戶 ID (例如 Amazon S3 儲存貯體 Amazon Resource Name (ARN))，則您必須使用這兩個全域條件內容索引鍵來限制許可。如果同時使用這兩個全域條件內容索引鍵，且 aws:SourceArn 值包含帳戶 ID，則在相同政策陳述式中使用 aws:SourceAccount 值和 aws:SourceArn 值中的帳戶時，必須使用相同的帳戶 ID。如果您想要僅允許一個資源與跨服務存取相關聯，則請使用 aws:SourceArn。如果您想要允許該帳戶中的任何資源與跨服務使用相關聯，請使用 aws:SourceAccount。

aws:SourceArn 的值必須使用擴充功能的 ARN。

防範混淆代理人問題的最有效方法是使用 aws:SourceArn 全域條件內容索引鍵，以及資源的完整 ARN。如果您不知道擴充功能的完整 ARN，或者如果您指定了多個擴充功能，請使用 aws:SourceArn 全域內容條件索引鍵搭配萬用字元 (*) 來表示 ARN 的未知部分。例如 arn:aws:cloudformation:*:123456789012:*。

注意

對於登錄服務 CloudFormation ，請呼叫 AWS Security Token Service (AWS STS) 以在您的帳戶中扮演角色。此角色在 RegisterType 操作中為 ExecutionRoleArn 設定，在LoggingConfig 操作中為 LogRoleArn 設定。

下列範例顯示如何在中使用aws:SourceArn和aws:SourceAccount全域條件前後關聯鍵字 AWS CloudFormation 來避免混淆的副問題。

{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": [
          "resources.cloudformation.amazonaws.com"
        ]
      },
      "Action": "sts:AssumeRole",
      "Condition": {
        "StringEquals": {
          "aws:SourceAccount": "123456789012"
        },
        "StringLike": {
          "aws:SourceArn": "arn:aws:cloudformation:us-east-1:123456789012:type/resource/Organization-Service-Resource/*"
        }
      }
    }
  ]
}
