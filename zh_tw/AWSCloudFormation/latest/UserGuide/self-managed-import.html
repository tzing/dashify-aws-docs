AWS CloudFormation StackSets 的自我管理堆疊匯入

AWS CloudFormation 堆疊匯入操作可以將現有堆疊匯入新的或現有的堆疊集，以便您可以在一次操作中將現有堆疊遷移至堆疊集。StackSets 擴充了堆疊功能，讓您可以跨多個帳戶和區域使用單次操作建立、更新或刪除堆疊。

例如，如果您有堆疊指定了跨多個帳戶的管理員 AWS Identity and Access Management (IAM) 角色，您可以將該堆疊匯入堆疊集。透過使用堆疊匯入，您可以避免停機時間和中斷，而無需刪除和重新建立這些資源。一旦堆疊匯入到堆疊集中，原始堆疊將成為新產生堆疊集的堆疊執行個體。

堆疊匯入的自我管理需求

除 堆疊匯入需求 區段外，自我管理堆疊匯入需要以下內容。

  • 堆疊匯入操作支援建立具有自我管理許可的堆疊集。

  • 堆疊匯入操作需要您建立堆疊集所在的管理員帳戶，以及包含堆疊的目標帳戶。

  • 目標帳戶必須具備許可來搭配使用 GetTemplate 操作與堆疊 ID 或 ARN 輸入。因此，您的管理員帳戶必須獲得 AWSCloudFormationStackSetsAdminstration 或 AWSCloudFormationStackSetsExectionRole 許可。

將堆疊匯入新的堆疊集

使用 AWS Management Console 將堆疊匯入新的堆疊集

若要將堆疊匯入堆疊集，請識別包含您要匯入之資源的堆疊。

  1. 請登入 AWS Management Console，開啟位於 https://console.aws.amazon.com/cloudformation 的 AWS CloudFormation 主控台。

  2. 從導覽窗格選擇 StackSets。

  3. 選擇 StackSets 頁面上方的 Create StackSet (建立 StackSet)。

  4. 在 Choose a template (選擇範本) 頁面上，使用下列其中一個選項指定範本，然後選取 Next (下一步)。

    • 選擇 Amazon S3 URL，然後在文字方塊中為您的範本指定 URL。

    • 選擇 Upload a template file (上傳範本檔案)，然後瀏覽您的範本。

    • 選擇 From stack ID (通過堆疊 ID)，然後輸入您的堆疊 ID。

  5. 在 Specify StackSet details (指定 StackSet 詳細資訊) 頁面上，輸入您要建立的堆疊集名稱並選取 Next (下一步)。

    (選用) 輸入堆疊集描述。

  6. 在 Configure StackSet options (設定 StackSet 選項) 頁面上檢閱所需選項，然後選取 Next (下一步)。

  7. 在 Set deployment options (設定部署選項) 頁面上，選取 Import stacks to stack set (將堆疊匯入堆疊集)。

  8. 在 Stacks to import (要匯入的堆疊) 欄位中輸入您想要匯入的堆疊的堆疊 ID。例如 arn:123456789101:us-east-1:StackToImport。

    (選用) 選取 Add another stack ID (新增另一個堆疊 ID)，然後輸入您想要匯入的另一個堆疊的堆疊 ID。每個堆疊匯入操作最多可以新增 10 個堆疊。

  9. 檢閱您的部署選項，然後選取 Next (下一步)。

  10. 在 Review (檢閱) 頁面上，檢閱您的選擇以及堆疊集的屬性。當您準備好將堆疊匯入堆疊集時，請選取 Submit (提交)。

結果：匯入的堆疊現在是指定堆疊集的堆疊執行個體。若要進一步了解堆疊匯入狀態，請參閱堆疊集和堆疊執行個體狀態碼。

將自我管理堆疊匯入現有堆疊集

使用 AWS Management Console 將堆疊匯入現有堆疊集

若要將現有堆疊匯入堆疊集，請識別包含您要匯入之資源的堆疊。

  1. 請登入 AWS Management Console，開啟位於 https://console.aws.amazon.com/cloudformation 的 AWS CloudFormation 主控台。

  2. 從導覽窗格選擇 StackSets。

  3. 在 StackSets 頁面上，選取您想要匯入堆疊的堆疊集。

  4. 選取堆疊集後，從 Actions (動作) 選單選擇 Add stacks to StackSet (將堆疊新增至 StackSet)。

  5. 在 Set deployment options (設定部署選項) 頁面上，選取 Import stacks to stack set (將堆疊匯入堆疊集)，然後在 Stacks to import (要匯入的堆疊) 欄位輸入您想要匯入之堆疊的堆疊 ID。例如 arn:123456789101:us-east-1:StackToImport。

    (選用) 選取 Add another stack ID (新增另一個堆疊 ID)，然後輸入您想要匯入的另一個堆疊的堆疊 ID。每個堆疊匯入操作最多可以新增 10 個堆疊。

  6. 選擇 Next (下一步)。

  7. 在 Specify overrides (指定覆寫) 頁面上，檢閱您的選擇，然後選取 Next (下一步)。

  8. 在 Review (檢閱) 頁面上，檢閱您的選擇以及堆疊集的屬性。當您準備好建立堆疊集時，請選擇 Submit (提交)。

結果：匯入的堆疊現在是指定堆疊集的堆疊執行個體。若要進一步了解堆疊匯入狀態，請參閱堆疊集和堆疊執行個體狀態碼。

將堆疊匯入堆疊集 (AWS CLI)

將堆疊匯入堆疊集

若要將現有堆疊匯入堆疊集，請識別包含您要匯入之資源的堆疊。在此範例中，要匯入的堆疊為 arn:123456789101:us-east-1:StackToImport。

  1. 透過指定要匯入的 CloudFormation 堆疊的完整 ARN，從堆疊 ID 建立堆疊集。

    aws cloudformation create-stack-set \
  --stack-id "arn:aws:cloudformation:us-east-1:123456789012:stack/StackToImport/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set-name "SingleStackSetName" \
  --permission-model "SELF_MANAGED" \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  2. 將指定堆疊匯入堆疊集。

    aws cloudformation import-stacks-to-stack-set \
  --stack-ids "arn:aws:cloudformation:us-east-1:123456789012:stack/StackToImport/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set "SingleStackSetName" \
  --permission-model SELF_MANAGED \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  3. 將匯入的堆疊複製到其他區域和帳戶。

    aws cloudformation create-stack-instances \
  --stack-set-name "StackSetToWhichStackimported" \
  --accounts "123556789101" \
  --regions "us-east-1"

將堆疊匯入堆疊集

若要將現有堆疊匯入堆疊集，請識別包含您要匯入之資源的堆疊。在此範例中，要匯入的堆疊為 arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport1/f449b250-b969-11e0-a185-5081d0136786 和 arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport2/f449b250-b969-11e0-a185-5081d0136786。

  1. 從堆疊 ID 建立堆疊集。

    aws cloudformation create-stack-set \
  --stack-id "arn:123456789101:us-east-1:StackToImport" \
  --stack-set-name "StackSetName" \
  --permission-model "SELF_MANAGED" \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  2. 將指定堆疊匯入堆疊集。

    aws cloudformation import-stacks-to-stackset 
  --stack-ids "arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport1/f449b250-b969-11e0-a185-5081d0136786, arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport2/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set "StackSetName" 
  --permission-model SELF_MANAGED \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  3. 將匯入的堆疊複製到其他區域和帳戶。

    aws cloudformation create-stack-instances \
  --stack-set-name "StackSetName" \
  --accounts "123456789012" \
  --regions "us-east-1"
