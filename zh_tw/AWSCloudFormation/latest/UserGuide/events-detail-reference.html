AWS CloudFormation 事件詳細參考

來自 AWS 服務的所有事件都有一組共同的欄位，其中包含事件的相關中繼資料，例如做為事件來源的 AWS 服務、產生事件的時間、發生事件的帳戶和地區，以及其他。如需這些一般欄位的定義，請參閱Amazon EventBridge 使用指南中的事件結構參考。

此外，每個事件都有一個 detail 欄位，其中包含該特定事件的特定資料。下面的參考定義了各種 AWS CloudFormation 事件的詳細信息字段。

使用 EventBridge 來選取和管理 AWS CloudFormation 事件時，請牢記下列事項：

  • 所有來源事件的source欄位 AWS CloudFormation 都設定為aws.cloudformation。

  • detail-type 欄位指定事件類型。

    例如 CloudFormation Resource Status Change 或 CloudFormation Drift Detection Status Change。

  • detail 欄位包含該特定事件的特定資料。

    例如，堆疊 ID、涉及的資源、各種資源的狀態，以及與特定類型的事件相關的其他資料。

如需有關建構啟用規則以符合 AWS CloudFormation 事件的事件模式的資訊，請參閱《Amazon EventBridge 使用指南》中的事件模式。

如需有關事件及其 EventBridge 處理方式的詳細資訊，請參閱《Amazon EventBridge 使用指南》中的Amazon EventBridge 事件。
