建立堆疊

若要建立堆疊，請執行 aws cloudformation create-stack 命令。您必須提供堆疊名稱、有效範本的位置和任何輸入參數。

參數以空格分隔，金鑰名稱要區分大小寫。如果您在執行時輸入錯誤的參數索引鍵名稱aws cloudformation create-stack，則 AWS CloudFormation 不會建立堆疊，並報告範本不包含該參數。

注意

您可以使用自己的儲存貯體，並手動上傳範本至 Amazon S3 或 AWS Systems Manager以管理其許可。然後，每當您建立或更新堆疊時，請指定 Amazon S3 或 AWS Systems Manager 範本檔案。

根據預設，aws cloudformation describe-stacks 會傳回參數值。若要防止傳回敏感參數值 (例如密碼)，請在 AWS CloudFormation 範本TRUE中包含設定的NoEcho屬性。

重要

使用 NoEcho 屬性不會遮罩任何儲存在下列資訊中的資訊：

  • 模Metadata板部分。 CloudFormation 不會轉換、修改或密文Metadata章節中包含的任何資訊。如需詳細資訊，請參閱 中繼資料。

  • Outputs 範本區段。如需詳細資訊，請參閱 輸出。

  • 資源定義的 Metadata 屬性。如需詳細資訊，請參閱 Metadata 屬性。

我們強烈建議您不要使用這些機制來包含敏感資訊，例如密碼或秘密。

重要

建議您使用堆疊 CloudFormation 範本中的動態參數來參照儲存和管理之外的敏感資訊 (例如 CloudFormation，在 Par AWS Systems Manager ameter Store 或) 中，而不是直接在範本中嵌入敏感資訊 AWS Secrets Manager。

如需詳細資訊，請參閱最請勿在您的範本中內嵌憑證佳做法。

下列範例會建立 myteststack 堆疊：

PROMPT> aws cloudformation create-stack \
  --stack-name myteststack \
  --template-body file:///home/testuser/mytemplate.json \
  --parameters ParameterKey=Parm1,ParameterValue=test1 ParameterKey=Parm2,ParameterValue=test2

CloudFormation 返回以下輸出：

{
  "StackId" : "arn:aws:cloudformation:us-east-1:123456789012:stack/myteststack/330b0120-1771-11e4-af37-50ba1b98bea6"
}

下列範例會在 AWS Systems Manager 文件中建立 myteststack 堆疊：

PROMPT> aws cloudformation create-stack \
 --stack-name myteststack \
 --template-url "ssm-doc://arn:aws:ssm:us-east-1:123456789012:document/documentName"
