AWS CloudFormation 配額

您的 AWS 帳戶擁有編寫範本和建立堆疊時可能需要知道的 AWS CloudFormation 配額。透過了解這些配額，您可以避免要求您重新設計範本或堆疊的限制錯誤。

StackSets

您可以在管理員帳戶中建立的 AWS CloudFormation 堆疊集數目上限。

1000 個堆疊集

若要建立更多堆疊集，請刪除不需要的堆疊集，或要求增加 AWS 帳戶中堆疊集數目上限。如需詳細資訊，請參閱《AWS 》中的 AWS 一般參考服務配額。

堆疊執行個體

您每個堆疊集可以建立的堆疊執行個體數目上限。

每個堆疊集 100,000 個堆疊執行個體

若要建立更多堆疊執行個體，請刪除您不需要的堆疊執行個體，或請求提高您 AWS 帳戶的堆疊執行個體數目上限。如需詳細資訊，請參閱《AWS 》中的 AWS 一般參考服務配額。

StackSets 例項作業

每個管理員帳戶可同時在每個區域中 (跨所有堆疊集) 執行作業的堆疊執行個體數目上限。

10,000 次操作

此限制適用於區域中涉及的所有堆疊集。包括受堆疊集建立和更新操作所影響的堆疊執行個體，以及直接建立、更新或刪除堆疊執行個體。

StackSets 排入佇列的作業

指定時間內堆疊集的佇列操作數目上限。

10,000 次操作

請求的範本內文大小

您可以在 CreateStack、UpdateStack 或 ValidateTemplate 請求中傳遞的範本內文大小上限。

51,200 位元組

若要使用較大的範本內文，請使用例如巢狀堆疊，將您的範本分成多個範本。或將範本上傳至 Amazon S3 儲存貯體。

Amazon S3 物件中的範本內文大小

您可以在 CreateStack、UpdateStack、ValidateTemplate 請求的 Amazon S3 物件中以 Amazon S3 範本 URL 傳遞的範本內文大小上限。

1 MB

若要使用較大的範本內文，請使用例如巢狀堆疊，將您的範本分成多個範本。或者使用縮小來減少 CloudFormation 模板大小。

範本描述

範本描述的大小上限。

1,024 位元組

每個勾點的版本

每個勾點的最大版本數量。

100 個版本

功能可用性

並非每個地區都提供的所有功 CloudFormation 能。如需區域的詳細資訊，請參閱全 AWS 域基礎結構區域表格。

  • 目前無法在下列區域中使用巨集：

    • 亞太區域 (雅加達)

  • 在下列區域目前未提供漂移偵測：

    • 亞太區域 (大阪)

  • 以下區域目前無法透過 CodeDeploy 使 AWS CloudFormation用執行 ECS 藍/綠部署：

    • 非洲 (開普敦)

    • 亞太區域 (大阪)

    • 歐洲 (米蘭)

StackSets 和巨集

StackSets 目前不支援從包含巨集的範本建立或更新具有服務管理權限的堆疊集。這包括轉換，這些轉換是由主控的巨集 AWS CloudFormation。如需巨集的詳細資訊，請參閱範本巨集。
