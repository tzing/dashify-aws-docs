AWS CloudFormation Git 同步逐步解說

透過下列逐步解說，學習使用 Git 同步。開始本教學課程之前，請先完成 Git 同步先決條件。

步驟 1：建立堆疊

  1. 存取 CloudFormation 主控台。

  2. 選擇建立堆疊。

  3. 在建立堆疊儀表板的先決條件 - 準備範本面板中，確認已選取範本已就緒。

  4. 在指定範本面板中，選擇從 Git 同步。

  5. 選擇下一步以指定堆疊詳細資訊。

步驟 2：指定堆疊詳細資訊

  1. 在提供堆疊名稱面板中，輸入堆疊的名稱。堆疊名稱可包含字母 (A-Z 和 a-z)、數字 (0-9) 和破折號 (-)。

  2. 堆疊部署檔案有兩個選項：

    • 使用下列參數建立檔案並將其放置在我的儲存庫中。

      如果您想要 Git 同步建立堆疊部署檔案並將其放置在儲存庫中，請選擇此選項。

    • 我在我的儲存庫中提供自己的檔案。

      如果您已建立堆疊部署檔案並將其新增至儲存庫，請選擇此選項。

    如需更多詳細資訊，請參閱 堆疊部署檔案。

  3. 範本定義儲存庫面板有兩個選項：

    • 選擇連結的 Git 儲存庫

    • 連結 Git 儲存庫

    如果您已將 Git 儲存庫連結至 CloudFormation，請選取選擇連結的 Git 儲存庫，然後繼續下一步。否則，請執行下列子步驟來連接 Git 儲存庫：

    1. 選擇連結 Git 儲存庫。

    2. 從下列選項選擇您的 Git 儲存庫提供者：

      • GitHub

      • GitHub Enterprise

      • GitLab

      • Bitbucket

    3. 從清單選擇連線。如果連線清單中沒有出現任何選項，請前往連線主控台來建立與儲存庫的連線。

  4. 在儲存庫清單中，選取包含堆疊範本檔案的 Git 儲存庫。

  5. 在分支清單中，選取您希望 Git 同步監視的分支。

    注意

    Git 同步只會監控所選分支是否有對 CloudFormation 範本和堆疊部署檔案的變更。您想要套用至堆疊的任何變更，均必須遞交到此分支。

  6. 若為部署檔案路徑，請指定完整路徑，包含儲存庫分支根目錄中的堆疊部署檔案名稱。

    如果 CloudFormation 正在為您產生檔案，這就是檔案將在您的儲存庫中遞交的位置。如果您要提供檔案，這是檔案在儲存庫中的位置。

  7. 新增 IAM 角色。IAM 角色包含 CloudFormation 從 Git 儲存庫同步堆疊所需的許可。您可以選擇新的 IAM 角色來產生新角色，或選擇現有 IAM 角色從 CloudFormation 帳戶中選取現有角色。如果您選擇產生新角色，所需的許可會包含在角色中。

    注意

    如需有關 Git 同步的必要許可和信任政策的重要資訊，請參閱 所需的許可。

  8. 若為範本檔案路徑，請為堆疊範本檔案指定儲存庫根目錄的完整路徑。

  9. 為您的堆疊新增您想從堆疊部署檔案中設定的參數。參數為鍵值組。例如，若要將 port 參數新增至堆疊部署檔案：

    1. 選擇 Add parameter (新增參數)。

    2. 在 Key (索引鍵) 欄位，輸入 port。

    3. 針對值，輸入 8080。

    4. 針對您要在堆疊部署檔案中使用的所有堆疊參數重複上述步驟。

  10. (選用) 新增標籤。標籤為鍵值組，您可以用來識別和分類堆疊中的資源。若要將標籤新增至堆疊部署，請遵循新增參數時所使用的相同程序。

  11. 選擇下一步繼續設定堆疊選項。

步驟 3：設定堆疊選項

如需組態堆疊選項的相關資訊，請參閱 設定 AWS CloudFormation 堆疊選項。

完成設定後，請選擇下一步繼續檢閱和部署。

步驟 4：檢閱和部署

  1. 檢閱堆疊設定並確認下列事項：

    • 堆疊範本已正確設定，並設為從 Git 同步。

    • 部署檔案設定正確。

    • 範本定義儲存庫已正確設定，尤其是選取了正確的儲存庫和分支名稱。

    • 部署檔案的預覽是正確的，且包含預期的參數與值。

  2. 選擇提交以建立堆疊。

    重要

    當您選擇提交時，系統會在 Git 儲存庫中自動建立一個提取請求。您必須將此提取請求合併到 Git 儲存庫中才能完成此過程。當您合併提取請求時，系統會建立堆疊，而 Git 同步會監控 CloudFormation 範本和堆疊部署檔案，以進行更新堆疊的變更。

修改堆疊

完成上述步驟之後，就會建立新的堆疊，而 CloudFormation 正在監控 Git 儲存庫中是否有變更。

若要修改堆疊，請直接變更堆疊部署檔案。然後，將它們遞交到您在 Git 同步中為堆疊設定的儲存庫和分支。

在 CloudFormation 儀表板中，您可以選取堆疊並選擇 Git 同步索引標籤，以檢視堆疊狀態和同步事件的相關資訊。如需更多詳細資訊，請參閱 AWS CloudFormation Git 同步狀態儀表板。
