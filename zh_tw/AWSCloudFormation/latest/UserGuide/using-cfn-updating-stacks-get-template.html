修改堆疊範本

您必須修改堆疊的範本，才能修改堆疊範本中所宣告的資源與屬性。為了確保您僅更新預計的資源，請將現有堆疊的範本當成起點並更新該範本。如果您要在原始程式碼控制系統中管理範本，則請將該範本的複本作為起點。不然，您亦可從 AWS CloudFormation取得堆疊範本的複本。

您可以重複使用現有堆疊範本，僅針對堆疊的參數或設定 (例如：堆疊的 Amazon SNS 主題) 進行修改。如此一來，便不需要取得堆疊範本的複本，或是修改堆疊範本。

注意

如果您的範本包含不受支援的變更，會 AWS CloudFormation 傳回一則訊息，指出不允許變更。不過，此訊息可能會以非同步方式發生，因為依預設會以非決定性 AWS CloudFormation 的順序建立和更新資源。

更新堆疊的範本 (主控台)

  1. 在 AWS CloudFormation 主控台的 Stacks (堆疊) 頁面上，按一下您要更新的堆疊名稱。

  2. 在所選堆疊的 stack details (堆疊詳細資訊) 窗格中，選取 Template (範本) 窗格，然後按一下 View in Designer (使用設計工具檢視)。

    AWS CloudFormation 在 AWS CloudFormation 設計器中開啟堆疊範本的複本。

  3. 修改範本。

    您可以使用設 AWS CloudFormation 計師 drag-and-drop 介面或整合式 JSON 和 YAML 編輯器來修改範本。如需有關使用 AWS CloudFormation 設計師的詳細資訊，請參閱什麼是 AWS CloudFormation 設計工具？。

    請務必只修改要更新的資源。針對沒有要更新的資源與屬性，請採用與目前堆疊組態相同的值。如需修改範本，請完成下列任一動作：

    • 新增新的資源，或是移除現有資源。

      對於大多數資源而言，變更資源的邏輯名稱相當於刪除該資源，且會用新資源取而代之。如果任何其他資源與重新命名的資源之間有依存關係，則需一併進行更新，亦可能會遭取代。為了啟動更新作業，您必須更新其他資源的屬性，而不是只變更邏輯名稱。

    • 新增、修改或刪除現有資源的屬性。

      如需有關更新特定資源屬性效果的資訊，請參閱 AWS 資源類型參考。更新各個屬性將產生下列其中一種效果：

      • 更新要求：不中斷

      • 更新要求：部分中斷

      • 更新要求：替換

    • 新增、修改或刪除資源的屬性 (Metadata、DependsOn、CreationPolicy、UpdatePolicy 與 DeletionPolicy)。

      重要

      您不能更新 CreationPolicy 或 UpdatePolicy 屬性本身。唯有涵蓋新增、修改或刪除資源等變更作業時，才能更新這些屬性。例如，您可以新增或修改資源的中繼資料屬性。

    • 新增、修改或刪除參數的宣告。然而，在資源不支援更新的情況下，您將無法新增、修改或刪除該資源所使用的參數。

    • 新增、修改或刪除映射的宣告。

      重要

      如果堆疊沒有使用映射中的值，您將無法更新映射本身。需要涵蓋新增、修改或刪除資源等變更作業，才能進行更新。例如，您可以新增或修改資源的中繼資料屬性。如果您要更新堆疊所使用的映射值，則不需進行任何其他變更，即可啟動更新作業。

    • 新增、修改或刪除條件的宣告。

      重要

      您無法更新條件本身。唯有涵蓋新增、修改或刪除資源等變更作業時，才能更新條件。例如，您可以新增或修改資源的中繼資料屬性。

    • 新增、修改或刪除輸出值的宣告。

    在屬性值或變更這些數值方面，部分資源或屬性可能會有所限制。舉例來說，變更 AWS::RDS::DBInstance 資源的 AllocatedStorage 屬性時，數值必須大於目前設定。如果更新作業所指定的數值不符合這些限制，則資源更新會失敗。如需了解 AllocatedStorage 變更的特定條件，請參閱 ModifyDBInstance。

    更新資源可能會影響其他資源的屬性。如果您使用 Ref 函數或Fn::GetAtt 函數，將已更新資源的屬性指定為範本中另一個資源中屬性值的一部分， AWS CloudFormation 也會更新包含已變更之屬性參照的資源。例如，如果您更新了AWS::RDS::DBInstance資源的MasterUsername屬性，而您的資AWS::AutoScaling::LaunchConfiguration源具有包含使用該Ref函數對資料庫執行個體名稱參考的UserData屬性，則 AWS CloudFormation 會使用新名稱重新建立資料庫執行個體，並更新LaunchConfiguration資源。

  4. 若要檢查範本中的語法錯誤，請從 AWS CloudFormation 設計工具列中選擇驗證範本 ( )。

    在 Messages (訊息) 窗格中檢視並修復所有錯誤，接著再次驗證範本。如果您沒有看到任何錯誤，表示該範本的語法有效。

  5. 從 AWS CloudFormation Designer 工具列中，選擇檔案功能表 ( )，然後選擇儲存，將範本儲存在 S3 儲存貯體或本機。

  6. 使用已修改的範本直接更新堆疊或使用更改集更新堆疊。

取得並更新堆疊範本 (CLI)

  1. 如需取得要更新的堆疊範本，請採用 aws cloudformation get-template 命令。

  2. 複製範本並將其貼至文字檔案，進行修改後加以儲存。請務必僅複製範本。該命令會以引號括住範本，請勿複製括住該範本的引號。範本本身會以左大括號開始，並以右大括號結束。您可以在此檔案中指定要進行的堆疊資源變更。

  3. 使用已修改的範本直接更新堆疊或使用更改集更新堆疊。
