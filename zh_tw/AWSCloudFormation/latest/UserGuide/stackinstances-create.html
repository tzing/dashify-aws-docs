將堆疊新增至堆疊集

當您建立堆疊集時，可為該堆疊集建立堆疊。AWS CloudFormation 也可以讓您在堆疊集建立之後的任何時間點，為其他帳戶和區域新增更多堆疊。您可以使用 AWS CloudFormation 主控台或在 AWS CLI 中使用 AWS CloudFormation 命令來新增堆疊執行個體。在此程序中，我們會將額外區域的堆疊執行個體新增至我們在建立堆疊集中建立的堆疊集。

主題
  • 使用 AWS Management Console 將堆疊執行個體新增至堆疊集
  • 使用 AWS CLI 將堆疊執行個體新增至堆疊集

使用 AWS Management Console 將堆疊執行個體新增至堆疊集

  1. 在以下網址開啟 AWS CloudFormation 主控台：https://console.aws.amazon.com/cloudformation。

  2. 從導覽窗格選擇 StackSets。在 StackSets 頁面上，選取您在 建立堆疊集 中建立的堆疊集。

  3. 選取堆疊集後，從 Actions (動作) 選單選擇 Add new stacks to StackSet (將新的堆疊新增至 StackSet)。

  4. 在 Set deployment options (設定部署選項) 頁面上，指定您要將堆疊集的堆疊新增至哪些帳戶和區域。

    AWS CloudFormation 會在第一個區域內的指定帳戶中部署堆疊，然後移至下一個區域，以此類推。只要一個區域的部署失敗不超過指定的容錯能力。

    1. [自我管理許可] 對於 Deployment targets (部署目標)，選擇 Deploy stacks in accounts (在帳戶中部署堆疊)。在文字方塊中貼上您的目標帳戶號碼，並以逗號分隔多個號碼。

      [服務管理許可] 對於 Deployment targets (部署目標)，選擇要部署到組織中的哪些帳戶。

      • 選擇 Deploy to organization (部署至組織)，以部署至組織中的所有帳戶。

      • 選擇 Deploy to organizational units (OUs) (部署至組織單位 (OU))，以部署至特定 OU 中的所有帳戶。選擇 Add another OU (新增另一個 OU)，然後將目標 OU ID 貼到文字方塊中。針對每個新的目標 OU 重複上述步驟。StackSets 也會以所選目標的任何子 OU 為目標。

      注意

      如果您新增的 OU 已是堆疊集的目標，StackSets 會在 OU 中的任何帳戶中 (尚無堆疊集中的堆疊執行個體) 建立新的堆疊執行個體 (例如，在建立堆疊集後新增至 OU 且已停用自動部署的帳戶)。

    2. 對於 Deployment regions (部署區域)，選擇 [US West (N. California) (美國西部 (加利佛尼亞北部))]。您將在美國西部 (加利佛尼亞北部) 區域，針對您指定的目標建立新的堆疊。

      如果您新增多個區域，則 Specify regions (指定區域) 下的區域順序決定其部署順序。

    3. 針對 Deployment options (部署選項)：

      • 針對 Maximum concurrent accounts (同時使用的帳戶上限)，保留 Number (數值) 和 1 的預設值。

        這表示 AWS CloudFormation 一次只在一個帳戶中部署您的堆疊。

      • 針對 Failure tolerance (容錯能力)，保留預設值 Number (數字) 和 0。

        這表示在您指定的其中一個區域中，堆疊部署最多只能失敗一次，否則 AWS CloudFormation 會停止目前區域中的部署，並取消在剩餘區域中的部署。如果您想 CloudFormation 要更容錯，請增加此值。

      選擇 Next (下一步)。

  5. 在 Specify Overrides (指定覆寫) 頁面上，依指定保留屬性值。您不需要覆寫您要建立之堆疊的任何屬性值。選擇 Next (下一步)。

  6. 在 Review (檢閱) 頁面上，檢閱您的選擇以及堆疊集的屬性。若要進行變更，請在您要變更屬性的區域中選擇 Edit (編輯)。在您可以建立堆疊之前，必須填入 Capabilities (功能) 區域中的核取方塊，確認您使用堆疊集所建立的一些資源可能需要新的 IAM 資源和許可。如需有關可能需要之許可的詳細資訊，請參閱本指南中的確認 AWS CloudFormation 範本中的 IAM 資源。當您準備好建立堆疊執行個體時，請選擇 Submit (提交)。

  7. AWS CloudFormation 開始建立您的堆疊執行個體。在選擇 Submit (提交) 時開啟的堆疊集詳細資訊頁面中，檢視您堆疊集中堆疊執行個體的建立進度和狀態。完成後，您的新堆疊執行個體應該會列在 Stack instances (堆疊執行個體) 標籤中。

使用 AWS CLI 將堆疊執行個體新增至堆疊集

當您身為委派管理員時，您必須在每次執行 StackSets 命令時將 --call-as 參數設定為 DELEGATED_ADMIN。

--call-as DELEGATED_ADMIN

  1. 開啟 AWS CLI。

  2. 執行 create-stack-instances 命令。

    [自我管理許可] 提供您要建立堆疊執行個體的帳戶 ID。

    aws cloudformation create-stack-instances --stack-set-name my-awsconfig-stackset --accounts '["account_id"]' --regions '["eu-west-1", "us-west-2"]'

    [服務管理許可] 提供您要建立堆疊執行個體的組織 (根) ID 或 OU ID。在此範例中，我們指定具有 ou-rcuk-1x5j1lwo 和 ou-rcuk-slr5lh0a ID 的 OU。

    aws cloudformation create-stack-instances --stack-set-name StackSet-myApp --deployment-targets OrganizationalUnitIds='["ou-rcuk-r1qi0wl7"]' --regions '["eu-west-1", "us-west-2"]'
    注意

    如果您新增的 OU 已是堆疊集的目標，StackSets 會在 OU 中的任何帳戶中 (尚無堆疊集中的堆疊執行個體) 建立新的堆疊執行個體 (例如，在建立堆疊集後新增至 OU 且已停用自動部署的帳戶)。
