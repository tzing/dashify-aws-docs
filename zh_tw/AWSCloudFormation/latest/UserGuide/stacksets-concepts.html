管理員和目標帳戶

管理員帳戶是您在其中建立堆疊集的 AWS 帳戶。對於具有服務受管許可的堆疊集，管理員帳戶是組織的 管理帳戶 或委派管理員帳戶。您可以登入建立堆疊集的 AWS 管理員帳戶，以管理堆疊集。

「目標帳戶」是您建立、更新或刪除堆疊集中一或多個堆疊的帳戶。先要設定管理員和目標帳戶之間的信任關係，才可以使用堆疊集在目標帳戶中建立堆疊。

AWS CloudFormation StackSets

堆疊集可讓您使用單一 CloudFormation 範本在不 AWS 帳戶 同區域中建立堆疊。堆棧集的 CloudFormation 模板定義了每個堆棧中的所有資源。當您建立堆疊集時，指定要使用的範本，以及範本要求的任何參數和功能。

定義堆疊集之後，您可以在目標帳戶中建立、更新或刪除堆疊並指 AWS 區域 定。當您建立、更新或刪除堆疊時，您也可以指定操作偏好設定。例如，包括您要執行操作的區域順序、堆疊操作停止前的失敗容錯閾值，以及同時執行堆疊操作的帳戶數目。

堆疊集是區域性資源。如果您在其中一個建立堆疊組合 AWS 區域，則只能在檢視該區域時看到或變更它。

堆疊集的許可模型

您可以使用自我管理許可或服務受管許可來建立堆疊集。

透過自我管理許可，您可 StackSets 以建立跨帳戶和區域部署所需的 IAM 角色。需要這些角色，才能在您用來管理堆疊集的帳戶與堆疊執行個體部署目的地帳戶之間建立信任關係。使用此許可模型， StackSets 可以部署到您有權建立 IAM 角色的任何 AWS 帳戶。

透過服務受管許可，您可以將堆疊執行個體部署到 AWS Organizations所管理的帳戶。使用此許可模型，您不必建立必要的 IAM 角色；代表您建 StackSets 立 IAM 角色。使用此模型時，您也可以自動部署到未來您新增至組織的帳戶。

AWS Organizations 整合 CloudFormation 並協助您集中管理和控管您的環境，同時擴充和擴充資 AWS 源。

  • 管理帳戶 – 您用於建立組織的帳戶。如需詳細資訊，請參閱 AWS Organizations 術語與概念。

  • 委派管理員 — 相容的 AWS 服務可以將組織中的 AWS 成員帳戶註冊為該服務中組織帳戶的管理員。如需詳細資訊，請參閱可搭配使用的AWS 服務 AWS Organizations。

如需使用服務受管許可建立和管理堆疊集的詳細資訊，請參閱下列主題：

  • 啟用使用 AWS Organizations進行受信任存取。

  • 註冊委派管理員

  • 建立具有服務管理許可的堆疊集

堆疊執行個體

「堆疊執行個體」是針對區域內某個目標帳戶中的堆疊所做的參考。堆疊執行個體可以在沒有堆疊的情況下存在。例如，如果因為某種原因而無法建立堆疊，則堆疊執行個體會顯示堆疊建立失敗的原因。堆疊執行個體僅與一個堆疊集相關聯。

下圖顯示堆疊集、堆疊操作和堆疊之間的邏輯關係。當您更新堆疊集時，所有帳戶和區域中所有相關聯的堆疊執行個體都會更新。

堆疊集操作

您可以對堆疊集執行下列操作。

建立堆疊集。

建立新的堆疊集包括指定要用來建立堆疊的 CloudFormation 範本、指定要在其中建立堆疊的目標帳戶，以及識別要 AWS 區域 在目標帳戶中部署堆疊的目標帳戶。堆疊集可確保在您選擇區域內的所有指定目標帳戶，相同的堆疊資源皆有一致的部署及相同的設定。

更新堆疊集

當您更新堆疊集時，是將變更推送到堆疊集中的堆疊。您可使用下列其中一種方法來更新堆疊集。您的範本更新一律影響所有堆疊；您無法選擇性更新堆疊集中某些堆疊的範本，讓其他範本不受影響。

  • 變更範本中的現有設定或新增新的資源，例如更新特定服務的參數設定，或者新增新的 Amazon EC2 執行個體。

  • 使用不同的範本來取代範本。

  • 跨現有或其他的區域，在現有或其他目標帳戶中新增堆疊。

刪除堆疊

當您刪除堆疊時，是從您指定區域內的指定目標帳戶，移除堆疊及其所有相關聯的資源。您可採用下列方式來刪除堆疊。

  • 從某些目標帳戶刪除堆疊，同時讓其他目標帳戶中的其他堆疊繼續執行。

  • 從某些區域刪除堆疊，同時讓其他區域中的堆疊繼續執行。

  • 從您的堆疊集刪除堆疊，但選擇 Retain Stacks (保留堆疊) 選項，儲存它們以繼續獨立執行您的堆疊集。隨後您可在 AWS CloudFormation中、堆疊集外，管理保留的堆疊。

  • 刪除您堆疊集中所有堆疊，準備刪除您的整個堆疊集。

刪除堆疊集

只有當堆疊集中沒有任何堆疊執行個體時才可以刪除它。

堆疊集操作選項

本節所述的這些選項，有助於控制為成功執行堆疊集操作所允許的故障時間和次數，並可防止您遺失堆疊資源。

同時帳戶上限

可用於建立、更新和刪除工作流程的此項設定，能讓您指定目標帳戶一次可執行之操作的數目或百分比上限。較低的數目或百分比，表示一次在較少的目標帳戶中執行某項操作。操作依 Deployment order (部署順序) 方塊中指定的順序，一次在一個區域中執行。例如，如果您要將堆疊部署到兩個區域內的 10 個目標帳戶，則 Maximum concurrent accounts (同時帳戶上限) 設定為 50 和 By percentage (依百分比) 就會將堆疊部署到第一個區域中的前五個帳戶，接著是第一個區域中的後五個帳戶，然後才會移至下一個區域並開始部署到前五個目標帳戶。

當您選擇 By percentage (依百分比) 時，如果指定的百分比不能代表您指定的帳戶整數， CloudFormation 會無條件捨去。例如，如果您要將堆疊部署到 10 個目標帳戶，並將最大並行帳戶設定為 25 和按百分比，則從同時部署 2.5 個堆疊 (這是不可能的) 向下 CloudFormation 捨入至同時部署兩個堆疊。

請注意，此設定可讓您指定操作的「上限」。針對大型的部署，因為服務調節之故，在某些情況下實際同時執行的帳戶數目會略低。部署速度上限為每個堆疊集 100 個並行堆疊執行個體。

並行模式

此設定可用於建立、更新和刪除工作流程，讓您能夠選擇堆疊集操作期間並行層級的運作方式。如需詳細資訊，請參閱 並行模式參數。

容錯能力

此設定可在建立、更新和刪除工作流程中使用，可讓您指定每個區域可能發生的堆疊作業失敗次數上限或百分比，在此之後會自動 CloudFormation 停止作業。較低的數目或百分比表示此操作在較少的堆疊上執行，但您能夠更快開始對失敗的操作進行疑難排解。例如，如果您要更新三個區域內 10 個目標帳戶中的 10 個堆疊，將容錯能力設定為 20 和依百分比，表示操作若要繼續，一個區域最多只能有兩個堆疊更新失敗。如果相同區域中的第三個堆疊失敗，就會 CloudFormation 停止作業。如果第一個區域中的堆疊無法更新，則該區域會繼續更新操作，然後移至下一個區域。如果第二個區域中的兩個堆疊無法更新，則失敗容忍度會達到 20%；如果「區域」中的第三個堆疊失敗，就會 CloudFormation 停止更新作業，且不會繼續前往後續的 Region。

當您選擇「按百分比」時，如果指定的百分比不代表每個區域內堆疊的整數，請向下 CloudFormation 舍入。例如，如果您要將堆疊部署到三個區域中的 10 個目標帳戶，並將「失敗容許度」設定為 25 和「按百分比」，則從 2.5 堆疊的失敗容許度 (這是不可能的) 向下 CloudFormation 捨入為每個區域兩個堆疊的失敗容忍度。

保留堆疊

可用於刪除堆疊工作流程的此項設定，讓堆疊及其資源即使從堆疊集移除後，仍能繼續執行。當您保留堆疊時，會保 AWS CloudFormation 留個別帳戶和區域中的堆疊不變。堆疊與堆疊集會取消關聯，但堆疊及其資源仍儲存。刪除堆疊作業完成後，您可以在 CloudFormation建立堆疊的目標帳戶 (非系統管理員帳戶) 中管理保留的堆疊。

區域並行

此設定可在建立、更新和刪除工作流程中使用，讓您選擇如何 StackSets 部署至區域。

循序 — 只要區域的部署失敗不超過指定的失敗容許，按照「區域部署順序」方塊中指定的方塊，一次將 StackSets作業部署到一個區域。順序部署是預設選項。

平行 — 只要區域的部署失敗不超過指定的容錯能力，就可以同時將 StackSets 作業部署到所有指定的區域。

標籤

您可以在堆疊集的建立和更新操作期間，透過指定金鑰值對來新增標籤。標籤有助於排序和篩選堆疊集資源，處理帳單和成本分配。有關如何在中使用標籤的詳細資訊 AWS，請參閱《使用AWS Billing and Cost Management 者指南》中的使用成本配置標籤。在您指定索引鍵/值組之後，請選擇 + 以儲存標籤。您可以選擇標籤右邊的紅色 X 來刪除您不再適用的標籤。

您套用至堆疊組合的標籤會套用至所有堆疊，以及堆疊所建立的資源。您也可以在中的僅堆疊層級新增標籤 CloudFormation，但這些標籤可能不會顯示在中 StackSets。

雖然 StackSets 不添加任何系統定義的標籤，但是您不應該以字符串aws:開始任何標籤的密鑰名稱。

StackSets 狀態碼

AWS CloudFormation StackSets 生成堆棧集操作的狀態碼。

下表說明堆疊集操作狀態碼。

堆疊集操作狀態 描述

RUNNING

操作正在進行中。

SUCCEEDED

操作已完成，未超過操作的容錯能力。

FAILED

操作無法完成的堆疊數目已超出使用者定義的容錯能力。在堆疊建立和更新操作期間，您設定的操作容錯能力適用於每個區域。如果某個區域內失敗的堆疊數目超過容錯能力，則區域中的操作狀態會變更為 FAILED。作業整體的狀態也會設定為FAILED，並 CloudFormation 取消任何剩餘區域中的作業。

QUEUED

[Service-managed permissions] 對於需要一系列操作的自動部署，操作會進入佇列中等待執行。例如：

  • 將帳戶從一個組織單位 (OU) 移至另一個組織單位 (OU) 會觸發自動部署。OU1 OU2 StackSets 執行刪除作業，從目標區域中的目標OU1帳戶移除堆疊執行個體，並將建立作業排入佇列，以將堆疊執行個體新增至目標區域中的目標OU2帳戶。

  • 將帳戶新增AccountA至 OU 會觸發自動部署。 StackSets 運行創建操作以將堆棧實例添加到目標區域AccountA中。如果您在此建立作業執行時將另一個帳戶新增AccountB至 OU，則會將第二個建立作業排入 StackSets 佇列。當第一個建立作業完成時， StackSets 執行第二個建立作業，將堆疊執行個體新增至AccountB目標區域。

STOPPING

應使用者請求，正在停止操作。

STOPPED

應使用者請求，已停止操作。

堆疊執行個體狀態碼

AWS CloudFormation StackSets 生成堆棧實例的狀態碼。

下表說明中堆疊執行個體的狀態碼 StackSets。

堆疊執行個體狀態 描述

CURRENT

堆疊集的堆疊現為最新。

OUTDATED

因為下列其中一個原因，堆疊集的堆疊不是最新。

  • 相關聯堆疊上的 CreateStackSet 或 UpdateStackSet 操作失敗。

  • 屬於 CreateStackSet 或 UpdateStackSet 操作的堆疊，在堆疊建立或更新前即已失敗或停止。

INOPERABLE

DeleteStackInstances 操作失敗，且堆疊的狀態不穩定。後續的 UpdateStackSet 操作會排除此狀態的堆疊。您可能需要將 DeleteStackInstances 設定為 RetainStacks 來執行 true 操作，以刪除堆疊執行個體，然後手動刪除堆疊。

CANCELLED

指定帳戶和區域中的操作已被取消。這可能是因為使用者已停止堆疊集操作，或是因為堆疊集操作超过了容錯能力。

FAILED

指定帳戶和區域中的操作失敗。如果區域內足夠多帳戶中的堆疊集操作失敗，則可能會超過堆疊集操作整體的容錯能力。

PENDING

指定帳戶和區域中的操作尚未開始。

RUNNING

指定帳戶和區域中的操作目前正在進行中。

SKIPPED_SUSPENDED_ACCOUNT

已跳過指定帳戶和區域中的操作，因為帳戶在操作時被暫停。

SUCCEEDED

指定帳戶和區域中的操作已順利完成。
