Fn::Sub

透過內部函數 Fn::Sub，即可用您指定的值替代輸入字串中的變數。您可以在範本中善用此函數，進而建構命令或輸出，其中包含建立或更新堆疊後才可供使用的數值。

宣告

以下各節會顯示該函數的語法。

JSON

{ "Fn::Sub" : [ String, { Var1Name: Var1Value, Var2Name: Var2Value } ] }

如果您僅有更換 String 參數中的範本參數、資源邏輯 ID 或資源屬性，則不需指定變數映射。

{ "Fn::Sub" : String }

YAML

完整函式名稱的語法：

Fn::Sub:
  - String
  - Var1Name: Var1Value
    Var2Name: Var2Value

短格式的語法：

!Sub
  - String
  - Var1Name: Var1Value
    Var2Name: Var2Value

如果您僅有更換 String 參數中的範本參數、資源邏輯 ID 或資源屬性，則不需指定變數映射。

完整函式名稱的語法：

Fn::Sub: String

短格式的語法：

!Sub String

參數

String

包含變數的字串，可在執行階段與其關聯值 AWS CloudFormation 取代。請使用 ${MyVarName} 格式編寫變數；變數可以指定為範本參數名稱、資源邏輯 ID、資源屬性，或索引鍵/值映射中的變數。如果您僅指定範本參數名稱、資源邏輯 ID 與資源屬性，便不需指定索引鍵/值映射。

如果您指定範本參數名稱或資源邏輯 ID${InstanceTypeParameter}，例如，會 CloudFormation 傳回與使用Ref內建函數相同的值。如果您指定資源屬性，例如${MyInstance.PublicIp}，會 CloudFormation 傳回與使用Fn::GetAtt內建函數相同的值。

要從字面上寫一個美元符號和大括號 (${})，在打開大括號後面添加一個驚嘆號 (!)，例如${!Literal}。 CloudFormation 將此文字解析為${Literal}。

VarName

String 參數中包含的變數名稱。

VarValue

在執行階段 CloudFormation 取代關聯變數名稱的值。

傳回值

CloudFormation 返回原始字符串，替換所有變量的值。

範例

以下範例會示範 Fn::Sub 函數的使用方式。

具有映射的 Fn::Sub

下方範例會使用映射，以將 ${Domain} 變數替換成 Ref 函數傳回的結果值。

JSON

{
    "Name": {
        "Fn::Sub": [
            "www.${Domain}",
            {
                "Domain": {
                    "Ref": "RootDomainName"
                }
            }
        ]
    }
}

YAML

Name: !Sub 
  - 'www.${Domain}'
  - Domain: !Ref RootDomainName

沒有映射的 Fn::Sub

下方範例的 Fn::Sub 會搭配 AWS::Region、AWS::AccountId 虛擬參數與 vpc 資源邏輯 ID 使用，藉此建立 VPC 的 Amazon Resource Name (ARN)。

JSON

{ "Fn::Sub": "arn:aws:ec2:${AWS::Region}:${AWS::AccountId}:vpc/${vpc}" }

YAML

!Sub 'arn:aws:ec2:${AWS::Region}:${AWS::AccountId}:vpc/${vpc}'

 

UserData 命令

以下範例會使用 Fn::Sub，進而將 AWS::StackName 與 AWS::Region 虛擬參數替換成執行期的實際堆疊名稱和區域。

JSON

為確保易讀性，JSON 範例會使用 Fn::Join 函數來分隔每個命令，而不是在單一字串值中指定整個使用者資料指令碼。

"UserData": { "Fn::Base64": { "Fn::Join": ["\n", [
  "#!/bin/bash -xe",
  "yum update -y aws-cfn-bootstrap",
  { "Fn::Sub": "/opt/aws/bin/cfn-init -v --stack ${AWS::StackName} --resource LaunchConfig --configsets wordpress_install --region ${AWS::Region}" },
  { "Fn::Sub": "/opt/aws/bin/cfn-signal -e $? --stack ${AWS::StackName} --resource WebServerGroup --region ${AWS::Region}" }]]
}}

YAML

YAML 範例則會使用常值區塊來指定使用者資料指令碼。

UserData:
  Fn::Base64:
    !Sub |
      #!/bin/bash -xe
      yum update -y aws-cfn-bootstrap
      /opt/aws/bin/cfn-init -v --stack ${AWS::StackName} --resource LaunchConfig --configsets wordpress_install --region ${AWS::Region}
      /opt/aws/bin/cfn-signal -e $? --stack ${AWS::StackName} --resource WebServerGroup --region ${AWS::Region}

支援的函數

對於 String 參數，您不能使用任何函數。您必須指定一個字串值。

對於 VarName 和 VarValue 參數，您可以使用下列函數：

  • Fn::Base64

  • Fn::FindInMap

  • Fn::GetAtt

  • Fn::GetAZs

  • Fn::If

  • Fn::ImportValue

  • Fn::Join

  • Fn::Select

  • Fn::Sub

  • Ref
