更新堆疊資源的行為

當您提交更新時，AWS CloudFormation 會根據您提交內容與堆疊目前範本之間的差異來更新資源。尚未變更的資源會在更新程序期間執行，無需中斷。對於更新的資源，AWS CloudFormation 使用下列其中一個更新行為：

更新而不中斷

AWS CloudFormation 會更新資源而不中斷該資源的操作，也不變更資源的實體 ID。例如，如果您更新 AWS::CloudTrail::Trail 資源上的某些屬性，AWS CloudFormation 會更新線索而不中斷。

更新並有些中斷

AWS CloudFormation 會更新資源並有些中斷。例如，如果您更新 AWS::EC2::Instance 資源上的特定屬性，當 AWS CloudFormation 和 Amazon EC2 重新設定執行個體時，執行個體可能有些中斷。

Replacement

AWS CloudFormation 會在更新期間重新建立資源，這也會產生新的實體 ID。AWS CloudFormation 通常會先建立取代資源、變更其他相依資源的參考以指向取代資源，然後刪除舊的資源。例如，如果您更新 AWS::EC2::Instance 資源類型的 AvailabilityZone 屬性，AWS CloudFormation 會建立新的資源，並將目前的 EC2 執行個體資源取代為此新的資源。

如果您要新增或移除需要取代的屬性，亦會觸發更新。即使屬性的實際值沒有改變，也會進行更新。

AWS CloudFormation 使用的方法取決於您為指定的資源類型更新的屬性。AWS 資源類型參考中將說明每個屬性的更新行為。

視更新行為而定，您可以決定何時修改資源，來降低這些變更對您的應用程式造成的影響。特別是，您可以規劃何時必須在更新期間「取代」資源。例如，如果您更新 AWS::RDS::DBInstance 資源類型的 Port 屬性，AWS CloudFormation 會透過使用更新的連接埠設定建立新的資料庫執行個體，然後刪除舊的資料庫執行個體來取代資料庫執行個體。更新之前，您可能會規劃執行下列作業以準備取代資料庫：

  • 擷取目前資料庫的快照。

  • 準備策略，讓使用該資料庫執行個體的應用程式可以處理資料庫執行個體被取代期間的中斷情況。

  • 確保使用該資料庫執行個體的應用程式將更新的連接埠設定及您所做的任何其他更新列入考量。

  • 使用資料庫快照在新的資料庫執行個體上還原資料庫。

此範例並不完整，旨在提供您想法來規劃何時在更新期間取代資源。

注意

如果範本包含一或多個巢狀堆疊，AWS CloudFormation 也會針對每個巢狀堆疊啟動一個更新。這對判斷是否已修改巢狀堆疊是必要的。AWS CloudFormation 只會更新巢狀堆疊中有對應範本中指定變更的這些資源。
