轉換

選用的 Transform 區段指定 AWS CloudFormation 用來處理您範本的一或多個巨集。Transform 區段的建置基礎是具有功能強大巨集系統之 AWS CloudFormation 的簡單宣告式語言。

您可以在範本中宣告一個或多個巨集。AWS CloudFormation 會依指定巨集時的順序來執行巨集。當您建立變更集時，AWS CloudFormation 產生的變更集會包含處理過的範本內容。然後，您可以檢閱變更並執行變更集。如需更多詳細資訊，請參閱 使用 AWS CloudFormation 巨集對範本執行自訂處理。

AWS CloudFormation 還支援轉換 (由 AWS CloudFormation 託管的巨集)。在執行順序和範圍方面，AWS CloudFormation 會將這些轉換與您建立的任何巨集視為相同。如需特定轉換的詳細資訊，請參閱 轉換參照。

若要宣告多個巨集，請使用清單格式，並指定一或多個巨集。

例如，在以下範例中，AWS CloudFormation 會評估範本 MyMacro，再評估 AWS::Serverless，這兩者都可以處理整個範本，因為它們都納入 Transform 區段中。

// Start of processable content for MyMacro and AWS::Serverless
Transform:
  - MyMacro
  - 'AWS::Serverless'
Resources:
  WaitCondition:
    Type: 'AWS::CloudFormation::WaitCondition'
  MyBucket:
    Type: 'AWS::S3::Bucket'
    Properties: 
        BucketName: MyBucket 
        Tags: [{"key":"value"}] 
        CorsConfiguration:[] 
  MyEc2Instance:
    Type: 'AWS::EC2::Instance' 
    Properties:
        ImageID: "ami-123"
// End of processable content for MyMacro and AWS::Serverless
