自訂資源請求物件

範本開發人員請求屬性

範本開發人員使用AWS CloudFormation資源來指定範本中的自訂資源。AWS::CloudFormation::CustomResource

在 AWS::CloudFormation::CustomResource 中，所有屬性皆是由 custom resource provider 所定義。且僅有一個必要屬性：ServiceToken。

ServiceToken

從自訂資源提供者處取得的服務權杖 (Amazon SNS 主題或 AWS Lambda Lambda 函數 Amazon Resource Name)，可用來存取服務。服務字符必須位於堆疊建立的相同區域。

必要：是

類型：字串

資源屬性中的所有其他欄位皆為選用，且系統會在請求的 ResourceProperties 欄位中，將這些欄位逐項傳送給custom resource provider。供應商則會定義這些欄位的名稱與有效內容。

自訂資源提供者請求欄位

系統會從 AWS CloudFormation 將這些欄位以 JSON 請求格式傳送至 SNS 主題中的自訂資源提供者，該主題是提供者為了因應此用途所設定。

RequestType

請求類型由 AWS CloudFormation 堆疊操作 (create-stack、update-stack 或 delete-stack) 所設定，範本開發人員則會針對包含自訂資源的堆疊啟動這項操作。

此屬性必須是下列其中之一：Create、Update 或 Delete。如需詳細資訊，請參閱 自訂資源請求類型。

必要：是

類型：字串

ResponseURL

回應 URL 可識別預先簽署的 S3 儲存貯體，該儲存貯體會接收來自訂資源提供者對 AWS CloudFormation 的回應。

必要：是

類型：字串

StackId

識別包含自訂資源之堆疊的 Amazon Resource Name (ARN)。

結合 StackId 與 RequestId 形成值，該值可用於唯一識別特定自訂資源上的請求。

必要：是

類型：字串

RequestId

請求的唯一 ID。

結合 StackId 與 RequestId 形成值，該值可用於唯一識別特定自訂資源上的請求。

必要：是

類型：字串

ResourceType

AWS CloudFormation 範本內由範本開發人員選擇之自訂資源的資源類型。自訂資源類型名稱的長度上限為 60 個字元，且可包含英數字元及以下字元：_@-。

必要：是

類型：字串

LogicalResourceId

AWS CloudFormation 範本內由範本開發人員選擇之自訂資源的名稱 (邏輯 ID)。此屬性能協助自訂資源提供者與範本開發人員之間的通訊。

必要：是

類型：字串

PhysicalResourceId

提供者專屬的 custom resource provider 定義之必要的實體 ID。

傳回的 PhysicalResourceId 值可以變更自訂資源更新操作。如果傳回的值相同，則視為正常更新。若傳回的值不同，AWS CloudFormation 會將該更新視為替代，並傳送刪除請求給舊資源。如需詳細資訊，請參閱 AWS::CloudFormation::CustomResource。

必要：始終與 Update 和 Delete 請求一起傳送；絕對不會與 Create 一起傳送。

類型：字串

ResourceProperties

此欄位包含由範本開發人員傳送的 Properties 物件內容。其內容由自訂資源提供者定義。

必要：否

類型：JSON 物件

OldResourceProperties

僅用於 Update 請求。包含在更新請求之前宣告的資源屬性。

必要：是

類型：JSON 物件
