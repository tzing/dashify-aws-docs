AWS CloudFormation 和介面 VPC 端點 (AWS PrivateLink)

您可以建立介面 VPC 端點，以在您的 VPC 與 AWS CloudFormation 間建立私有連線。介面端點採用這項技術 AWS PrivateLink，可讓您在沒有網際網路閘道、NAT 裝置、VPN 連線或AWS直 Connect 連線的情況下私密存取 CloudFormation API。VPC 中的執行個體不需要公有 IP 位址即可與 CloudFormation API 通訊。您的 VPC 和 CloudFormation 不會離開 Amazon 網絡之間的流量。

每個介面端點都是由您子網路中的一或多個彈性網路介面表示。

如需詳細資訊，請參閱《Amazon VPC 使用者指南》中的介面 VPC 端點 (AWS PrivateLink)。

CloudFormation VPC 端點的考量

在為其設定介面 VPC 端點之前 CloudFormation，請務必先查看 Amazon VPC 使用者指南中的界面端點屬性和限制。

CloudFormation 支援從您的 VPC 呼叫其所有 API 動作。

為 CloudFormation 建立介面 VPC 端點

您可以使用 Amazon VPC 主控台或 AWS Command Line Interface () AWS CLI 建立 CloudFormation 服務的 VPC 端點。如需詳細資訊，請參閱《Amazon VPC 使用者指南》中的建立介面端點。

建立 VPC 端點以 CloudFormation 使用下列服務名稱：

  • com.amazonaws.region.cloudformation

如果您為該端點啟用私有 DNS，您可以使用其區域的預設 DNS 名稱 (例如 cloudformation.us-east-1.amazonaws.com)，向 CloudFormation 發出 API 請求。

如需詳細資訊，請參閱《Amazon VPC 使用者指南》中的透過介面端點存取服務。

為 CloudFormation 建立 VPC 端點政策

您可以將端點政策連接至控制 CloudFormation 存取權限的 VPC 端點。此政策會指定下列資訊：

  • 可執行動作的主體。

  • 可執行的動作。

  • 可供執行動作的資源。

如需詳細資訊，請參閱 Amazon VPC 使用者指南中的使用 VPC 端點控制對服務的存取。

範例：用於動作的 VPC 端點原則 CloudFormation

以下是的端點策略範例 CloudFormation。連接至端點時，此策略會授與所有資源上所有主參與者所列 CloudFormation 動作的存取權。下列範例會拒絕所有使用者透過 VPC 端點建立堆疊的權限，並允許對服務上的所有其他動作進行完整存取。 CloudFormation 

{
  "Statement": [
    {
      "Action": "cloudformation:*", 
      "Effect": "Allow", 
      "Principal": "*", 
      "Resource": "*"
    },
    {
      "Action": "cloudformation:CreateStack", 
      "Effect": "Deny", 
      "Principal": "*", 
      "Resource": "*"
    }
  ]
}
