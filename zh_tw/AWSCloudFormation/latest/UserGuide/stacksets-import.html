將堆疊匯入 AWS CloudFormation StackSets

AWS CloudFormation 堆疊匯入操作可以將現有堆疊匯入新的或現有的堆疊集，以便您可以在一次操作中將現有堆疊遷移至堆疊集。StackSets 擴充了堆疊功能，讓您可以跨多個帳戶和區域使用單次操作建立、更新或刪除堆疊。

針對自我管理或服務受管 StackSets，使用堆疊匯入操作。針對自我管理 StackSets，匯入操作可以在管理員帳戶或不同的目標帳戶和 AWS 區域 中匯入堆疊。針對服務受管 StackSets，匯入操作可以在與管理帳戶相同的 AWS Organizations 中匯入任何堆疊。匯入操作最多可以使用內嵌堆疊 ID 匯入 10 個堆疊，或使用 Amazon S3 物件匯入最多 200 個堆疊。

只要支援 StackSets，就可以使用堆疊匯入。如需有關 StackSets 區域支援的資訊，請參閱 StackSets 區域支援。

堆疊匯入需求

由於堆疊集跨多個帳戶執行堆疊操作，您需要有 AWS 帳戶 中定義的必要許可，才能建立您的第一個堆疊集。

  • 若要設定必要許可，以建立具有自我管理許可的堆疊集，請參閱 執行堆疊集操作，此操作涉及預設停用的區域 和 授與自我管理許可。

  • 若要設定必要許可，以建立具有服務受管許可的堆疊集，請參閱 執行堆疊集操作，此操作涉及預設停用的區域 和 啟用使用 AWS Organizations進行受信任存取。。

重要

NoEcho 屬性不受 StackSet 匯入支援。包含 NoEcho 的堆疊將不會透過 StackSet 匯入匯入到新的堆疊集。

如需有關 StackSets 需求的詳細資訊，請參閱堆疊集操作的先決條件。

在將堆疊匯入堆疊集之前，請確定您了解下列需求：

  • StackSet 只能屬於一個堆疊集。

  • 您可以通過在堆疊匯入操作中明確指定標籤作為參數來實作堆疊集的堆疊標籤。

  • 匯入操作期間不會影響堆疊的自訂參數覆寫。

  • 匯入堆疊時，StackSets 配額和堆疊執行個體會套用。如需有關配額的詳細資訊，請參閱 AWS CloudFormation 配額。
