資料共用的限制 (預覽)

這是針對 Amazon Redshift 的多資料倉儲透過資料共用功能進行寫入的發行前版本文件，可在 PREVIEW_2023 追蹤公開預覽中取得。文件和功能會隨時變更。我們建議僅搭配測試叢集使用此功能，不要在生產環境中使用。如需預覽版條款與條件，請參閱 AWS 服務條款中的 Beta 版服務參與。

如果您尚未在預覽軌道上建立資料保護，請前往在帳戶內或跨 AWS 帳戶共用讀取和寫入資料以開始使用。

以下是在 Redshift 中使用資料共用時的限制：

  • ra3.xlplus 叢集目前無法使用透過資料共用的多倉儲寫入功能。若要使用此功能，請使用下列節點類型建立叢集：

    • ra3.4xl

    • ra3.16xl

    也可以與具有預覽追蹤的 Amazon Redshift Serverless 工作群組共同運作。

  • 對於跨帳戶和跨區域資料共用，生產者和取用者叢集及無伺服器命名空間都必須加密。這是出於安全目的。但是，他們不需要共用相同的加密金鑰。

  • 您只能透過資料共用來共用 SQL UDF。不支援 Python 和 Lambda UDF。

  • 如果生產者資料庫具有特定的定序，請為取用者資料庫使用相同的定序設定。

  • Redshift 不支援將外部結構描述、資料表或外部資料表上的近期繫結視觀表新增至資料共用。

  • Redshift 不支援生產者叢集上的巢狀 SQL 使用者定義函式。

  • Redshift 不支援使用交錯排序索引鍵的共用資料表，以及參考具有交錯排序索引鍵之資料表的檢視。

  • 取用者不能將資料共用物件新增到另一個資料共用。此外，取用者無法將參考資料共用物件的檢視新增到另一個資料共用中。

  • Redshift 不支援存取同時在準備和執行存取之間發生 DDL 的資料共用物件。

  • Redshift 不支援透過資料共用來共用預存程序。

如需資料共用限制的詳細資訊，請參閱 預覽版資料共用的要求和限制。
