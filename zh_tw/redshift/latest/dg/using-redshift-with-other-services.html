搭配其他服務使用 Amazon Redshift

Amazon Redshift 整合了其他的 AWS 服務，以協助您使用資料安全功能，快速可靠地移動、轉換和載入資料。

在 Amazon Redshift 和 Amazon S3 之間移動資料

Amazon Simple Storage Service (Amazon S3) 是將資料存放在雲端中的 Web 服務。Amazon Redshift 利用平行處理功能，從存放在 Amazon S3 儲存貯體中的多個資料檔案讀取和載入資料。如需詳細資訊，請參閱從 Amazon S3 載入資料。

您也可以利用平行處理模式，從 Amazon Redshift 資料倉儲將資料匯出至 Amazon S3 上的多個資料檔案。如需詳細資訊，請參閱卸載資料。

搭配 Amazon DynamoDB 使用 Amazon Redshift

Amazon DynamoDB 是全受管的 NoSQL 資料庫服務。您可以使用 COPY 命令，從單一 Amazon DynamoDB 資料表載入具有資料的 Amazon Redshift 資料表。如需詳細資訊，請參閱從 Amazon DynamoDB 資料表載入資料。

透過 SSH 從遠端主機匯入資料

您可以在 Amazon Redshift 中使用 COPY 指令，從一個或多個遠端主機載入資料，例如 Amazon EMR 叢集、Amazon EC2 執行個體或其他電腦。COPY 會使用 SSH 連接至遠端主機，然後在遠端主機執行命令來產生資料。Amazon Redshift 支援多個同時連線。COPY 指令會以平行方式，從多個主機資源讀取和載入輸出的資料。如需詳細資訊，請參閱從遠端主機載入資料。

使用 AWS Data Pipeline 自動化資料載入

您可以使用 AWS Data Pipeline，來自動將資料移出和移入 Amazon Redshift，並進行轉換。藉由使用 AWS Data Pipeline 的內建排程功能，您可以執行重複的任務並進行排程，而不需自行編寫複雜的資料傳輸或轉換邏輯。例如，您可以設定重複性的任務，自動將資料從 Amazon DynamoDB 複製到 Amazon Redshift。如需如何建立管道來將資料從 Amazon S3 定期移動到 Amazon Redshift 的教學課程，請參閱 AWS Data Pipeline 開發人員指南中的使用 AWS Data Pipeline 將資料複製到 Amazon Redshift。

使用 AWS Database Migration Service (AWS DMS) 遷移資料

您可以使用 AWS Database Migration Service，將資料遷移到 Amazon Redshift。AWS DMS 可在最廣為使用的商用和開放原始碼資料庫之間遷移您的資料，例如 Oracle、PostgreSQL、Microsoft SQL Server、Amazon Redshift、Aurora、DynamoDB、Amazon S3、MariaDB 和 MySQL。如需詳細資訊，請參閱使用 Amazon Redshift 資料庫做為 AWS Database Migration Service 的目標。
