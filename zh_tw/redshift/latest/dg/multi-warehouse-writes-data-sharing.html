在 AWS 帳戶 內或跨帳戶共用讀取和寫入資料 (預覽)

這是針對 Amazon Redshift 的多資料倉儲透過資料共用功能進行寫入的發行前版本文件，可在 PREVIEW_2023 追蹤公開預覽中取得。文件和功能會隨時變更。我們建議僅搭配測試叢集使用此功能，不要在生產環境中使用。如需預覽版條款與條件，請參閱 AWS 服務條款中的 Beta 版服務參與。
注意

ra3.xlplus 叢集目前無法使用透過資料共用的多倉儲寫入功能。若要使用此功能，請使用預覽追蹤建立 ra3.4xl 叢集、ra3.16xl 叢集或 Amazon Redshift Serverless 工作群組。

如需設定預覽追蹤的詳細資訊，請參閱下列之一：

  • 對於 Amazon Redshift Serverless 預覽：建立預覽工作群組

  • 對於 Amazon Redshift 佈建的叢集預覽：建立預覽叢集

您可以在相同的 AWS 帳戶 內或從一個叢集 AWS 帳戶 到另一個叢集，跨不同的 Amazon Redshift 叢集或 Amazon Redshift Serverless 工作群組共用讀取和寫入的資料庫物件。您也可以跨區域寫入資料。本主題中的程序展示設定資料共同的方法。您可以針對不同的資料表授予 SELECT、INSERT 和 UPDATE 等權限，並針對不同的結構描述授予 USAGE 和 CREATE 權限。一旦確認寫入交易，資料即可供所有倉儲使用。跨帳戶共用資料時，生產者帳戶管理員可選擇是否讓這些帳戶取得資料的唯讀存取權。此外，生產者帳戶管理員可以判斷特定的命名空間或區域是否取得唯讀 read-and-write，或是資料的任何存取權。

以下各節說明如何設定資料共用。這些過程假設您正在佈建的叢集或 Amazon Redshift Serverless 工作群組中的資料庫中工作。

唯讀資料共用與讀寫資料共用

之前資料共用中的物件在所有情況下都是唯讀的。寫入資料共用中的物件是一項新功能。只有當生產者特別授與寫入權限 (如 INSERT 或 CREATE) 給資料共用的物件時，資料共用中的物件才能啟用寫入功能。本主題後續章節中的詳細資訊如下。

如果您正在尋找唯讀資料共用的現有文件，請參閱 Amazon Redshift 中的跨叢集共用資料。

先決條件和使用須知

隔離設定 — 您的資料庫必須設定為快照隔離。預設情況下，在 PREVIEW_2023 追蹤中建立的資料庫為快照隔離。您可以透過使用 CREATE DATABASE 或 ALTER DATABASE 陳述式中的 ISOLATION 關鍵字來變更隔離設定。

注意

ra3.xlplus 叢集目前無法使用透過資料共用的多倉儲寫入功能。若要使用此功能，請使用預覽追蹤建立 ra3.4xl 叢集、ra3.16xl 叢集或 Amazon Redshift Serverless 工作群組。

以生產者資料庫管理員的身分共用資料庫物件，以便在資料共用中進行讀取和寫入作業

若要開始資料共用，生產者上的管理員會建立資料共用，並將物件加入至其中：

  1. 生產者資料庫擁有者或超級使用者會建立資料共用。資料共用是資料庫物件、許可權和取用者的邏輯容器。(取用者是您帳戶和其他帳戶中的叢集或 Amazon Redshift Serverless 命名空間)。每個資料共用都與其建立的資料庫相關聯，只能新增來自該資料庫的物件。以下命令會建立資料共用。

    CREATE DATASHARE my_datashare [PUBLICACCESSIBLE = TRUE];

    設定 PUBLICACCESSIBLE = TRUE 可讓取用者從可公開存取的叢集和已佈建的工作群組查詢您的資料共用。如果您不允許，請將其忽略，或明確地將其設定為 false。

    資料共用所有者必須授予他們想要新增到資料共用的結構描述的 USAGE。GRANT 命令是新的。它用於授予結構描述上的各種操作，包括 CREATE 和 USAGE.用。結構描述包含共用物件：

    CREATE SCHEMA myshared_schema1;
CREATE SCHEMA myshared_schema2;
 
GRANT USAGE ON SCHEMA myshared_schema1 TO DATASHARE my_datashare;
GRANT CREATE, USAGE ON SCHEMA myshared_schema2 TO DATASHARE my_datashare;

    或者，系統管理員也可以繼續執行 ALTER 命令，將結構描述新增至資料共用。以這種方式新增結構描述時，只會授與 USAGE 權限。

    ALTER DATASHARE my_datashare ADD SCHEMA myshared_schema1;

  2. 管理員新增結構描述之後，就可以授與結構描述中物件的資料共用權限。這些是讀取和寫入權限 GRANT ALL 範例顯示如何授與所有權限。

    GRANT SELECT, INSERT ON TABLE myshared_schema1.table1, myshared_schema1.table2, myshared_schema2.table1
TO DATASHARE my_datashare;
                     
GRANT ALL ON TABLE myshared_schema1.table4 TO DATASHARE my_datashare;

    您可以繼續執行像 ALTER DATASHARE 這樣的命令來新增資料表。當您這麼做時，只會授與新增物件的 SELECT 權限。

    ALTER DATASHARE my_datashare ADD TABLE myshared_schema1.table1, myshared_schema1.table2, myshared_schema2.table1;

  3. 系統管理員會將資料共用的使用權授與帳戶中的特定命名空間。您可以在叢集詳細資料頁面、Amazon Redshift Serverless 命名空間詳細資料頁面，或藉由執行命令 SELECT current_namespace;，來尋找命名空間 ID 以做為 ARN 的一部分。如需詳細資訊，請參閱 CURRENT_NAMESPACE前。

    GRANT USAGE ON DATASHARE my_datashare TO NAMESPACE '86b5169f-012a-234b-9fbb-e2e24359e9a8';

將資料共用授權給另一個帳戶

在生產者上建立資料共用，並為其新增物件後，您可能需要與另一個 AWS 帳戶共用該物件。將資料共用授權給其他帳戶使用，這是次要安全程序，必須由資料安全管理員在生產者上執行。它將資料共用授權給取用者帳戶取用。您可以將資料共用授權給另一個帳戶，以用於唯讀目的，或用於讀寫目的。如果授權寫入，則允許該帳戶具有授予資料共用的寫入權限。若非如此，則只能使用授予資料共用的讀取權限。如需如何執行此作業的資訊，請參閱 以生產者安全管理員身分授權資料共用以進行寫入 (預覽)。

將來自另一個帳戶的資料共用相關聯為取用者

如果資料共用是來自其他帳戶，您必須使其與帳戶中的特定叢集或命名空間產生關聯，才能與這些叢集或命名空間搭配使用。您可以使其與帳戶中的所有命名空間、帳戶中特定區域中的所有命名空間，或帳戶中的特定命名空間產生關聯。您可以選擇是否允許命名空間唯讀存取，或允許讀取和寫入存取。如果與寫入相關聯，則允許相關聯的叢集和 Serverless 命名空間具有授與資料共用的寫入權限。若非如此，則只能使用授予資料共用的讀取權限。有關如何執行此操作的詳細資訊，請參閱將共用資料關聯為取用者資料安全管理員 (預覽)。

您可以授予資料共用的權限

您可以在資料共用內容中授與不同的物件類型和各種權限。

結構描述：

  • USAGE

  • CREATE

資料表：

  • SELECT

  • INSERT

  • UPDATE

  • DELETE

  • TRUNCATE

  • DROP

  • REFERENCES

函數：

  • EXECUTE

讓叢集或 Serverless 工作群組查詢資料共用

此步驟假設資料共用源自您帳戶中的其他叢集或 Amazon Redshift Serverless 命名空間，或者來自另一個帳戶，且已與您正在使用的命名空間相關聯。

  1. 取用者資料庫管理員可以從資料共用建立資料庫。

    CREATE DATABASE my_ds_db [WITH PERMISSIONS] FROM DATASHARE my_datashare OF NAMESPACE 'abc123def';

    如果您建立具有權限的資料庫，您可以將資料共用物件的細微權限授與不同的使用者和角色。如果沒有這個功能，所有被授與資料共用資料庫 USAGE 權限的使用者和角色，都會被授與資料共用資料庫內所有物件的所有權限。

  2. 以下說明如何將權限授與 Redshift 資料庫使用者或角色。您必須連線到本機資料庫，才能執行這些陳述式。如果您在執行授予陳述式之前，在資料共用資料庫上執行 USE 命令，則無法執行這些陳述式。

    GRANT USAGE ON DATABASE my_ds_db TO ROLE data_eng;
GRANT CREATE, USAGE ON SCHEMA my_ds_db.my_shared_schema TO ROLE data_eng;
GRANT ALL ON ALL TABLES IN SCHEMA my_ds_db.my_shared_schema TO ROLE data_eng;
 
GRANT USAGE ON DATABASE my_ds_db TO bi_user;
GRANT USAGE ON SCHEMA my_ds_db.my_shared_schema TO bi_user;
GRANT SELECT ON my_ds_db.my_shared_schema.table1 TO bi_user;

寫入資料共用中的物件

有兩種方法可以在資料共用內寫入物件。

使用 USE 命令，連接到本機資料庫

使用此命令，您可以使用兩部分符號查詢 datashARE 的對象，並運行多語句事務。您只能在單一資料庫中執行多重陳述式交易。

USE database_name;

您可以透過執行 RESET USE 命令返回使用連線的資料庫。

RESET USE;

一旦使用資料庫，您可以開始在該資料庫中交易。您無法在單一交易中寫入多個資料庫。

重要

如果您在沒有第一次認可或結束目前交易的情況下執行 USE 命令，資料倉儲將會重新啟動。請務必在執行 USE 陳述式之前，結束或認可開展交易。

USE database_name;    --set the context to the datashare database
 
INSERT INTO datashare_schema.table1 VALUES (1);
 
BEGIN; --start a transaction
 
INSERT INTO datashare_schema.table2 VALUES (1);
INSERT INTO datashare_schema.table3 VALUES (1);
 
COMMIT;  --end transaction; you can also use the END command.
注意

您無法將資料從 parquet 和 ORC 檔案複製到資料共用表。

透過 Redshift JDBC，ODBC 或 Python 驅動程式直接連接到資料共用資料庫

如果您直接連接到資料共用資料庫，則可以使用兩部分記號查詢表，並立即在資料共用資料庫中執行多陳述式交易。

--connect to the datashare database via JDBC, ODBC, or the Python Driver
 
INSERT INTO datashare_schema.table1 VALUES (1);
 
BEGIN; --start a transaction
 
INSERT INTO datashare_schema.table2 VALUES (1);
INSERT INTO datashare_schema.table3 VALUES (1);
 
COMMIT;  -- end transaction; can also use the END command.
重要

支援執行 INSERT INTO SELECT 陳述式，但若要這麼做，您必須讓查詢中參照的所有資料表具有 INSERT 權限。

注意

即將提供對透過資料 API 連線至資料共用資料庫的支援。

預覽版資料共用的要求和限制

  • 您必須直接連接至資料共用資料庫，或執行 USE 命令以寫入資料庫。不過我們很快將能夠透過三部分記號來實現這項操作。

  • 您必須使用 Serverless 工作群組、ra3.4xl 叢集或 ra3.16xl 叢集才能使用此功能。已規劃為 ra3.xlplus 叢集提供支援。

  • 中繼資料探索 — 當您是透過 Redshift JDBC、ODBC 或 Python 驅動程式直接連線至資料存取資料庫的取用者時，您可以透過下列方式檢視目錄資料：

    • SQL SHOW 命令。

    • 查詢 information_schema 資料表和視觀表。

    • 查詢 SVV 中繼資料檢視。

  • 您無法透過資料 API 連線至資料共用資料庫。即將提供相關支援。

  • 取用者看不到授與資料共用的權限。我們很快就會新增這項功能。

  • 對於跨帳戶資料共用，生產者和取用者叢集都必須加密。

  • 資料庫的隔離層級必須是快照隔離，才能允許其他 Serverless 工作群組和叢集寫入資料庫。

  • 寫入資料共用物件的取用者不會觸發自動分析作業。因此，生產者必須在將資料插入資料表後手動執行分析，才能更新資料表統計資料。如果沒這麼做，查詢計劃可能不是最好的。

支援的 SQL 陳述式

以下陳述式適用於透過寫入進行資料共用的公開預覽版本：

  • BEGIN | START TRANSACTION

  • END | COMMIT | ROLLBACK

  • COPY 不含 COMPUPDATE

  • { CREATE | DROP } SCHEMA

  • { CREATE | DROP | SHOW } TABLE

  • CREATE TABLE table_name AS

  • DELETE

  • { GRANT | REVOKE } privilege_name ON OBJECT_TYPE object_name TO consumer_user

  • INSERT

  • SELECT

  • INSERT INTO SELECT

  • TRUNCATE

  • UPDATE

  • 超級資料類型列欄

不支援的陳述式類型 — 不支援下列項目：

  • 寫入生產者時，對取用者倉庫進行多陳述式查詢。

  • 並行擴縮查詢從取用者寫入生產者。

  • 從取用者寫入到生產者的自動複製作業。

  • 從取用者到生產者的串流工作寫入。

  • 在生產者叢集上建立零 ETL 整合表的取用者。如需零 ETL 整合的相關資訊，請參閱使用零 ETL 整合。

  • 使用交錯排序索引鍵寫入資料表。
