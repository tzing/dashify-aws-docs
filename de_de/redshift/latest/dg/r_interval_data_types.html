Intervall-Datentypen

Sie können einen Intervalldatentyp verwenden, um Zeiträume oder Zeitbereiche anzugeben, z. B. 12 hours6 weeks, oder 1 month. Mit INTERVAL-Datentypen können Sie Zeiträume oder Zeitbereiche angeben, z. B. 12 hours6 weeks, oder 1 month. Sie können Intervalldatentypen in Berechnungen verwenden, z. B. die Berechnung von Unterschieden zwischen Datetimes oder Intervallen, Summen von Intervallen oder Summen einer Datetime und eines Intervalls.

Die folgenden Intervalle werden unterstützt:

  • JAHR ZU MONAT

  • TAG AUF SEKUNDE

Syntax

So erstellen Sie ein Intervall vom Typ JEAR TO MONTH:

INTERVAL <year_to_month_qualifier>

So erstellen Sie ein Intervallliteral vom Typ JEAR TO MONTH:

INTERVAL <interval-string> <qualifier>

Verwenden Sie die folgende Syntax, um ein Intervall vom Typ DAY TO SECOND zu erstellen. Der fractional_precision Parameter ist nur erforderlich, wenn Ihr Intervall SECOND enthält.

INTERVAL <qualifier> [ <fractional_precision> ]

Verwenden Sie die folgende Syntax, um ein Intervallliteral vom Typ DAY TO SECOND zu erstellen.

INTERVAL <interval-string> <qualifier> [ <fractional_precision> ]

Argumente

Intervallzeichenfolge

Gibt den Wert des Intervalls an. Nur erforderlich, wenn eine Intervallkonstante erstellt wird

Qualifikator

Gibt den Bereich des Intervalls an. Wenn Sie einen Qualifizierer verwenden und ein Intervall mit Zeiteinheiten erstellen, die kleiner als der Qualifizierer sind, kürzt Amazon Redshift die kleineren Teile des Intervalls und verwirft sie. Das folgende Beispiel zeigt die Verwendung eines MINUTE-Qualifizierers und Amazon Redshift verwirft die Zeiteinheiten, die kleiner als MINUTE sind.

INTERVAL '1 day 1 hour 1 minute 1.123 seconds' MINUTE

wird auf gekürzt 

'1 day 01:01:00'
Bruchwert_Genauigkeit

Optionaler Parameter, der die Anzahl der im Intervall zulässigen Nachkommastellen angibt. SECOND (3) erstellt beispielsweise ein Intervall, das nur drei Nachkommastellen zulässt, z. B. 1,234 Sekunden. Gilt nur für Intervalle von DAY TO SECOND, die Sekunden enthalten. Die maximale Anzahl unterstützter Ziffern beträgt sechs.

Der Datentyp YEAR TO MONTH unterstützt die folgenden Qualifikatoren:

  • JAHR

  • MONAT

  • JAHR ZU MONAT

Der Datentyp DAY TO SECOND unterstützt die folgenden Qualifikatoren:

  • TAG

  • STUNDE

  • MINUTE

  • SECOND

  • TAG AUF STundE

  • TAG AUF MINUTE

  • TAG AUF SEKUNDE

  • STUNDE AUF MINUTE

  • STUNDE AUF SEKUNDE

  • MINUTEN AUF SEKUNDEN

Beispiele

Das folgende Beispiel zeigt, wie eine INTERVAL-Konstante von 1 Jahr erstellt wird.

INTERVAL '1' YEAR

Wenn Sie eine Intervallzeichenfolge angeben, die den Qualifizierer überschreitet, werden die verbleibenden Zeiteinheiten aus dem Intervall gekürzt. Im folgenden Beispiel wird ein Intervall von 13 Monaten zu 1 Jahr und 1 Monat, aber die verbleibenden 1 Monat werden aufgrund des YEAR-Qualifizierers ausgelassen.

INTERVAL '13 months' YEAR
'1 year'

Wenn Sie einen Qualifizierer verwenden, der niedriger als Ihre Intervallzeichenfolge ist, sind die verbleibenden Einheiten enthalten.

INTERVAL '13 months' MONTH
'1 year 1 month'

Die Angabe einer Genauigkeit in Ihrem Intervall kürzt die Anzahl der Nachkommastellen auf die angegebene Genauigkeit.

INTERVAL '1.234567' SECOND (3)
'00:00:01.234'

Wenn Sie keine Genauigkeit angeben, verwendet Amazon Redshift die maximale Genauigkeit von 6.

INTERVAL '1.23456789' SECOND
'00:00:01.234567'

Das folgende Beispiel zeigt, wie Sie ein Bereichsintervall erstellen.

INTERVAL '2:2' MINUTE TO SECOND
'00:02:02'

Qualifikatoren bestimmen die Einheiten, die Sie angeben. Obwohl das folgende Beispiel beispielsweise dieselbe Intervallzeichenfolge von „2:2“ wie das vorherige Beispiel verwendet, erkennt Amazon Redshift aufgrund des Qualifikators, dass es unterschiedliche Zeiteinheiten verwendet

INTERVAL '2:2' HOUR TO MINUTE
'02:02:00'

Die Einheiten können abgekürzt und in ihren Pluralformen verwendet werden. Beispiele: 5s, 5 second und 5 seconds drücken dasselbe Intervall aus. Unterstützte Einheiten sind Jahre, Monate, Stunden, Minuten und Sekunden.

INTERVAL '5s' SECOND
'00:00:05'

INTERVAL '5 second' SECOND
'00:00:05'
                     
INTERVAL '5 HOURS' HOUR
'05:00:00'
                     
INTERVAL '5 h' HOUR
'05:00:00

Die folgenden Beispiele zeigen, wie INTERVAL-Datentypen mit Tabellen verwendet werden

create table sample_intervals (y2m interval month, m2s interval hour to minute);
insert into sample_intervals values (interval '20' month, interval '2 days 1:1:1.123456' day to second);
select y2m::text, m2s::text from sample_intervals;

      y2m      |       m2s       
---------------+-----------------
 1 year 8 mons | 2 days 01:01:00

update sample_intervals set y2m = interval '2' year where y2m = interval '1-8' year to month;
select * from sample_intervals;

   y2m   |       m2s       
---------+-----------------
 2 years | 2 days 01:01:00

delete from sample_intervals where m2s = interval '2 1:1:0' day to second;
select * from sample_intervals;

 y2m | m2s 
-----+-----

Intervallstile

Sie können den SET Befehl verwenden, um das Ausgabeformat Ihrer Intervallzeichenfolgen zu ändern. Verfügbare Optionen sind: postgressql_standard, und postgres_verbose. Der Standardwert ist postgres. Beispiele für jedes Ausgabeformat sind unten aufgeführt.

Ausgabeformat sql_standard

Die Werte für das Intervall von Jahr zu Monat sind wie folgt formatiert. Wenn Sie negativ vor dem Intervall angeben, ist das Intervall ein negativer Wert und gilt für das gesamte Intervall.

'[-]yy-mm'

Die Werte für das Intervall von Tag zu Sekunde sind wie folgt formatiert.

'[-]dd hh:mm:ss.ffffff'
INTERVAL '1-2' YEAR TO MONTH outputs as '1-2'
                     
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 2:03:04.5678'

postgres-Ausgabeformat

Im Folgenden finden Sie das Ausgabeformat für postgres. Jeder numerische Wert kann negativ sein.

[@] '<numeric> <unit> [, <numeric> <unit> ...]'
INTERVAL '1-2' YEAR TO MONTH outputs as '1 year 2 mons'
                  
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 day 02:03:04.5678'

postgres_verbose-Ausgabeformat

Die postgres_verbose-Syntax ähnelt postgres, aber postgres_verbose-Ausgaben enthalten auch die Zeiteinheit.

'[@] <numeric> <unit> [, <numeric> <unit> ...] [direction]'
INTERVAL '1-2' YEAR TO MONTH outputs as '@ 1 year 2 mons'

INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '@ 1 day 2 hours 3 mins 4.5678 secs'

Der folgende Befehl zeigt, wie der Intervallstil auf festgelegt wirdsql_standard.

SET intervalstyle to 'sql_standard';

Einschränkungen

Diese INTERVAL-Funktion vor der Veröffentlichung hat die folgenden Einschränkungen:

  • Wenn Sie den JDBC-Treiber verwenden, können Sie die Intervalle von Jahr zu Monat und Tag zu Sekunde nicht als Parameter für PREPARE-Anweisungen verwenden.

  • Der JDBC-Treiber interpretiert alle Intervallwerte als Zeichenfolgen, sodass Sie die zurückgegebenen Intervalle nur mit der getString Methode aus einer SELECT-Abfrage abrufen können.

  • Gespeicherte Prozeduren akzeptieren keine Intervallwerte als Funktionsparameter.

  • Sie können keine Umwandlung einer Zahl in ein Intervall durchführen.
