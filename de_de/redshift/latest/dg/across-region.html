Freigeben von Daten über AWS-Regionen hinweg

Sie können Daten für Lesezwecke übergreifend über Amazon-Redshift-Cluster in AWS-Regionen freigeben. Mit regionsübergreifender Datenfreigabe können Sie Daten übergreifend über AWS-Regionen ohne manuelles Kopieren von Daten freigeben. Sie müssen für Ihre Daten kein Unload in Amazon S3 durchführen und die Daten nicht in einen neuen Amazon-Redshift-Cluster kopieren oder eine regionsübergreifende Snapshot-Kopie durchführen.

Mit regionsübergreifender Datenfreigabe können Sie Daten über Cluster hinweg im selben AWS-Konto oder in einem anderen AWS-Konten freigeben, auch wenn sich die Cluster in verschiedenen Regionen befinden. Bei der Freigabe von Daten für Amazon-Redshift-Cluster, die sich in demselben AWS-Konto, aber anderen AWS-Regionen befinden, folgen Sie demselben Workflow wie bei der Freigabe von Daten innerhalb von einem AWS-Konto. Weitere Informationen finden Sie unter Freigeben von Daten innerhalb eines AWS-Konto.

Wenn Cluster die Daten in unterschiedlichen AWS-Konten und AWS-Regionen freigeben, können Sie demselben Workflow wie bei der Freigabe von Daten übergreifend über AWS-Konten folgen und auf Regionsebene im Konsumentencluster Zuordnungen einschließen. Die regionsübergreifende Datenfreigabe unterstützt die Datashare-Zuordnung mit dem gesamten AWS-Konto, der gesamten AWS-Region oder bestimmten Cluster-Namespaces innerhalb einer AWS-Region. Weitere Informationen zur Datenfreigabe übergreifend über AWS-Konten finden Sie unter Freigeben von Daten über AWS-Konten hinweg.

Beim Verbrauchen von Daten aus einer anderen Region zahlt der Verbraucher die regionsübergreifende Datenübertragungsgebühr von der Produzentenregion zur Konsumentenregion.

Um das Datashare zu verwenden, kann ein Administrator des Konsumentenkontos das Datashare auf eine der folgenden drei Arten zuordnen.

  • Zuordnung mit einem gesamten AWS-Konto erstreckt sich über all seine AWS-Regionen

  • Zuordnung mit einer bestimmten AWS-Region in einem AWS-Konto

  • Zuordnung mit bestimmten Cluster-Namespaces innerhalb einer AWS-Region

Wenn der Administrator das gesamte AWS-Konto auswählt, haben alle vorhandenen und zukünftigen Cluster-Namespaces in verschiedenen AWS-Regionen im Konto Zugriff auf die Datashares. Ein Administrator eines Konsumentenkontos kann auch bestimmte AWS-Regionen auswählen oder Cluster-Namespaces innerhalb einer Region, um ihnen Zugriff auf die Datashares zu gewähren.

Wenn Sie ein Administrator eines Produzenten-Clusters oder Datenbankbesitzer sind, erstellen Sie ein Datashare, fügen Sie dem Datashare Datenbankobjekte und Datenkonsumenten hinzu und erteilen Sie den Datenkonsumenten Berechtigungen. Weitere Informationen finden Sie unter Aktionen des Administrators des Produzenten-Clusters.

Wenn Sie ein Administrator des Produzentenkontos sind, autorisieren Sie Datashares über die AWS Command Line Interface (AWS CLI) oder über die Amazon-Redshift-Konsole und wählen Sie die Datenkonsumenten aus.

Wenn Sie in Administrator eines Konsumentenkontos sind, gehen Sie wie folgt vor:

Verwenden Sie die Amazon-Redshift-Konsole, um einen oder mehrere Datashares zuzuordnen, die von anderen Konten für Ihr gesamtes AWS-Konto oder bestimmte AWS-Regionen oder Cluster-Namespaces in einer AWS-Region freigegeben werden.

Mit der regionsübergreifenden Datenfreigabe können Sie mit der AWS Command Line Interface (AWS CLI) oder der Amazon-Redshift-Konsole Cluster in einer bestimmten AWS-Region hinzufügen.

Um eine oder mehrere AWS Regionen anzugeben, können Sie den CLI-Befehl associate-data-share-consumer mit der optionalen Option consumer-region verwenden.

Im folgenden Beispiel wird Salesshare über die CLI mit der Option associate-entire-account dem ganzen AWS-Konto zugeordnet. Sie können jeweils nur eine Region zuordnen.

aws redshift associate-data-share-consumer
--region {PRODUCER_REGION}
--data-share-arn arn:aws:redshift:{PRODUCER_REGION}:{PRODUCER_ACCOUNT}:datashare:{PRODUCER_CLUSTER_NAMESPACE}/Salesshare
--associate-entire-account

Im folgenden Beispiel wird Salesshare der Region USA Ost (Ohio) (us-east-2) zugeordnet.

aws redshift associate-data-share-consumer
--region {PRODUCER_REGION}
--data-share-arn arn:aws:redshift:{PRODUCER_REGION}:0123456789012:datashare:{PRODUCER_CLUSTER_NAMESPACE}/Salesshare
--consumer-region 'us-east-2'

Im folgenden Beispiel wird Salesshare einem bestimmten Konsumenten-Cluster-Namespace in einem anderen AWS-Konto in der Region Asien-Pazifik (Sydney) (ap-southeast-2) zugeordnet.

aws redshift associate-data-share-consumer
--data-share-arn arn:aws:redshift:{PRODUCER_REGION}:{PRODUCER_ACCOUNT}:datashare:{PRODUCER_CLUSTER_NAMESPACE}/Salesshare
--consumer-arn 'arn:aws:redshift:ap-southeast-2:{CONSUMER_ACCOUNT}:namespace:{ConsumerImmutableClusterId}'

Sie können die Amazon-Redshift-Konsole verwenden, um Datashares Ihrem gesamten AWS-Konto oder spezifischen AWS-Regionen oder Cluster-Namespaces innerhalb einer AWS-Region zuzuordnen. Melden Sie sich dazu bei https://console.aws.amazon.com/redshift/ an. Ordnen Sie dann einen oder mehrere Datashares zu, die von anderen Konten für Ihr gesamtes AWS-Konto, die gesamte AWS-Region oder oder bestimmte Cluster-Namespaces in einer AWS-Region freigegeben werden. Weitere Informationen finden Sie unter Zuordnen von Datashares.

Nachdem das AWS-Konto oder bestimmte Cluster-Namespaces zugeordnet sind, stehen die Datashares zur Verfügung. Sie können die Zuordnung des Datashares auch jederzeit ändern. Beim Ändern der Zuordnung von einzelnen Cluster-Namespaces zu einem AWS-Konto überschreibt Amazon Redshift die Cluster-Namespaces mit den AWS-Konto-Informationen. Beim Ändern der Zuordnung von einem AWS-Konto zu einzelnen Cluster-Namespaces überschreibt Amazon Redshift die AWS-Konto-Informationen mit den Cluster-Namespaces. Beim Ändern der Zuordnung von einem gesamten AWS-Konto zu einzelnen AWS-Regionen und Cluster-Namespaces überschreibt Amazon Redshift die AWS-Konto-Informationen mit den einzelnen Regionen und Cluster-Namespaces.

Wenn Sie Administrator für einen Konsumenten-Cluster sind, können Sie lokale Datenbanken erstellen, die auf die Datashares verweisen, und Benutzern oder Rollen im Konsumenten-Cluster nach Bedarf Berechtigungen für Datenbanken gewähren, die aus den Datashares erstellt wurden. Sie können auch Ansichten für freigegebene Objekte und externe Schemas erstellen, um auf bestimmte Schemas in der Konsumenten-Datenbank zu verweisen, die im Konsumenten-Cluster importiert wurden, und granulare Berechtigungen zuweisen. Weitere Informationen finden Sie unter Aktionen für Administratoren von Konsumenten-Clustern.
