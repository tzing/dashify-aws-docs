Verwalten von AWS CloudFormation Ereignissen mit Amazon EventBridge

Amazon EventBridge ist ein Serverless-Service, der Ereignisse verwendet, um Anwendungskomponenten miteinander zu verbinden, sodass Sie einfacher skalierbare ereignisgesteuerte Anwendungen erstellen können. Bei der ereignisgesteuerten Architektur werden lose gekoppelte Softwaresysteme entwickelt, die zusammenarbeiten, indem sie Ereignisse senden und darauf reagieren. Ereignisse stellen eine Änderung in einer Ressource oder Umgebung dar.

Wie bei vielen - AWS Services AWS CloudFormation generiert Ereignisse und sendet sie an den EventBridge Standard-Event-Bus. (Der Standard-Event-Bus wird automatisch in jedem AWS Konto bereitgestellt.) Ein Event Bus ist ein Router, der Ereignisse empfängt und sie an null oder mehr Ziele weiterleitet. Regeln, die Sie für den Event Bus angeben, werten Ereignisse aus, sobald sie eintreffen. Jede Regel prüft, ob ein Ereignis mit dem Ereignismuster der Regel übereinstimmt. Wenn das Ereignis übereinstimmt, sendet der Event Bus das Ereignis an das/die angegebenen Ziel(e).

CloudFormation -Ereignisse

AWS CloudFormation sendet Ereignisse an , EventBridge wenn eine Operation zum Erstellen, Aktualisieren, Löschen oder Erkennen von Abweichungen auf einem Stack ausgeführt wird. sendet AWS CloudFormation auch Ereignisse an EventBridge , um Statusänderungen an Stack-Sets und Stack-Set-Instances zu erhalten. Sie können EventBridge Regeln verwenden, um Ereignisse an Ihre definierten Ziele weiterzuleiten. Diese Ereignisse werden auf Grundlage einer optimalen Leistung übermittelt und können in falscher Reihenfolge geliefert werden.

Da CloudFormation Ereignisse Änderungen an Stacks oder Stack-Sets und ihren Ressourcen darstellen, können Sie sie verwenden, um Workflows zu initiieren, die den jeweiligen Ereignissen zugeordnet sind. Beispielsweise:

  • Erstellen Sie Stack- oder Stack-Set-spezifische Tags für alle Ressourcen, die über AWS CloudFormationbereitgestellt werden.

  • Stellen Sie eine Zuordnung zwischen einem CloudFormation Stack oder Stack-Set und einem Amazon WorkSpaces Application Manager (Amazon WAM) her.

  • Geben Sie eine Zuordnung zu einem AppRegistry für den erstellten Stack oder das erstellte Stack-Set an.

Die folgenden Ereignisse werden von generiert CloudFormation und an den Standard-Event-Bus in gesendet EventBridge. Weitere Informationen finden Sie unter AWS CloudFormation Referenz zu Ereignisdetails.

Senden von AWS CloudFormation Ereignissen mithilfe von EventBridge Regeln

Damit der EventBridge Standard-Event-Bus AWS CloudFormation Ereignisse an ein Ziel sendet, müssen Sie eine Regel erstellen, die ein Ereignismuster enthält, das den Daten in den gewünschten AWS CloudFormation Ereignissen entspricht.

Das Erstellen einer Regel besteht aus den folgenden allgemeinen Schritten:

  1. Erstellen eines Ereignismusters für die Regel, das Folgendes angibt:

    • AWS CloudFormation ist die Quelle der Ereignisse, die von der Regel ausgewertet werden.

    • (Optional): Alle anderen Ereignisdaten, mit denen verglichen werden soll.

    Weitere Informationen finden Sie unter Erstellen von Ereignismustern für AWS CloudFormation Ereignisse.

  2. (Optional): Erstellen eines Eingabe-Transformators, der die Daten aus dem Ereignis anpasst, bevor die Informationen an das Ziel der Regel EventBridge übergibt.

    Weitere Informationen finden Sie unter Eingabetransformation im EventBridge -Benutzerhandbuch.

  3. Geben Sie das/die Ziel(e) an, an das/die Ereignisse übermittelt EventBridge werden sollen, die dem Ereignismuster entsprechen.

    Ziele können andere - AWS Services, software-as-a-service (SaaS)-Anwendungen, API-Ziele oder andere benutzerdefinierte Endpunkte sein. Weitere Informationen finden Sie unter Ziele im Benutzerhandbuch fürEventBridge .

Umfassende Anweisungen zum Erstellen von Event-Bus-Regeln finden Sie unter Erstellen von Regeln, die auf Ereignisse reagieren im EventBridge -Benutzerhandbuch.

Erstellen von Ereignismustern für AWS CloudFormation Ereignisse

Wenn ein Ereignis an den Standard-Event-Bus AWS CloudFormation übermittelt, EventBridge verwendet das für jede Regel definierte Ereignismuster, um zu bestimmen, ob das Ereignis an das Ziel(e) der Regel übermittelt werden soll. Ein Ereignismuster entspricht den Daten in den gewünschten AWS CloudFormation Ereignissen. Jedes Ereignismuster ist ein JSON-Objekt, das Folgendes enthält:

  • Ein source-Attribut, das den Service identifiziert, der das Ereignis sendet. Bei - AWS CloudFormation Ereignissen ist die Quelle aws.cloudformation.

  • (Optional): Ein detail-type-Attribut, das ein Array der zuzuordnenden Ereignistypen enthält.

  • (Optional): Ein detail-Attribut, das alle anderen Ereignisdaten für den Abgleich enthält.

    Dazu gehören zum Beispiel die Stack-ID, die beteiligten Ressourcen, der Status verschiedener Ressourcen und andere Daten, die für einen bestimmten Ereignistyp relevant sind.

Das folgende Ereignismuster beispielsweise passt zu allen Ereignissen zur Änderung des Ressourcenstatus:

{
  "source": ["aws.cloudformation"],
  "detail-type": ["CloudFormation Resource Status Change"]
}

Während das folgende Ereignismuster Ereignisdetaildaten verwendet, um nur mit Ereignissen zur Änderung des Ressourcenstatus übereinzustimmen, bei denen eine neue - AWS::S3::Bucket oder -AWS::SNS::TopicRessource CloudFormation erstellt:

{
  "source": ["aws.cloudformation"],
  "detail-type": ["CloudFormation Resource Status Change"],
  "detail": {
    "status-details": {
      "status": ["CREATE_COMPLETE"]
    },
    "resource-type": ["AWS::S3::Bucket", "AWS::SNS::Topic"]
  }
}

Weitere Informationen zum Schreiben von Ereignismustern finden Sie unter Ereignismuster im EventBridge -Benutzerhandbuch.

Testen von Ereignismustern auf AWS CloudFormation Ereignisse in EventBridge

Sie können die EventBridge Sandbox verwenden, um schnell ein Ereignismuster zu definieren und zu testen, ohne den größeren Prozess zum Erstellen oder Bearbeiten einer Regel abschließen zu müssen. Mit der Sandbox können Sie ein Ereignismuster definieren und anhand eines Beispielereignisses bestätigen, dass das Muster mit den gewünschten Ereignissen übereinstimmt. EventBridge Geben Sie Ihnen die Möglichkeit, eine neue Regel mit diesem Ereignismuster direkt aus der Sandbox zu erstellen.

Weitere Informationen finden Sie unter Testen eines Ereignismusters mit der EventBridge Sandbox im EventBridge -Benutzerhandbuch.

Amazon EventBridge -Berechtigungen

AWS CloudFormation benötigt keine zusätzlichen Berechtigungen, um Ereignisse an zu übermitteln Amazon EventBridge. Die Ereignisse enthalten Informationen, die bereits über die API-Operationen CloudFormationvon verfügbar sind.

Die von Ihnen angegebenen Ziele benötigen möglicherweise bestimmte Berechtigungen oder Konfigurationen. Weitere Informationen zur Verwendung bestimmter Services für Ziele finden Sie unter Amazon EventBridge Ziele im Amazon EventBridge -Benutzerhandbuch.
