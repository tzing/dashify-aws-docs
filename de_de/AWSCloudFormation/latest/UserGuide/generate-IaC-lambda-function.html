Aktualisieren von AWS::Lambda::Function Ressourcen

Wenn eine generierte Vorlage AWS::Lambda::Function Ressourcen enthält, werden Warnungen generiert, die besagen, dass - Code/S3Bucket und -Code/S3KeyEigenschaften als identifiziert werdenMUTUALLY_EXCLUSIVE_PROPERTIES. Darüber hinaus erhält die -Code/S3ObjectVersionEigenschaft eine UNSUPPORTED_PROPERTIES Warnung. Möglicherweise stellen Sie fest, dass die Code/ZipFile Eigenschaften Code/ImageUri und in der generierten Vorlage fehlen. Dies liegt daran, dass es sich um sich gegenseitig ausschließende Schreibeigenschaften handelt. Jede AWS::Lambda::Function Ressource kann nur einen dieser exklusiven Eigenschaften haben:

  • Code/S3Bucket - und -Code/S3KeyEigenschaften und optional die -Code/S3ObjectVersionEigenschaft

  • Code/ImageUri Eigenschaft

  • Code/ZipFile Eigenschaft

Der IaC-Generator kann nicht bestimmen, welcher Satz exklusiver schreibgeschützter Eigenschaften zum Erstellen oder Aktualisieren der Ressource verwendet wurde. Daher enthält der IaC-Generator nur den ersten Satz von Eigenschaften in der generierten Vorlage. Um AWS::Lambda::Function Ressourcen in eine generierte Vorlage aufzunehmen, muss die generierte Vorlage heruntergeladen und mit den richtigen Informationen für den Lambda-Code aktualisiert werden.

Wenn Sie Ihren Lambda-Code in einem Amazon S3-Bucket speichern und die S3ObjectVersion Eigenschaft nicht verwenden, können Sie die generierte Vorlage ohne Änderungen importieren. Der IaC-Generator fragt Sie während des Importvorgangs nach dem Amazon S3-Bucket und -Schlüssel als Vorlagenparameter.

Wenn Sie Ihren Lambda-Code als Amazon-ECR-Repository speichern, können Sie Ihre Vorlage mithilfe der folgenden Anweisungen aktualisieren:

  1. Laden Sie die generierte Vorlage herunter.

  2. Entfernen Sie die Eigenschaften und die entsprechenden Parameter für die Code/S3Key Eigenschaften Code/S3Bucket und aus der generierten Vorlage.

  3. Ersetzen Sie die entfernten Eigenschaften in der generierten Vorlage durch die -Code/ImageUriEigenschaft, wobei Sie die URL für das Amazon-ECR-Repository angeben.

  4. Öffnen Sie die generierte Vorlage in der IaC-Generator-Konsole und wählen Sie die Schaltfläche Bearbeitene Vorlage importieren.

Wenn Sie Ihren Lambda-Code als in einer ZIP-Datei speichern, können Sie Ihre Vorlage mithilfe der folgenden Anweisungen aktualisieren:

  1. Laden Sie die generierte Vorlage herunter.

  2. Entfernen Sie die Eigenschaften und die entsprechenden Parameter für die Code/S3Key Eigenschaften Code/S3Bucket und aus der generierten Vorlage.

  3. Ersetzen Sie die entfernten Eigenschaften in der generierten Vorlage durch die -Code/ZipFileEigenschaft.

  4. Öffnen Sie die generierte Vorlage in der IaC-Generator-Konsole und wählen Sie die Schaltfläche Bearbeitene Vorlage importieren.

Wenn Sie keine Kopie Ihres Lambda-Codes haben, können Sie Ihre Vorlage mithilfe der folgenden Anweisungen aktualisieren:

  1. Verwenden Sie die AWS Lambda GetFunction -API-Aktion (z. B. mit dem aws lambda get-function AWS CLI Befehl .

  2. In der Antwort lautet der RepositoryType Parameter , S3 wenn sich der Code in einem Amazon S3-Bucket befindet, oder , ECR wenn sich der Code in einem Amazon-ECR-Repository befindet.

  3. In der Antwort enthält der Location Parameter eine vorsignierte URL, mit der Sie das Bereitstellungspaket 10 Minuten lang herunterladen können. Laden Sie den Code herunter.

  4. Laden Sie den Code in einen Amazon S3-Bucket hoch.

  5. Führen Sie einen Importvorgang mit der generierten Vorlage aus und geben Sie den Bucket-Namen und den Schlüssel als Parameterwerte an.
