StackSet Details zum Vorgangsstatus ändern

Im Folgenden finden Sie die Detailfelder für Ereignisse zur Änderung des StackSet Vorgangsstatus.

Die detail-type Felder source und sind enthalten, da sie bestimmte Werte für AWS CloudFormation Ereignisse enthalten.

{
  . . .,
  "detail-type": "CloudFormation StackSet Operation Status Change",
  "source": "aws.cloudformation",
  . . .,
  "detail": {
    "stack-set-arn" : "string",
    "stack-set-operation-id" : "string",
    "status-details": {
        "status": "string"
      }
    }
  }
}
detail-type

Identifiziert den Ereignistyp.

Bei Ereignissen mit dem StackSet Betriebsstatus lautet dieser Wert CloudFormation StackSet Operation Status Change.

source

Identifiziert den Service, aus dem das Ereignis stammt. Für CloudFormation -Ereignisse ist dieser Wert aws.cloudformation.

detail

Ein JSON-Objekt, das Informationen zum Ereignis enthält. Der Service, der das Ereignis generiert, bestimmt den Inhalt dieses Feldes.

Für Ereignisse mit dem StackSet Betriebsstatus umfassen diese Daten:

stack-set-arn

Der Amazon-Ressourcenname (ARN), der dem zugeordnet ist StackSet.

stack-set-operation-id

Die eindeutige ID, die der StackSet Operation zugeordnet ist.

status-details
status

Der StackSet Betriebsstatus.

Weitere Details finden Sie unter StackSets Statuscodes.

Zulässige Werte: RUNNING | SUCCEEDED | FAILED | STOPPING | STOPPED | QUEUED
