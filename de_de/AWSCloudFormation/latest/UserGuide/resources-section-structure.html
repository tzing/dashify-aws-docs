Ressourcen

Der Abschnitt Resources ist ein erforderlicher Abschnitt. In ihm deklarieren Sie die AWS-Ressourcen, die Ihr Stack enthalten soll, beispielsweise eine Amazon-EC2-Instance oder einen Amazon-S3-Bucket.

Syntax

Der Abschnitt Resources besteht aus dem Schlüsselnamen Resources. Die folgende Pseudovorlage veranschaulicht den Abschnitt Resources:

JSON

"Resources" : {
    "Logical ID" : {
        "Type" : "Resource type",
        "Properties" : {
            Set of properties
        }
    }
}

YAML

Resources:
  Logical ID:
    Type: Resource type
    Properties:
      Set of properties

Ressourcenfelder

Logical ID

Die logische ID muss alphanumerisch (A-Za-z0-9) und innerhalb der Vorlage eindeutig sein. Über diesen logischen Namen referenzieren Sie die Ressource in anderen Abschnitten der Vorlage. Wenn Sie zum Beispiel ein Amazon Elastic Block Store-Volume einer Amazon EC2-Instance zuweisen möchten, referenzieren Sie die logischen IDs, um die Blockspeicher der Instance zuzuordnen.

Neben einer logischen ID haben bestimmte Ressourcen auch eine physische ID. Sie ist der tatsächliche Name, der dieser Ressource zugewiesen ist, zum Beispiel die ID einer EC2-Instance oder der Name eines S3-Buckets. Nutzen Sie die physische ID, um Ressourcen außerhalb von AWS CloudFormation-Vorlagen zu referenzieren. Das funktioniert jedoch erst, nachdem diese Ressourcen erstellt wurden. Angenommen, Sie geben einer EC2-Instance-Ressource eine logische ID von MyEC2Instance. Wenn AWS CloudFormation die Instance erzeugt, generiert AWS CloudFormation automatisch eine physische ID (z. B. i-28f9ba55) und weist dieser zu. Diese physische ID können Sie verwenden, um die Instance zu referenzieren und ihre Eigenschaften (beispielsweise den DNS-Namen) über die Amazon-EC2-Konsole abzurufen. Ressourcen mit Unterstützung für benutzerdefinierte Namen können Sie eigene Namen (physische IDs) zuweisen, um sie schneller referenzieren zu können. So können Sie einem S3-Bucket, in dem Protokolle gespeichert werden, zum Beispiel den Namen MyPerformanceLogs geben. Weitere Informationen finden Sie unter Namenstyp.

Ressourcentyp

Der Ressourcentyp bezeichnet, welchen Typ von Ressource Sie deklarieren. Beispielsweise deklariert AWS::EC2::Instance eine EC2-Instance. Eine Liste aller Ressourcentypen finden Sie unter AWS Ressourcen- und Eigenschaftstypen – Referenz.

ResourceProperties

Ressourceneigenschaften sind zusätzliche Optionen, die Sie für eine Ressource angeben können. Beispielsweise müssen Sie für jede EC2-Instance eine Amazon Machine Image (AMI)-ID angeben. Diese AMI-ID deklarieren Sie als Eigenschaft der Instance, wie im folgenden Beispiel dargestellt:

Beispiel JSON
"Resources" : {
  "MyEC2Instance" : {
    "Type" : "AWS::EC2::Instance",
    "Properties" : {
      "ImageId" : "ami-0ff8a91507f77f867"
    }
  }
}
Beispiel YAML
Resources:
  MyEC2Instance:
    Type: "AWS::EC2::Instance"
    Properties:
      ImageId: "ami-0ff8a91507f77f867"

Müssen für eine Ressource keine Eigenschaften deklariert werden, können Sie den Eigenschaftenabschnitt für diese Ressource weglassen.

Eigenschaftswerte können Literalzeichenfolgen, Listen von Zeichenfolgen, boolesche Werte, Parameterreferenzen, Pseudoreferenzen oder von einer Funktion zurückgegebene Werte sein. Das folgende Beispiel demonstriert, wie Sie verschiedene Typen von Eigenschaftswerten deklarieren können:

Beispiel JSON
"Properties" : {
    "String" : "one-string-value",
    "Number" : 123,
    "LiteralList" : [ "first-value", "second-value" ],
    "Boolean" : true,
    "ReferenceForOneValue" :  { "Ref" : "MyLogicalResourceName" } ,
    "FunctionResultWithFunctionParams" : {
        "Fn::Join" : [ "%", [ "Key=", { "Ref" : "MyParameter" } ] ] }
}
Beispiel YAML
Properties:
  String: OneStringValue
  String: A longer string value 
  Number: 123
  LiteralList:
    - "[first]-string-value with a special characters"
    - "[second]-string-value with a special characters"
  Boolean: true
  ReferenceForOneValue:
    Ref: MyLogicalResourceName
  ReferenceForOneValueShortCut: !Ref MyLogicalResourceName
  FunctionResultWithFunctionParams: !Sub |
    Key=%${MyParameter}

Sie können Ressourcen bedingungsbasiert erstellen, indem Sie eine Bedingung mit ihnen verknüpfen. Diese Bedingung müssen Sie im Abschnitt Conditions der Vorlage definieren.

Beispiele

Das folgende Beispiel zeigt eine Ressourcendeklaration. Es werden zwei Ressourcen definiert. Die Ressource MyInstance enthält die Ressource MyQueue als Teil ihrer Eigenschaft UserData :

JSON

  "Resources" : {
    "MyInstance" : {
        "Type" : "AWS::EC2::Instance",
        "Properties" : {
            "UserData" : {
                "Fn::Base64" : {
                    "Fn::Join" : [ "", [ "Queue=", { "Ref" : "MyQueue" } ] ]
                 } },
            "AvailabilityZone" : "us-east-1a",
            "ImageId" : "ami-0ff8a91507f77f867"
        }
    },

    "MyQueue" : {
        "Type" : "AWS::SQS::Queue",
        "Properties" : {
        }
    }
}

YAML

Resources: 
  MyInstance: 
    Type: "AWS::EC2::Instance"
    Properties: 
      UserData: 
        "Fn::Base64":
          !Sub |
            Queue=${MyQueue}
      AvailabilityZone: "us-east-1a"
      ImageId: "ami-0ff8a91507f77f867"
  MyQueue: 
    Type: "AWS::SQS::Queue"
    Properties: {}
