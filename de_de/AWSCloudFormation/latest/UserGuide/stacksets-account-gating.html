Konfigurieren eines Zielkonto-Gates in AWS-CloudFormation-StackSets

Ein Konto-Gate ist eine optionale Funktion, mit der Sie eine AWS Lambda-Funktion angeben können, die überprüft, ob ein Zielkonto bestimmte Anforderungen erfüllt, bevor AWS mit CloudFormation-StackSets-Stack-Vorgängen in diesem Konto beginnt. Ein gängiges Beispiel für ein Konto-Gate ist die Sicherstellung, dass keine aktiven oder nicht aufgelösten CloudWatch-Alarme im Zielkonto aktiv sind. Immer wenn Sie damit beginnen, Stack-Vorgänge im Zielkonto auszuführen, ruft StackSets die Funktion auf und fährt nur fort, wenn die Funktion einen SUCCEEDED-Code zurückgibt. Wenn die Lambda-Funktion den Status FAILED zurückgibt, setzt StackSets den angeforderten Vorgang nicht fort. Wenn Sie keine Lambda-Funktion für das Konto-Gate konfiguriert haben, überspringt StackSets die Überprüfung und fährt mit Ihrem Vorgang fort.

Wenn die Konto-Gate-Überprüfung des Zielkontos fehlschlägt, wird der fehlgeschlagene Vorgang Ihrer angegebene Ausfalltoleranz bzw. dem Prozentsatz für Stacks angerechnet. Weitere Informationen zur Fehlertoleranz finden Sie unter Optionen in den Operationen für Stack-Sets.

Konto-Gates sind nur für StackSets-Vorgänge verfügbar. Diese Funktion ist nicht für andere AWS CloudFormation-Vorgänge außerhalb von StackSets verfügbar.

Einrichtungsvoraussetzungen

In der folgenden Liste werden die Einrichtungsvoraussetzungen für Konto-Gating beschrieben.

  • Um mit den StackSets-Konto-Gate-Funktionen arbeiten zu können, muss der Name Ihrer Lambda-Funktion AWSCloudFormationStackSetAccountGate sein.

  • AWSCloudFormationStackSetExecutionRole benötigt Berechtigungen zum Aufrufen Ihrer Lambda-Funktion. Ohne diese Berechtigungen überspringt StackSets die Überprüfung des Konto-Gates und fährt mit den Stack-Vorgängen fort.

  • Damit Konto-Gating funktioniert, muss Zielkonten die Lambda-Berechtigung InvokeFunction hinzugefügt werden. Die Vertrauensbeziehungs-Richtlinie des Zielkontos muss über eine Vertrauensbeziehung mit dem Administratorkonto verfügen. Es folgt ein Beispiel für eine Richtlinienanweisung, die die Lambda-Berechtigungen invokefunction erteilt.

    {
    "Version": "2012-10-17",
    "Statement": [
        {
            "Effect": "Allow",
            "Action": [
                "lambda:InvokeFunction"
            ],
            "Resource": "*"
        }
    ]
}
