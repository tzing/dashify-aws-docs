Exemplarische AWS CloudFormation-Vorgehensweise für Git Sync

In der folgenden Anleitung erfahren Sie, wie Sie Git Sync verwenden. Bevor Sie beginnen, müssen Sie die Voraussetzungen für Git Sync erfüllen.

Schritt 1: Erstellen eines Stacks

  1. Greifen Sie auf die CloudFormation-Konsole zu.

  2. Wählen Sie Stack erstellen aus.

  3. Vergewissern Sie sich, dass im Dashboard Stack erstellen im Bereich Voraussetzung – Vorlage vorbereiten die Option Vorlage ist bereit ausgewählt ist.

  4. Wählen Sie im Bereich Vorlage angeben die Option Von Git synchronisieren.

  5. Wählen Sie Weiter aus, um Stack-Details festzulegen.

Schritt 2: Festlegen von Stack-Details

  1. Geben Sie unter Stack-Name einen Namen für Ihren Stack ein. Stack-Namen können Buchstaben (A–Z und a–z), Ziffern (0–9) und Bindestriche (-) enthalten.

  2. Die Stack-Bereitstellungsdatei hat zwei Optionen:

    • Datei mit den folgenden Parametern erstellen und sie in meinem Repository platzieren.

      Wählen Sie diese Option, wenn Git Sync die Stack-Bereitstellungsdatei erstellen und in Ihrem Repository ablegen soll.

    • Ich stelle meine eigene Datei in meinem Repository bereit.

      Wählen Sie diese Option, wenn Sie die Stack-Bereitstellungsdatei erstellt und in Ihrem Repository abgelegt haben.

    Weitere Informationen finden Sie unter Stack-Bereitstellungsdatei.

  3. Der Bereich Projektarchiv für bietet zwei Optionen:

    • Verlinktes Git-Repository auswählen

    • Git-Repository verlinken

    Wenn Sie bereits ein Git-Repository mit CloudFormation verlinkt haben, wählen Sie Ein verlinktes Git-Repository auswählen aus und fahren Sie mit dem nächsten Schritt fort. Führen Sie andernfalls die folgenden Teilschritte aus, um ein Git-Repository zu verbinden:

    1. Wählen Sie Git-Repository verlinken.

    2. Wählen Sie Ihren Git-Repository-Anbieter aus den folgenden Optionen aus:

      • GitHub

      • GitHub Enterprise

      • GitLab

      • Bitbucket

    3. Wählen Sie Verbindung aus der Liste aus. Wenn in der Verbindungsliste keine Optionen angezeigt werden, rufen Sie die Verbindungskonsole auf, um eine Verbindung zu Ihrem Repository herzustellen.

  4. Wählen Sie in der Repository-Liste das Git-Repository aus, das Ihre Stack-Vorlagendatei enthält.

  5. Wähle in der Liste Verzweigung die Verzweigung aus, die Git Sync überwachen soll.

    Anmerkung

    Git Sync überwacht nur die ausgewählte Verzweigung auf Änderungen an der CloudFormation-Vorlage und den Stack-Bereitstellungsdateien. Alle Änderungen, die Sie auf Ihren Stack anwenden möchten, müssen für diese Verzweigung übernommen werden.

  6. Geben Sie für den Pfad der Bereitstellungsdatei den vollständigen Pfad einschließlich des Namens der Stack-Bereitstellungsdatei aus dem Stammverzeichnis Ihrer Repository-Verzweigung an.

    Wenn CloudFormation die Datei für Sie generiert, wird die Datei hier in Ihrem Repository festgeschrieben. Wenn Sie die Datei bereitstellen, ist dies der Speicherort der Datei in Ihrem Repository.

  7. Fügen Sie eine IAM-Rolle hinzu. Die IAM-Rolle umfasst Berechtigungen, die CloudFormation benötigt, um den Stack aus Ihrem Git-Repository zu synchronisieren. Sie können Neue IAM-Rolle wählen, um eine neue Rolle zu generieren, oder Vorhandende IAM-Rolle wählen, um eine bestehende Rolle aus Ihrem CloudFormation-Konto auszuwählen. Wenn Sie sich dafür entscheiden, eine neue Rolle zu generieren, sind die erforderlichen Berechtigungen in der Rolle enthalten.

    Anmerkung

    Wichtige Informationen zu den erforderlichen Berechtigungen und der Vertrauensrichtlinie für Git Sync finden Sie unter Erforderliche Berechtigungen.

  8. Geben Sie für den Pfad der Vorlagendatei den vollständigen Pfad aus dem Stammverzeichnis Ihres Repositorys für die Stack-Vorlagendatei an.

  9. Fügen Sie Parameter für Ihren Stack hinzu, die Sie aus Ihrer Stack-Bereitstellungsdatei festlegen möchten. Parameter sind Schlüssel-Wert-Paare, Um beispielsweise den Port-Parameter zu Ihrer Stack-Bereitstellungsdatei hinzuzufügen:

    1. Wählen Sie Parameter hinzufügen aus.

    2. Geben Sie für Key (Schlüssel) port ein.

    3. Geben Sie für Wert 8080 ein.

    4. Wiederholen Sie die vorherigen Schritte für alle Stack-Parameter, die Sie in Ihrer Stack-Bereitstellungsdatei verfügbar machen möchten.

  10. (Optional) Fügen Sie Tags hinzu. Tags sind Schlüssel-Wert-Paare, die Sie zum Identifizieren und Kategorisieren von Ressourcen im Stack verwenden können. Um der Stack-Bereitstellung Tags hinzuzufügen, gehen Sie genauso vor, wie Sie Parameter hinzugefügt haben.

  11. Wählen Sie Weiter, um mit der Konfiguration der Stack-Optionen fortzufahren.

Schritt 3: Konfigurieren von Stack-Optionen

Informationen zum Konfigurieren von Stack-Optionen finden Sie unter Festlegen von AWS CloudFormation-Stack-Optionen.

Wenn Sie die Konfiguration abgeschlossen haben, wählen Sie Weiter, um mit der Überprüfung und Bereitstellung fortzufahren.

Schritt 4: Überprüfen und Bereitstellen

  1. Überprüfen Sie Ihre Stack-Einstellungen und bestätigen Sie Folgendes:

    • Die Stack-Vorlage ist korrekt konfiguriert und auf Von Git synchronisieren gesetzt.

    • Die Bereitstellungsdatei ist korrekt konfiguriert.

    • Das Repository mit der Vorlagendefinition ist korrekt konfiguriert, insbesondere, das richtige Repository und der richtige Verzweigungsname wurden ausgewählt.

    • Die Vorschau der Bereitstellungsdatei ist korrekt und enthält die erwarteten Parameter und Werte.

  2. Wählen Sie Absenden, um den Stack zu erstellen.

    Wichtig

    Eine Pull-Anfrage wird automatisch in Ihrem Git-Repository erstellt, wenn Sie Absenden wählen. Sie müssen diese Pull-Anfrage mit Ihrem Git-Repository zusammenführen, um den Vorgang abzuschließen. Wenn Sie die Pull-Anforderung zusammenführen, wird der Stack erstellt und Git Sync überwacht die CloudFormation-Vorlage und die Stack-Bereitstellungsdatei auf Änderungen, um den Stack zu aktualisieren.

Ändern des Stacks

Nachdem Sie die vorherigen Schritte abgeschlossen haben, wird ein neuer Stack erstellt, und CloudFormation überwacht Ihr Git-Repository auf Änderungen.

Um den Stack zu ändern, nehmen Sie Änderungen direkt an der Stack-Bereitstellungsdatei vor. Dann übertragen Sie sie in das Repository und die Verzweigung, die Sie für den Stack in Git Sync konfiguriert haben.

Im CloudFormation-Dashboard können Sie den Stack auswählen und die Registerkarte Git-Synchronisierung auswählen, um Informationen zum Status des Stacks und zu Synchronisierungsereignissen anzuzeigen. Weitere Informationen finden Sie unter AWS CloudFormation Git-Sync-Status-Dashboard.
