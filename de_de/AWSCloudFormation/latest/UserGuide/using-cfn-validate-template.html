Validierung einer Vorlage

Um Ihre Vorlagen-Datei auf Syntaxfehler zu überprüfen, können Sie den aws cloudformation validate-template-Befehl verwenden.

Anmerkung

Der aws cloudformation validate-template-Befehl wurde entwickelt, um nur die Syntax Ihrer Vorlage zu überprüfen. Er stellt nicht sicher, dass die Eigenschaftswerte, die Sie für eine Ressource angegeben haben, für diese Ressource gültig sind. Er bestimmt auch nicht die Anzahl der Ressourcen, die bei der Erstellung des Stacks vorhanden sein werden.

Um die Betriebsgültigkeit zu überprüfen, müssen Sie versuchen, den Stack zu erstellen. Es gibt keine Sandbox oder Testfläche für AWS CloudFormation-Stacks, daher werden Ihnen die Ressourcen verrechnet, die Sie beim Testen erstellen.

Bei der Validierung wird von AWS CloudFormation geprüft, ob die Vorlage gültiger JSON-Code ist. Wenn dies nicht der Fall ist, wird von CloudFormation geprüft, ob die Vorlage gültiger YAML-Code ist. Wenn beide Prüfungen fehlschlagen, gibt CloudFormation einen Vorlagenvalidierungsfehler zurück. Sie können Vorlagen lokal durch Angabe des Parameters --template-body validieren oder remote mit dem Parameter --template-url.

Gültige Vorlage

Im folgenden Beispiel wird der --template-url-Parameter übergeben, um eine Vorlage an einem Remote-Speicherort zu validieren:

aws cloudformation validate-template --template-url https://s3.amazonaws.com/cloudformation-templates-us-east-1/S3_Bucket.template

Im Folgenden sehen Sie die Ausgabe des vorherigen Befehls.

{
    "Description": "AWS CloudFormation Sample Template S3_Bucket: Sample template showing how to create a publicly accessible S3 bucket. **WARNING** This template creates an S3 bucket.
You will be billed for the AWS resources used if you create a stack from this template.",
    "Parameters": [],
    "Capabilities": []
}

Das erwartete Ergebnis enthält keine Fehlermeldung, aber Informationen über alle Parameter.

Validierungsfehler

Dieser Abschnitt erzeugt einen Validierungsfehler beim Ausführen des aws cloudformation validate-template-Befehls.

Im folgenden Beispiel wird der --template-body-Parameter übergeben, um eine Vorlage an einem lokalen Speicherort zu validieren.

aws cloudformation validate-template --template-body file:///home/local/test/sampletemplate.json

Im Folgenden sehen Sie die Ausgabe des vorherigen Befehls.

{
    "ResponseMetadata": {
        "RequestId": "4ae33ec0-1988-11e3-818b-e15a6df955cd"
    },
    "Errors": [
        {
            "Message": "Template format error: JSON not well-formed. (line 11, column 8)",
            "Code": "ValidationError",
            "Type": "Sender"
        }
    ],
    "Capabilities": [],
    "Parameters": []
}
A client error (ValidationError) occurred: Template format error: JSON not well-formed. (line 11, column 8)

Das erwartete Ergebnis enthält eine Fehlermeldung und Informationen zum Fehler.
