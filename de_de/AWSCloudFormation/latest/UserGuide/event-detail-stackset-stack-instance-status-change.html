StackSet Ereignisdetails zur Änderung des Stack-Instance-Status

Im Folgenden finden Sie die Detailfelder für StackSet Stack-Instance-Statusereignisse.

Die detail-type Felder source und sind enthalten, da sie bestimmte Werte für AWS CloudFormation Ereignisse enthalten.

{
  . . .,
  "detail-type": "CloudFormation StackSet StackInstance Status Change",
  "source": "aws.cloudformation",
  . . .,
  "detail": {
    "stack-set-arn" : "string",
    "stack-id" : "string",
    "status-details": {
        "status": "string",
        "status-reason": "string",
        "detailed-status": "string"
      }
    }
  }
}
detail-type

Identifiziert den Ereignistyp.

Bei StackSet Stack-Instance-Statusereignissen lautet dieser Wert CloudFormation StackSet StackInstance Status Change.

source

Identifiziert den Service, aus dem das Ereignis stammt. Für CloudFormation -Ereignisse ist dieser Wert aws.cloudformation.

detail

Ein JSON-Objekt, das Informationen zum Ereignis enthält. Der Service, der das Ereignis generiert, bestimmt den Inhalt dieses Feldes.

Für StackSet Stack-Instance-Statusereignisse umfassen diese Daten:

stack-set-arn

Der Amazon-Ressourcenname (ARN), der dem zugeordnet ist StackSet.

stack-id

Die eindeutige Stack-ID, die der Stack-Instance zugeordnet ist.

status-details
status

Der StackSet Instance-Status.

Weitere Details finden Sie unter Stack-Instance-Statuscodes.

Zulässige Werte: CURRENT | OUTDATED | INOPERABLE

status-reason

Statusgrund der StackSet Instance.

detailed-status

Der detaillierte Status der StackSet Instance.

Zulässige Werte: CANCELLED | FAILED | FAILED_IMPORT | INOPERABLE | PENDING | RUNNING | SKIPPED_SUSPENDED_ACCOUNT | SUCCEEDED
