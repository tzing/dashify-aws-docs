Überprüfen Ihres Stacks auf der AWS CloudFormation-Konsole

Der letzte Schritt, bevor ein Stack gestartet wird, ist die Prüfung der während der Stack-Erstellung eingegebenen Werte.

  1. Prüfen Sie auf der Seite Review die Details des Stacks.

    Wenn Sie einen der Werte vor dem Start des Stacks ändern möchten, wählen Sie im entsprechenden Abschnitt Edit (Bearbeiten) aus, um zu der Seite mit der Einstellung, die Sie ändern möchten, zurückzukehren.

  2. Nachdem Sie die Einstellungen für die Stackerstellung überprüft haben, wählen Sie Stack erstellen aus, um den Stack zu starten.

    Anmerkung

    An dieser Stelle haben Sie auch die Möglichkeit, einen neuen Änderungssatz anstelle eines neuen Stacks zu erstellen. Um dies zu tun, klicken Sie auf Create change set (Änderungssatz erstellen) anstelle von Create Stack (Stack erstellen). Weitere Informationen finden Sie unter Erstellen von Stacks mithilfe von Änderungssätzen.

    CloudFormation zeigt den Bereich Events (Ereignisse) der Seite Stack details (Stack-Details) für Ihren neuen Stack an. Von hier aus können Sie Die Ereignisse, Daten oder Ressourcen Ihres Stacks anzeigen. CloudFormation aktualisiert die Stack-Ereignisse automatisch im Minutentakt. Darüber hinaus zeigt CloudFormation die Kennzeichnung New events available (Neue Ereignisse verfügbar) an, wenn neue Stack-Ereignisse auftreten. Klicken Sie auf das Aktualisierungssymbol, um diese Ereignisse in die Liste zu laden. Verfolgen Sie die Stack-Erstellungsereignisse, um die Abfolge der Ereignisse nachzuvollziehen, die zur Erstellung des Stacks (oder zu einem Fehler im Fall von Stack-Debugging) geführt haben.

    Während der Stack erstellt wird, wird er auf der Seite Stacks mit dem Status CREATE_IN_PROGRESS aufgelistet.

    Nachdem der Stack erfolgreich erstellt wurde, wechselt der Status zu CREATE_COMPLETE. Sie können dann die Registerkarte Outputs (Ausgaben) auswählen, um die Stack-Ausgaben einzusehen, wenn Sie welche in der Vorlage definiert haben.

Ermittlung der Fehlerursache

Wenn ein Stack nicht erstellt wird, können Sie mithilfe von AWS CloudFormation das Ereignis ermitteln, das wahrscheinlich die Ursache für den Stack-Fehler ist. Wählen Sie dazu in der Liste der Stacks den fehlerhaften Stack und dann die Option Ursache erkennen aus.

CloudFormation analysiert den Fehler und gibt das Ereignis an, das die wahrscheinliche Ursache dafür ist.
