IaC-Generator und schreibgeschützte Eigenschaften

Schreibgeschützte Eigenschaften sind Ressourceneigenschaften, die geschrieben werden können, aber nicht von gelesen werden könnenAWS CloudFormation. (Ein Beispiel wäre ein Datenbankpasswort.) Dies führt zu Problemen beim Generieren von Vorlagen aus vorhandenen Ressourcen. Im Allgemeinen werden schreibgeschützte Eigenschaften in Parameter in der generierten Vorlage konvertiert. Auf diese Weise können Sie die Eigenschaften während Importvorgängen als Parameterwerte eingeben. Schreibgeschützte Eigenschaften können in den folgenden Fällen jedoch nicht in Parameter konvertiert werden:

  1. Es gibt mehrere exklusive Eigenschaftensätze, von denen mindestens einige schreibgeschützt sind. Der IaC-Generator kann nicht bestimmen, welcher Satz exklusiver Eigenschaften während der Erstellung auf die Ressource angewendet wurde. Sie können beispielsweise den Code für einen AWS::Lambda::Function mit einer dieser Eigenschaften angeben.

    • Code/S3Bucket, Code/S3Keyund optional Code/S3ObjectVersion

    • Code/ImageUri

    • Code/ZipFile

    Alle diese Eigenschaften sind schreibgeschützt. Der IaC-Generator wählt einen der exklusiven Eigenschaftensätze aus und fügt sie der generierten Vorlage hinzu. Parameter werden für jede der Nur-Schreib-Eigenschaften hinzugefügt. Die Parameternamen enthalten OneOf und die Parameterbeschreibungen geben an, dass die entsprechende Eigenschaft durch andere exklusive Eigenschaften ersetzt werden kann. Der IaC-Generator legt eine Warnung Type von MUTUALLY_EXCLUSIVE_PROPERTIES für die enthaltenen Eigenschaften fest.

  2. Die Schreibeigenschaft kann einer von mehreren Datentypen sein. Beispielsweise AWS::ApiGateway::RestApi kann die Body Eigenschaft von entweder ein object oder seinstring. In diesem Fall legt der IaC-Generator eine Warnung Type von fest MUTUALLY_EXCLUSIVE_TYPES und schließt die -Eigenschaft in die generierte Vorlage ein, die den Typ verwendetstring.

  3. Die Schreibeigenschaft hat den Typ array. Parameter können nur skalare Werte sein, sodass es nicht möglich ist, der Vorlage für Arrays einen Parameter hinzuzufügen. In diesem Fall schließt der IaC-Generator die -Eigenschaft nicht in die generierte Vorlage ein und legt eine Warnung Type von festUNSUPPORTED_PROPERTIES.

  4. Die Schreibeigenschaft ist optional. Der IaC-Generator kann nicht erkennen, ob die schreibgeschützte Eigenschaft jemals beim Einrichten der Ressource verwendet wurde. In diesem Fall schließt der IaC-Generator die -Eigenschaft nicht in die generierte Vorlage ein und legt eine Warnung Type von festUNSUPPORTED_PROPERTIES.

Wenn die generierte Vorlage Ressourcen mit reinen Schreibeigenschaften enthält. Die IaC-Generatorkonsole zeigt eine Warnung mit einer Zusammenfassung der Art von Problemen an. Beispielsweise:

Sie können Warndetails anzeigen auswählen, um weitere Details anzuzeigen. Die Ressourcen mit schreibgeschützten Eigenschaften werden durch die logische ID identifiziert, die in der generierten Vorlage und dem Ressourcentyp verwendet wird.

Verwenden Sie die Liste der Warnungen, um Ressourcen mit reinen Schreibeigenschaften zu identifizieren, und sehen Sie sich jede Ressource an, um festzustellen, welche Änderungen (falls vorhanden) an der generierten Vorlage vorgenommen werden müssen. Sie können die generierte Vorlage herunterladen, indem Sie auf die Schaltfläche Herunterladen klicken. Nachdem die Änderungen vorgenommen wurden, können Sie die Schaltfläche Bearbeitene Vorlage importieren auswählen, um fortzufahren.

Wichtig

Derzeit gibt die AWS Ressourcen- und Eigenschaftstypen – Referenz Dokumentation nicht an, ob eine Eigenschaft schreibgeschützt ist oder mehrere Typen unterstützt. Sie müssen sich die Warnungen ansehen, die von der IaC-Generatorkonsole oder dem Ressourcenanbieterschema zurückgegeben werden, um festzustellen, welche Eigenschaften schreibgeschützt sind.

Weitere Informationen zum Ressourcenanbieterschema finden Sie unter Ressourcenanbieterschema im CloudFormation Benutzerhandbuch für die -Befehlszeilenschnittstelle. Informationen zum Herunterladen der Ressourcenanbieterschemata finden Sie unter CloudFormation -Ressourcenanbieterschemata.
