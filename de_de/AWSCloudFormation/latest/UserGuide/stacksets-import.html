Importieren eines Stacks in AWS CloudFormation StackSets

Der AWS CloudFormation-Stack-Importvorgang kann vorhandene Stacks in neue oder vorhandene Stack-Sets importieren, sodass Sie vorhandene Stacks in einem einzelnen Vorgang zu einem Stack-Set migrieren können. StackSets erweitert die Funktionalität von Stacks, indem es Ihnen ermöglicht, Stacks über mehrere Konten und Regionen innerhalb eines einzigen Vorgangs zu erstellen, zu aktualisieren oder zu löschen.

Verwenden Sie die Stack-Importvorgänge für selbstverwaltete oder serviceverwaltete StackSets. Für selbstverwaltete StackSets kann der Importvorgang Stacks in das Administratorkonto oder in verschiedene Zielkonten und AWS-Regionen importieren. Für serviceverwaltete StackSets kann der Importvorgang jeden Stack in der gleichen AWS Organizations importieren, wie das Verwaltungskonto. Der Importvorgang kann bis zu 10 Stacks mithilfe von Inline-Stack-IDs oder bis zu 200 Stacks mithilfe eines Amazon S3-Objekts importieren.

Der Stackimport ist überall dort verfügbar, wo StackSets unterstützt werden. Informationen zur regionalen Unterstützung von StackSets finden Sie unter StackSets Regionale Unterstützung.

Anforderungen für den Stack-Import

Da Stack-Sets Stack-Vorgänge über mehrere Konten hinweg ausführen, benötigen Sie vor der Erstellung Ihres ersten Stack-Sets die erforderlichen Berechtigungen, die in Ihrem AWS-Konto definiert sind.

  • Informationen zum Einrichten der erforderlichen Berechtigungen zum Erstellen eines Stack-Sets mit selbstverwalteten Berechtigungen finden Sie unter Durchführen von Stack-Set-Operationen mit Regionen, die standardmäßig deaktiviert sind und Selbstverwaltete Berechtigungen erteilen.

  • Informationen zum Einrichten der erforderlichen Berechtigungen zum Erstellen eines Stack-Sets mit serviceverwalteten Berechtigungen finden Sie unter Durchführen von Stack-Set-Operationen mit Regionen, die standardmäßig deaktiviert sind und Aktivieren des vertrauenswürdigen Zugriffs mit AWS Organizations.

Wichtig

Die Eigenschaft NoEcho wird vom StackSet-Import nicht unterstützt. Stacks, die NoEcho enthalten, werden nicht über den StackSet-Import in neue Stack-Sets importiert.

Weitere Informationen zu StackSets-Voraussetzungen finden Sie unter Voraussetzungen für Stack-Set-Vorgänge.

Bevor Sie einen Stack in ein Stack-Set importieren, sollten Sie die folgenden Anforderungen verstehen:

  • Stacks kann nur zu einem einzelnen Stack-Set gehören.

  • Sie können Stack-Tags für das Stack-Set implementieren, indem Sie Tags explizit als Parameter im Stack-Importvorgang angeben.

  • Die benutzerdefinierten Parameter-Überschreibungen eines Stacks sind während des Importvorgangs nicht betroffen.

  • Die StackSet-Kontingente und Stack-Instances gelten beim Import von Stacks. Weitere Informationen zu Kontingenten finden Sie unter Kontingente für AWS CloudFormation.
