Realización de una prueba de concepto de Amazon Redshift

Amazon Redshift es un almacenamiento de datos rápido y escalable que permite analizar todos los datos de forma sencilla y rentable mediante SQL estándar junto con las herramientas de inteligencia empresarial (BI) existentes. Amazon Redshift ofrece un rendimiento rápido en un almacenamiento de datos en la nube de bajo costo. Utiliza optimización de consultas sofisticada, almacenamiento en caché acelerado, almacenamiento en columnas en discos locales de alto rendimiento y ejecución masiva en paralelo de consultas.

En las siguientes secciones, encontrará un marco para crear una prueba de concepto con Amazon Redshift. El marco lo ayuda a usar las prácticas recomendadas de arquitectura para diseñar y operar un almacenamiento de datos seguro, de alto rendimiento y rentable. Esta guía se basa en la revisión de diseños de miles de arquitecturas de clientes en una amplia variedad de tipos de empresas y casos de uso. Hemos compilado experiencias del cliente para desarrollar este conjunto de prácticas recomendadas y ayudarlo así a desarrollar criterios para evaluar la carga de trabajo de su almacenamiento de datos.

Información general sobre el proceso

El proceso para realizar una prueba de concepto tiene tres pasos:

  1. Identificar los objetivos de la prueba de concepto: puede trabajar primero con los requisitos empresariales y los criterios de éxito, para luego llevarlos a un plan de proyecto técnico de la prueba de concepto.

  2. Configurar el entorno de prueba de concepto: la mayor parte del proceso de configuración consiste en hacer clic en unos pocos botones para crear sus recursos. En cuestión de minutos, puede tener listo un entorno de almacenamiento de datos con los datos cargados.

  3. Complete el plan de proyecto de la prueba de concepto para asegurarse de que se cumplan los objetivos.

En las siguientes secciones, se explican en detalle cada uno de los pasos.

Identificar los objetivos empresariales y los criterios de éxito

La identificación de los objetivos de la prueba de concepto juega un papel crucial a la hora de determinar lo que debe medir como parte del proceso de evaluación. Los criterios de evaluación deben incluir los desafíos de escalado actuales, las mejoras en la experiencia del cliente con el almacenamiento de datos y los métodos para abordar sus problemas operativos actuales. Puede utilizar las siguientes preguntas para identificar los objetivos de la prueba de concepto:

  • ¿Cuáles son los objetivos en lo relativo al escalado del almacenamiento de datos de ?

  • ¿Cuáles son los acuerdos de nivel de servicios específicos cuyos términos desea mejorar?

  • ¿Qué nuevos conjuntos de datos necesita incluir en el almacenamiento de datos?

  • ¿Cuáles son las consultas SQL críticas para la empresa que necesita probar y medir? Asegúrese de incluir toda la gama de complejidades de SQL, como los diferentes tipos de consultas (por ejemplo, selección, inserción, actualización y eliminación).

  • ¿Cuáles son los tipos generales de cargas de trabajo que tiene previsto probar? Los ejemplos pueden incluir cargas de trabajo de extracción, transformación y carga (ETL); consultas de informes; y extracciones por lotes.

Después de responder estas preguntas, debería ser capaz de establecer objetivos SMART y criterios de éxito para crear su prueba de concepto. Para obtener información sobre cómo establecer objetivos, consulte SMART criteria en Wikipedia.

Configurar la prueba de concepto

Como eliminamos el aprovisionamiento de hardware, la instalación de redes y software de un almacenamiento de datos en las instalaciones, probar Amazon Redshift con su propio conjunto de datos nunca ha sido tan fácil. Muchas de las decisiones y los cálculos en cuanto a tamaño que solían requerirse ahora están a un clic de distancia. Puede cambiar el tamaño de su clúster de forma flexible o ajustar la relación entre el almacenamiento y el cálculo.

En términos generales, la configuración del entorno de prueba de concepto de Amazon Redshift es un proceso de dos pasos. Implica el lanzamiento de un almacenamiento de datos y, luego, la conversión del esquema y los conjuntos de datos para su evaluación.

Elegir un tamaño inicial para el clúster

Puede elegir el tipo de nodo y la cantidad de nodos a través de la consola de Amazon Redshift. Le recomendamos que también pruebe el cambio de tamaño del clúster como parte de su plan de prueba de concepto. Para obtener el tamaño inicial del clúster, siga estos pasos:

  1. Inicie sesión en la AWS Management Console y abra la consola de Amazon Redshift en https://console.aws.amazon.com/redshift/.

  2. En el menú de navegación, elija Create cluster (Crear clúster) para abrir la página de configuración.

  3. En Cluster identifier (Identificador de clúster), ingrese un nombre para su clúster.

  4. En el siguiente paso, se describe una consola de Amazon Redshift que se ejecuta en una región de AWS que admite los tipos de nodo RA3. Para obtener una lista de las regiones de AWS que admiten los tipos de nodo RA3, consulte Información general sobre los tipos de nodo RA3 en la Guía de administración de Amazon Redshift.

    Si no sabe cuál sería el tamaño adecuado para el clúster, elija Help me choose (Ayúdeme a elegir). De este modo, se inicia una calculadora de tamaño que le hace preguntas sobre el tamaño y las características de consulta de los datos que planea almacenar en el almacenamiento de datos. Si conoce el tamaño requerido para su clúster (es decir, el tipo de nodo y la cantidad de nodos), elija I’ll choose (Yo elegiré). A continuación, elija el Node type (Tipo de nodo) y la cantidad de Nodes (nodos) para determinar el tamaño del clúster para la prueba de concepto.

    nota

    Si su organización reúne los requisitos necesarios y su clúster se crea en una Región de AWS donde no está disponible Amazon Redshift sin servidor, es posible que pueda crear un clúster en el programa de prueba gratuita de Amazon Redshift. Elige Producción o Prueba gratuita para responder la pregunta ¿Para qué planifica usar este clúster? Si elige Prueba gratuita, cree una configuración con el tipo de nodo dc2.large. Para obtener más información sobre la elección de una prueba gratuita, consulte Prueba gratuita de Amazon Redshift. Para obtener una lista de las Regiones de AWS donde Amazon Redshift sin servidor está disponible, consulte los puntos de conexión enumerados para la API de Redshift sin servidor en la Referencia general de Amazon Web Services.

  5. Después de ingresar todas las propiedades necesarias del clúster, elija Create cluster (Crear clúster) para lanzar su almacenamiento de datos.

    Para obtener más detalles sobre la creación de clústeres con la consola de Amazon Redshift, consulte Creación de un clúster en la Guía de administración de Amazon Redshift.

Convertir el esquema y configurar los conjuntos de datos para la prueba de concepto

Si no tiene un almacenamiento de datos existente, omita esta sección y consulte la Guía de introducción a Amazon Redshift. La Guía de introducción a Amazon Redshift, proporciona un tutorial para crear un clúster y ejemplos de configuración de datos en Amazon Redshift.

Cuando migra desde el almacenamiento de datos existente, puede convertir el esquema, el código y los datos mediante AWS Schema Conversion Tool y AWS Database Migration Service. Las herramientas que elija dependen del origen de los datos y de las replicaciones continuas opcionales. Para obtener más información, consulte ¿Qué es AWS Schema Conversion Tool? en la Guía del usuario de AWS Schema Conversion Tool y ¿Qué es AWS Database Migration Service? en la Guía del usuario de AWS Database Migration Service. Los siguientes recursos pueden ayudarlo a configurar sus datos en Amazon Redshift:

  • Migrar el almacenamiento de datos a Amazon Redshift mediante la AWS Schema Conversion Tool: esta publicación de blog proporciona información general sobre cómo puede utilizar los extractores de datos de AWS SCT para migrar el almacenamiento de datos existente a Amazon Redshift. La herramienta AWS SCT puede migrar sus datos desde muchas plataformas heredadas (como Oracle, Greenplum, Netezza, Teradata, Microsoft SQL Server o Vertica).

  • Si así lo desea, también puede utilizar AWS Database Migration Service para las replicaciones continuas de los datos modificados de la fuente. Para obtener más información, consulte Uso de una base de datos de Amazon Redshift como destino para AWS Database Migration Service en la Guía del usuario de AWS Database Migration Service.

Amazon Redshift es un sistema de administración de bases de datos relacionales (RDBMS). Como tal, puede ejecutar muchos tipos de modelos de datos, incluidos los esquemas en estrella; los esquemas en copo de nieve; los modelos de bóveda de datos; y las tablas simples, planas o normalizadas. Después de configurar los esquemas en Amazon Redshift, puede aprovechar el procesamiento masivo en paralelo y el almacenamiento de datos en columnas para realizar consultas analíticas rápidas de forma inmediata. Para obtener información acerca de los tipos de esquema, consulte Esquema en estrella, Esquema en copo de nieve y Modelos de bóveda de datos en Wikipedia.

Lista de comprobación para una evaluación completa

Asegúrese de que la evaluación completa satisfaga todas las necesidades de su almacenamiento de datos. Considere la posibilidad de incluir los siguientes elementos en sus criterios de éxito:

  • Tiempo de carga de datos: el comando COPY es una forma común de probar cuánto tardan en cargarse los datos. Para obtener más información, consulte Prácticas recomendadas de Amazon Redshift para la carga de datos.

  • Rendimiento del clúster: una forma habitual de determinar el rendimiento es medir las consultas por hora. Para ello, configure una prueba para que ejecute consultas típicas para su carga de trabajo.

  • Seguridad de los datos: con Amazon Redshift, puede cifrar con facilidad datos en reposo y en tránsito. También dispone de una serie de opciones para administrar las claves. Amazon Redshift también admite la integración del inicio de sesión único. Los precios de Amazon Redshift incluyen la seguridad integrada, la compresión de datos, el almacenamiento de copias de seguridad y la transferencia de datos.

  • Integración de herramientas de terceros: puede usar una conexión JDBC u ODBC para integrarse a herramientas externas de inteligencia empresarial y otros tipos.

  • Interoperabilidad con otros servicios de AWS: Amazon Redshift se integra a otros servicios de AWS, como Amazon EMR, Amazon QuickSight, AWS Glue, Amazon S3 y Amazon Kinesis. Puede usar esta integración a la hora de configurar y administrar su almacenamiento de datos.

  • Copias de seguridad e instantáneas: las copias de seguridad y las instantáneas se crean de manera automática. También puede crear una instantánea a un momento dado en cualquier momento o según una programación. Intente usar una instantánea y crear un segundo clúster como parte de su evaluación. Evalúe si sus organizaciones de desarrollo y pruebas pueden usar el clúster.

  • Ajuste del tamaño: su evaluación debe incluir un aumento del número o los tipos de nodos de Amazon Redshift. Evalúe que el rendimiento de la carga de trabajo antes y después del cambio del tamaño coincida con la variabilidad del volumen de su carga de trabajo. Para obtener más información, consulte Cambio de tamaño de los clústeres de Amazon Redshift en la Guía de administración de Amazon Redshift.

  • Escalado de simultaneidad: esta característica lo ayuda a manejar la variabilidad del volumen de tráfico en su almacenamiento de datos. Con el escalado de simultaneidad, puede permitir un número prácticamente ilimitado de consultas y usuarios simultáneos con un rendimiento rápido y contante para las consultas. Para obtener más información, consulte Trabajo con escalado de simultaneidad.

  • Administración de cargas de trabajo (WLM) automática: priorice sus consultas críticas para la empresa sobre otras consultas a través de la WLM automática. Intente configurar colas en función de sus cargas de trabajo (por ejemplo, una cola para ETL y otra para generar informes). A continuación, habilite la WLM automática para asignar los recursos de simultaneidad y memoria de forma dinámica. Para obtener más información, consulte Implementación de WLM automática.

  • Amazon Redshift Advisor: Advisor desarrolla recomendaciones personalizadas para potenciar el rendimiento y optimizar los costos analizando la carga de trabajo y las métricas de uso de su clúster. Inicie sesión en la consola de Amazon Redshift para ver las recomendaciones de Advisor. Para obtener más información, consulte Uso de las recomendaciones de Amazon Redshift Advisor.

  • Diseño de tablas: Amazon Redshift proporciona un excelente rendimiento desde el primer momento para la mayoría de las cargas de trabajo. A la hora de crear una tabla, la clave de ordenación y la clave de distribución predeterminadas son AUTO. Para obtener más información, consulte Uso de la optimización automática de tablas.

  • Soporte: recomendamos encarecidamente que considere AWS Support como parte de su evaluación. Además, asegúrese de hablar con el gerente de su cuenta sobre la prueba de concepto. AWS puede ayudar con orientación técnica y créditos para la prueba de concepto si reúne los requisitos necesarios. Si no encuentra la ayuda que busca, puede comunicarse directamente con el equipo de Amazon Redshift. Para obtener ayuda, envíe el formulario a Solicitar soporte para la prueba de concepto de Amazon Redshift.

  • Integración de lake house: con la integración incorporada, intente utilizar la característica Amazon Redshift Spectrum que funciona desde el principio. Con Redshift Spectrum, puede llevar el almacenamiento de datos al lago de datos y ejecutar consultas en petabytes de datos en Amazon S3 utilizando el clúster existente. Para obtener más información, consulte Consulta de datos externos mediante Amazon Redshift Spectrum.

Desarrollar un plan de proyecto para la evaluación

Algunas de las siguientes técnicas para crear puntos de referencia para las consultas pueden ayudarlo con su evaluación de Amazon Redshift:

  • Reúna una lista de consultas para cada categoría de tiempo de ejecución. Contar con un número suficiente (por ejemplo, 30 por categoría) le permite asegurarse de que su evaluación refleje la implementación de un almacenamiento de datos real. Añada un identificador único para asociar cada consulta que incluya en su evaluación con una de las categorías que estableció para la evaluación. A continuación, puede usar estos identificadores únicos para determinar el rendimiento de las tablas de sistema.

    También puede crear un grupo de consultas para organizar sus consultas de evaluación. Por ejemplo, si ha establecido una categoría "Informes" en la evaluación, puede crear un sistema de codificación para etiquetar sus consultas de evaluación con la palabra "Informe". A continuación, puede identificar consultas individuales dentro del informe como I1, I2, etc. El siguiente ejemplo demuestra esta estrategia.

    
SELECT 'Reporting' AS query_category, 'R1' as query_id, * FROM customers;              

    
SELECT query, datediff(seconds, starttime, endtime)
  FROM stl_query 
  WHERE 
  querytxt LIKE '%Reporting%' 
  and starttime >= '2018-04-15 00:00' 
  and endtime < '2018-04-15 23:59';

    Una vez que haya asociado una consulta a una categoría de evaluación, puede usar un identificador único con el objetivo de determinar el rendimiento de las tablas de sistema para cada categoría.

  • Pruebe el rendimiento con consultas de ETL o usuario históricas que tengan una serie de tiempos de ejecución en su almacenamiento de datos existente. Puede usar una utilidad de prueba de carga, como JMeter de código abierto, o una utilidad personalizada. De ser así, asegúrese de que su utilidad cumpla los siguientes requisitos:

    • Puede tener en cuenta el tiempo de transmisión de la red.

    • Evalúa el tiempo de ejecución en función del rendimiento de las tablas internas de sistema. Para obtener información acerca de cómo hacerlo, consulte Análisis del resumen de consultas.

  • Identifique todas las diversas permutaciones que tiene previsto probar durante la evaluación. En la siguiente lista se ofrecen variables comunes:

    • Tamaño del clúster

    • Tipo de nodo

    • Duración de la prueba de carga

    • Configuración de simultaneidad

  • Reduzca el costo de su prueba de concepto pausando su clúster durante las horas no laborables y los fines de semana. Cuando un clúster está en pausa, se suspende la facturación de la informática bajo demanda. Para realizar pruebas en el clúster, reanude la facturación por segundo. También puede crear una programación para pausar y reanudar el clúster de forma automática. Para obtener más información, consulte Pausa y reanudación de clústeres en la Guía de administración de Amazon Redshift.
