Tipos de datos de intervalo

Puede usar un tipo de datos de intervalo para especificar períodos o intervalos de tiempo, como 12 hours, 6 weeks o 1 month. Los tipos de datos INTERVAL le permiten especificar períodos o intervalos de tiempo, como 12 hours, 6 weeks o 1 month. Puede usar tipos de datos de intervalos en los cálculos, como calcular las diferencias entre fechas y horas o intervalos, sumas de intervalos o sumas de una fecha y hora y un intervalo.

Los siguientes intervalos son compatibles:

  • YEAR TO MONTH

  • DAY TO SECOND

Sintaxis

Para crear un intervalo de tipo YEAR TO MONTH:

INTERVAL <year_to_month_qualifier>

Para crear un literal de intervalo de tipo YEAR TO MONTH:

INTERVAL <interval-string> <qualifier>

Utilice la siguiente sintaxis para crear un intervalo de tipo DAY TO SECOND. El parámetro fractional_precision solo es necesario si el intervalo contiene SECOND.

INTERVAL <qualifier> [ <fractional_precision> ]

Utilice la siguiente sintaxis para crear un literal de intervalo de tipo DAY TO SECOND.

INTERVAL <interval-string> <qualifier> [ <fractional_precision> ]

Argumentos

Interval-string

Especifica el valor del intervalo. Solo es necesario si se crea una constante de intervalo

Qualifier

Especifica el rango del intervalo. Si usa un calificador y crea un intervalo con unidades de tiempo más pequeñas que el calificador, Amazon Redshift corta y descarta las partes más pequeñas del intervalo. En el siguiente ejemplo, se muestra el uso de un calificador MINUTE y Amazon Redshift descarta las unidades de tiempo inferiores a MINUTE.

INTERVAL '1 day 1 hour 1 minute 1.123 seconds' MINUTE

se corta a 

'1 day 01:01:00'
Fractional_precision

Parámetro opcional que especifica el número de dígitos fraccionales permitidos en el intervalo. Por ejemplo, SECOND (3) crea un intervalo que permite solo tres dígitos fraccionales, como 1,234 segundos. Aplicable solo a los intervalos DAY TO SECOND que contengan segundos. El número máximo de dígitos compatibles es seis.

El tipo de datos YEAR TO MONTH admite los siguientes calificadores:

  • YEAR

  • MONTH

  • YEAR TO MONTH

El tipo de datos DAY TO SECOND admite los siguientes calificadores:

  • DAY

  • HOUR

  • MINUTE

  • SECOND

  • DAY TO HOUR

  • DAY TO MINUTE

  • DAY TO SECOND

  • HOUR TO MINUTE

  • HOUR TO SECOND

  • MINUTE TO SECOND

Ejemplos

En el siguiente ejemplo, se muestra cómo crear una constante INTERVAL de 1 año.

INTERVAL '1' YEAR

Si especifica una cadena de intervalo que supere el calificador, las unidades de tiempo restantes se truncan con respecto al intervalo. En el ejemplo siguiente, un intervalo de 13 meses se convierte en 1 año y 1 mes, pero el mes restante se omite debido al calificador YEAR.

INTERVAL '13 months' YEAR
'1 year'

Si utiliza un calificador inferior a la cadena de intervalos, se incluyen las unidades sobrantes.

INTERVAL '13 months' MONTH
'1 year 1 month'

Al especificar una precisión en el intervalo, se trunca el número de dígitos fraccionarios hasta alcanzar la precisión especificada.

INTERVAL '1.234567' SECOND (3)
'00:00:01.234'

Si no especifica una precisión, Amazon Redshift utiliza la precisión máxima de 6.

INTERVAL '1.23456789' SECOND
'00:00:01.234567'

En el siguiente ejemplo, se muestra cómo crear un intervalo con rangos.

INTERVAL '2:2' MINUTE TO SECOND
'00:02:02'

Los calificadores dictan las unidades que se especifican. Por ejemplo, aunque en el siguiente ejemplo se utiliza la misma cadena de intervalos de “2:2” que en el ejemplo anterior, Amazon Redshift reconoce que utiliza diferentes unidades de tiempo debido al calificador

INTERVAL '2:2' HOUR TO MINUTE
'02:02:00'

También se admiten abreviaturas y plurales de cada unidad; por ejemplo: 5s, 5 second y 5 seconds son intervalos equivalentes. Las unidades admitidas son años, meses, horas, minutos y segundos.

INTERVAL '5s' SECOND
'00:00:05'

INTERVAL '5 second' SECOND
'00:00:05'
                     
INTERVAL '5 HOURS' HOUR
'05:00:00'
                     
INTERVAL '5 h' HOUR
'05:00:00

En los siguientes ejemplos, se muestra cómo usar tipos de datos INTERVAL con tablas

create table sample_intervals (y2m interval month, m2s interval hour to minute);
insert into sample_intervals values (interval '20' month, interval '2 days 1:1:1.123456' day to second);
select y2m::text, m2s::text from sample_intervals;

      y2m      |       m2s       
---------------+-----------------
 1 year 8 mons | 2 days 01:01:00

update sample_intervals set y2m = interval '2' year where y2m = interval '1-8' year to month;
select * from sample_intervals;

   y2m   |       m2s       
---------+-----------------
 2 years | 2 days 01:01:00

delete from sample_intervals where m2s = interval '2 1:1:0' day to second;
select * from sample_intervals;

 y2m | m2s 
-----+-----

Estilos de intervalo

Puede utilizar el comando SET para cambiar el formato de salida de sus cadenas de intervalos. Las opciones disponibles son postgres, sql_standard, y postgres_verbose. El valor predeterminado es postgres. A continuación, se muestran ejemplos de cada formato de salida.

Formato de salida sql_standard

Los valores del intervalo de año a mes tienen el siguiente formato. Si se especifica un valor negativo antes del intervalo, eso indica que el intervalo es un valor negativo y se aplica a todo el intervalo.

'[-]yy-mm'

Los valores del intervalo de día a segundo tienen el siguiente formato.

'[-]dd hh:mm:ss.ffffff'
INTERVAL '1-2' YEAR TO MONTH outputs as '1-2'
                     
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 2:03:04.5678'

Formato de salida postgres

A continuación figura el formato de salida para postgres. Cada valor numérico puede ser negativo.

[@] '<numeric> <unit> [, <numeric> <unit> ...]'
INTERVAL '1-2' YEAR TO MONTH outputs as '1 year 2 mons'
                  
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 day 02:03:04.5678'

Formato de salida postgres_verbose

La sintaxis de postgres_verbose es similar a la de postgres, pero las salidas de postgres_verbose también contienen la unidad de tiempo.

'[@] <numeric> <unit> [, <numeric> <unit> ...] [direction]'
INTERVAL '1-2' YEAR TO MONTH outputs as '@ 1 year 2 mons'

INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '@ 1 day 2 hours 3 mins 4.5678 secs'

El siguiente comando muestra cómo establecer el estilo de intervalo como sql_standard.

SET intervalstyle to 'sql_standard';

Limitaciones 

Esta característica de INTERVAL en versión preliminar tiene las siguientes limitaciones:

  • Si utiliza el controlador JDBC, no puede usar los intervalos de año a mes y de día a segundo como parámetros para las instrucciones PREPARE.

  • El controlador JDBC interpreta todos los valores de los intervalos como cadenas, por lo que solo puede obtener los intervalos devueltos por una consulta SELECT mediante el método getString.

  • Los procedimientos almacenados no aceptan valores de intervalo como parámetros de función.

  • No puede convertir un número en un intervalo.
