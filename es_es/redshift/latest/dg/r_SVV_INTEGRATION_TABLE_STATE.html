SVV_INTEGRATION_TABLE_STATE

SVV_INTEGRATION_TABLE_STATE muestra detalles sobre la información de integración en el nivel de tabla.

SVV_INTEGRATION_TABLE_STATE solo es visible para los superusuarios. Para obtener más información, consulte Visibilidad de datos en las tablas y vistas de sistema.

Para obtener más información, consulte Uso de integraciones sin ETL.

Columnas de la tabla

Nombre de la columna Tipo de datos Descripción
integration_id character (128) Identificador asociado con la integración.
target_database character (128) Se trata del nombre de la base de datos de Amazon Redshift.
schema_name character (128) Nombre del esquema de Amazon Redshift.
table_name character (128) El nombre de la tabla.
table_state character (128) Estado de la tabla. Los valores posibles son Synced, Failed, Deleted, ResyncRequired y ResyncInitiated.
table_last_replicated_checkpoint character (128) Coordenadas del registro sincronizado actual.
reason character (256) Motivo de la última transición de estado. Los motivos más comunes pueden ser tipos de datos no compatibles en las tablas, ya que las tablas no tienen claves principales. Para obtener más información sobre cómo solucionar problemas comunes, consulte Solución de problemas de integraciones sin ETL en Amazon Redshift.
last_updated_timestamp TIMESTAMP sin zona horaria Hora (UTC) en la que se actualizó la tabla por última vez.

Consultas de ejemplo

El siguiente comando SQL muestra el registro de las integraciones. 

select * from svv_integration_table_state;
