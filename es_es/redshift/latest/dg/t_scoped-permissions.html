Permisos acotados

Utilice los permisos acotados para modificar los permisos de un usuario o rol en el ámbito de una base de datos o un esquema.

Los permisos acotados se aplican a los objetos del ámbito seleccionado al conceder o revocar el permiso, así como a los objetos nuevos creados después de conceder o revocar el permiso. Por ejemplo, conceder a un rol un permiso acotado para seleccionar entre las tablas de una base de datos permite a los usuarios con ese rol seleccionar entre las tablas que se creen en el futuro y también podrán seleccionar entre las tablas que existen actualmente en la base de datos.

Puede usar los permisos acotados para establecer los permisos predeterminados en una base de datos compartida creada a partir de un recurso compartido de datos. Un usuario de un recurso compartido de datos del lado del consumidor al que se le conceden permisos acotados en una base de datos compartida los recibirá automáticamente para cualquier objeto nuevo que se añada al recurso compartido de datos del lado del productor.

Para obtener más información sobre la aplicación de permisos acotados, consulte GRANT y REVOKE.

Consideraciones para utilizar permisos acotados

Cuando utilice permisos acotados, tenga en cuenta lo siguiente:

  • Puede utilizar permisos acotados para conceder permisos en el ámbito de una base de datos o un esquema a un usuario o rol elegido, o también puede revocarlos. No puede conceder permisos acotados a grupos de usuarios, por lo que no puede revocarlos.

  • Al conceder o revocar permisos acotados, se modifican los permisos de todos los objetos actuales y futuros del ámbito.

  • Los permisos acotados se aplican independientemente de los permisos en el nivel de objeto. No se excluyen mutuamente. Por ejemplo, pensemos en un usuario que tiene permisos en el nivel de objeto para acceder a una tabla y también permisos acotados para el esquema en el que se encuentra la tabla. En las dos situaciones siguientes, el usuario conserva el acceso a la tabla.

    • Se revoca el permiso acotado del usuario para el esquema en el que se encuentra la tabla, pero el usuario sigue teniendo acceso a la tabla mediante un permiso en el nivel de objeto para la tabla.

    • Se revoca el permiso en el nivel de objeto del usuario para la tabla, pero el usuario sigue teniendo acceso a la tabla mediante un permiso acotado para el esquema.

  • Para conceder o revocar permisos acotados, debe cumplir uno de los siguientes criterios:

    • Superusuarios

    • Usuarios con la opción de conceder ese permiso Para obtener más información sobre las opciones de concesión, consulte el parámetro WITH GRANT OPTION en GRANT.

  • Los permisos acotados solo se pueden conceder o revocar a los objetos de la base de datos conectada o a las bases de datos importadas desde un recurso compartido de datos.

  • Puede conceder permisos acotados que se apliquen a las bases de datos que se hayan creado a partir de los recursos compartidos de datos.
