Fragmentos de la plantilla de Amazon CloudWatch Logs

Amazon CloudWatch Logs puede monitorizar sus archivos de registro personalizados, de las aplicaciones y del sistema desde instancias Amazon EC2 u otros orígenes. Puede usar AWS CloudFormation para aprovisionar y administrar grupos de registro y filtros de métricas. Para obtener más información de introducción a Amazon CloudWatch Logs, consulte la sección de Monitorización de archivos del sistema, de aplicaciones y de registro personalizados en la Guía del usuario de Amazon CloudWatch.

Enviar registros a CloudWatch Logs desde una instancia Linux

La siguiente plantilla describe un servidor web y sus métricas personalizadas. Los eventos de registros del registro del servidor web proporcionan los datos para las métricas personalizadas. Para enviar eventos de registro a una métrica personalizada, el campo UserData instala un agente de CloudWatch Logs en la instancia Amazon EC2. La información de configuración para el agente, como la ubicación del archivo de registro del servidor, el nombre del grupo de registro y el nombre del flujo de registros, se definen en el archivo /tmp/cwlogs/apacheaccess.conf. El flujo de registros se crea después de que el servidor web comienza a enviar eventos de registro al archivo /var/log/httpd/access_log.

nota

Una nota sobre permisos: la instancia WebServerHost hace referencia al perfil de la instancia LogRoleInstanceProfile, lo que, a su vez, hace referencia al rol LogRole. LogRole especifica el permiso s3:GetObject para arn:aws:s3:::*.

Este permiso es necesario porque WebServerHost descarga el agente de CloudWatch Logs (awslogs-agent-setup.py) de Amazon S3 en la sección UserData.

Los dos filtros de métrica describen cómo la información de registro se transforma en métricas de CloudWatch. La métrica 404 cuenta el número de casos de 404. La métrica de tamaño métrica controla el tamaño de una solicitud. Las dos alarmas de CloudWatch enviarán notificaciones si hay más de dos 404 en un plazo de 2 minutos o si el tamaño de solicitud media es de más de 3500 KB en 10 minutos.

Enviar registros a CloudWatch Logs desde una instancia Windows

La siguiente plantilla configura CloudWatch Logs para una instancia de Windows 2012 R2.

El agente CloudWatch Logs en Windows (agente SSM en AMI de Windows 2012R2 y Windows 2016) solo envía registros después de iniciarse, para que no se envíen los registros que se generan antes del inicio. Para solucionar esto, la plantilla ayuda a garantizar que el agente comience antes de que se escriban registros:

  • Configurando los parámetros del agente como primer elemento de config en cfn-init configSets.

  • Utilizando waitAfterCompletion para insertar una pausa tras el comando que inicia el agente.
