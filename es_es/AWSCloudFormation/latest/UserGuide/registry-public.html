Uso de extensiones públicas en CloudFormation

Las extensiones públicas son extensiones de CloudFormation, como los tipos de recursos o módulos, que se publican en el registro para que las usen todos los usuarios de CloudFormation. Esto incluye todas las extensiones publicadas por AWS. Los terceros también pueden publicar extensiones públicas.

Existen dos tipos de extensiones públicas:

  • Extensiones públicas de AWS: las extensiones publicadas por AWS son siempre públicas y están habilitadas de forma predeterminada, por lo que no tiene que hacer ninguna acción antes de utilizarlas en la cuenta. Además, AWS controla el control de versiones de la extensión, por lo que siempre se utiliza la última versión disponible.

  • Extensiones de terceros: son extensiones que están disponibles para uso general por editores distintos de AWS. Para usar una extensión pública, antes debe activarla en su cuenta y región.

    Puede publicar sus propias extensiones de terceros para que estén disponibles para los usuarios generales de CloudFormation. Para obtener más información, consulte Publishing extensions (Publicación de extensiones) en la Guía del usuario de la interfaz de línea de comandos de CloudFormation.

nota

Al igual que las extensiones privadas, las extensiones públicas de editores externos pueden implementar controladores de eventos que se ejecutan durante las operaciones de creación de recursos, lectura, actualización, listado y eliminación de pilas. Por eso, el uso de estas extensiones en sus pilas de CloudFormation puede incurrir en cargos en su cuenta. Estos se suman a los cargos generados por los recursos creados. Para más información, consulte Precios de AWS CloudFormation.

Activación de extensiones públicas para usarlas en su Cuenta de AWS

Para utilizar una extensión pública de terceros en su plantilla, antes debe activar la extensión para la cuenta y región en la que desea utilizarla. Al habilitar una extensión, esta se podrá utilizar en operaciones de pila en la cuenta y la región en la que se active. Cuando se activa una extensión pública de terceros, CloudFormation crea una entrada en el registro de extensiones de su cuenta para la extensión activada como una extensión privada. Esto permite personalizar la extensión tal y como se activó en su cuenta de las siguientes formas:

  • Especifique un alias para usar en lugar del nombre público de la extensión de terceros. Esto puede ayudar a evitar colisiones de nombres entre extensiones de terceros.

  • Especifique si la extensión se actualiza automáticamente cuando esté disponible una nueva versión secundaria o de parche.

  • Especifique el rol de ejecución que CloudFormation utiliza para activar la extensión, además de configurar el registro para la extensión.

Cuando se activa una extensión pública de terceros, también puede establecer cualquier propiedad de configuración que incluya la extensión. Las propiedades de configuración definen cómo se configura la extensión para una cuenta y región determinadas. Consulte lo siguiente para obtener más información sobre la configuración:

  • Defining the account-level configuration of an extension (Definición de la configuración de cuenta de una extensión)

  • Hooks configuration schema (Esquema de configuración de enlaces)

Configuración de CloudFormation para utilizar automáticamente nuevas versiones de extensiones

Cuando se activa una extensión, también puede especificar el tipo de extensión para utilizar la versión secundaria más reciente. El tipo de extensión actualiza la versión secundaria siempre que el editor publique una nueva versión en la extensión activada.

Por ejemplo, la próxima vez que lleve a cabo una operación de pila, como crear o actualizar una pila, con una plantilla que incluya esa extensión, CloudFormation utilizará la nueva versión secundaria.

La actualización a una nueva versión de extensión, ya sea de forma automática o manual, no afecta a ninguna instancia de extensión que ya esté aprovisionada en pilas.

CloudFormation considera que las actualizaciones de versiones principales de las extensiones pueden contener cambios importantes y, por lo tanto, requiere que actualice manualmente a una nueva versión principal de una extensión.

Las extensiones publicadas por AWS están activadas de forma predeterminada para todas las cuentas y regiones en las que están disponibles y siempre utilizan la última versión disponible en cada Región de AWS.

importante

Dado que usted controla si las extensiones se actualizan a la última versión en su cuenta y cuándo, puede acabar con diferentes versiones de la misma extensión implementadas en diferentes cuentas y regiones.

Esto podría dar lugar a resultados inesperados al usar la misma plantilla, que contiene esa extensión, en esas cuentas y regiones.

Especificación de alias para hacer referencia a extensiones

No puede activar más de una extensión con un nombre de pila en una cuenta y región determinadas. Dado que diferentes editores pueden ofrecer extensiones públicas con el mismo nombre de extensión, CloudFormation le permite especificar un alias para cualquier extensión pública de terceros que active.

Si especifica un alias para la extensión, CloudFormation trata el alias como el nombre del tipo de extensión dentro de la cuenta y la región. Debe usar el alias para hacer referencia a la extensión en las plantillas, las llamadas a la API y la consola de CloudFormation.

Los alias de extensión tienen que ser únicos en una cuenta y región determinadas. Puede activar el mismo recurso público varias veces en la misma cuenta y región, mediante alias de nombres de tipos diferentes.

importante

Si bien los alias de extensión solo deben ser únicos en una cuenta y región determinadas, recomendamos encarecidamente a los usuarios no asignar el mismo alias a diferentes extensiones públicas de terceros en todas las cuentas y regiones. Esto podría generar resultados inesperados al usar una plantilla que contenga el alias de extensión en varias cuentas o regiones.

Especificación de un rol de ejecución al activar extensiones

Al activar un recurso, puede especificar el rol de ejecución de IAM que CloudFormation debe asumir al invocar esa extensión en su cuenta y región.

Para que CloudFormation asuma el rol de ejecución, el rol debe tener una política de confianza definida con CloudFormation. Además, se concede permiso para llevar a cabo una operación al crear una política de IAM y adjuntarla al rol de ejecución. Los permisos necesarios se definen en la sección del controlador del esquema de extensión.

A continuación, se muestra un ejemplo de política de confianza de roles de IAM para la extensión del tipo de recurso:

{
    "Version": "2012-10-17",
    "Statement":[
        {
            "Effect": "Allow",
            "Principal":{
                "Service": "resources.cloudformation.amazonaws.com"
            },
            "Action":"sts:AssumeRole",
            "Condition":{
                "StringEquals":{
                    "aws:SourceAccount":"123456789012"
                },
                "StringLike":{
                    "aws:SourceArn":"arn:aws:cloudformation:us-east-1:123456789012:type/resource/Organization-Service-Resource/*"
                }
            }
        }
    ]
}

A continuación, se muestra un ejemplo de política de confianza de roles de IAM para la extensión de enlace:

{
    "Version":"2012-10-17",
    "Statement":[
        {
            "Effect":"Allow",
            "Principal": {
                "Service": [
                    "resources.cloudformation.amazonaws.com",
                    "hooks.cloudformation.amazonaws.com"
                ]
            },
            "Action":"sts:AssumeRole",
            "Condition":{
                "StringEquals":{
                    "aws:SourceAccount":"123456789012"
                },
                "StringLike":{
                     "aws:SourceArn":"arn:aws:cloudformation:us-east-1:123456789012:type/hook/Organization-Service-Hook/*"
                }
            }
        }
    ]
}

Para obtener más información, consulte Modifying a role trust policy (Modificación de una política de confianza de rol) en la Guía del usuario de AWS Identity and Access Management.

Para activar una extensión pública en el registro

Active su extensión en el registro de AWS CloudFormation para que esté disponible para el uso.

Para activar una extensión pública para usarla en su cuenta mediante la consola de CloudFormation

  1. Inicie sesión en la AWS Management Console y abra la consola de AWS CloudFormation en https://console.aws.amazon.com/cloudformation.

  2. En el panel de navegación de CloudFormation, en CloudFormation Registry (Registro de CloudFormation), seleccione Public extensions (Extensiones públicas).

  3. Use Filter (Filtrar) para seleccionar el tipo de extensión y seleccione Third party (Terceros). (Las extensiones publicadas por AWS están activadas de manera predeterminada).

  4. Seleccione la extensión y, a continuación, seleccione Activate (Activar).

    nota

    Si hay varias versiones de una extensión disponibles, puede utilizar el menú Version (Versión) para seleccionar la versión de la extensión que desea activar. La versión predeterminada es la versión más actualizada.

  5. Utilice las opciones de la página Activate extension (Activar extensión) para personalizar cómo se activa la extensión en esta cuenta.

    • Para activar la extensión mediante un alias de tipo extensión:

      1. Seleccione Override default (Anular configuración predeterminada).

      2. Ingrese el alias del tipo de extensión que desea usar con esta extensión. El alias debe seguir el formato recomendado para el tipo de extensión.

      Para obtener más información, consulte Specifying aliases to refer to extensions (Especificación de alias para hacer referencia a extensiones).

    • Si va a habilitar un recurso privado, puede especificar el rol de IAM que CloudFormation debe asumir al invocar la extensión. Para obtener más información, consulte Specifying an execution role when activating extensions (Especificación de un rol de ejecución al activar extensiones).

    • En Logging config (Configuración de recopilación de registros), especifique la información de configuración de registro para una extensión, si lo desea. Por ejemplo:

      {
    "logRoleArn": "arn:aws:iam::account:role/rolename",
    "logGroupName": "log-group-name"
}

      No es necesario registrar la información de configuración, pero se recomienda para fines de depuración. Para usar la configuración de registro con enlaces, agregue la misma política de confianza que el rol de ejecución especificado, de modo que el rol de registro pueda escribir registros en el grupo de registros.

      Los nombres de clave LogRoleARN y logGroupName distinguen entre mayúsculas y minúsculas.

    • En Versioning (Control de versiones), especifique cómo recibirá las actualizaciones.

      • On (Activado): se actualiza automáticamente a la última versión secundaria. Las versiones principales se actualizan manualmente.

      • Off (Desactivado): nunca se actualiza automáticamente a la última versión secundaria. Todas las versiones se actualizan manualmente.

      Para obtener más información, consulte Configuración de CloudFormation para que utilice automáticamente nuevas versiones de extensiones.

  6. Especifique los datos de configuración.

    nota

    Este paso es obligatorio para los enlaces. Debe especificar ALL para la propiedad TargetStack. Esta operación habilita las propiedades del enlace que se definen en Hook’s schema properties (Propiedades del esquema del enlace).

    Si la extensión requiere una configuración adicional en la cuenta y la región, CloudFormation incluye la sección Configuration (Configuración) en la página Activate extension (Activar extensión). Tiene la opción de especificar los datos de configuración ahora o después de que se haya activado la extensión. Si la extensión no requiere configuración, no se mostrará la sección Configuration (Configuración).

    • Para especificar los datos de configuración mientras se habilita la extensión:

      1. Seleccione Configure now (Configurar ahora) y, a continuación, Activate extension (Activar extensión).

        CloudFormation mostrará la página Configure extension (Configurar extensión). Para ver el esquema de configuración actual de la extensión, asegúrese de que la opción View configuration schema (Ver esquema de configuración) esté activada.

      2. En el cuadro de texto Configuration JSON (JSON de configuración), ingrese una cadena JSON que represente el esquema de configuración que desea especificar para esta extensión.

      3. Seleccione Configure extension (Configurar extensión).

    • Para especificar los datos más adelante:

      • Seleccione Configure later (Configurar más tarde) y, a continuación, Activate extension (Activar extensión).

      Para especificar los datos de configuración de una extensión que ya está activada, visite la página de detalles de la extensión activada.

    Una vez que la extensión se haya activado correctamente, CloudFormation mostrará la página de detalles de esa extensión.

Para actualizar una extensión pública activada en su cuenta mediante la consola de CloudFormation

Actualice su extensión en el registro de AWS CloudFormation para que esté disponible para el uso.

Después de activar una extensión en su cuenta y región, podrá actualizar las siguientes propiedades de dicha extensión a medida que se active:

  • El esquema de configuración.

  • Si desea activar o no las actualizaciones automáticas de versiones.

  • Para las extensiones registradas privadas, la función de ejecución que se va a usar.

  • La información de configuración de registro.

  1. Inicie sesión en la AWS Management Console y abra la consola de AWS CloudFormation en https://console.aws.amazon.com/cloudformation.

  2. En el panel de navegación de CloudFormation, en CloudFormation Registry (Registro de CloudFormation), seleccione Public extensions (Extensiones públicas).

  3. En el menú Actions (Acciones), seleccione Edit (Editar) y, a continuación, la opción de edición adecuada:

    • Para actualizar el esquema de configuración, consulte To specify configuration properties for an extension in the CloudFormation console (Para especificar las propiedades de configuración de una extensión en la consola de CloudFormation).

    • Para activar o desactivar las actualizaciones automáticas:

      1. Seleccione Edit automatic updates (Editar actualizaciones automáticas).

      2. Seleccione On (Activado) u Off (Desactivado) y, a continuación, Save (Guardar).

    • Para actualizar el rol de ejecución:

      1. Seleccione Edit execution role (Editar rol de ejecución).

      2. Especifique el ARN del rol de IAM que desea que CloudFormation utilice al invocar esta extensión y, a continuación, seleccione Save (Guardar).

    • Para actualizar la configuración de registro:

      1. Seleccione Edit logging config (Editar configuración de recopilación de registros).

      2. Edite el JSON de configuración de registro y, a continuación, seleccione Save (Guardar).

Para activar una extensión pública para usarla en su cuenta mediante la AWS CLI

Active su extensión en el registro de AWS CloudFormation mediante la AWS CLI.

Para activar enlaces

Una vez que haya desarrollado y registrado su enlace, puede activarlo en su Cuenta de AWS mediante la publicación en el registro.

  • Para activar un enlace en su cuenta, use la operación SetTypeConfiguration. Esta operación habilita las propiedades del enlace que se definen en la sección properties del esquema del enlace.

    nota

    Al habilitar los enlaces en su cuenta, autoriza al enlace a usar los permisos definidos de su Cuenta de AWS. CloudFormation elimina los permisos no necesarios antes de pasar los permisos al enlace. CloudFormation recomienda a los clientes o usuarios de enlaces que revisen los permisos de los enlaces y estén al tanto de los permisos que tienen concedidos los enlaces antes de habilitarlos en sus cuentas.

    Especifique los datos de configuración de la extensión de enlace registrada en la misma cuenta y Región de AWS.

    aws cloudformation --region us-west-2 set-type-configuration \
  --configuration "{"CloudFormationConfiguration":{"HookConfiguration":{"TargetStacks": "ALL", "FailureMode": "FAIL", "Properties":{}}}}" \
  --type-arn $HOOK_TYPE_ARN
    importante

    Para permitir que el enlace inspeccione la configuración de la pila de forma proactiva, debe configurar TargetStacks en ALL en la sección HookConfiguration.

Para activar los tipos y módulos de recursos
Para activar una extensión pública para usarla en su cuenta mediante la AWS CLI

  • Use activate-type para activar la extensión y especificar si se actualizará de forma automática cada vez que se publique una nueva versión secundaria de la extensión.

    En el ejemplo siguiente se especifica el nombre de recurso de Amazon (ARN) público de una extensión pública para activar para esta cuenta. Además, se especifica que CloudFormation actualiza la extensión cada vez que se publica una nueva versión secundaria.

    aws cloudformation activate-type \
  --public-type-arn public_extension_ARN \
  --auto-update true

    Este comando devuelve un ARN de la extensión activada, específico de esta cuenta y región.

    {
    "Arn": "624af370-311a-11e8-b6b7-500cexample"
}
Para actualizar la versión de una extensión pública mediante la AWS CLI

Utilice activate-type para volver a activar la extensión.

Utilice el parámetro --version-bump para especificar si se debe actualizar la extensión a la versión más reciente de MAJOR, la versión más reciente de MINOR.

aws cloudformation activate-type \
    --region us-west-2 \
    --type RESOURCE \
    --type-name Example::Test::1234567890abcdef0 \
    --type-name-alias Example::Test::Alias \
    --version-bump MAJOR

Para desactivar las extensiones públicas en su cuenta

En las secciones siguientes se muestra cómo desactivar extensiones públicas en su cuenta.

Para desactivar una extensión pública en su cuenta mediante la consola de CloudFormation

  1. Use el registro de CloudFormation para buscar la extensión.

  2. En el menú Actions (Acciones), seleccione Deactivate (Desactivar).

  3. Seleccione Deactivate (Desactivar).

Para desactivar una extensión pública en su cuenta mediante la AWS CLI

Para desactivar una extensión pública en la cuenta, especifique el valor de deactivate-type. A continuación, se muestra un ejemplo de comando de desactivación de la AWS CLI.

aws cloudformation deactivate-type \
    --region us-west-2 \
    --type MODULE \
    --type-name Example::Test::Type::MODULE

Para desactivar un enlace en su cuenta mediante la AWS CLI

La desactivación de un enlace evita que este ejecute su Cuenta de AWS.

Si se establece TargetStacks en NONE, se desactiva el enlace en su cuenta, por lo que no se aplica a las operaciones de pila.

Utilice la operación set-type-configuration y especifique TargetStacks como NONE para desactivar un enlace.

En el siguiente ejemplo, se especifica la Región de AWS y el nombre de recurso de Amazon (ARN) del enlace que se va a desactivar.

aws cloudformation set-type-configuration \
  --region us-west-2 \
  --configuration "{"CloudFormationConfiguration":{"HookConfiguration":{"TargetStacks": "NONE", "FailureMode": "FAIL", "Properties":{}}}}" \
  --type-arn HOOK_TYPE_ARN
