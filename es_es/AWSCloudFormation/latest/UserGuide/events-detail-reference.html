Referencia detallada de los eventos de AWS CloudFormation

Todos los eventos de los servicios de AWS tienen un conjunto común de campos que contienen metadatos sobre el evento, como el servicio de AWS que es el origen del evento, la hora en que se generó el evento, la cuenta y la región en las que tuvo lugar el evento, etc. Para ver las definiciones de estos campos generales, consulte Referencia de estructura de eventos en la Guía del usuario de Amazon EventBridge.

Además, cada evento tiene un campo detail que contiene datos específicos de ese evento en particular. La siguiente referencia define los campos de detalle para los distintos eventos de AWS CloudFormation.

Cuando se utiliza EventBridge para seleccionar y administrar eventos de AWS CloudFormation, es útil tener en cuenta lo siguiente:

  • El campo source para todos los eventos de AWS CloudFormation está establecido en aws.cloudformation.

  • El campo detail-type especifica el tipo de evento.

    Por ejemplo, CloudFormation Resource Status Change o CloudFormation Drift Detection Status Change.

  • El campo detail contiene los datos específicos de ese evento en particular.

    Por ejemplo, el ID de la pila, los recursos involucrados, el estado de varios recursos y otros datos relevantes para un tipo concreto de eventos.

Para obtener más información sobre cómo construir patrones de eventos que permitan que las reglas coincidan con los eventos de AWS CloudFormation, consulte Patrones de eventos en la Guía del usuario de Amazon EventBridge.

Para obtener más información sobre los eventos y cómo los procesa EventBridge, consulte Eventos de Amazon EventBridge en la Guía del usuario de Amazon EventBridge.
