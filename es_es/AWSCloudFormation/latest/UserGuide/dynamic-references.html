Uso de referencias dinámicas para especificar valores de plantillas

Las referencias dinámicas son una forma consistente y compacta de especificar valores externos almacenados y administrados en otros servicios, como Systems Manager Parameter Store y AWS Secrets Manager, en sus plantillas de pila. Cuando se utiliza una referencia dinámica, CloudFormation recupera el valor de la referencia especificada cuando es necesario durante las operaciones de pila y de conjunto de cambios.

CloudFormation admite actualmente los siguientes patrones de referencia dinámica:

  • ssm, para valores de texto no cifrado almacenados en AWS Systems Manager Parameter Store.

  • ssm-secure, para proteger las cadenas almacenadas en AWS Systems Manager Parameter Store.

  • secretsmanager, para valores secretos completos o específicos que se almacenan en AWS Secrets Manager.

Consideraciones cuando se utilizan referencias dinámicas

A continuación se indican las siguientes consideraciones que debe tener en cuenta al utilizar las referencias dinámicas:

importante

Recomendamos encarecidamente no incluir referencias dinámicas ni ningún dato confidencial en las propiedades de los recursos que formen parte del identificador principal de un recurso.

Cuando se incluye un parámetro de referencia dinámica en una propiedad que forma un identificador de recursos principal, CloudFormation puede usar el valor de texto sin formato real en el identificador del recurso principal. Este ID de recurso puede aparecer en cualquier destino o salida derivada.

Para determinar qué propiedades de recursos comprenden el identificador principal de un tipo de recurso, consulte la documentación de referencia de recursos para ese recurso. En la sección Return values (Valores de devolución), el valor de devolución de la función Ref representa las propiedades del recurso que comprenden el identificador principal del tipo de recurso.

  • Puede incluir hasta 60 referencias dinámicas en una plantilla de pila.

  • Para transformaciones, como AWS::Include y AWS::Serverless, AWS CloudFormation no resuelve las referencias dinámicas antes de invocar cualquier transformación. En su lugar, AWS CloudFormation pasa la cadena literal de la referencia dinámica a la transformación. Las referencias dinámicas (incluidas las que se han insertado en la plantilla procesada como consecuencia del resultado de una transformación) se resuelven al ejecutar el conjunto de cambios usando la plantilla.

  • Las referencias dinámicas de valores seguros, como ssm-secure y secretsmanager, no se admiten actualmente en los recursos personalizados.

nota

No cree una referencia dinámica que tenga una barra inversa (\) como valor final. AWS CloudFormation no puede resolver estas referencias, lo que resulta en un error de recurso.

Especificación de referencias dinámicas en plantillas de pilas

Las referencias dinámicas siguen el patrón que indicamos a continuación:

'{{resolve:service-name:reference-key}}' o bien '{{resolve:ssm:[a-zA-Z0-9_.\-/]+(:\d+)?}}'.

nombre-servicio

Especifica el servicio en el que se almacena y administra el valor.

Obligatorio.

Actualmente, los valores válidos son:

  • ssm: parámetro de texto no cifrado del almacén de parámetros de Administrador de sistemas.

  • ssm-secure: parámetro de cadena segura del almacén de parámetros de Administrador de sistemas.

    nota

    En la actualidad, Administrador de sistemas no admite los parámetros SecureString en las regiones cn-north-1 y cn-northwest-1.

    Para obtener más información, consulte AWS Systems Manager Parameter Store en la Guía del usuario de AWS Systems Manager.

  • secretsmanager: secreto de Secrets Manager.

clave-referencia

La clave de referencia. En función del tipo de referencia dinámica, la clave de referencia puede estar compuesta por varios segmentos.

Obligatorio.

Parámetros de SSM

Utilice la referencia dinámica ssm para incluir los valores almacenados en el almacén de parámetros de Administrador de sistemas del tipo String o StringList en sus plantillas.

Patrón de referencia

Para los parámetros de SSM, el segmento reference-key está formado por el nombre del parámetro y el número de versión. Use el siguiente patrón:

'{{resolve:ssm:parameter-name:version}}'

Su referencia debe cumplir el siguiente patrón de expresión regular para nombre-parámetro y versión:

'{{resolve:ssm:[a-zA-Z0-9_.-/]+:\\d+}}'

nombre-parámetro

El nombre del parámetro en Systems Manager Parameter Store. El nombre de parámetro distingue entre mayúsculas y minúsculas.

Obligatorio.

version

Un número entero que especifica la versión del parámetro que se va a utilizar. Si no especifica la versión exacta, CloudFormation utiliza la última versión del parámetro cada vez que cree o actualice la pila. Para obtener más información, consulte Trabajo con versiones de parámetros en la Guía del usuario de AWS Systems Manager.

Opcional.

Ejemplo

En el siguiente ejemplo se utiliza una referencia dinámica ssm para establecer el control de acceso de un bucket de S3 a un valor de parámetro almacenado en el almacén de parámetros de Administrador de sistemas. Tal y como se especifica, CloudFormation usará la versión 2 del parámetro S3AccessControl para las operaciones de pila y de conjunto de cambios.

JSON

  "MyS3Bucket": {
    "Type": "AWS::S3::Bucket",
    "Properties": {
      "AccessControl": "{{resolve:ssm:S3AccessControl:2}}"
    }
  }

YAML

  MyS3Bucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      AccessControl: '{{resolve:ssm:S3AccessControl:2}}'

Para especificar un parámetro almacenado en el almacén de parámetros de Administrador de sistemas, debe tener acceso para llamar a GetParameters para el parámetro especificado. Para obtener más información, consulte Controlling access to Systems Manager parameters (Control del acceso a los parámetros de Systems Manager) en la Guía del usuario de AWS Systems Manager.

A continuación indicamos algunas consideraciones que deben tenerse en cuenta cuando se utiliza el patrón de referencia dinámica ssm:

  • Actualmente, CloudFormation no admite el acceso transversal en cuentas del parámetro SSM.

  • En el caso de recursos personalizados, resuelve las referencias ssm dinámicas CloudFormation antes de enviar la solicitud al recurso personalizado. Para obtener más información, consulte Recursos personalizados.

  • CloudFormation no admite el uso de etiquetas de parámetros o parámetros públicos en las referencias dinámicas.

    Una etiqueta de parámetro es un alias definido por el usuario que ayuda a administrar las distintas versiones de un parámetro. Para obtener más información, consulte Labeling parameters (Etiqueta de parámetros) en la Guía del usuario de AWS Systems Manager.

    Un parámetro público es un parámetro proporcionado por un servicio de AWS para usarlo con dicho servicio, que se almacena en AWS Systems Manager Parameter Store. Para ver un ejemplo de parámetros públicos, consulte Recuperación de los metadatos de Amazon ECS-Optimized AMI en la Amazon Elastic Container Service Developer Guide.

  • CloudFormation no admite actualmente la detección de desviaciones en referencias dinámicas. Para las referencias dinámicas de ssm en las que no se ha especificado la versión del parámetro, se recomienda que, si actualiza la versión del parámetro en SSM, también lleve a cabo una operación de actualización de la pila en cualquier pila que incluya la referencia dinámica de ssm, con el fin de obtener la versión más reciente del parámetro.

  • Para verificar qué versión de una referencia dinámica de ssm se utilizará en una operación de pila, cree un conjunto de cambios para la operación de la pila. A continuación, revise la plantilla procesada en la pestaña Template (Plantilla).

  • Los parámetros de SSM sin una versión no se admiten en Parameters block (Bloque de parámetros), use SSM parameter types (Tipos de parámetros SSM) en su lugar. Si utiliza los parámetros de SSM, debe especificar una versión del parámetro de Systems Manager para que AWS CloudFormation lo use.

Parámetros de la cadena segura de SSM

Utilice el patrón de referencia dinámica de ssm-secure para especificar parámetros del tipo AWS Systems Manager SecureString en sus plantillas. Para las referencias dinámicas ssm-secure, AWS CloudFormation nunca almacena el valor del parámetro real. AWS CloudFormation obtiene acceso al valor del parámetro durante las operaciones de creación y actualización para las pilas y conjuntos de cambios. Actualmente los parámetros SecureString solo se pueden usar para las propiedades de recursos que admiten el patrón de referencia dinámica ssm-secure.

Un parámetro SecureString es toda información confidencial que debe almacenarse o a la que se hace referencia de forma segura. Es decir, los datos que no desea que los usuarios modifiquen o remitan a ellos como texto sin cifrar, por ejemplo, contraseñas o claves de licencia. Para obtener más información sobre cadenas seguras, consulte Use secure string parameters (Uso de los parámetros de cadena segura) en la Guía del usuario de AWS Systems Manager.

Los valores de los parámetros Cadena segura no se almacenan en CloudFormation ni se devuelven en ningún resultado de llamada a la API.

Patrón de referencia

Para las referencias dinámicas ssm-secure, el segmento reference-key está formado por el nombre del parámetro y el número de versión. Use el siguiente patrón:

'{{resolve:ssm-secure:parameter-name:version}}'

Su referencia debe cumplir el siguiente patrón de expresión regular para nombre-parámetro y versión:

'{{resolve:ssm-secure:[a-zA-Z0-9_.-/]+:\\d+}}'

nombre-parámetro

El nombre del parámetro en Systems Manager Parameter Store. El nombre de parámetro distingue entre mayúsculas y minúsculas.

Obligatorio.

version

Un número entero que especifica la versión del parámetro que se va a utilizar. Si no especifica la versión exacta, AWS CloudFormation utiliza la última versión del parámetro cada vez que cree o actualice la pila. Para obtener más información, consulte Trabajo con versiones de parámetros en la Guía del usuario de AWS Systems Manager.

Opcional.

Ejemplo

En el siguiente ejemplo se utiliza una referencia dinámica ssm-secure para establecer la contraseña de un usuario de IAM para una cadena segura almacenada en el almacén de parámetros de Administrador de sistemas. Tal y como se especifica, CloudFormation usará la versión 10 del parámetro IAMUserPassword para las operaciones de pila y de conjunto de cambios.

JSON

  "MyIAMUser": {
    "Type": "AWS::IAM::User",
    "Properties": {
      "UserName": "MyUserName",
      "LoginProfile": {
        "Password": "{{resolve:ssm-secure:IAMUserPassword:10}}"
      }
    }
  }

YAML

  MyIAMUser:
    Type: AWS::IAM::User
    Properties:
      UserName: 'MyUserName'
      LoginProfile:
        Password: '{{resolve:ssm-secure:IAMUserPassword:10}}'

A continuación indicamos algunas consideraciones que deben tenerse en cuenta cuando se utiliza el patrón de referencia dinámica ssm-secure:

  • CloudFormation no devuelve el valor del parámetro real para las cadenas seguras en ninguna llamada a la API, sino más bien devuelve la referencia dinámica literal.

  • CloudFormation no almacena la referencia dinámica literal, que contiene el nombre de parámetro en texto sin formato de la cadena segura.

  • Para los conjuntos de cambios, CloudFormation compara la cadena de referencia dinámica literal. No resuelve ni compara los valores reales de las referencias ssm-secure.

  • Las referencias dinámicas de valores seguros, como ssm-secure y secretsmanager, no se admiten actualmente en los recursos personalizados.

  • En los casos en que CloudFormation debe restaurar una actualización de la pila, se producirá un error en dicha operación de restauración de actualización si la versión especificada anteriormente de un parámetro de cadena segura ya no está disponible. En estos casos, lleve a cabo una de las siguientes operaciones:

    • Use CONTINUE_UPDATE_ROLLBACK para omitir el recurso.

    • Vuelva a crear el parámetro SecureString o de cadena segura en el almacén de parámetros de Administrador de sistemas y actualícelo hasta que la versión del parámetro llegue a la versión que se utiliza en la plantilla. Luego use CONTINUE_UPDATE_ROLLBACK sin omitir el recurso.

  • Actualmente, AWS CloudFormation no admite el acceso transversal en cuentas del parámetro SSM.

  • CloudFormation no admite el uso de etiquetas de parámetros o parámetros públicos en las referencias dinámicas.

    Una etiqueta de parámetro es un alias definido por el usuario que ayuda a administrar las distintas versiones de un parámetro. Para obtener más información, consulte Labeling parameters (Etiqueta de parámetros) en la Guía del usuario de AWS Systems Manager.

    Un parámetro público es un parámetro proporcionado por un servicio de AWS para usarlo con dicho servicio, que se almacena en AWS Systems Manager Parameter Store. Para ver un ejemplo de parámetros públicos, consulte Recuperación de los metadatos de Amazon ECS-Optimized AMI en la Amazon Elastic Container Service Developer Guide.

Secretos de Secrets Manager

Utilice la referencia dinámica secretsmanager para recuperar todos los valores de un secreto o la totalidad de un secreto que se almacenan en Secrets Manager para usarlos en las plantillas. Los secretos pueden ser credenciales de base de datos, contraseñas, claves de API de terceros e incluso texto arbitrario. Con Secrets Manager, puede almacenar y controlar el acceso a estos secretos de forma centralizada, de modo que pueda reemplazar las credenciales codificadas en su código (incluidas las contraseñas), con una llamada API a Secrets Manager para recuperar el secreto mediante programación. Para obtener más información, consulte ¿Qué es AWS Secrets Manager? en la Guía del usuario de AWS Secrets Manager.

Consideraciones importantes sobre el uso de referencias dinámicas con secretos de Secrets Manager

Deben tenerse en cuenta las siguientes consideraciones de seguridad importantes al utilizar referencias dinámicas para especificar secretos de Secrets Manager en las plantillas de pilas:

  • La referencia dinámica secretsmanager se pueden utilizar en todas las propiedades de recursos. El uso de la referencia dinámica secretsmanager indica que ni Secrets Manager ni CloudFormation registren o mantengan valores de secretos resueltos. Sin embargo, el valor del secreto puede mostrarse en el servicio donde se está usando el recurso. Debe revisar el uso para evitar fugas de datos secretos.

  • Cuando se actualiza un secreto en Secrets Manager, el secreto no se actualiza automáticamente en CloudFormation. Para que CloudFormation actualice una referencia dinámica de secretsmanager, debe ejecutar una actualización de pila que actualice el recurso que contiene la referencia dinámica, ya sea actualizando la propiedad del recurso que contenga la referencia dinámica secretsmanager o actualizando otra de las propiedades del recurso.

    Por ejemplo, supongamos que especifica en la plantilla la propiedad MasterPassword de un recurso AWS::RDS::DBInstance para que sea una referencia dinámica de secretsmanager y después crea una pila a partir de la plantilla. Más tarde, actualice el valor de ese secreto en Secrets Manager, pero no actualice el recurso de AWS::RDS::DBInstance en la plantilla. En ese caso, aunque actualice la pila, el valor del secreto de la propiedad MasterPassword no se actualizará y seguirá siendo el valor anterior.

    Considere también la posibilidad de utilizar Secrets Manager para que cambie automáticamente el secreto en las bases de datos o los servicios protegidos. Para obtener más información, consulte Rotar secretos de AWS Secrets Manager.

  • Las referencias dinámicas de valores seguros, como, por ejemplo, secretsmanager no se admiten actualmente en los recursos personalizados.

Permisos necesarios

Para especificar un secreto almacenado en Secrets Manager, debe tener acceso para llamar a GetSecretValue para el secreto.

Patrón de referencia

Para los secretos de Secrets Manager, el segmento reference-key se compone de varios segmentos, como el id del secreto, la clave del valor del secreto, la fase de versión y el id de versión. Use el siguiente patrón:

{{resolve:secretsmanager:secret-id:secret-string:json-key:version-stage:version-id}}

id-secreto

El nombre o el ARN del secreto.

Para obtener acceso a un secreto en su cuenta de AWS, solo debe especificar el nombre del secreto. Para obtener acceso a un secreto en otra cuenta de AWS, especifique el ARN completo del secreto.

Obligatorio.

cadena-secreta

Actualmente el único valor admitido es SecretString. El valor predeterminado es SecretString.

clave-json

El nombre de la clave del par clave-valor cuyo valor desea recuperar. Si no se especifica una json-key, CloudFormation recupera todo el texto secreto.

Este segmento no puede incluir el signo de dos puntos (:).

fase-versión

La etiqueta de fase de la versión del secreto que se usará. Secrets Manager utiliza etiquetas provisionales para realizar un seguimiento de las diferentes versiones durante el proceso de rotación. Si usa version-stage, no especifique version-id. Si no especifica version-stage ni version-id, la versión predeterminada es la AWSCURRENT.

Este segmento no puede incluir el signo de dos puntos (:).

id-versión

El identificador único de la versión del secreto a utilizar. Si especifica version-id, no especifique version-stage. Si no especifica version-stage ni version-id, la versión predeterminada es la AWSCURRENT.

Este segmento no puede incluir el signo de dos puntos (:).

Ejemplos

En el siguiente ejemplo, se utilizan los segmentos secret-name y json-key para recuperar los valores del nombre de usuario y la contraseña almacenados en el secreto MyRDSSecret. De forma predeterminada, la versión del secreto recuperada es la versión con el valor de fase de versión de AWSCURRENT.

JSON

{
    "MyRDSInstance": {
        "Type": "AWS::RDS::DBInstance",
        "Properties": {
            "DBName": "MyRDSInstance",
            "AllocatedStorage": "20",
            "DBInstanceClass": "db.t2.micro",
            "Engine": "mysql",
            "MasterUsername": "{{resolve:secretsmanager:MyRDSSecret:SecretString:username}}",
            "MasterUserPassword": "{{resolve:secretsmanager:MyRDSSecret:SecretString:password}}"
        }
    }
}

YAML

  MyRDSInstance:
    Type: 'AWS::RDS::DBInstance'
    Properties:
      DBName: MyRDSInstance
      AllocatedStorage: '20'
      DBInstanceClass: db.t2.micro
      Engine: mysql
      MasterUsername: '{{resolve:secretsmanager:MyRDSSecret:SecretString:username}}'
      MasterUserPassword: '{{resolve:secretsmanager:MyRDSSecret:SecretString:password}}'

La especificación de los siguientes segmentos recuperaría SecretString para MySecret.

'{{resolve:secretsmanager:MySecret}}' o '{{resolve:secretsmanager:MySecret::::}}'

La especificación de los siguientes segmentos recuperaría el valor de password para MySecret.

'{{resolve:secretsmanager:MySecret:SecretString:password}}'

La especificación de los siguientes segmentos recuperaría SecretString para MySecret que está en otra cuenta de AWS. Debe especificar el ARN del secreto completo para obtener acceso a otra cuenta de AWS.

'{{resolve:secretsmanager:arn:aws:secretsmanager:us-west-2:123456789012:secret:MySecret-a1b2c3}}'

La especificación de los siguientes segmentos recuperaría el valor de password para MySecret que está en otra cuenta de AWS. Debe especificar el ARN del secreto completo para obtener acceso a otra cuenta de AWS.

'{{resolve:secretsmanager:arn:aws:secretsmanager:us-west-2:123456789012:secret:MySecret-a1b2c3:SecretString:password}}'

La especificación de los siguientes segmentos recuperaría el valor de password para la versión AWSPREVIOUS de MySecret.

'{{resolve:secretsmanager:MySecret:SecretString:password:AWSPREVIOUS}}'
