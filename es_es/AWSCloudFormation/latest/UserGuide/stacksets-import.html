Importación de una pila en AWS CloudFormation StackSets

La operación de importación de pilas de AWS CloudFormation puede importar pilas existentes en conjuntos de pilas nuevos o existentes, de modo que pueda migrar las pilas existentes a un juego de pilas en una sola operación. StackSets amplía la funcionalidad de las pilas para que pueda crear, actualizar o eliminar pilas de varias cuentas y regiones con una sola operación.

Use las operaciones de importación de pilas para StackSets autoadministrados o administrados por servicios. Para StackSets autoadministrado, la operación de importación puede importar pilas en la cuenta de administrador o en diferentes cuentas de destino y Regiones de AWS. Para StackSets administrados por servicio, la operación de importación puede importar cualquier pila en el mismo AWS Organizations como cuenta de administración. La operación de importación puede importar hasta 10 pilas con los ID de pila en línea o hasta 200 pilas mediante un objeto de Amazon S3.

La importación de pilas está disponible siempre que StackSets sea compatible. Para obtener información sobre la compatibilidad de la región de StackSets, consulte StackSets regional support (Compatibilidad regional de StackSets).

Requisitos para la importación de pila

Dado que los conjuntos de pilas llevan a cabo operaciones en varias cuentas, antes de que pueda empezar a crear su primer conjunto de pilas debe tener los permisos necesarios definidos en su Cuenta de AWS.

  • Para configurar los permisos necesarios para crear un conjunto de pilas con permisos autoadministrados, consulte Realización de operaciones de conjunto de pila que implican regiones desactivadas de forma predeterminada y Concesión de permisos autoadministrados.

  • Para configurar los permisos necesarios para crear un conjunto de pilas con permisos administrados por servicios, consulte Realización de operaciones de conjunto de pila que implican regiones desactivadas de forma predeterminada y Activar el acceso de confianza con AWS Organizations.

La propiedad NoEcho no es compatible con la importación de StackSet. Las pilas que contengan NoEcho no se importarán en nuevos conjuntos de pilas mediante la importación de StackSet.

Para obtener más información sobre requisitos de StackSets, consulte Requisitos previos para las operaciones con conjuntos de pilas.

Antes de importar una pila a un conjunto de pilas, asegúrese de comprender los siguientes requisitos:

  • Las pilas solo pueden pertenecer a un conjunto de pilas.

  • Puede implementar etiquetas de pila en el conjunto de pilas especificando etiquetas explícitamente como parámetros en la operación de importación de pila.

  • Las anulaciones de parámetros personalizados de una pila no se ven afectadas durante la operación de importación.

  • Las cuotas de StackSets y las instancias de pila se aplican al importar pilas. Para obtener más información sobre las cuotas, consulte Cuotas de AWS CloudFormation.
