AWS::DynamoDB::GlobalTable CapacityAutoScalingSettings

Configura un objetivo escalable y una política de escalado automático para la capacidad de lectura o escritura de una tabla o índice secundario global.

Sintaxis

Para declarar esta entidad en su plantilla de AWS CloudFormation, use la siguiente sintaxis:

JSON

{
  "MaxCapacity" : Integer,
  "MinCapacity" : Integer,
  "SeedCapacity" : Integer,
  "TargetTrackingScalingPolicyConfiguration" : TargetTrackingScalingPolicyConfiguration
}

YAML

  MaxCapacity: Integer
  MinCapacity: Integer
  SeedCapacity: Integer
  TargetTrackingScalingPolicyConfiguration: 
    TargetTrackingScalingPolicyConfiguration

Propiedades

MaxCapacity

Las unidades de capacidad máxima aprovisionadas para la tabla global.

Obligatorio: sí

Tipo: entero

Mínimo: 1

Requisitos de actualización: sin interrupción

MinCapacity

Las unidades de capacidad mínima aprovisionadas para la tabla global.

Obligatorio: sí

Tipo: entero

Mínimo: 1

Requisitos de actualización: sin interrupción

SeedCapacity

Al cambiar el modo de facturación de PAY_PER_REQUEST a PROVISIONED, DynamoDB requiere que especifique los valores de las unidades de capacidad de lectura y escritura para la tabla y para cada índice secundario global. Estos valores se aplicarán a todas las réplicas. La tabla utilizará estos valores aprovisionados hasta que CloudFormation cree las políticas de escalado automático que configuró en la plantilla. CloudFormation no puede determinar qué capacidad necesitarán la tabla y sus índices secundarios globales en este período de tiempo, ya que dependen de la aplicación.

Si desea cambiar el modo de facturación de una tabla de PAY_PER_REQUEST a PROVISIONED, debe especificar un valor para esta propiedad para cada uno de los recursos con escala automática. Si especifica valores diferentes para el mismo recurso en diferentes regiones, CloudFormation utilizará el valor más alto que se encuentre en las SeedCapacity propiedades o. ReadCapacityUnits Por ejemplo, si su índice secundario global myGSI tiene un valor SeedCapacity de 10 en us-east-1 y un valor fijo ReadCapacityUnits de 20 en eu-west-1, inicialmente CloudFormation establecerá la capacidad de lectura en 20. myGSI Tenga en cuenta que si deshabilita ScaleIn para myGSI en us-east-1, es posible que sus unidades de capacidad de lectura no se devuelvan a 10.

También debe especificar un valor para SeedCapacity cuando vaya a cambiar el modo de facturación de una tabla de PROVISIONED aPAY_PER_REQUEST, ya CloudFormation que es posible que necesite revertir la operación (revertir el modo de facturación aPROVISIONED) y esto no podrá realizarse sin especificar un valor para.
