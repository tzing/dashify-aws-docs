数据共享的限制（预览版）

以下是通过 PREVIEW_2023 版本中公开预览版 Amazon Redshift 数据共享功能写入多数据仓库的预发行文档。文档和特征都可能会更改。我们建议您只在测试集群中使用此功能，而不要在生产环境中使用。有关预览条款和条件，请参阅 AWS 服务条款中的测试版服务参与。

如果您尚未在预览版跟踪上创建数据共享，请前往 AWS 账户内或跨账户共享读取和写入数据开始使用。

以下是在 Redshift 中使用数据共享时的限制：

  • 目前，ra3.xlplus 集群无法通过数据共享实现多仓库写入。要使用此功能，请创建具有以下节点类型的集群：

    • ra3.4xl

    • ra3.16xl

    它还可以与带有预览版跟踪的 Amazon Redshift Serverless 工作组配合使用。

  • 对于跨账户以及跨区域数据共享，必须对创建者和使用者集群和无服务器命名空间进行加密。这是出于安全目的。但是，二者不需要共享相同的加密密钥。

  • 通过数据共享，您只能共享 SQL UDF。不支持 Python 和 Lambda UDF。

  • 如果生产者数据库有特定的排序规则，请在使用者数据库中使用相同的排序规则设置。

  • Redshift 不支持将外部架构、表或外部表上的后期绑定视图添加到数据共享。

  • Redshift 不支持生产者集群上的嵌套 SQL 用户定义的函数。

  • Redshift 不支持共享具有交错排序键的表以及引用具有交错排序键的表的视图。

  • 使用者无法将数据共享对象添加到另一个数据共享。此外，使用者无法将引用数据共享对象的视图添加到另一个数据共享。

  • Redshift 不支持访问在“准备”和“执行”访问之间发生了并发 DDL 的数据共享对象。

  • Redshift 不支持通过数据共享来共享存储过程。

有关数据共享限制的更多信息，请参阅预览版数据共享的要求和限制。
