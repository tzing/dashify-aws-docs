AWS 账户内或跨账户共享读取和写入数据（预览版）

以下是通过 PREVIEW_2023 版本中公开预览版 Amazon Redshift 数据共享功能写入多数据仓库的预发行文档。文档和特征都可能会更改。我们建议您只在测试集群中使用此功能，而不要在生产环境中使用。有关预览条款和条件，请参阅 AWS 服务条款中的测试版服务参与。
注意

目前，ra3.xlplus 集群无法通过数据共享实现多仓库写入。要使用此功能，请使用预览版跟踪创建 ra3.4xl 集群、ra3.16xl 集群或 Amazon Redshift Serverless 工作组。

有关设置预览版跟踪的更多信息，请参阅以下任一内容：

  • 对于 Amazon Redshift Serverless 预览：创建预览工作组

  • 对于 Amazon Redshift 预置集群预览版：创建预览版集群

您可以跨同一 AWS 账户内的不同 Amazon Redshift 集群或 Amazon Redshift Serverless 工作组共享数据库对象以进行读取和写入，也可以在不同 AWS 账户之间进行共享。您也可以跨区域写入数据。本主题中的过程显示了如何设置数据共享。您可以为不同的表授予 SELECT、INSERT 和 UPDATE 等权限，为不同的架构授予 USAGE 和 CREATE 权限。一旦提交写入事务，所有仓库都可获得实时数据。在跨账户共享数据时，生产者账户管理员可以选择这些账户是否获得对数据的只读访问权限。此外，生产者账户管理员可以确定特定的命名空间或区域是否获得对数据的只读、读写或任何访问权限。

以下各节介绍如何配置数据共享。这些过程假设您在处理预置集群或 Amazon Redshift Serverless 工作组中的数据库。

只读数据共享与读写数据共享

以前，数据共享中的对象在所有情况下都是只读的。写入数据共享中的对象是一项新功能。只有当生产者特别授予数据共享中对象的 INSERT 或 CREATE 等写入权限时，数据共享中的对象才会启用写入功能。详细信息将在本主题的后续章节中介绍。

如果您要查找只读数据存储的现有文档，请访问 Amazon Redshift 跨集群共享数据。

先决条件和使用注意事项

隔离设置 – 您的数据库必须设置为快照隔离。默认情况下，在 PREVIEW_2023 库中创建的数据库是快照隔离的。可以通过在 CREATE DATABASE 或 ALTER DATABASE 语句中使用 ISOLATION 关键字来更改隔离设置。

注意

目前，ra3.xlplus 集群无法通过数据共享实现多仓库写入。要使用此功能，请使用预览版跟踪创建 ra3.4xl 集群、ra3.16xl 集群或 Amazon Redshift Serverless 工作组。

以生产者数据库管理员的身份在数据共享中共享数据库对象以进行读写访问

要开始数据共享，生产者上的管理员会创建一个数据共享并向其中添加对象：

  1. 生产者数据库所有者或超级用户创建数据共享。数据共享是数据库对象、权限和使用者的逻辑容器。（使用者是指您的账户和其他账户中的集群或 Amazon Redshift Serverless 命名空间。） 每个数据共享都与在其中创建的数据库相关联，并且只能添加该数据库中的对象。下面的命令将创建一个数据共享：

    CREATE DATASHARE my_datashare [PUBLICACCESSIBLE = TRUE];

    设置 PUBLICACCESSIBLE = TRUE 允许使用者从可公开访问的集群和预置工作组中查询您的数据共享。如果您不想允许查询，请将其省略或明确设置为 false。

    数据共享所有者必须对要添加到数据共享的架构授予 USAGE 权限。GRANT 命令是新的。它用于授予对架构的各种操作，包括 CREATE 和 USAGE。架构中包含共享对象：

    CREATE SCHEMA myshared_schema1;
CREATE SCHEMA myshared_schema2;
 
GRANT USAGE ON SCHEMA myshared_schema1 TO DATASHARE my_datashare;
GRANT CREATE, USAGE ON SCHEMA myshared_schema2 TO DATASHARE my_datashare;

    或者，管理员可以继续运行 ALTER 命令将架构添加到数据共享。以这种方式添加架构时，仅授予 USAGE 权限。

    ALTER DATASHARE my_datashare ADD SCHEMA myshared_schema1;

  2. 管理员添加架构后，可以授予对架构中对象的数据共享权限。这些权限可以是读取和写入权限。GRANT ALL 示例显示了如何授予所有权限。

    GRANT SELECT, INSERT ON TABLE myshared_schema1.table1, myshared_schema1.table2, myshared_schema2.table1
TO DATASHARE my_datashare;
                     
GRANT ALL ON TABLE myshared_schema1.table4 TO DATASHARE my_datashare;

    您可以继续运行 ALTER DATASHARE 等命令来添加表。这样做时，只会对添加的对象授予 SELECT 权限。

    ALTER DATASHARE my_datashare ADD TABLE myshared_schema1.table1, myshared_schema1.table2, myshared_schema2.table1;

  3. 管理员将数据共享的使用权限授予账户中的特定命名空间。您可以在集群详细信息页面、Amazon Redshift Serverless 命名空间详细信息页面中或通过运行命令 SELECT current_namespace; 来找到作为 ARN 一部分的命名空间 ID。有关更多信息，请参阅 CURRENT_NAMESPACE。

    GRANT USAGE ON DATASHARE my_datashare TO NAMESPACE '86b5169f-012a-234b-9fbb-e2e24359e9a8';

授权与其他账户共享数据共享

在生产者上创建数据共享并向其中添加对象后，您可能希望与另一个 AWS 账户共享该数据共享。授权其他账户使用数据共享是一个辅助安全流程，必须由数据安全管理员在生产者身上执行。它授权使用者账户使用数据共享。您可以将数据共享授权给其他账户，用于只读或读写目的。如果授权写入，则允许该账户拥有授予数据共享的写入权限。否则，只能使用授予数据共享的读取权限。有关此操作的更多信息，请参阅以生产者安全管理员身份授权写入数据共享（预览版）。

以使用者身份关联来自其他账户的数据共享

如果数据共享来自其他账户，则必须将该数据共享与账户中的特定集群或命名空间相关联，然后才能将该数据共享与这些集群或命名空间一起使用。您可以将该数据共享与账户中的所有命名空间、账户中特定区域的所有命名空间或账户中的特定命名空间相关联。您可以选择是否允许命名空间具有只读访问权限或允许读写访问权限。如果关联写入，则允许关联的集群和 Serverless 命名空间获得授予数据共享的写入权限。否则，只能使用授予数据共享的读取权限。有关如何执行此操作的更多信息，请参阅以使用者数据安全管理员身份关联共享数据（预览版）。

您可以向数据共享授予的权限

在数据共享上下文中，您可以向它们授予不同的对象类型和各种权限。

架构：

  • USAGE

  • CREATE

表：

  • SELECT

  • INSERT

  • UPDATE

  • DELETE

  • TRUNCATE

  • DROP

  • REFERENCES

函数：

  • EXECUTE

允许集群或 Serverless 工作组查询数据共享

此步骤假设数据共享来自另一个集群或您账户中的 Amazon Redshift Serverless 命名空间，或者来自另一个账户并已与您正在使用的命名空间相关联。

  1. 使用者数据库管理员可以从数据共享中创建数据库。

    CREATE DATABASE my_ds_db [WITH PERMISSIONS] FROM DATASHARE my_datashare OF NAMESPACE 'abc123def';

    如果您使用 WITH PERMISSIONS 创建数据库，则可以向不同的用户和角色授予对数据共享对象的精细权限。否则，所有获得对数据共享数据库的 USAGE 权限的用户和角色都将获得对数据共享数据库中所有对象的所有权限。

  2. 以下内容显示如何向 Redshift 数据库用户或角色授予权限。您必须连接到本地数据库才能运行这些语句。如果在运行 grant 语句之前在数据共享数据库上执行 USE 命令，则无法运行这些语句。

    GRANT USAGE ON DATABASE my_ds_db TO ROLE data_eng;
GRANT CREATE, USAGE ON SCHEMA my_ds_db.my_shared_schema TO ROLE data_eng;
GRANT ALL ON ALL TABLES IN SCHEMA my_ds_db.my_shared_schema TO ROLE data_eng;
 
GRANT USAGE ON DATABASE my_ds_db TO bi_user;
GRANT USAGE ON SCHEMA my_ds_db.my_shared_schema TO bi_user;
GRANT SELECT ON my_ds_db.my_shared_schema.table1 TO bi_user;

写入数据共享中的对象

写入数据共享中的对象有两种方法。

使用 USE 命令连接到本地数据库

借助此命令，您可以使用由两部分组成的表示法查询数据共享的对象并运行多语句事务。您只能在单个数据库中运行多语句事务。

USE database_name;

您可以通过运行 RESET USE 命令重新使用所连接的数据库。

RESET USE;

使用数据库后，即可在该数据库中开始事务。您不能在单个事务中写入多个数据库。

重要

如果您在没有先提交或结束当前事务的情况下运行 USE 命令，则数据仓库将重新启动。在运行 USE 语句之前，请务必结束或提交未完成的事务。

USE database_name;    --set the context to the datashare database
 
INSERT INTO datashare_schema.table1 VALUES (1);
 
BEGIN; --start a transaction
 
INSERT INTO datashare_schema.table2 VALUES (1);
INSERT INTO datashare_schema.table3 VALUES (1);
 
COMMIT;  --end transaction; you can also use the END command.
注意

您无法将数据从 parquet 和 ORC 文件复制到数据共享表。

通过 Redshift JDBC、ODBC 或 Python 驱动程序直接连接到数据共享数据库

如果您直接连接到数据共享数据库，则可以使用由两部分组成的表示法查询表，并立即在数据共享数据库中运行多语句事务。

--connect to the datashare database via JDBC, ODBC, or the Python Driver
 
INSERT INTO datashare_schema.table1 VALUES (1);
 
BEGIN; --start a transaction
 
INSERT INTO datashare_schema.table2 VALUES (1);
INSERT INTO datashare_schema.table3 VALUES (1);
 
COMMIT;  -- end transaction; can also use the END command.
重要

支持运行 INSERT INTO SELECT 语句，但要执行此操作，您必须对查询中引用的所有表具有 INSERT 权限。

注意

即将支持通过 Data API 连接数据共享数据库。

预览版数据共享的要求和限制

  • 必须直接连接到数据共享数据库或运行 USE 命令才能写入数据共享。但是，我们很快就会启用使用由三部分组成的表示法进行此操作的功能。

  • 您必须使用 Serverless 工作组、ra3.4xl 集群或 ra3.16xl 集群才能使用此功能。计划支持 ra3.xlplus 集群。

  • 元数据发现 – 如果您是通过 Redshift JDBC、ODBC 或 Python 驱动程序直接连接到数据共享数据库的使用者，您可以通过以下方式查看目录数据：

    • SQL SHOW 命令。

    • 查询 information_schema 表和视图。

    • 查询 SVV 元数据视图。

  • 您无法通过 Data API 连接到数据共享数据库。我们将很快提供这方面的支持。

  • 使用者看不到授予数据共享的权限。我们将很快添加这一功能。

  • 对于跨账户数据共享，必须对创建器和使用者集群进行加密。

  • 您数据库的隔离级别必须是快照隔离，以便允许其他 Serverless 工作组和集群对其进行写入。

  • 写入数据共享对象的使用者不会触发自动分析操作。因此，在向表中插入数据后，生产者必须手动运行分析才能更新表统计信息。否则，查询计划可能无法达到最佳效果。

支持的 SQL 语句

通过写入共享数据的公开预览版支持以下语句：

  • BEGIN | START TRANSACTION

  • END | COMMIT | ROLLBACK

  • COPY without COMPUPDATE

  • { CREATE | DROP } SCHEMA

  • { CREATE | DROP | SHOW } TABLE

  • CREATE TABLE table_name AS

  • DELETE

  • { GRANT | REVOKE } privilege_name ON OBJECT_TYPE object_name TO consumer_user

  • INSERT

  • SELECT

  • INSERT INTO SELECT

  • TRUNCATE

  • UPDATE

  • 超级数据类型列

不支持的语句类型 - 不支持以下语句类型：

  • 写入生产者时，对使用者仓库进行多语句查询。

  • 并发扩展查询从使用者写入生产者。

  • 自动复制作业从使用者写入生产者。

  • 流式作业从使用者写入生产者。

  • 使用者在生产者集群上创建零 ETL 集成表。有关零 ETL 集成的更多信息，请参阅使用零 ETL 集成。

  • 用交错排序键写入表。
