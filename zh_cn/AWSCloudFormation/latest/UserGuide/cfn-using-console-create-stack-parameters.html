指定堆栈名称和参数

选择堆栈模板后，指定已在此模板中定义的堆栈名称和参数值。

利用参数，您可在创建时自定义您的堆栈。可在堆栈模板中使用参数值来修改资源的配置方式。这样一来，您无需对多个模板中的代码值进行硬编码来指定其他设置。有关 AWS CloudFormation 模板中的参数的更多信息，请参阅 参数。

指定堆栈名称和参数值

  1. 在 Specify stack details (指定堆栈详细信息) 页面上，在堆栈名称框中键入堆栈名称。

    堆栈名称是帮助您从堆栈列表中查找特定堆栈的标识符。堆栈名称只能包含字母数字字符（区分大小写）和连字符。该名称必须以字母字符开头，且不得超过 128 个字符。

  2. 在 Parameters 部分中，指定在堆栈模板中定义的参数。

    您可使用或更改带默认值的任何参数。

  3. 如果您对参数值满意，请选择下一步，以继续为堆栈设置选项。

特定于 AWS 的参数类型

在创建包含特定于 AWS 的参数类型的堆栈时，CloudFormation 控制台提供这些参数的有效值的下拉列表。根据参数类型，您可按 ID、名称或 Name 标签的值来搜索值。例如，利用 AWS::EC2::VPC::Id 参数类型，您可搜索特定的 VPC ID，如 vpc-b47658d1。如果使用一个名称（如 Name:TestVPC）标记 VPC，则也可以搜索 TestVPC。目前，您只能搜索带有 Name 键的标签值。

注意

控制台不提供下拉列表，也不允许您搜索带 AWS::EC2::Image::Id 参数类型的值。CloudFormation 仅验证输入值是否为有效的 Amazon Elastic Compute Cloud 映像 ID。

组和排序参数

控制台按照输入参数逻辑 ID 的字母顺序列出输入参数。创建模板时，您可以使用 AWS::CloudFormation::Interface 元数据键覆盖默认排序方式。有关更多信息和 AWS::CloudFormation::Interface 元数据键的示例，请参阅 AWS::CloudFormation::Interface。
