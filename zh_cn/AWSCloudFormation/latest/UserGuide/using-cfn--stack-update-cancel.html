取消堆栈更新

堆栈更新开始后，如果堆栈仍处于 UPDATE_IN_PROGRESS 状态，则您可以取消堆栈更新。更新完成之后，将无法取消它。但是，您可以通过任何先前的设置再次更新堆栈。

如果您取消堆栈更新，那么堆栈将回滚到启动堆栈更新之前已存在的堆栈配置。

要取消堆栈更新，请使用控制台

  1. 从 AWS CloudFormation 控制台上的堆栈列表中，选择当前正在更新的堆栈。其状态必须为 UPDATE_IN_PROGRESS。

  2. 选择 Stack actions (堆栈操作)，然后选择 Cancel update stack (取消更新堆栈)。

  3. 要继续取消更新，请选择 Cancel update (取消更新)。否则，选择 Cancel (取消) 以恢复更新。

堆栈将进入 UPDATE_ROLLBACK_IN_PROGRESS 状态。更新取消过程完成后，堆栈状态将被设置为 UPDATE_ROLLBACK_COMPLETE。

要取消堆栈更新，请使用命令行

  • 使用命令 aws cloudformation cancel-update-stack 可取消更新。
