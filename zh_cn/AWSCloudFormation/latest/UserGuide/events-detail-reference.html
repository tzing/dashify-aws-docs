AWS CloudFormation 事件详细信息参考

来自 AWS 服务的所有事件都有一组公共字段，其中包含有关事件的元数据，例如作为事件来源的 AWS 服务、事件的生成时间、事件发生的账户和区域。有关这些常规字段的定义，请参阅《Amazon EventBridge 用户指南》中的事件结构参考。

此外，每个事件都有一个 detail 字段，其中包该特定事件专有的数据。下面的参考定义了各种 AWS CloudFormation 事件的详细信息字段。

在使用 EventBridge 选择和管理 AWS CloudFormation 事件时，请记住以下几点：

  • 来自 AWS CloudFormation 的所有事件的 source 字段都设置为 aws.cloudformation。

  • detail-type 字段指定事件类型。

    例如，CloudFormation Resource Status Change 或 CloudFormation Drift Detection Status Change。

  • detail 字段包含该特定事件专有的数据。

    例如，堆栈 ID、涉及的资源、各种资源的状态以及与特定类型事件相关的其他数据。

有关如何构造使规则能够匹配 AWS CloudFormation 事件的事件模式的信息，请参阅《Amazon EventBridge 用户指南》中的事件模式。

有关事件以及 EventBridge 如何处理事件的更多信息，请参阅《Amazon EventBridge 用户指南》中的 Amazon EventBridge 事件。
