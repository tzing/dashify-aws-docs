データベース開発者向けの情報

Amazon Redshift を初めて使用する場合は、「Amazon Redshift Serverless」をお読みして開始方法について調べることをお勧めします。

データベースユーザー、データベース設計者、データベース開発者、データベース管理者向けの情報を見つけるには、次の表を参考にしてください。

目的が 推奨設定

Amazon Redshift データウェアハウスの内部的なアーキテクチャについて説明します。

システムとアーキテクチャの概要 で、Amazon Redshift の内部的なアーキテクチャの概要を説明しています。

Amazon Redshift ウェブサービスの全体的な概要については、Amazon Redshift の製品詳細ページを参照してください。

データベース、テーブル、ユーザー、およびその他のデータベースオブジェクトを作成する。

「Getting started using databases」では、SQL 開発の基本を紹介しています。

Amazon Redshift SQL には、Amazon Redshift の SQL コマンドと関数およびその他の SQL 要素の構文と例があります。

「Amazon Redshift テーブル設計のベストプラクティス」は、ソートキー、分散キー、および圧縮エンコーディングの選択に関する推奨事項の要約です。

パフォーマンスが最適になるようにテーブルを設計する方法を知る。

「自動テーブル最適化の使用」で、テーブル列内のデータへの圧縮適用に関する、および分散キーとソートキーの選択に関する考慮事項を詳しく説明しています。

データをロードする。

データのロード で、大容量のデータセットを Amazon DynamoDB のテーブルから、または Amazon S3 のバケットに格納されたフラットファイルからロードする手順を説明しています。

「データをロードするための Amazon Redshift のベストプラクティス」には、データのロードの時間短縮と効率向上のためのヒントがあります。

ユーザー、グループ、およびデータベースのセキュリティを管理する。

「データベースセキュリティの管理」で、データベースのセキュリティに関するトピックを取り上げています。

システムのパフォーマンスを監視して最適化する。

「システムテーブルとビューのリファレンス」で、システムテーブルおよびビューについて詳しく説明しています。これらに対してクエリを実行すると、データベースのステータスが分かり、クエリやプロセスの監視を行うことができます。

また、「Amazon Redshift 管理ガイド」を参照して、AWS Management Console を使用してシステムのヘルスチェック、メトリクスのモニタリング、クラスターのバックアップと復元を行う方法をお調べください。

大容量データセットからの情報を分析してレポートを作成する。

多くのソフトウェアベンダーから、各社の製品と Amazon Redshift の組み合わせが認定されているため、現在お使いのツールを引き続きご使用いただけます。詳細については、Amazon Redshift パートナーページを参照してください。

Amazon Redshift でサポートされるすべての SQL 式、コマンド、関数の詳細については、「SQL リファレンス」を参照してください。

Amazon Redshift のリソースとテーブルを操作する。

プログラムでリソースを操作してオペレーションを実行する方法の詳細については、Amazon Redshift Serverless API ガイド、Amazon Redshift API ガイド、Amazon Redshift Data API ガイドを参照してください。

チュートリアルに従って Amazon Redshift についてより理解してください。

 Amazon Redshift のチュートリアルに従って、Amazon Redshift の機能についてお調べください。
