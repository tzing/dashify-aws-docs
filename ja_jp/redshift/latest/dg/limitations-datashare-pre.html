データ共有に関する制限事項 (プレビュー)

これは Amazon Redshift のデータ共有機能によるマルチデータウェアハウス書き込みに関するプレリリースドキュメントで、PREVIEW_2023 トラックでパブリックプレビュー版が提供されています。ドキュメントと機能はどちらも変更されることがあります。この特徴は、本番環境ではなくテストクラスターでのみ使用することをお勧めします。プレビューの契約条件については、AWS のサービス条件の「ベータサービスへの参加」を参照してください。

プレビュートラックでまだデータ共有を作成していない場合は、開始するには、「AWS アカウント内またはアカウント間での読み取りデータと書き込みデータの共有」に移動します。

Redshift でデータ共有を操作する場合、以下の制限事項があります。

  • データ共有によるマルチウェアハウス書き込みは、現在 ra3.xlplus クラスターでは使用できません。この機能を使用するには、以下のノードタイプでクラスターを作成します。

    • ra3.4xl

    • ra3.16xl

    また、プレビュートラックを備えた Amazon Redshift Serverless ワークグループでも機能します。

  • アカウント間およびリージョン間でのデータ共有の場合、プロデューサークラスターとコンシューマークラスターおよびサーバーレス名前空間の両方を暗号化する必要があります。これはセキュリティ上の目的です。ただし、同じ暗号キーを共有する必要はありません。

  • SQL UDF はデータ共有を通じてのみ共有できます。Python と Lambda UDF はサポートされていません。

  • プロデューサデータベースに特定の照合順序がある場合は、コンシューマーデータベースでも同様な照合順序設定を使用します。

  • Redshift は、データ共有に対する外部スキーマやテーブルの追加、または外部テーブルの遅延バインディングビューの追加はサポートしていません。

  • Redshift は、プロデューサークラスターでネストされたユーザー定義の SQL 関数をサポートしていません。

  • Redshift は、インターリーブソートキーを持つテーブルと、インターリーブソートキーを持つテーブルを参照するビューの共有をサポートしていません。

  • コンシューマーは、データ共有オブジェクトを別のデータ共有に追加することはできません。さらに、コンシューマーはデータ共有オブジェクトを参照するビューを別のデータ共有に追加することはできません。

  • Redshift は、アクセスの準備と実行の間に同時に DDL が発生した、データ共有オブジェクトへのアクセスをサポートしていません。

  • Redshift は、データ共有によるストアドプロシージャの共有をサポートしていません。

データ共有の制限事項の詳細については、「プレビューでのデータ共有の要件と制限」を参照してください。
