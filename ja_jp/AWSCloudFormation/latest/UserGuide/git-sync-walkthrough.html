AWS CloudFormation Git 同期チュートリアル

以下のチュートリアルで Git 同期の使い方を紹介します。開始する前に、Git 同期の前提条件を満たしていることを確認してください。

ステップ 1: スタックを作成する

  1. CloudFormation コンソールにアクセスします。

  2. [スタックの作成] を選択します。

  3. [スタックの作成]ダッシュボードの [前提条件-テンプレートの準備] パネルで、[テンプレートの準備完了] が選択されていることを確認します。

  4. [テンプレートを指定] パネルで、[Git から同期] を選択します。

  5. [次へ] を選択してスタック詳細を指定します。

ステップ 2: スタックの詳細を指定する

  1. [スタック名を指定] に、スタックの名前を入力します。スタック名には、文字 (A～Z および a～z)、数字 (0～9)、およびダッシュ (-) を使用できます。

  2. スタックのデプロイファイルには 2 つのオプションがあります。

    • 以下のパラメータを使用してファイルを作成し、リポジトリに配置します。

      Git 同期でスタックデプロイファイルを作成してリポジトリに配置する場合は、このオプションを選択します。

    • 自分のリポジトリに自分のファイルを置いています。

      スタックデプロイファイルを作成してリポジトリに追加済みの場合は、このオプションを選択します。

    詳細については、「スタックデプロイファイル」を参照してください。

  3. [テンプレート定義リポジトリ] パネルには次の 2 つのオプションがあります。

    • リンクされている Git リポジトリを選択する

    • Git リポジトリをリンクする

    すでに Git リポジトリを CloudFormation にリンクしている場合は、[リンクされている Git リポジトリを選択] を選択し、次のステップに進みます。それ以外の場合は、以下のサブステップを実行して Git リポジトリに接続します。

    1. [Git リポジトリをリンクする] を選択します。

    2. Git リポジトリプロバイダーを以下から選択してください。

      • GitHub

      • GitHub エンタープライズ

      • GitLab

      • Bitbucket

    3. リストから [接続] を選択します。[接続] リストにオプションが表示されない場合は、接続コンソールに移動してリポジトリへの接続を作成してください。

  4. [リポジトリ] では、スタックテンプレートファイルが含まれている Git リポジトリを選択します。

  5. [ブランチ] リストで、Git 同期に監視させるブランチを選択します。

    注記

    Git 同期は、CloudFormation テンプレートとスタックデプロイファイルへの変更について、選択したブランチのみを監視します。スタックに適用する変更はすべてこのブランチにコミットする必要があります。

  6. デプロイファイルパスには、リポジトリブランチのルートにあるスタックデプロイファイル名を含むフルパスを指定します。

    CloudFormation がファイルを生成している場合、ここでファイルがリポジトリにコミットされます。ご自身でファイルを設置する場合、これがリポジトリ内のファイルの場所です。

  7. IAM ロールを追加します。IAM ロールには、CloudFormation が Git リポジトリからスタックを同期するために必要な権限が含まれています。[新規 IAM ロール] を選択して新しいロールを生成するか、[既存の IAM ロール] を選択して CloudFormation アカウントから既存のロールを選択できます。新しいロールの生成を選択すると、必要な権限がロールに含まれます。

    注記

    Git 同期に必要な権限と信頼ポリシーに関する重要な情報については、「必要なアクセス許可」を参照してください。

  8. テンプレートファイルパスには、リポジトリのルートからのスタックテンプレートファイルのフルパスを指定します。

  9. スタックのデプロイファイルから設定するスタックのパラメータを追加します。パラメータはキーと値のペアです。たとえば、port パラメータをスタックデプロイファイルに追加するには:

    1. [Add parameter] (パラメータの追加) を選択します。

    2. [キー] に「port」と入力します。

    3. [Value (値)] に「8080」と入力します。

    4. スタックデプロイファイルで使用できるようにするすべてのスタックパラメータについて、前のステップを繰り返します。

  10. (オプション) タグを追加します。タグは、スタックにあるリソースの特定、および分類に使用できるキーと値のペアです。スタックのデプロイにタグを追加するには、パラメータを追加したときと同じプロセスに従います。

  11. [次へ] を選択して [スタックオプションの設定] に進みます。

ステップ 3: スタックオプションの設定

スタックオプションの設定については、「AWS CloudFormation スタックオプションの設定」を参照してください。

設定が完了したら、[次へ] を選択してレビューとデプロイを続行します。

ステップ 4: レビューとデプロイ

  1. スタック設定を確認し、以下を確認します。

    • スタックテンプレートは正しく設定され、[Git から同期] に設定されています。

    • デプロイファイルは正しく設定されています。

    • テンプレート定義リポジトリは正しく設定されており、具体的には、正しいリポジトリとブランチ名が選択されています。

    • デプロイファイルのプレビューは正しく、期待通りのパラメータと値が含まれています。

  2. [送信] を選択してスタックを作成します。

    重要

    [送信] を選択すると、プルリクエストが Git リポジトリに自動的に作成されます。プロセスを完了するには、このプルリクエストを Git リポジトリにマージする必要があります。プルリクエストをマージすると、スタックが作成され、Git 同期は CloudFormation テンプレートとスタックデプロイファイルの変更を監視してスタックを更新します。

スタックの変更

前述のステップを完了すると、新しいスタックが作成され、CloudFormation が Git リポジトリの変更を監視します。

スタックを変更するには、スタックのデプロイファイルに直接変更を加えます。次に、Git 同期でスタック用に設定したリポジトリとブランチにコミットします。

CloudFormation ダッシュボードで、スタックを選択し、[Git 同期] タブを選択すると、スタックと同期イベントのステータスに関する情報が表示されます。詳細については、「AWS CloudFormation Git 同期ステータスダッシュボード」を参照してください。
