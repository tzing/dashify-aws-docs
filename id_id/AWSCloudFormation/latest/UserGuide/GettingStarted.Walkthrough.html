Memulai

Dengan template yang tepat, Anda dapat menyebarkan sekaligus semua AWS sumber daya yang Anda butuhkan untuk suatu aplikasi. Di bagian ini, Anda akan memeriksa template yang mendeklarasikan sumber daya untuk WordPress blog, membuat WordPress blog sebagai tumpukan, memantau proses pembuatan tumpukan, memeriksa sumber daya di tumpukan, dan kemudian menghapus tumpukan. Anda menggunakan AWS Management Console untuk menyelesaikan tugas-tugas ini.

Langkah 1: Pilih templat

Pertama, Anda akan membutuhkan templat yang menentukan sumber daya yang diinginkan dalam tumpukan. Untuk langkah ini, Anda menggunakan templat contoh yang sudah disiapkan. Template sampel membuat WordPress blog dasar yang menggunakan instans Amazon EC2 tunggal dengan database MySQL lokal untuk penyimpanan. Template ini juga membuat grup keamanan Amazon EC2 untuk mengontrol pengaturan firewall untuk instans Amazon EC2.

penting

AWS CloudFormation gratis, tetapi AWS sumber daya yang CloudFormation menciptakan hidup (dan tidak berjalan di kotak pasir). Anda akan dikenakan biaya penggunaan standar untuk sumber daya ini sampai Anda mengakhirinya dalam tugas terakhir tutorial ini. Total biaya akan minimal. Untuk informasi tentang bagaimana Anda dapat meminimalkan biaya apa pun, kunjungi http://aws.amazon.com/free/.

Untuk melihat templat

  • Anda dapat melihat template WordPress sampel JSON atau YAMAL. Anda tidak perlu mengunduhnya karena Anda akan menggunakan URL templat nanti dalam panduan ini. Untuk informasi lebih lanjut tentang format templat, lihat Format templat AWS CloudFormation.

Template adalah file teks JSON atau YAMAL yang berisi informasi konfigurasi tentang AWS sumber daya yang ingin Anda buat di tumpukan. Untuk panduan ini, templat sampel mencakup enam bagian tingkat atas: AWSTemplateFormatVersion, Description, Parameters, Mappings, Resources, dan Outputs; namun, hanya bagian Resources yang wajib diisi.

Bagian Resources berisi definisi AWS sumber daya yang ingin Anda buat dengan template. Setiap sumber daya terdaftar secara terpisah dan menentukan properti yang diperlukan untuk menciptakan sumber daya tertentu. Deklarasi sumber daya berikut adalah konfigurasi untuk Instans EC2, yang dalam contoh ini memiliki nama logis WebServer:

Jika Anda telah membuat Instans EC2 sebelumnya, Anda dapat mengenali properti, seperti ImageId, InstanceType, dan KeyName, yang menentukan konfigurasi instans. Deklarasi sumber daya adalah cara yang efisien untuk menentukan semua pengaturan konfigurasi ini sekaligus. Ketika deklarasi sumber daya ditempatkan dalam templat, Anda dapat membuat dan mengonfigurasi semua sumber daya yang dinyatakan dengan menggunakan templat untuk membuat tumpukan. Buat tumpukan baru yang menggunakan templat yang sama untuk meluncurkan konfigurasi sumber daya yang sama.

Deklarasi sumber daya dimulai dengan string yang menentukan nama logis untuk sumber daya. Seperti yang akan Anda lihat, nama logis dapat digunakan untuk merujuk ke sumber daya dalam templat.

Anda menggunakan bagian Parameter untuk menyatakan nilai-nilai yang dapat dilewatkan ke templat ketika Anda membuat tumpukan. Parameter adalah cara yang efektif untuk menentukan informasi sensitif, seperti nama pengguna dan kata sandi, yang tidak ingin Anda simpan dalam templat itu sendiri. Ini juga merupakan cara untuk menentukan informasi yang mungkin unik untuk aplikasi tertentu atau konfigurasi yang Anda gunakan, misalnya, nama domain atau tipe instans. Saat Anda membuat WordPress tumpukan nanti di bagian ini, Anda akan melihat kumpulan parameter yang dideklarasikan dalam templat muncul di halaman Tentukan Detail dari wizard Buat Tumpukan, di mana Anda dapat menentukan parameter sebelum membuat tumpukan.

Di WebServer deklarasi sumber daya, Anda melihat KeyName properti yang ditentukan dengan KeyName parameter:

contoh JSON
"WebServer" : {
  "Type": "AWS::EC2::Instance",
  "Properties": {
    "KeyName" : { "Ref" : "KeyName" },
    ...
  }
},
contoh YAML
WebServer:
  Type: AWS::EC2::Instance
  Properties:
    KeyName:
      Ref: KeyName
    ...

Kurung kurawal berisi panggilan ke fungsi Ref dengan KeyName sebagai inputnya. Fungsi Ref mengembalikan nilai objek yang dirujuknya. Dalam hal ini, fungsi Ref menetapkan KeyName properti dengan nilai yang ditentukan untuk KeyName saat tumpukan dibuat.

Fungsi Ref juga dapat mengatur properti sumber daya untuk nilai sumber daya lain. Misalnya, deklarasi sumber daya WebServer berisi deklarasi properti berikut:

contoh JSON
"WebServer" : {
  "Type": "AWS::EC2::Instance",
  "Properties": {
    ...
    "SecurityGroups" : [ {"Ref" : "WebServerSecurityGroup"} ],
    ...
  }
},
contoh YAML
WebServer:
  Type: AWS::EC2::Instance
  Properties:
    SecurityGroups:
    - Ref: WebServerSecurityGroup
    ...

Properti SecurityGroups mengambil daftar grup keamanan EC2. Fungsi Ref memiliki input dari WebServerSecurityGroup, yang merupakan nama logis grup keamanan dalam templat, dan menambahkan nama WebServerSecurityGroup ke SecurityGroups properti.

Dalam templat, Anda juga akan menemukan bagian Pemetaan. Anda menggunakan pemetaan untuk mendeklarasikan nilai bersyarat yang dievaluasi dengan cara yang sama seperti pernyataan tabel pencarian. Templat menggunakan pemetaan untuk memilih Amazon Machine Image (AMI) yang benar untuk wilayah dan tipe arsitektur untuk tipe instans. Output menentukan nilai kustom yang dikembalikan oleh aws cloudformation describe-stacks perintah dan di tab Output CloudFormation konsol setelah pembuatan tumpukan. Anda dapat menggunakan nilai output untuk mengembalikan informasi dari sumber daya dalam tumpukan, seperti URL untuk situs web yang dibuat dalam templat. Kami mencakup pemetaan, keluaran, dan hal-hal lain tentang template secara lebih detail di Mempelajari dasar-dasar templat.

Sekian tentang templat untuk saat ini. Mari kita mulai membuat tumpukan.

Langkah 2: Pastikan Anda telah menyiapkan item yang diperlukan untuk tumpukan

Sebelum Anda membuat tumpukan dari templat, Anda harus memastikan bahwa semua sumber daya dependen yang diperlukan templat sudah tersedia. Template dapat menggunakan atau merujuk ke AWS sumber daya dan sumber daya yang ada yang dideklarasikan dalam template itu sendiri. CloudFormation mengurus memeriksa referensi ke sumber daya dalam template dan juga memeriksa referensi ke sumber daya yang ada untuk memastikan bahwa mereka ada di wilayah tempat Anda membuat tumpukan. Jika template anda mengacu pada sumber daya depended yang tidak ada, pembuatan tumpukan akan gagal.

Contoh WordPress template berisi parameter inputKeyName, yang menentukan key pair yang digunakan untuk instans Amazon EC2 yang dideklarasikan dalam template. Template bergantung pada pengguna yang membuat tumpukan dari template untuk menyediakan key pair Amazon EC2 yang valid untuk parameter tersebutKeyName. Jika Anda menyediakan nama pasangan kunci yang valid, tumpukan akan berhasil dibuat. Jika Anda tidak menyediakan namapasangan kunci yang valid, tumpukan akan dibatalkan.

Pastikan Anda memiliki key pair Amazon EC2 yang valid dan rekam nama key pair sebelum membuat stack.

Untuk melihat pasangan kunci Anda, buka konsol Amazon EC2, lalu pilih Pasangan Kunci di panel navigasi.

catatan

Jika Anda tidak memiliki key pair Amazon EC2, Anda harus membuat key pair di wilayah yang sama tempat Anda membuat tumpukan. Untuk informasi tentang membuat key pair, lihat Mendapatkan key pair SSH di Panduan Pengguna Amazon EC2 untuk Instans Linux.

Sekarang Anda memiliki key pair yang valid, mari kita gunakan WordPress template untuk membuat tumpukan.

Langkah 3: Buat tumpukan

Anda akan membuat tumpukan Anda berdasarkan file WordPress-1.0.0 yang dibahas sebelumnya. Template berisi beberapa AWS sumber daya, seperti instans EC2.

Untuk membuat WordPress tumpukan

  1. Masuk ke AWS Management Console dan buka AWS CloudFormation konsol di https://console.aws.amazon.com/cloudformation.

  2. Pilih Buat tumpukan.

  3. Di bagian Tentukan templat, pilih URL Templat Amazon S3 untuk mengetik atau menempelkan URL untuk WordPress templat sampel, lalu pilih Berikutnya:

    https://s3.us-west-2.amazonaws.com/cloudformation-templates-us-west-2/WordPress_Single_Instance.template

    catatan

    AWS CloudFormation template yang disimpan dalam bucket S3 harus dapat diakses oleh pengguna yang membuat tumpukan, dan harus berada di Wilayah yang sama dengan tumpukan yang sedang dibuat.

  4. Di bagian Tentukan detail tumpukan, masukkan nama di bidang Nama tumpukan. Untuk contoh ini, gunakan MyWPTestStack. Nama tumpukan tidak dapat berisi spasi.

  5. Di bagian Parameter, Anda harus memberikan nilai untuk semua parameter yang tidak memiliki nilai default, termasuk DBUser, DBPassword, DB, dan. RootPassword KeyName Di KeyNamebidang, masukkan nama pasangan Amazon EC2 yang valid di wilayah yang sama tempat Anda membuat tumpukan.

  6. Pilih Berikutnya.

  7. Dalam skenario ini, kami tidak akan menambahkan tag apapun. Pilih Berikutnya. Tag, yang merupakan pasangan nilai kunci, dapat membantu Anda mengidentifikasi tumpukan. Untuk informasi selengkapnya, lihat Menambahkan tag ke CloudFormation tumpukan.

  8. Tinjau informasi untuk tumpukan. Setelah Anda puas dengan pengaturan, pilih Kirim.

Langkah 4: Pantau kemajuan penciptaan tumpukan

Setelah Anda menyelesaikan wizard Create Stack, CloudFormation mulailah membuat sumber daya yang ditentukan dalam template. Tumpukan baru AndaMyWPTestStack,, muncul dalam daftar di bagian atas CloudFormationkonsol. Statusnya harus CREATE_IN_PROGRESS. Anda dapat melihat status detail untuk tumpukan dengan melihat peristiwanya.

Untuk melihat peristiwa sebuah tumpukan

  1. Di CloudFormation konsol, pilih tumpukan MyWPTestStack dalam daftar.

  2. Pilih tab Peristiwa di panel detail klaster.

    Konsol secara otomatis menyegarkan daftar peristiwa dengan peristiwa terbaru setiap 60 detik.

Tab Peristiwa menampilkan setiap langkah besar dalam pembuatan tumpukan yang diurutkan berdasarkan waktu setiap peristiwa dengan kejadian terbaru berada di atas.

Acara pertama (di bagian bawah daftar acara) adalah awal dari proses pembuatan tumpukan:

2013-04-24 18:54 UTC-7 CREATE_IN_PROGRESS AWS::CloudFormation::Stack MyWPTestStack User initiated

Berikutnya adalah peristiwa yang menandai awal dan penyelesaian penciptaan setiap sumber daya. Sebagai contoh, pembuatan instans EC2 memberikan hasil sebagai berikut:

2013-04-24 18:59 UTC-7 CREATE_COMPLETE AWS::EC2::Instance...

2013-04-24 18:54 UTC-7 CREATE_IN_PROGRESS AWS::EC2::Instance...

CREATE_IN_PROGRESSAcara dicatat ketika CloudFormation melaporkan bahwa ia telah mulai membuat sumber daya. Peristiwa CREATE_COMPLETE dicatat ketika sumber daya berhasil dibuat.

Ketika CloudFormation telah berhasil membuat tumpukan, Anda akan melihat peristiwa berikut di bagian atas tab Events:

2013-04-24 19:17 UTC-7 CREATE_COMPLETE AWS::CloudFormation::Stack MyWPTestStack

Jika tidak CloudFormation dapat membuat sumber daya, ia melaporkan CREATE_FAILED peristiwa dan, secara default, memutar kembali tumpukan dan menghapus sumber daya apa pun yang telah dibuat. Parameter kolom Alasan Status menampilkan masalah yang menyebabkan kegagalan.

Langkah 5: Gunakan sumber daya tumpukan Anda

Ketika tumpukan MyWPTestStack mempunyai status CREATE_COMPLETE, CloudFormation telah selesai membuat tumpukan, dan Anda dapat mulai menggunakan sumber dayanya.

WordPress Tumpukan sampel membuat WordPress situs web. Anda dapat melanjutkan dengan WordPress setup dengan menjalankan script WordPress instalasi.

Untuk menyelesaikan WordPress instalasi

  1. Pada tab Output, di baris WebsiteURL, pilih tautan di kolom Nilai.

    Nilai WebsiteURL output adalah URL skrip instalasi untuk WordPress situs web yang Anda buat dengan tumpukan.

  2. Pada halaman web untuk WordPress instalasi, ikuti petunjuk di layar untuk menyelesaikan WordPress instalasi. Untuk informasi lebih lanjut tentang menginstal WordPress, lihat https://wordpress.org/support/article/how-to-install-wordpress/.

    Setelah Anda menyelesaikan instalasi dan masuk, Anda diarahkan ke dasbor tempat Anda dapat mengatur opsi tambahan untuk WordPress blog Anda. Kemudian, Anda dapat mulai menulis kiriman untuk blog Anda yang berhasil dibuat dengan menggunakan CloudFormation templat.

Langkah 6: Bersihkan

Anda telah menyelesaikan tugas CloudFormation memulai. Untuk memastikan Anda tidak dikenakan biaya layanan yang tidak diinginkan, Anda dapat membersihkannya dengan menghapus tumpukan dan sumber dayanya.

Untuk menghapus tumpukan dan sumber dayanya

  1. Dari CloudFormation konsol, pilih MyWPTestStack tumpukan.

  2. Pilih Hapus Tumpukan.

  3. Pada pesan konfirmasi yang muncul, pilih Ya, hapus.

Status untuk MyWPTestStack perubahan ke DELETE_IN_PROGRESS. Dengan cara yang sama Anda memantau pembuatan tumpukan, Anda dapat memantau penghapusan dengan menggunakan tab Peristiwa. Ketika CloudFormation menyelesaikan penghapusan tumpukan, itu menghapus tumpukan dari daftar.

Selamat! Anda berhasil memilih templat, membuat tumpukan, melihat, dan menggunakan sumber dayanya, dan menghapus tumpukan dan sumber dayanya.
