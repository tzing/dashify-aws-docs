Transformasi

Bagian opsional Transform menentukan satu atau lebih makro yang digunakan AWS CloudFormation untuk memproses templat Anda. Bagian Transform dibangun pada bahasa deklaratif yang sederhana dari AWS CloudFormation dengan sistem makro yang kuat.

Anda dapat mendeklarasikan satu atau lebih makro dalam template. AWS CloudFormationmengeksekusi makro dalam urutan bahwa mereka ditentukan. Saat Anda membuat set perubahan, AWS CloudFormation menghasilkan set perubahan yang mencakup konten templat yang diproses. Anda kemudian dapat meninjau perubahan dan mengeksekusi set perubahan. Untuk informasi selengkapnya, lihat Menggunakan AWS CloudFormation makro untuk melakukan pemrosesan khusus pada templat.

AWS CloudFormation juga mendukung transformasi, yaitu makro yang di-host olehAWS CloudFormation. AWS CloudFormation memperlakukan transformasi ini sama seperti makro yang Anda buat dalam hal urutan eksekusi dan ruang lingkup. Untuk informasi rinci mengenai transformasi spesifik, lihatTransformasi referensi.

Untuk mendeklarasikan beberapa makro, gunakan format daftar dan tentukan satu atau lebih makro.

Sebagai contoh, dalam sampel templat di bawah ini, AWS CloudFormation mengevaluasi MyMacro kemudian AWS::Serverless, keduanya dapat memproses isi dari seluruh templat karena penyertaan mereka dalam bagian Transform.

// Start of processable content for MyMacro and AWS::Serverless
Transform:
  - MyMacro
  - 'AWS::Serverless'
Resources:
  WaitCondition:
    Type: 'AWS::CloudFormation::WaitCondition'
  MyBucket:
    Type: 'AWS::S3::Bucket'
    Properties: 
        BucketName: MyBucket 
        Tags: [{"key":"value"}] 
        CorsConfiguration:[] 
  MyEc2Instance:
    Type: 'AWS::EC2::Instance' 
    Properties:
        ImageID: "ami-123"
// End of processable content for MyMacro and AWS::Serverless
