Menerapkan aplikasi di Amazon EC2 dengan AWS CloudFormation

Anda dapat menggunakan AWS CloudFormation untuk menginstal, mengonfigurasi, dan memulai aplikasi secara otomatis di instans Amazon EC2. Melakukan hal ini memungkinkan Anda untuk dengan mudah menduplikasi penyebaran dan memperbarui instalasi yang ada tanpa terhubung langsung ke instans, yang dapat menghemat banyak waktu dan usaha.

CloudFormation termasuk satu set skrip pembantu (cfn-init, cfn-signal,, dan cfn-hup) yang didasarkan cfn-get-metadata pada cloud-init. Anda memanggil skrip pembantu ini dari CloudFormation templat Anda untuk menginstal, mengonfigurasi, dan memperbarui aplikasi di instans Amazon EC2 yang ada di templat yang sama.

Walkthrough berikut menjelaskan cara membuat templat yang meluncurkan tumpukan LAMP menggunakan skrip cfn helper untuk menginstal, mengonfigurasi, dan memulai Apache, MySQL, dan PHP. Anda akan mulai dengan template sederhana yang menyiapkan instans Amazon EC2 dasar yang menjalankan Amazon Linux, dan kemudian terus menambahkan ke template hingga menjelaskan tumpukan LAMP penuh.

Konfigurasi LAMP

Dengan templat yang kita miliki sekarang yang menginstal Linux, Apache, MySQL, dan PHP, kita harus memperluas templat agar secara otomatis mengonfigurasi dan menjalankan Apache, MySQL, dan PHP. Pada contoh berikut, kami memperluas bagian Parameters, sumber daya AWS::CloudFormation::Init, dan properti UserData untuk menyelesaikan konfigurasi. Dengan templat sebelumnya, bagian yang ditandai dengan elipsis (...) dihilangkan agar ringkas. Tambahan untuk templat ditampilkan dalam teks huruf miring merah.

Contoh menetapkan parameter DBUsername dan DBPassword dengan properti NoEcho-nya diatur menjadi true. Jika Anda menyetel NoEcho atribut ketrue, CloudFormation mengembalikan nilai parameter yang disamarkan sebagai tanda bintang (*****) untuk setiap panggilan yang menggambarkan peristiwa tumpukan atau tumpukan, kecuali untuk informasi yang disimpan di lokasi yang ditentukan di bawah ini.

penting

Menggunakan NoEcho atribut tidak menutupi informasi apa pun yang disimpan sebagai berikut:

  • Bagian Metadata template. CloudFormation tidak mengubah, memodifikasi, atau menyunting informasi apa pun yang Anda sertakan di Metadata bagian ini. Untuk informasi selengkapnya, lihat Metadata.

  • Bagian Outputs template. Untuk informasi selengkapnya, lihat Output.

  • MetadataAtribut definisi sumber daya. Untuk informasi selengkapnya, lihat Metadataatribut.

Kami sangat menyarankan Anda untuk tidak menggunakan mekanisme ini untuk memasukkan informasi sensitif, seperti kata sandi atau rahasia.

penting

Daripada menyematkan informasi sensitif secara langsung di CloudFormation template Anda, kami sarankan Anda menggunakan parameter dinamis dalam template tumpukan untuk mereferensikan informasi sensitif yang disimpan dan dikelola di luar CloudFormation, seperti di AWS Systems Manager Parameter Store atau AWS Secrets Manager.

Untuk informasi lebih lanjut, lihat praktik Jangan menanamkan kredensial dalam templat Anda terbaik.

CreationPolicy atribut

Akhirnya, Anda memerlukan cara untuk menginstruksikan CloudFormation untuk menyelesaikan pembuatan tumpukan hanya setelah semua layanan (seperti Apache dan MySQL) berjalan dan tidak setelah semua sumber daya tumpukan dibuat. Dengan kata lain, jika Anda menggunakan templat dari bagian sebelumnya untuk meluncurkan tumpukan, CloudFormation menetapkan status tumpukan sebagai CREATE_COMPLETE setelah berhasil membuat semua sumber daya. Namun, jika satu atau beberapa layanan gagal untuk memulai, CloudFormation masih menetapkan status tumpukan sebagaiCREATE_COMPLETE. Untuk mencegah agar status berubah menjadi CREATE_COMPLETE sebelum semua layanan telah berhasil dimulai, Anda dapat menambahkan atribut CreationPolicyatribut ke instans. Atribut ini menempatkan status instans CREATE_IN_PROGRESS sampai CloudFormation menerima jumlah sinyal sukses yang diperlukan atau periode batas waktu terlampaui, sehingga Anda dapat mengontrol kapan instance telah berhasil dibuat.
