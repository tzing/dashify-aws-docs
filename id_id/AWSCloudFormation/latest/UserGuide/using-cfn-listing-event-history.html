Penglihatan riwayat acara tumpukan

Anda dapat melacak status sumber daya yang AWS CloudFormation dibuat dan dihapus dengan aws cloudformation describe-stack-events perintah. Jumlah waktu untuk membuat atau menghapus tumpukan tergantung dari kompleksitas tumpukan Anda.

Dalam contoh berikut, sampel tumpukan dibuat dari file templat dengan menggunakan perintah aws cloudformation create-stack. Setelah tumpukan dibuat, acara yang dilaporkan selama pembuatan tumpukan ditampilkan dengan menggunakan perintah aws cloudformation describe-stack-events.

$ aws cloudformation create-stack \
    --stack-name myteststack \

catatan

Anda dapat menjalankan perintah aws cloudformation describe-stack-events selagi tumpukan sedang dibuat untuk melihat acara yang dilaporkan.

Acara terbaru dilaporkan terlebih dahulu. Tabel berikut menjelaskan bidang yang dikembalikan oleh perintah aws cloudformation describe-stack-events:

Bidang Deskripsi
EventId

Pengidentifikasi acara.
StackName

Nama tumpukan yang sesuai dengan acara tersebut.
StackId

Pengidentifikasi tumpukan yang sesuai dengan acara tersebut.
LogicalResourceId

Pengidentifikasi logis sumber daya.
PhysicalResourceId

Pengidentifikasi fisik sumber daya.
ResourceProperties

Properti sumber daya.
ResourceType

Jenis sumber daya.
Timestamp

Waktu ketika peristiwa itu terjadi.
ResourceStatus

Status sumber daya, yang dapat menjadi salah satu kode status berikut: CREATE_COMPLETE | | | CREATE_FAILED | CREATE_IN_PROGRESS DELETE_COMPLETE | DELETE_FAILED | DELETE_IN_PROGRESS | DELETE_SKIPPED | IMPORT_COMPLETE | IMPORT_IN_PROGRESS | IMPORT_ROLLBACK_COMPLETE | IMPORT_ROLLBACK_FAILED | IMPORT_ROLLBACK_IN_PROGRESS | REVIEW_IN_PROGRESS | ROLLBACK_COMPLETE | ROLLBACK_FAILED | ROLLBACK_IN_PROGRESS UPDATE_COMPLETE | UPDATE_COMPLETE_CLEANUP_IN_PROGRESS | UPDATE_FAILED | UPDATE_IN_PROGRESS | UPDATE_ROLLBACK_COMPLETE | UPDATE_ROLLBACK_COMPLETE_CLEANUP_IN_PROGRESS | UPDATE_ROLLBACK_FAILED | UPDATE_ROLLBACK_IN_PROGRESS

DELETE_SKIPPEDStatus berlaku untuk sumber daya dengan atribut kebijakan penghapusan dari pertahankan.
DetailedStatus

Status rinci tumpukan. Jika CONFIGURATION_COMPLETE ada, fase konfigurasi sumber daya tumpukan telah selesai dan stabilisasi sumber daya sedang berlangsung.
ResourceStatusReason

Informasi lebih lanjut tentang status.
