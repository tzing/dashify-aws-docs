Memantau dan mengembalikan operasi tumpukan

Pemicu rollback memungkinkan Anda untuk membuatAWS CloudFormation memantau status aplikasi Anda selama pembuatan dan pembaruan tumpukan, dan untuk mengembalikan operasi itu jika aplikasi melanggar ambang batas alarm yang telah Anda tentukan. Untuk setiap pemicu rollback yang Anda buat, Anda menentukan CloudWatch alarm yang CloudFormation harus dipantau. CloudFormation memantau alarm yang ditentukan selama operasi pembuatan atau pembaruan tumpukan, dan untuk jumlah waktu tertentu setelah semua sumber daya telah di-deploy. Jika salah satu alarm memasukiALARM status selama operasi tumpukan atau periode pemantauan, CloudFormation mengembalikan seluruh operasi tumpukan.

Anda dapat mengatur waktu pemantauan dari default 0 hingga 180 menit. Selama waktu ini, CloudFormation memantau semua pemicu rollback setelah operasi pembuatan atau pembaruan tumpukan men-deploy semua sumber daya yang diperlukan. Jika salah satu alarm memasukiALARM status selama operasi tumpukan atau periode pemantauan ini CloudFormation , mengembalikan seluruh operasi tumpukan. Kemudian, untuk operasi pembaruan, jika periode pemantauan berakhir tanpa alarm memasukiALARM status, CloudFormation mulai membuang sumber daya lama seperti biasa. Jika Anda mengatur waktu pemantauan tetapi tidak menentukan pemicu rollback apa pun, CloudFormation masih menunggu periode waktu yang ditentukan sebelum membersihkan sumber daya lama untuk operasi pembaruan. Anda dapat menggunakan periode pemantauan ini untuk melakukan validasi tumpukan manual yang diinginkan, dan membatalkan pembuatan atau pembaruan tumpukan secara manual jika diperlukan. Jika Anda mengatur waktu pemantauan 0 menit, CloudFormation masih memantau pemicu rollback selama operasi pembuatan dan pembaruan tumpukan dan mengembalikan operasi jika alarm memasuki status ALARM. Kemudian, untuk operasi pembaruan tanpa alarm yang melanggar, ini mulai membuang sumber daya lama segera setelah operasi selesai.

Secara default, CloudFormation hanya mengembalikan operasi tumpukan jika alarm memasukiALARM status, bukanINSUFFICIENT_DATA status. Agar CloudFormation mengembalikan operasi tumpukan jika alarm memasukiINSUFFICIENT_DATA status juga, edit CloudWatch alarm untuk menangani data yang hilangbreaching. Untuk informasi lebiih lanjut, lihat Mengonfigurasi CloudWatch alarm menangani data yang hilang di Panduan CloudWatch Pengguna Amazon.

CloudFormation tidak memantau pemicu rollback ketika mengembalikan tumpukan selama operasi pembaruan.

Anda dapat menambahkan maksimum 5 pemicu rollback. Untuk menambahkan pemicu rollback, tentukan Amazon Resource Name (ARN) CloudWatch alarm. Saat ini,AWS::CloudWatch::Alarm danAWS::CloudWatch::CompositeAlarm jenis dapat digunakan sebagai pemicu rollback. Untuk informasi selengkapnya tentang CloudWatch alarm, lihat Menggunakan CloudWatch alarm Amazon di Panduan CloudWatch Pengguna Amazon.

Jika CloudWatch alarm yang diberikan hilang, seluruh operasi tumpukan gagal dan mengembalikan.

Ketahuilah bahwa akses ke Amazon CloudWatch memerlukan kredensi. Kredensi tersebut harus memiliki izin untuk mengaksesAWS sumber daya, seperti mengambil data CloudWatch metrik tentang sumber daya Anda. Untuk informasi lebiih lanjut, lihat Autentikasi dan pengendalian akses untuk Amazon CloudWatch di Panduan CloudWatch Pengguna Amazon.

Untuk menambahkan pemicu rollback selama pembuatan atau pembaruan tumpukan

Untuk menambahkan pemicu rollback ke pembuatan atau pembaruan tumpukan, tentukan CloudWatch alarm Amazon di bagian konfigurasi Rollback.

  1. Selama membuat atau memperbarui tumpukan, pada halaman Configure stack options, di bawah Advanced options, luaskan bagian Rollback configuration.

  2. Masukkan waktu pemantauan antara0 -180 menit. Nilai default-nya adalah 0.

  3. Tentukan ARN dari CloudWatch alarm atau alarm komposit yang ingin digunakan sebagai pemicu rollback, lalu pilih Tambahkan CloudWatch alarm ARN.

    Misalnya, berikut ini adalah ARN untuk CloudWatch alarm atau alarm komposit,arn:aws:cloudwatch:us-east-1:123456789012:alarm:MyAlarmName.

  4. Pilih Berikutnya dan tinjau detail tumpukan Anda.

  5. Setelah Anda meninjau pengaturan pembuatan tumpukan dan perkiraan biaya tumpukan, pilih Buat tumpukan untuk meluncurkan tumpukan Anda.

Hasil: Anda telah berhasil menambahkan CloudWatch alarm Amazon ke tumpukan Anda.

Untuk menambahkan pemicu rollback untuk satu set perubahan

Untuk menambahkan pemicu rollback ke set perubahan, buat perubahan yang ditetapkan dengan memilih template tumpukan Anda dan tentukan CloudWatch alarm Amazon di bagian konfigurasi Rollback.

  1. Selama membuat atau memperbarui set perubahan, pada halaman Konfigurasi opsi tumpukan, di bawah Opsi lanjutan, perluas bagian konfigurasi Rollback.

  2. Masukkan waktu pemantauan antara0 -180 menit. Nilai default-nya adalah 0.

  3. Tentukan ARN dari CloudWatch alarm atau alarm komposit yang ingin digunakan sebagai pemicu rollback, lalu pilih Tambahkan CloudWatch alarm ARN.

    Misalnya, berikut ini adalah ARN untuk CloudWatch alarm atau alarm komposit,arn:aws:cloudwatch:us-east-1:123456789012:alarm:MyAlarmName.

  4. Pilih Berikutnya dan tinjau detail tumpukan Anda.

  5. Setelah Anda meninjau pengaturan pembuatan tumpukan dan perkiraan biaya tumpukan, pilih Buat set perubahan.

  6. Tinjau modul Create change set dan pilih Create change set.

  7. (Opsional) Untuk menyelesaikan pembuatan tumpukan baru berdasarkan set perubahan, pilih Jalankan, tentukan konfigurasi rollback Anda, lalu pilih Jalankan set perubahan.

Hasil: Anda telah berhasil menambahkan CloudWatch alarm Amazon ke set perubahan Anda.

Untuk melihat pemicu rollback untuk tumpukan

Untuk melihat pemicu rollback untuk tumpukan, lihat bagian Konfigurasi rollback.

  1. Di halaman Tumpukan, pilih tumpukan yang ingin Anda lihat dari daftar di sebelah kiri.

  2. Pada tab Stack info, luaskan bagian konfigurasi Rollback.

Hasil: Pemicu rollback ditampilkan di bagian konfigurasi Rollback, jika pemicu rollback ditentukan.

Untuk melihat pemicu rollback untuk set perubahan

Untuk melihat pemicu rollback untuk set perubahan, lihat bagian Konfigurasi Rollback.

  1. Di halaman Tumpukan, pilih tumpukan yang ingin Anda lihat dari daftar di sebelah kiri.

  2. Pilih tab Ubah set, dan pilih set perubahan yang ingin Anda lihat.

  3. Pilih tab Input, dan lihat bagian Rollback configuration.

Hasil: Pemicu rollback ditampilkan di bagian konfigurasi Rollback, jika pemicu rollback ditentukan.
