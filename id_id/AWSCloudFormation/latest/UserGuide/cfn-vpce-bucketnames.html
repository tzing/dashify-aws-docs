Mengatur VPC endpoint untuk AWS CloudFormation

Anda dapat meningkatkan postur keamanan VPC Anda dengan mengkonfigurasi AWS CloudFormation untuk menggunakan VPC endpoint antarmuka. Titik akhir antarmuka didukung oleh PrivateLink, teknologi yang memungkinkan Anda mengakses CloudFormation API privat melalui alamat IP privat. PrivateLinkmembatasi semua lalu lintas jaringan antara VPC Anda dan CloudFormation ke jaringan Amazon. Selain itu, Anda tidak memerlukan gateway internet, perangkat NAT, atau virtual private gateway.

Anda tidak diwajibkan untuk mengonfigurasi PrivateLink, sebaiknya direkomendasikan. Untuk informasi lebih lanjut tentang PrivateLink dan VPC endpoints, lihat MengaksesAWS layanan melalui PrivateLink.

Sebelum Anda memulai

Sebelum Anda mengkonfigurasi VPC endpoints untuk CloudFormation, perhatikan pertimbangan berikut.

  • Saat menggunakan fitur VPC endpoint, berikan akses ke bucket S3 CloudFormation khusus untuk sumber daya di VPC yang harus menanggapi permintaan sumber daya kustom atau kondisi menunggu.

    Jika Anda menggunakan CloudFormation untuk membuat sumber daya di VPC dengan VPC endpoint, Anda mungkin perlu memodifikasi kebijakan titik akhir IAM sehingga memungkinkan akses ke bucket S3 tertentu.

    CloudFormation memiliki bucket S3 di setiap Region untuk memantau respons ke permintaan sumber daya kustom atau kondisi menunggu. Jika templat menyertakan sumber daya kustom atau kondisi menunggu di VPC, kebijakan akhir VPC endpoint harus mengizinkan pengguna untuk mengirim respons ke bucket berikut:

    • Untuk sumber daya kustom, izinkan lalu lintas ke bucket cloudformation-custom-resource-response-region. Saat menggunakan sumber daya kustom, nama region tidak mengandung tanda hubung. Sebagai contoh, uswest2.

    • Untuk kondisi menunggu, izinkan lalu lintas ke bucket cloudformation-waitcondition-region. Ketika menggunakan kondisi menunggu, nama region mengandung tanda hubung. Sebagai contoh, us-west-2.

    Jika kebijakan titik akhir memblokir lalu lintas ke bucket ini, CloudFormation tidak akan menerima respons dan operasi tumpukan gagal. Misalnya, jika Anda memiliki sumber daya VPC di us-west-2 Region yang harus menanggapi kondisi menunggu, sumber daya harus dapat mengirim respons ke bucket cloudformation-waitcondition-us-west-2.

    Untuk daftar Region yang CloudFormation mendukung, lihat halaman Regions dan endpoint di halaman Referensi Umum Amazon Web.

  • VPC endpoint saat ini tidak mendukung permintaan lintas Region — pastikan Anda membuat titik akhir Anda di Region yang sama tempat Anda berencana untuk mengeluarkan panggilan API CloudFormation.

  • VPC endpoint hanya mendukung DNS yang disediakan Amazon melalui Route 53. Jika Anda ingin menggunakan DNS Anda sendiri, Anda dapat menggunakan penerusan DNS bersyarat. Untuk informasi selengkapnya, tinjau Pengaturan opsi DHCP di Panduan Pengguna Amazon VPC.

  • Grup keamanan yang dilampirkan ke VPC endpoint harus mengizinkan koneksi masuk pada port 443 dari subnet privat VPC.

Untuk membuat VPC endpoint untuk CloudFormation layanan, gunakan prosedur Membuat Titik Akhir Antarmuka di Panduan Pengguna Amazon VPC untuk membuat titik akhir berikut:
