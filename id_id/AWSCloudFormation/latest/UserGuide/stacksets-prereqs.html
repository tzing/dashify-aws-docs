Prasyarat untuk operasi set tumpukan

Karena set tumpukan melakukan operasi tumpukan di beberapa akun, sebelum Anda dapat membuat kumpulan tumpukan pertama, Anda memerlukan izin yang diperlukan yang ditentukan di AWS akun Anda.

Untuk menyiapkan izin yang diperlukan untuk membuat set tumpukan dengan izin yang dikelola sendiri, lihat Melakukan operasi set tumpukan melibatkan wilayah yang dinonaktifkan secara default dan Memberikan izin yang dikelola sendiri.

Untuk menyiapkan izin yang diperlukan untuk membuat set tumpukan dengan izin yang dikelola layanan, lihat Melakukan operasi set tumpukan melibatkan wilayah yang dinonaktifkan secara default dan Aktifkan akses tepercaya dengan AWS Organizations.

catatan

Mengaktifkan akses tepercaya dengan AWS Organizations for saat ini AWS CloudFormation StackSets tidak didukung di Wilayah China Beijing dan Ningxia.

Melakukan operasi set tumpukan melibatkan wilayah yang dinonaktifkan secara default

Wilayah AWS diperkenalkan setelah 20 Maret 2019, seperti Asia Pasifik (Hong Kong), dinonaktifkan secara default. Anda harus mengaktifkan Wilayah ini untuk akun Anda sebelum dapat menggunakannya. Karenanya, pertimbangkan hal berikut sebelum melakukan operasi kumpulan tumpukan yang melibatkan akun di Kawasan yang dinonaktifkan secara default:

  • Untuk membuat set tumpukan dari akun administrator set tumpukan (jika menggunakan izin yang dikelola sendiri) atau akun manajemen organisasi (jika menggunakan izin yang dikelola layanan) di Wilayah yang dinonaktifkan secara default, Anda harus terlebih dahulu mengaktifkan Wilayah tersebut untuk administrator atau akun manajemen.

  • AWS CloudFormation Untuk berhasil membuat atau memperbarui instance tumpukan:

    • Akun target harus berada di Wilayah yang saat ini diaktifkan untuk akun target tersebut.

    • Akun administrator set tumpukan atau akun manajemen organisasi harus mengaktifkan Wilayah yang sama dengan akun target.

penting

Ketahuilah bahwa selama operasi set tumpukan, administrator dan akun target bertukar metadata mengenai akun itu sendiri, selain contoh set tumpukan dan set tumpukan yang terlibat.

Selain itu, jika Anda menonaktifkan Wilayah yang berisi akun tempat instans kumpulan tumpukan berada, Anda bertanggung jawab untuk menghapus instans atau sumber daya tersebut, jika diinginkan. Selain itu, ketahuilah bahwa metadata mengenai akun target di Wilayah yang dinonaktifkan akan disimpan di akun administrator.

Untuk informasi selengkapnya tentang mengaktifkan dan menonaktifkan wilayah, lihat Mengelola Wilayah AWS di Referensi UmumAWS .
