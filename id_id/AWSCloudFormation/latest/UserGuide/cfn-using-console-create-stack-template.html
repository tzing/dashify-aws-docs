Memilih templat tumpukan

Setelah memulai panduan Membuat Stack, tentukan templat yang Anda ingin agar digunakan AWS CloudFormation untuk membuat tumpukan Anda.

CloudFormation templat adalah file berformat JSON- atau YAML yang menentukan AWS sumber daya yang membentuk tumpukan Anda. Untuk informasi selengkapnya tentang CloudFormation templat, lihatBekerja dengan templat AWS CloudFormation.

Untuk memilih templat tumpukan:

  1. Di halaman Tentukan templat, pilih templat tumpukan dengan menggunakan salah satu opsi berikut:

    Templat sudah siap

    Tentukan templat yang telah selesai Anda siapkan untuk membuat tumpukan.

    Di bagian Tentukan templat, pilih opsi yang sesuai berdasarkan lokasi templat:

    • URL Amazon S3

      Tentukan URL untuk templat dalam bucket S3.

      Masukkan URL di bidang URL Amazon S3.

      penting

      Jika templat Anda termasuk tumpukan bersarang (misalnya, tumpukan yang dijelaskan dalam dokumen templat lain yang terletak di subdirektori), pastikan bahwa bucket S3 berisi file dan direktori yang diperlukan.

      Jika Anda memiliki templat di bucket yang diaktifkan versioning-nya, Anda dapat menentukan versi khusus templat, seperti https://s3.amazonaws.com/templates/myTemplate.template?versionId=123ab1cdeKdOW5IH4GAcYbEngcpTJTDW. Untuk informasi selengkapnya, lihat Mengelola objek dalam bucket berkemampuan versi di Panduan Pengguna Layanan Penyimpanan Sederhana Amazon.

      URL harus menunjuk ke templat dengan ukuran maksimal 1 MB yang tersimpan dalam bucket S3 yang memiliki izin membaca. URL maksimal sepanjang 1024 karakter. Beberapa sumber daya mungkin mengharuskan bucket berada di Wilayah yang sama dengan tumpukan.

    • Mengunggah file templat

      Pilih CloudFormation templat di komputer lokal Anda.

      Pilih Pilih File untuk memilih file templat yang ingin Anda unggah. Templat dapat berukuran hingga 1 MB. Setelah memilih templat, CloudFormation unggah file dan tampilkan URL S3.

      Jika Anda menggunakan AWS CLI atau API untuk membuat tumpukan, Anda dapat mengunggah templat dengan ukuran hingga 51.200 byte.

      catatan

      Jika Anda mengunggah file templat lokal, CloudFormation unggah file templat Amazon Simple Storage Service (Amazon S3) di akun Anda. AWS Jika belum memiliki bucket S3 yang dibuat oleh CloudFormation, ini membuat bucket unik untuk setiap Wilayah tempat Anda mengunggah file templat. Jika Anda sudah memiliki bucket S3 yang dibuat AWS CloudFormation di AWS akun Anda, CloudFormation tambahkan templat untuk bucket itu.

      Pertimbangan yang perlu diingat tentang bucket S3 yang dibuat oleh CloudFormation

      • Bucket dapat diakses oleh siapa saja dengan izin Amazon S3 di akun Anda. AWS

      • CloudFormation membuat bucket dengan enkripsi sisi server diaktifkan secara default, sehingga mengenkripsi semua objek yang tersimpan dalam bucket.

        Anda dapat langsung mengelola opsi enkripsi untuk bucket yang CloudFormation telah dibuat; misalnya, menggunakan konsol Amazon S3 di https://console.aws.amazon.com/s3/, atau. AWS CLI Untuk informasi selengkapnya, lihat enkripsi default Amazon S3 untuk bucket S3 di Panduan Pengguna Layanan Penyimpanan Sederhana Amazon.

      • Anda dapat menggunakan bucket sendiri dan mengelola perizinannya dengan mengunggah templat secara manual ke Amazon S3. Saat Anda membuat atau memperbarui tumpukan, tentukan URL Amazon S3 dari file templat.

    Gunakan templat contoh

    • Untuk melihat lebih banyak contoh dan snippet templat, yang diatur menurut layanan AWS, klik Lihat templat contoh lainnya.

    Membuat templat di Designer

    Membuat atau memodifikasi templat menggunakan AWS CloudFormation Designer, antarmuka seret-jatuhkan untuk membuat diagram grafis templat Anda. Untuk informasi selengkapnya, lihat ApaAWS CloudFormationDesainer?.

  2. Untuk menerima pengaturan Anda, pilih Selanjutnya, dan lanjutkan dengan menentukan nama tumpukan dan parameter.

    Sebelum membuat sumber daya, CloudFormation validasi templat untuk menangkap sintaks dan beberapa kesalahan semantik, seperti dependensi melingkar. Selama validasi, periksa CloudFormation terlebih dahulu apakah templat JSON valid. Jika bukan, CloudFormation periksa apakah templat YAMB valid. Jika kedua pengecekan gagal, CloudFormation mengembalikan kesalahan validasi templat.
