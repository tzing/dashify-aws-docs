Dalam Memulai , Anda mempelajari cara menggunakan templat untuk membuat tumpukan. Anda melihat sumber daya dinyatakan dalam templat dan bagaimana sumber daya tersebut dipetakan ke sumber daya dalam tumpukan. Kita juga membahas parameter input dan bagaimana parameter tersebut memungkinkan Anda meneruskan nilai tertentu saat Anda membuat tumpukan dari templat. Di bagian ini, kita akan membahas lebih dalam tentang sumber daya dan parameter. Kami juga akan membahas komponen template lainnya sehingga Anda akan tahu cara menggunakan komponen ini bersama-sama untuk membuat template yang menghasilkan AWS sumber daya yang Anda inginkan.

Apa itu AWS CloudFormation template?

Template adalah deklarasi AWS sumber daya yang membentuk tumpukan. Template disimpan sebagai file teks yang formatnya sesuai dengan standar JavaScript Object Notation (JSON) atau YAMAL. Karena mereka adalah file teks, Anda dapat membuat dan mengeditnya di editor teks apa pun dan mengelolanya di sistem kontrol sumber Anda dengan sisa kode sumber Anda. Untuk informasi selengkapnya tentang format templat, lihat Format templat AWS CloudFormation.

Dalam template, Anda mendeklarasikan AWS sumber daya yang ingin Anda buat dan konfigurasikan. Anda menyatakan objek sebagai pasangan nama-nilai atau pasangan nama dengan sekumpulan objek anak tertutup. Sintaks tergantung pada format yang Anda gunakan. Untuk informasi selengkapnya, lihat Anatomi templat. Satu-satunya objek tingkat atas yang diperlukan adalah objek Sumber Daya, yang harus menyatakan setidaknya satu sumber daya. Mari kita mulai dengan templat paling dasar yang hanya berisi objek Sumber Daya, yang berisi satu pernyataan sumber daya.

Sumber Daya: Hello Bucket!

Objek Sumber Daya berisi daftar objek sumber daya. Sebuah pernyataan sumber daya berisi atribut sumber daya, yang dinyatakan sendiri sebagai objek anak. Sumber daya harus memiliki Type atribut, yang mendefinisikan jenis AWS sumber daya yang ingin Anda buat. Atribut Type memiliki format khusus:

AWS::ProductIdentifier::ResourceType

Misalnya, jenis sumber daya untuk bucket Amazon S3 adalah. AWS::S3::Bucket Untuk daftar lengkap jenis sumber daya, lihat Referensi templat.

Mari kita lihat template dasar. Template berikut mendeklarasikan sumber daya tunggal tipe AWS::S3::Bucket: dengan nama HelloBucket.

JSON

{
    "Resources": {
        "HelloBucket": {
            "Type": "AWS::S3::Bucket"
        }
    }
}

YAML

Resources:
  HelloBucket:
    Type: 'AWS::S3::Bucket'

Jika Anda menggunakan template ini untuk membuat tumpukan, AWS CloudFormation akan membuat bucket Amazon S3. Membuat bucket sederhana, karena CloudFormation dapat membuat bucket dengan pengaturan default. Untuk sumber daya lain, seperti Amazon EC2 Auto Scaling grup atau instans EC2, CloudFormation memerlukan informasi lebih lanjut. Pernyataan sumber daya menggunakan atribut Properties untuk menentukan informasi yang digunakan untuk membuat sumber daya.

Bergantung pada jenis sumber daya, beberapa properti diperlukan, seperti ImageId properti untuk AWS::EC2::Instancesumber daya, dan lainnya bersifat opsional. Beberapa properti memiliki nilai default, seperti AccessControl properti AWS::S3::Bucket sumber daya, sehingga menentukan nilai untuk properti tersebut adalah opsional. Properti lain tidak diperlukan tetapi dapat menambahkan fungsionalitas yang Anda inginkan, seperti WebsiteConfiguration properti AWS::S3::Bucket sumber daya. Menentukan nilai untuk properti tersebut sepenuhnya opsional dan berdasarkan kebutuhan Anda. Dalam contoh di atas, karena sumber daya AWS::S3::Bucket hanya memiliki properti opsional dan kita tidak memerlukan fitur opsional apa pun, kita dapat menerima default-nya dan menghilangkan atribut Properti.

Untuk melihat properti untuk setiap jenis sumber daya, lihat topik di AWS referensi jenis sumber daya dan properti.

Properti Resource dan menggunakan sumber daya bersama-sama

Biasanya, properti untuk sumber daya hanyalah nilai string. Misalnya, template berikut menentukan ACL (PublicRead) kalengan untuk AccessControl properti bucket.

JSON

{
    "Resources": {
        "HelloBucket": {
            "Type": "AWS::S3::Bucket",
            "Properties": {
                "AccessControl": "PublicRead"
            }
        }
    }
}

YAML

Resources:
  HelloBucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      AccessControl: PublicRead

Beberapa sumber daya dapat memiliki beberapa properti, dan beberapa properti dapat memiliki satu atau lebih subproperti. Misalnya, AWS::S3::Bucketsumber daya memiliki dua properti: AccessControl dan WebsiteConfiguration. WebsiteConfiguration Properti ini memiliki dua subproperti: IndexDocument dan ErrorDocument. Templat berikut menunjukkan sumber daya bucket asli kami dengan properti tambahan.

JSON

{
    "Resources": {
        "HelloBucket": {
            "Type": "AWS::S3::Bucket",
            "Properties": {
                "AccessControl": "PublicRead",
                "WebsiteConfiguration": {
                    "IndexDocument": "index.html",
                    "ErrorDocument": "error.html"
                }
            }
        }
    }
}

YAML

Resources:
  HelloBucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      AccessControl: PublicRead
      WebsiteConfiguration:
        IndexDocument: index.html
        ErrorDocument: error.html

Salah satu manfaat terbesar dari template dan CloudFormation kemampuan untuk membuat satu set sumber daya yang bekerja sama untuk membuat aplikasi atau solusi. Nama yang digunakan untuk sumber daya dalam templat adalah nama logis. Saat CloudFormation membuat sumber daya, itu menghasilkan nama fisik yang didasarkan pada kombinasi nama logis, nama tumpukan, dan ID unik.

Anda mungkin bertanya-tanya bagaimana Anda mengatur properti pada satu sumber daya berdasarkan nama atau properti sumber daya lain. Misalnya, Anda dapat membuat CloudFront distribusi yang didukung oleh bucket S3 atau instans EC2 yang menggunakan grup keamanan EC2, dan semua sumber daya ini dapat dibuat dalam templat yang sama. CloudFormation memiliki sejumlah fungsi intrinsik yang dapat Anda gunakan untuk merujuk ke sumber daya lain dan propertinya. Anda dapat menggunakan fungsi Ref untuk mereferensikan ke properti pengidentifikasi sumber daya. Seringkali, ini adalah nama fisik sumber daya; namun, terkadang dapat berupa pengenal, seperti alamat IP untuk sumber daya AWS: :EC2: :EIP atau Nama Sumber Daya Amazon (ARN) untuk topik Amazon SNS. Untuk daftar nilai yang dikembalikan oleh fungsi Ref, lihat Fungsi ref. Template berikut berisi sumber AWS::EC2::Instancedaya. SecurityGroups Properti sumber daya memanggil fungsi Ref untuk merujuk ke AWS::EC2::SecurityGroup sumber InstanceSecurityGroup daya.

JSON

{
    "Resources": {
        "Ec2Instance": {
            "Type": "AWS::EC2::Instance",
            "Properties": {
                "SecurityGroups": [
                    {
                        "Ref": "InstanceSecurityGroup"
                    }
                ],
                "KeyName": "mykey",
                "ImageId": ""
            }
        },
        "InstanceSecurityGroup": {
            "Type": "AWS::EC2::SecurityGroup",
            "Properties": {
                "GroupDescription": "Enable SSH access via port 22",
                "SecurityGroupIngress": [
                    {
                        "IpProtocol": "tcp",
                        "FromPort": 22,
                        "ToPort": 22,
                        "CidrIp": "0.0.0.0/0"
                    }
                ]
            }
        }
    }
}

Resources:

    Type: 'AWS::EC2::Instance'
    Properties:
      SecurityGroups:
        - !Ref InstanceSecurityGroup

MyExistingSecurityGroup adalah string yang mengacu pada grup keamanan EC2 yang ada alih-alih grup keamanan yang dideklarasikan dalam template. Anda menggunakan string literal untuk mereferensikan ke sumber daya AWS yang ada.

Pada contoh di atas, KeyName properti dari AWS::EC2::Instanceadalah string mykey literal. Ini berarti bahwa pasangan kunci dengan nama mykey harus ada di wilayah tempat tumpukan dibuat; jika tidak, pembuatan tumpukan akan gagal karena pasangan kunci tidak ada. Key pair yang Anda gunakan dapat bervariasi dengan wilayah tempat Anda membuat tumpukan, atau Anda mungkin ingin berbagi template dengan orang lain sehingga mereka dapat menggunakannya dengan AWS akun mereka. Jika demikian, Anda dapat menggunakan parameter input sehingga nama pasangan kunci dapat ditentukan saat tumpukan dibuat. Fungsi Ref dapat merreferensikan ke parameter input yang ditentukan pada waktu pembuatan tumpukan. Template berikut menambahkan objek Parameter yang berisi KeyName parameter, yang digunakan untuk menentukan KeyName properti untuk AWS::EC2::Instance sumber daya. Jenis parameter adalah AWS::EC2::KeyPair::KeyName, yang menjamin pengguna menentukan nama pasangan kunci yang valid dalam akunnya dan di wilayah di mana tumpukan sedang dibuat.

Fungsi Ref berguna jika parameter atau nilai yang dikembalikan untuk sumber daya persis seperti yang Anda inginkan; namun, Anda mungkin memerlukan atribut sumber daya lainnya. Misalnya, jika Anda ingin membuat CloudFront distribusi dengan asal S3, Anda perlu menentukan lokasi bucket dengan menggunakan alamat gaya DNS. Sejumlah sumber daya memiliki atribut tambahan yang nilainya dapat Anda gunakan di templat Anda. Untuk mendapatkan atribut ini, Anda menggunakan GetAtt fungsi Fn::. Template berikut membuat sumber daya CloudFront distribusi yang menentukan nama DNS sumber daya bucket S3 menggunakan Fn::GetAtt fungsi untuk mendapatkan atribut bucket. DomainName

JSON

{
  "Resources": {
    "myBucket": {
      "Type": "AWS::S3::Bucket"
    },
    "myDistribution": {
      "Type": "AWS::CloudFront::Distribution",
      "Properties": {
        "DistributionConfig": {
          "Origins": [
            {
              "DomainName": {
                "Fn::GetAtt": [
                  "myBucket",
                  "DomainName"
                ]
              },
              "Id": "myS3Origin",
              "S3OriginConfig": {}
            }
          ],
          "Enabled": "true",
          "DefaultCacheBehavior": {
            "TargetOriginId": "myS3Origin",
            "ForwardedValues": {
              "QueryString": "false"
            },
            "ViewerProtocolPolicy": "allow-all"
          }
        }
      }
    }
  }
}

    Type: 'AWS::S3::Bucket'

    Type: 'AWS::CloudFront::Distribution'

Fn::GetAtt Fungsi ini mengambil dua parameter, nama logis sumber daya dan nama atribut yang akan diambil. Untuk daftar lengkap atribut yang tersedia untuk sumber daya, lihat Fn:: GetAtt. Anda akan melihat bahwa Fn::GetAtt fungsi tersebut mencantumkan dua parameternya dalam sebuah array. Untuk fungsi yang mengambil beberapa parameter, gunakan array untuk menentukan parameternya.

Menerima input pengguna menggunakan parameter input

Sejauh ini, Anda telah mempelajari tentang sumber daya dan sedikit tentang bagaimana menggunakannya bersama-sama dalam templat. Anda telah mempelajari cara mereferensikan ke parameter input, tetapi kita belum membahas secara mendalam tentang cara mendefinisikan parameter input itu sendiri. Mari kita lihat pernyataan parameter dan bagaimana Anda dapat membatasi dan memvalidasi input pengguna.

Anda menyatakan parameter dalam objek Parameter templat. Parameter berisi daftar atribut yang menentukan nilai dan batasan terhadap nilainya. Satu-satunya atribut yang diperlukan adalah Type, yang dapat berupa String, Number, atau tipe AWS-spesifik. Anda juga dapat menambahkan atribut Description yang memberi tahu pengguna lebih lanjut tentang jenis nilai apa yang harus mereka tentukan. Nama dan deskripsi parameter muncul di halaman Tentukan Parameter saat pengguna menggunakan templat di panduan Membuat Tumpukan.

Fragmen templat berikut adalah objek Parameter yang menyatakan parameter yang digunakan di halaman Tentukan Parameter di atas.

JSON

  "Parameters": {
    "KeyName": {
      "Description" : "Name of an existing EC2 KeyPair to enable SSH access into the WordPress web server",
      "Type": "AWS::EC2::KeyPair::KeyName"
    },
    "WordPressUser": {
      "Default": "admin",
      "NoEcho": "true",
      "Description" : "The WordPress database admin account user name",
      "Type": "String",
      "MinLength": "1",
      "MaxLength": "16",
      "AllowedPattern" : "[a-zA-Z][a-zA-Z0-9]*"
    },
    "WebServerPort": {
      "Default": "8888",
      "Description" : "TCP/IP port for the WordPress web server",
      "Type": "Number",
      "MinValue": "1",
      "MaxValue": "65535"
    }
  }

YAML

Parameters:
  KeyName:
    Description: Name of an existing EC2 KeyPair to enable SSH access into the WordPress web server
    Type: AWS::EC2::KeyPair::KeyName
  WordPressUser:
    Default: admin
    NoEcho: true

Untuk parameter dengan nilai default, CloudFormation gunakan nilai default kecuali pengguna menentukan nilai lain. Jika Anda menghilangkan atribut default, pengguna harus menentukan nilai untuk parameter tersebut; namun, mengharuskan pengguna untuk memasukkan nilai tidak memastikan bahwa nilai tersebut valid. Untuk memvalidasi nilai parameter, Anda dapat mendeklarasikan batasan atau menentukan tipe parameter -spesifik. AWS

Anda akan melihat bahwa parameter KeyName tidak memiliki atribut Default dan parameter lainnya memiliki. Misalnya, parameter WordPressUser memiliki atribut Default: admin, tetapi parameter KeyName tidak memiliki. Pengguna harus menentukan nilai nama kunci pada pembuatan tumpukan. Jika tidak, CloudFormation gagal membuat tumpukan dan melempar pengecualian:Parameters: [KeyName] must have values.

Untuk AWStipe parameter spesifik, CloudFormation memvalidasi nilai input terhadap nilai yang ada di AWS akun pengguna dan di wilayah tempat mereka membuat tumpukan sebelum membuat sumber daya tumpukan apa pun. Dalam template sampel, KeyName parameter adalah tipe parameter AWS-spesifik dari. AWS::EC2::KeyPair::KeyName CloudFormation memeriksa bahwa pengguna menentukan nama key pair EC2 yang valid sebelum membuat tumpukan. Contoh lain dari tipe parameter AWS-specific adalahAWS::EC2::VPC::Id, yang mengharuskan pengguna untuk menentukan ID VPC yang valid. Selain validasi di muka, AWS konsol menampilkan daftar drop-down nilai valid untuk tipe parameter AWS-specific, seperti nama key pair EC2 yang valid atau ID VPC, saat pengguna menggunakan wizard Create Stack.

Untuk jenis String, Anda dapat menggunakan atribut berikut untuk menyatakan batasan: MinLength, MaxLength, Default, AllowedValues, dan AllowedPattern. Pada contoh di atas, parameter WordPressUser memiliki tiga batasan: nilai parameter harus sepanjang 1 hingga 16 karakter (MinLength, MaxLength) dan harus dimulai dengan huruf diikuti dengan kombinasi huruf dan angka (AllowedPattern).

Untuk jenis Number, Anda dapat menyatakan batasan berikut: MinValue, MaxValue, Default, dan AllowedValues. Angka dapat berupa bilangan bulat atau nilai float. Di contoh di atas, parameter WebServerPort harus berupa angka antara 1 dan 65535 inklusif (MinValue, MaxValue).

Sebelumnya di bagian ini, kami menyebutkan bahwa parameter adalah cara yang baik untuk menentukan data sensitif atau spesifik implementasi, seperti kata sandi atau nama pengguna, yang perlu Anda gunakan tetapi tidak ingin disematkan di templat itu sendiri. Jika Anda menyetel NoEcho atribut ketrue, CloudFormation mengembalikan nilai parameter yang disamarkan sebagai tanda bintang (*****) untuk setiap panggilan yang menggambarkan peristiwa tumpukan atau tumpukan, kecuali untuk informasi yang disimpan di lokasi yang ditentukan di bawah ini. Dalam contoh di atas, nilai WordPressUser parameter tidak terlihat oleh siapa pun yang melihat pengaturan tumpukan, dan nilainya dikembalikan sebagai tanda bintang.

penting

Menggunakan NoEcho atribut tidak menutupi informasi apa pun yang disimpan sebagai berikut:

  • Bagian Metadata template. CloudFormation tidak mengubah, memodifikasi, atau menyunting informasi apa pun yang Anda sertakan di Metadata bagian ini. Untuk informasi selengkapnya, lihat Metadata.

  • Bagian Outputs template. Untuk informasi selengkapnya, lihat Output.

  • MetadataAtribut definisi sumber daya. Untuk informasi selengkapnya, lihat Metadataatribut.

Kami sangat menyarankan Anda untuk tidak menggunakan mekanisme ini untuk memasukkan informasi sensitif, seperti kata sandi atau rahasia.

penting

Daripada menyematkan informasi sensitif secara langsung di CloudFormation template Anda, kami sarankan Anda menggunakan parameter dinamis dalam template tumpukan untuk mereferensikan informasi sensitif yang disimpan dan dikelola di luar CloudFormation, seperti di AWS Systems Manager Parameter Store atau AWS Secrets Manager.

Untuk informasi lebih lanjut, lihat praktik Jangan menanamkan kredensial dalam templat Anda terbaik.

Menentukan nilai bersyarat menggunakan pemetaan

Parameter adalah cara yang bagus untuk memungkinkan pengguna menentukan nilai unik atau sensitif untuk digunakan dalam properti sumber daya tumpukan; tetapi, mungkin ada pengaturan yang bergantung pada wilayah atau agak rumit untuk diketahui pengguna karena syarat atau dependensi lain. Dalam kasus ini, Anda ingin meletakkan beberapa logika dalam templat itu sendiri sehingga pengguna dapat menentukan nilai yang lebih sederhana (atau tidak sama sekali) untuk mendapatkan hasil yang mereka inginkan. Dalam contoh sebelumnya, kami membuat kode keras ID AMI untuk ImageId properti instans EC2 kami. Ini bekerja dengan baik di wilayah US-East, di mana ini mewakili AMI yang kita inginkan. Namun, jika pengguna mencoba membangun tumpukan di wilayah yang berbeda, mereka akan mendapatkan AMI yang salah atau tidak ada AMI sama sekali. (ID AMI bersifat unik untuk suatu wilayah, sehingga ID AMI yang sama di wilayah yang berbeda mungkin tidak mewakili AMI atau AMI yang sama sekali berbeda.)

Untuk menghindari masalah ini, Anda memerlukan cara untuk menentukan ID AMI yang tepat berdasarkan input bersyarat (dalam contoh ini, wilayah tempat tumpukan dibuat). Ada dua fitur templat yang dapat membantu, objek Pemetaan dan parameter pseudo AWS::Region.

Parameter AWS::Region pseudo adalah nilai yang CloudFormation diselesaikan sebagai wilayah tempat tumpukan dibuat. Parameter pseudo diselesaikan CloudFormation saat Anda membuat tumpukan. Pemetaan memungkinkan Anda untuk menggunakan nilai input sebagai syarat yang menentukan nilai lain. Serupa dengan pernyataan switch, pemetaan mengaitkan satu set nilai dengan yang lain. Menggunakan parameter AWS::Region bersama-sama dengan pemetaan, Anda dapat memastikan bahwa ID AMI yang sesuai dengan wilayah telah ditentukan. Template berikut berisi objek Pemetaan dengan nama pemetaan RegionMap yang digunakan untuk memetakan ID AMI ke wilayah yang sesuai.

JSON

{
  "Parameters": {
    "KeyName": {
      "Description": "Name of an existing EC2 KeyPair to enable SSH access to the instance",
      "Type": "String"
    }
  },
  "Mappings": {
    "RegionMap": {
      "us-east-1": {
        "AMI": "ami-76f0061f"
      },
      "us-west-1": {
        "AMI": "ami-655a0a20"

Di RegionMap, setiap wilayah dipetakan ke pasangan nama-nilai. Pasangan nama-nilai adalah label, dan nilai untuk memetakan. Dalam RegionMap, AMI adalah label dan ID AMI adalah nilainya. Untuk menggunakan peta untuk mengembalikan nilai, Anda menggunakan FindInMap fungsi Fn::, meneruskan nama peta, nilai yang digunakan untuk menemukan nilai yang dipetakan, dan label nilai yang dipetakan yang ingin Anda kembalikan. Pada contoh di atas, ImageId properti sumber daya Ec2Instance menggunakan FindInMap fungsi Fn:: untuk menentukan nilainya dengan menentukan RegionMap sebagai peta yang akan digunakan, AWS::Region sebagai nilai input untuk dipetakan, dan AMI sebagai label untuk mengidentifikasi nilai yang akan dipetakan. Misalnya, jika template ini digunakan untuk membuat tumpukan di Wilayah AS Barat (California N.), ImageId akan disetel keami-655a0a20.

Tip

Parameter pseudo AWS::Region memungkinkan Anda untuk mendapatkan wilayah tempat tumpukan dibuat. Beberapa sumber daya, seperti AWS::EC2::Instance, AWS::AutoScaling::AutoScalingGrup, dan AWS::ElasticLoadBalancing::LoadBalancer, memiliki properti yang menentukan zona ketersediaan. Anda dapat menggunakan fungsi Fn::GetAZsuntuk mendapatkan daftar semua availability zone di suatu wilayah.

Nilai yang dibangun dan nilai output

Parameter dan pemetaan adalah cara terbaik untuk meneruskan atau menentukan nilai tertentu pada waktu pembuatan tumpukan, tetapi ada situasi di mana nilai dari parameter atau atribut sumber daya lainnya hanya sebagian dari nilai yang Anda butuhkan. Misalnya, dalam fragmen berikut dari WordPress template, fungsi Fn: :Join membangun Target subproperti HealthCheck properti untuk ElasticLoadBalancer sumber daya dengan menggabungkan WebServerPort parameter dengan string literal lainnya untuk membentuk nilai yang dibutuhkan.

            "Ref": "Ec2Instance2"

Fungsi Fn::Join mengambil dua parameter, pembatas yang memisahkan nilai yang ingin Anda gabungkan dan array nilai sesuai urutan yang Anda inginkan. Dalam contoh di atas, fungsi Fn: :Join menentukan string kosong sebagai pembatas dan HTTP:, nilai WebServerPort parameter, dan karakter/sebagai nilai untuk digabungkan. Jika WebServerPort memiliki nilai 8888, properti Target akan disetel ke nilai berikut:

HTTP:8888/

Fungsi Fn::Join juga berguna untuk menyatakan nilai output untuk tumpukan. Objek Output dalam templat berisi pernyataan untuk nilai yang ingin Anda sediakan setelah tumpukan dibuat. Output adalah cara mudah untuk menangkap informasi penting tentang sumber daya atau parameter input Anda. Misalnya, dalam WordPress template, kami mendeklarasikan objek Output berikut.

Langkah selanjutnya

Kita baru saja membahas bagian dasar templat dan cara menggunakannya. Anda telah mempelajari hal-hal berikut tentang templat:

  • Mendeklarasikan sumber daya dan propertinya.

  • Mereferensikan sumber daya lain dengan fungsi Ref dan atribut sumber daya menggunakan fungsi. Fn::GetAtt

  • Menggunakan parameter untuk memungkinkan pengguna menentukan nilai pada waktu pembuatan tumpukan dan menggunakan kendala untuk memvalidasi input parameter.

  • Menggunakan pemetaan untuk menentukan nilai bersyarat.

  • Menggunakan fungsi Fn: :Join untuk membangun nilai berdasarkan parameter, atribut sumber daya, dan string lainnya.

  • Menggunakan nilai output untuk menangkap informasi tentang sumber daya tumpukan.

Kami tidak membahas dua objek tingkat atas dalam template: AWSTemplateFormatVersion dan Description. AWSTemplateFormatVersion hanyalah versi format template - jika Anda tidak menentukannya, CloudFormation akan menggunakan versi terbaru.

Deskripsi adalah string JSON atau YAML yang valid. Deskripsi ini muncul di halaman Tentukan Parameter dari panduan Membuat Tumpukan. Untuk informasi selengkapnya, lihat Versi format dan Deskripsi.

Tentu saja, ada fitur templat dan tumpukan yang lebih canggih. Berikut adalah daftar beberapa hal penting yang ingin Anda pelajari lebih lanjut:

Atribut Optional yang dapat digunakan dengan sumber daya apapun:

  • DependsOn atribut memungkinkan Anda untuk menentukan bahwa satu sumber daya harus dibuat setelah yang lain.

  • DeletionPolicy atribut memungkinkan Anda untuk menentukan bagaimana CloudFormation harus menangani penghapusan sumber daya.

  • Atribut Metadata memungkinkan Anda untuk menentukan data terstruktur dengan sumber daya.

AWS::CloudFormation::Stackmemungkinkan Anda untuk sarang tumpukan lain sebagai sumber daya dalam template Anda.
