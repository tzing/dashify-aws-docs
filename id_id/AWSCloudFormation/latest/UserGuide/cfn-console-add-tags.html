Mengatur opsi tumpukan AWS CloudFormation

Setelah menentukan parameter yang ditentukan dalam templat, Anda dapat mengatur opsi tambahan untuk tumpukan Anda.

Anda dapat mengatur opsi tumpukan berikut:

Tag

Tanda adalah pasangan nilai kunci arbitrer yang dapat digunakan untuk mengidentifikasi tumpukan Anda untuk tujuan seperti alokasi biaya. Untuk informasi lebih lanjut tentang apa itu tanda dan bagaimana cara penggunaannya, lihat Menandai sumber daya Anda di Panduan Pengguna Amazon EC2.

Kunci terdiri dari karakter alfanumerik atau spasi. Kunci tanda dapat berisi hingga 127 karakter.

Nilai terdiri dari karakter alfanumerik atau spasi. Nilai tanda dapat berisi hingga 255 karakter.

Izin

Yang adaAWS Identity and Access Management(IAM) peran layanan yang CloudFormation dapat berasumsi.

Alih-alih menggunakan kredensi akun Anda, CloudFormation menggunakan kredenial peran untuk membuat tumpukan. Untuk informasi selengkapnya, lihat AWS CloudFormation peran layanan.

Opsi kegagalan tumpukan

Menentukan opsi kegagalan penyediaan untuk semua penyebaran tumpukan dan mengubah operasi set. Untuk informasi selengkapnya, lihatOpsi kegagalan tumpukan.

KlasterMengatur kembali semua sumber daya tumpukanpilihan akan memutar kembali semua sumber daya yang ditentukan dalam template ketika status stackCREATE_FAILEDatauUPDATE_FAILED.

Untuk membuat operasi,Pertahankan sumber daya yang berhasil disediakanopsi mempertahankan keadaan sumber daya yang berhasil, sementara sumber daya gagal akan tetap dalam keadaan gagal sampai operasi pembaruan berikutnya dilakukan.

Untuk memperbarui dan mengubah operasi set,Pertahankan sumber daya yang berhasil disediakanpilihan melestarikan keadaan sumber daya sukses sementara memutar kembali sumber daya gagal untuk negara stabil terakhir diketahui. Sumber daya yang gagal akan berada diUPDATE_FAILEDstatus. Sumber daya tanpa status stabil terakhir yang diketahui akan dihapus pada operasi tumpukan berikutnya.

Anda juga dapat mengatur opsi lanjutan berikut untuk pembuatan tumpukan:

Kebijakan tumpukan

Menetapkan sumber daya yang Anda ingin lindungi dari pembaruan yang tidak disengaja selama pembaruan tumpukan. Secara default, semua sumber daya dapat diperbarui selama pembaruan tumpukan.

Anda dapat memasukkan kebijakan tumpukan secara langsung sebagai JSON, atau mengunggah file JSON yang berisi kebijakan tumpukan. Untuk informasi selengkapnya, lihat Mencegah pembaruan ke sumber daya tumpukan.

Konfigurasi rollback

Memungkinkan Anda untuk memiliki CloudFormation memantau status tumpukan selama pembuatan dan pembaruan tumpukan, dan untuk mengembalikan operasi itu jika tumpukan melanggar ambang batas alarm yang telah Anda tentukan. Tentukan CloudWatch alarm itu CloudFormation harus memantau. Jika salah satu alarm pergi keALARMnegara selama operasi stack atau periode pemantauan, CloudFormation gulungan kembali seluruh operasi stack. Untuk informasi selengkapnya, lihat Memantau dan mengembalikan operasi tumpukan.

Opsi notifikasi

Anda dapat menentukan topik Amazon Simple Notification Service baru atau yang sudah ada di mana notifikasi tentang peristiwa tumpukan dikirim.

Jika Anda membuat topik Amazon SNS, Anda harus menentukan nama dan alamat email tempat pemberitahuan peristiwa tumpukan akan dikirim.

Opsi pembuatan tumpukan

Opsi berikut disertakan untuk pembuatan tumpukan, tetapi tidak tersedia sebagai bagian dari pembaruan tumpukan.

Waktu habis

Menentukan jumlah waktu, dalam menit, yang CloudFormation harus membagikan sebelum waktu keluar operasi pembuatan stack. Jika CloudFormation tidak dapat membuat seluruh tumpukan dalam waktu yang diberikan, pembuatan tumpukan akan gagal karena waktu habis dan mengembalikan tumpukan.

Secara default, tidak ada waktu habis untuk pembuatan tumpukan. Namun, sumber daya individu mungkin memiliki waktu habis sendiri berdasarkan sifat layanan yang diterapkan. Misalnya, jika waktu sumber daya individual di tumpukan Anda habis, waktu pembuatan tumpukan juga akan habis meskipun waktu habis yang Anda tentukan untuk pembuatan tumpukan belum tercapai.

Untuk mengatur opsi tumpukan

  1. Di halaman Konfigurasi opsi tumpukan dari panduan Membuat tumpukan, Anda dapat menentukan tanda dan izin. Gunakan bagian Opsi lanjutan untuk mengatur opsi konfigurasi tambahan untuk tumpukan Anda.

  2. Ketika Anda telah memasukkan semua opsi tumpukan, pilihLangkah Selanjutnya untuk melanjutkan dengan meninjau tumpukan Anda.
