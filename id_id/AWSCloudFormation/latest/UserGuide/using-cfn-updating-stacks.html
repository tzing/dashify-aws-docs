Pembaruan tumpukan AWS CloudFormation

Jika Anda perlu membuat perubahan pada pengaturan tumpukan atau mengubah sumber dayanya, lebih baik Anda memperbarui tumpukan daripada menghapusnya dan membuat tumpukan baru. Misalnya, jika Anda memiliki tumpukan dengan instans EC2, Anda dapat memperbarui tumpukan untuk mengubah ID AMI instans.

Ketika memperbarui tumpukan, Anda mengirimkan perubahan, seperti nilai parameter input baru atau templat yang diperbarui. AWS CloudFormation membandingkan perubahan yang Anda kirimkan dengan tahapan tumpukan Anda saat ini dan memperbarui sumber daya yang diubah saja. Untuk rangkuman alur kerja pembaruan, lihat Bagaimana cara AWS CloudFormation kerjanya?.

catatan

Ketika memperbarui tumpukan, AWS CloudFormation mungkin mengganggu sumber daya atau mengganti sumber daya yang diperbarui, tergantung dari properti yang Anda perbarui. Untuk informasi lebih lanjut tentang perilaku pembaruan sumber daya, lihat Memperbarui perilaku sumber daya tumpukan.

Metode pembaruan

AWS CloudFormation menyediakan dua metode untuk memperbarui tumpukan: pembaruan langsung atau pembuatan dan eksekusi set perubahan. Jika Anda langsung memperbarui tumpukan, Anda mengirimkan perubahan dan AWS CloudFormation langsung men-deploy-nya. Gunakan pembaruan langsung jika Anda ingin men-deploy pembaruan dengan cepat.

Dengan set perubahan, Anda dapat mem-pratinjau perubahan AWS CloudFormation yang akan Anda buat ke tumpukan, lalu memutuskan apakah akan menerapkan perubahan tersebut. Set perubahan adalah dokumen berformat JSON yang merangkum perubahan AWS CloudFormation yang akan dibuat ke tumpukan. Gunakan set perubahan jika Anda ingin memastikan bahwa AWS CloudFormation tidak membuat perubahan yang tidak disengaja atau jika Anda ingin mempertimbangkan beberapa pilihan. Misalnya, Anda dapat menggunakan set perubahan untuk memverifikasi bahwa AWS CloudFormation tidak akan menggantikan instans basis data tumpukan Anda selama pembaruan.
