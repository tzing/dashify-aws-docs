AWS CloudFormationGambaran umum antarmuka desainer

Desainermemiliki empat panel. Panel kanvas menampilkan diagram sumber daya templat Anda sehingga Anda dapat melihat sumber daya templat dan hubungannya secara sekilas. Untuk menambahkan sumber daya ke templat, seret sumber daya dari panel Jenis sumber daya ke panel kanvas. Gunakan panel Editor JSON dan YAML terintegrasi untuk menentukan detail templat, seperti properti sumber daya atau parameter templat. Setelah mengubah templat, Anda dapat menyimpannya ke file lokal atau ke bucket Amazon S3. Ketika Anda mengonversi templat yang valid dari JSON ke YAML atau sebaliknya, panel Pesan menampilkan pesan sukses atau kegagalan. Ketika Anda membuka atau memvalidasi templat yang tidak valid, panel Pesan menampilkan kesalahan validasi.

catatan

Desainer tidak dapat menampilkan atau mengubah sumber daya yang berjalan di tumpukan Anda; menggunakannya hanya untuk membuat, memodifikasi, dan menyimpan templat.

Gambar berikut mengilustrasikan panel Desainer dan komponen utamanya.

Panel dan komponen desainer

1. Toolbar

Toolbar menyediakan akses cepat ke perintah untuk tindakan umum, seperti membuka dan menyimpan templat, membatalkan atau mengulangi perubahan, membuat tumpukan, dan memvalidasi templat Anda. Anda juga dapat mengunduh diagram sebagai gambar, mendapatkan bantuan, atau menyegarkan diagram di panel kanvas.

2. Panel jenis sumber daya

Panel Jenis sumber daya mencantumkan semua sumber daya templat yang dapat Anda tambahkan ke templat, dikategorikan berdasarkan nama layanan AWS. Tambahkan sumber daya dengan menyeretnya dari panel Jenis sumber daya ke kanvas. Sebagian besar sumber daya yang didukung tercantum dalam AWS referensi jenis sumber daya dan properti. Panel Jenis sumber daya tidak mencantumkan sumber daya penghubung, seperti sumber daya AWS::EC2::SubnetRouteTableAssociation. Anda membuat sumber daya ini ketika menghubungkan sumber daya yang relevan, seperti ketika menghubungkan tabel rute ke subnet. Untuk informasi selengkapnya, lihat Panel kanvas.

catatan

Desainer hanya dapat menampilkanAWS CloudFormationjenis sumber daya yang didukung. Tidak dapat menampilkan entitas lain, seperti Availability Zone (AZ) atau sumber daya dari tumpukan nested.

3. Panel kanvas

Panel kanvas menampilkan sumber daya templat Anda sebagai diagram. Anda menggunakannya untuk menambah atau menghapus sumber daya, membuat hubungan antara sumber daya, dan mengatur tata letaknya. Perubahan yang Anda buat di kanvas secara otomatis mengubah JSON atau YAML templat. Untuk informasi selengkapnya, lihat Panel kanvas.

4. Tombol pas ke jendela

Tombol yang mengubah ukuran panel kanvas agar sesuai dengan diagram template anda.

5. Tombol Layar penuh dan Layar terpisah

Tombol untuk memilih tampilan yang berbeda dari Desainer. Anda dapat memilih tampilan layar penuh dari kanvas, tampilan layar penuh dari Editor JSON dan YAML terintegrasi, atau tampilan layar terpisah dari kanvas dan editor.

6. Panel editor JSON dan YAML terintegrasi

Di editor terintegrasi, Anda menentukan detail templat, seperti properti sumber daya atau parameter templat. Saat Anda memilih item dikanvas, Desainer menyoroti JSON atau YAKL yang terkait di editor. Setelah mengedit JSON atau YAML, Anda harus me-refresh kanvas (pilih ) untuk memperbarui diagram. Anda dapat mengonversi templat yang valid antara JSON dan YAKL dengan memilih tombol radio yang sesuai diMemilih Bahasa Templat. Desainer hanya dapat mengonversi templat YAKL atau JSON yang valid. Jika konversi berhasil,Pesanpanel menampilkan pesan seperti: Berhasil dikonversi template untuk YAKL.AWS CloudFormation Desainer tidak mempertahankan pemformatan saat mengonversi templat.

penting

Kami menyarankan agar Anda tidak menambahkan#YAKL komentar untuk template Anda di Designer. Jika template YAKL Anda memiliki#komentar, Desainer tidak mempertahankan komentar tersebut ketika mengedit YAKL atau mengonversi ke JSON. Jika Anda mengedit atau mengubah templat Anda di Desainer (misalnya, jika Anda menyeret sumber daya pada kanvas), komentar Anda akan hilang.

Setelah Anda memilih bahasa templat, sumber daya baru yang Anda seret ke kanvas akan dibuat dalam bahasa yang telah Anda pilih. Untuk mengubah kembali ke bahasa lain, pastikan templat Anda valid, kemudian pilih YAML atau JSON yang bertuliskan Pilih bahasa templat.

7. Panel pesan

Ketika Anda mengonversi templat dari JSON ke YAML atau sebaliknya, panel Pesan menampilkan pesan sukses atau kegagalan. Ketika Anda membuka, memvalidasi, mencoba untuk membuat tumpukan dengan templat yang tidak valid, panel Pesan menampilkan kesalahan validasi.
