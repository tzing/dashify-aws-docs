Menggunakan ekstensi pribadi di CloudFormation

Ekstensi pribadi adalah ekstensi yang secara eksplisit diizinkan untuk digunakan di ekstensi Anda. Akun AWS

Ada dua jenis ekstensi pribadi:

  • Ekstensi pribadi yang diaktifkan - Adalah salinan lokal ekstensi pihak ketiga yang telah Anda aktifkan untuk akun dan wilayah Anda. Saat Anda mengaktifkan ekstensi publik pihak ketiga, CloudFormation buat salinan lokal ekstensi tersebut di registri akun Anda.

  • Ekstensi pribadi terdaftar — Dapat juga mengaktifkan ekstensi pribadi yang tidak terdaftar di CloudFormation registri publik. Ekstensi ini mungkin adalah ekstensi yang Anda buat sendiri, atau ekstensi yang dibagikan dengan Anda oleh organisasi Anda atau pihak ketiga lainnya. Untuk menggunakan ekstensi pribadi semacam itu di akun Anda, Anda harus terlebih dahulu mendaftarkannya. Mendaftarkan ekstensi mengunggah salinannya ke CloudFormation registri di akun Anda dan mengaktifkannya.

Setiap penggunaan ekstensi pribadi di akun Anda analog dengan menggunakannya di lingkungan kotak pasir. Ini karena ekstensi dikendalikan versi, dan perilaku penyediaan khusus versi, yang berarti ekstensi pribadi berperilaku sama seperti jika dipublikasikan.

catatan

Ekstensi pribadi, dan ekstensi publik yang diaktifkan dari penerbit pihak ketiga, dapat menerapkan handler peristiwa yang berjalan selama membuat, membaca, memperbarui, daftar, dan menghapus operasi. Karena itu, menggunakan ekstensi ini di CloudFormation tumpukan Anda menimbulkan biaya ke akun Anda. Ini adalah tambahan untuk biaya yang dikeluarkan untuk sumber daya yang dibuat. Untuk informasi selengkapnya, lihat Harga AWS CloudFormation.

Untuk informasi tentang mengembangkan ekstensi pribadi Anda sendiri, lihat Panduan Pengguna Antarmuka Baris CloudFormation Perintah.

Mendaftarkan ekstensi pribadi

Untuk menggunakan ekstensi pribadi yang tidak tercantum dalam CloudFormation registri - baik yang Anda kembangkan sendiri, atau yang telah dibagikan dengan Anda - Anda harus terlebih dahulu mendaftarkannya CloudFormation, di akun dan wilayah tempat Anda ingin menggunakannya. Mendaftarkan ekstensi mengunggah salinannya ke CloudFormation registri di akun Anda, dan mengaktifkannya. Setelah Anda mendaftarkan ekstensi pribadi, itu akan muncul di CloudFormation registri untuk akun dan wilayah itu, dan Anda dapat menggunakannya di templat tumpukan Anda.

Anda dapat mendaftarkan ekstensi menggunakan perintah tipe register dariAWS CLI, atau menggunakan submit perintah CLI. CloudFormation

Untuk mendaftarkan ekstensi pribadi menggunakan CloudFormation CLI, lihat Mendaftarkan ekstensi di Panduan Pengguna CloudFormation CLI.

Izin IAM untuk mendaftarkan ekstensi pribadi

Sebagai bagian dari mendaftarkan ekstensi pribadi, Anda menentukan bucket Amazon S3 yang berisi paket proyek ekstensi. Paket ini berisi file sumber yang diperlukan untuk ekstensi yang ingin Anda daftarkan. Pengguna yang mendaftarkan ekstensi harus dapat mengakses paket proyek di bucket Amazon S3 itu. Artinya, pengguna perlu memiliki GetObjectizin untuk paket ekstensi.

Ini benar apakah Anda menggunakan perintah tipe register dariAWS CLI, atau perintah submit CLI. CloudFormation

Untuk informasi selengkapnya, lihat Tindakan, Sumber Daya, dan Kunci Kondisi Amazon S3 di Panduan AWS Identity and Access Management Pengguna.

Mendaftarkan ekstensi

Daftarkan ekstensi Anda di AWS CloudFormation registri sehingga tersedia untuk digunakan.

Untuk mendaftarkan kait menggunakan AWS CLI

Daftarkan hook Anda dengan CloudFormation, sehingga tersedia untuk digunakan dalam AWS CloudFormation registri.

  1. (Opsional) Konfigurasikan Wilayah AWS nama default Anda keus-west-2, dengan mengirimkan operasi. configure

    aws configure
AWS Access Key ID [None]: <Your Access Key ID>
AWS Secret Access Key [None]: <Your Secret Key>
Default region name [None]: us-west-2
Default output format [None]: json

  2. (Opsional) Perintah berikut membangun dan mengemas proyek hook Anda tanpa mendaftarkannya.

    $ cfn submit --dry-run

  3. Daftarkan hook Anda dengan menggunakan CloudFormation CLIcfn submit.

    cfn submit --set-default

    Perintah mengembalikan perintah berikut.

    {‘ProgressStatus’: ‘COMPLETE’}

    Hasil: Anda telah berhasil mendaftarkan hook Anda.

Untuk mendaftarkan jenis sumber daya menggunakan AWS CLI

Daftarkan jenis sumber daya Anda CloudFormation, sehingga tersedia untuk digunakan dalam AWS CloudFormation registri.

  1. Temukan Amazon S3 bucket yang berisi paket proyek untuk ekstensi pribadi yang ingin Anda daftarkan di akun Anda.

  2. Gunakan perintah register-type untuk mendaftarkan ekstensi di akun Anda:

    RegisterTypeadalah operasi asinkron, dan mengembalikan token pendaftaran yang dapat Anda gunakan untuk melacak kemajuan permintaan pendaftaran Anda.

    catatan

    Jika ekstensi memanggil AWS API sebagai bagian dari fungsinya, Anda harus membuat peran eksekusi IAM yang menyertakan izin yang diperlukan untuk memanggil AWS API tersebut, dan menyediakan peran eksekusi tersebut di akun Anda. Anda kemudian dapat menentukan peran eksekusi ini menggunakan --execution-role-arn parameter. CloudFormation kemudian mengasumsikan bahwa peran eksekusi untuk menyediakan jenis sumber daya Anda dengan kredensi yang sesuai.

    Misalnya, perintah berikut mendaftarkan jenis My::Resource::Example sumber daya saat iniAkun AWS:

    aws cloudformation register-type --type-name My::Resource::Example --schema-handler-package [s3 object path] --type RESOURCE                
{
    "RegistrationToken": "f5525280-104e-4d35-bef5-8f1fexample"
}

  3. Opsional: Gunakan token pendaftaran dengan perintah describe-type-registration untuk melacak kemajuan permintaan pendaftaran Anda.

    Ketika CloudFormation menyelesaikan permintaan pendaftaran, itu menetapkan status kemajuan permintaan keCOMPLETE.

    Contoh berikut menggunakan token registrasi yang dikembalikan oleh describe-type-registration perintah di atas untuk mengembalikan informasi status pendaftaran.

    aws cloudformation describe-type-registration --registration-token f5525280-104e-4d35-bef5-8f1fexample
{
    "ProgressStatus": "COMPLETE",
    "TypeArn": "arn:aws:cloudformation:us-east-1:012345678910:type/resource/My-Resource-Example",
    "Description": "Deployment is currently in DEPLOY_STAGE of status COMPLETED; ",
    "TypeVersionArn": "arn:aws:cloudformation:us-east-1:111122223333:type/resource/My-Resource-Example/00000001"
}

Mengonfigurasi ekstensi di tingkat akun

Ekstensi dapat menyertakan properti konfigurasi yang dimaksudkan untuk diterapkan ke semua instance ekstensi untuk akun dan Wilayah tertentu. Penulis ekstensi mendefinisikan ini dalam konfigurasi definisi ekstensi. Jika ada properti yang diperlukan dalam definisi konfigurasi ekstensi, Anda harus menentukan properti tersebut sebelum dapat menggunakan ekstensi di akun dan Wilayah Anda.

catatan

Jika konfigurasi tipe Anda menyertakan referensi dinamis ke nilai yang disimpan di AWS Systems Manager atauAWS Secrets Manager, peran apa pun yang digunakan untuk menyediakan tipe (misalnya, saat membuat atau memperbarui tumpukan) harus memiliki izin yang tepat untuk mengambil nilai tersebut. Secara khusus:

  • Jika konfigurasi tipe berisi parameter yang disimpan di AWS Systems Manager Parameter Store, pengguna atau peran yang digunakan untuk menyediakan tipe harus memiliki izin untuk memanggil GetParameter.

  • Jika konfigurasi tipe berisi rahasia yang disimpanAWS Secrets Manager, pengguna atau peran yang digunakan untuk menyediakan tipe harus memiliki izin untuk memanggil GetSecretValue.

Untuk informasi selengkapnya tentang referensi dinamis, lihat Menggunakan referensi dinamis untuk menentukan nilai templat.

Untuk informasi selengkapnya tentang bagaimana definisi konfigurasi didefinisikan saat mengembangkan ekstensi, lihat Mendefinisikan konfigurasi tingkat akun ekstensi di Panduan CloudFormation Pengguna untuk Pengembangan Ekstensi.

Untuk melihat data konfigurasi saat ini untuk ekstensi di CloudFormation konsol

  1. Gunakan CloudFormation registri untuk menemukan ekstensi.

  2. Pilih ekstensi untuk melihat detail ekstensi.

  3. Pada halaman detail ekstensi, pilih tab Konfigurasi.

  4. Perluas tab skema Konfigurasi untuk melihat skema konfigurasi yang ditentukan untuk ekstensi.

  5. Perluas tab Konfigurasi untuk melihat konfigurasi saat ini yang telah Anda tetapkan untuk ekstensi ini.

Untuk melihat data konfigurasi saat ini untuk ekstensi di AWS CLI

  • Gunakan perintah describe-type untuk mengembalikan informasi mendetail tentang ekstensi. ConfigurationSchemaElemen output berisi definisi konfigurasi ekstensi saat ini di Wilayah tertentu.

    Atau, gunakan batch-describe-type-configurationsperintah untuk mengembalikan data konfigurasi tentang beberapa ekstensi.

Untuk menentukan properti konfigurasi untuk jenis ekstensi

Untuk menentukan properti konfigurasi untuk ekstensi menggunakan CloudFormation konsol

  1. Gunakan CloudFormation registri untuk menemukan ekstensi di ekstensi yang diaktifkan akun Anda.

  2. Pilih ekstensi untuk melihat detail ekstensi.

  3. Pada halaman detail ekstensi, dari tab Konfigurasi, pilih Edit konfigurasi.

    Atau, dari Tindakan, pilih Edit, lalu pilih Edit konfigurasi.

    CloudFormation menampilkan halaman ekstensi Konfigurasi. Pastikan skema konfigurasi View diaktifkan untuk melihat skema definisi konfigurasi ekstensi saat ini.

  4. Di kotak teks Konfigurasi JSON, masukkan string JSON yang mewakili skema konfigurasi yang ingin Anda atur untuk ekstensi ini. Itu harus memvalidasi terhadap skema yang didefinisikan dalam skema Konfigurasi.

  5. Pilih Konfigurasi ekstensi.

Untuk menentukan properti konfigurasi untuk ekstensi menggunakan AWS CLI

  • Gunakan set-type-configuration perintah untuk menentukan konfigurasi yang akan digunakan dengan ekstensi di akun dan Wilayah Anda.

    JSON yang Anda lewati --configuration harus memvalidasi terhadap skema definisi konfigurasi ekstensi. Berikut ini adalah contoh dari set-type-configuration perintah yang menentukan konfigurasi.

    aws cloudformation set-type-configuration \
    --region us-west-2 \
    --type RESOURCE \
    --type-name My::Resource::Example \
    --configuration-alias default \
    --configuration "{"CredentialKey": "testUserCredential"}"

Menentukan versi ekstensi pribadi untuk digunakan menggunakan AWS CLI

Seiring waktu, Anda dapat mendaftarkan beberapa versi ekstensi yang sama. Anda dapat menentukan versi ekstensi mana yang ingin Anda gunakan untuk operasi CloudFormation.

Untuk menentukan versi ekstensi

Gunakan set-type-default-version perintah untuk menentukan versi ekstensi mana yang akan digunakan untuk CloudFormation operasi di akun Anda.

Misalnya, perintah berikut menetapkan versi default dari jenis sumber daya My::Resource::Example untuk akun 00000003 saat ini.

aws cloudformation set-type-default-version \
  --type RESOURCE \
  --type-name My::Resource::Example \
  --version-id 00000003

Untuk menentukan versi hook default

Untuk menentukan versi default hook Anda di akun Anda, gunakan set-type-default-version perintah dan tentukan jenis, jenis nama, dan versi ID.

aws cloudformation set-type-default-version \
    --type HOOK \
    --type-name MyCompany::Testing::MyTestHook \
    --version-id 00000003
