Menggunakan referensi dinamis untuk menentukan nilai templat

Referensi dinamis menyediakan cara yang ringkas dan ampuh bagi Anda untuk menentukan nilai eksternal yang disimpan dan dikelola dalam layanan lainnya, seperti Systems Manager Parameter Store danAWS Secrets Manager, di templat tumpukan Anda. Ketika Anda menggunakan referensi dinamis, CloudFormation mengambil nilai referensi yang ditentukan saat diperlukan selama operasi tumpukan dan set perubahan.

CloudFormation saat ini mendukung pola referensi dinamis berikut ini:

  • ssm, Untuk nilai plaintext yang disimpan diAWS Systems Manager Penyimpanan Parameter.

  • ssm-secure, untuk string aman yang disimpan diAWS Systems Manager Parameter Store.

  • secretsmanager, untuk seluruh rahasia atau nilai rahasia yang tersimpan di dalamnyaAWS Secrets Manager.

Pertimbangan saat menggunakan referensi dinamis

Berikut adalah hal yang harus Anda pertimbangkan saat menggunakan referensi dinamis:

penting

Kami sangat menyarankan agar tidak menyertakan parameter, atau data sensitif apa pun, di properti sumber daya yang merupakan bagian dari pengenal utama sumber daya.

Saat parameter referensi dinamis termasuk dalam properti yang membentuk pengenal sumber daya utama, CloudFormation dapat menggunakan nilai plaintext aktual pada pengenal sumber daya utama. ID sumber daya ini mungkin muncul dalam output turunan atau tujuan.

Untuk menentukan properti sumber daya yang terdiri dari pengenal primer tipe sumber daya ini, lihat dokumentasi referensi sumber daya untuk sumber daya itu. Di bagian Kembalikan nilai, nilai Ref pengembalian fungsi mewakili properti sumber daya yang terdiri dari tipe pengidentifikasi utama sumber daya.

  • Anda dapat menyertakan hingga 60 referensi dinamis dalam templat tumpukan.

  • Untuk transformasi, sepertiAWS::Include danAWS::Serverless,AWS CloudFormation tidak menyelesaikan referensi dinamis sebelum menerapkan transformasi apa pun. Sebaliknya, AWS CloudFormation melewati string literal referensi dinamis untuk transformasi. Referensi dinamis (termasuk yang dimasukkan ke dalam templat diproses sebagai hasil dari transformasi) diselesaikan ketika Anda menjalankan perubahan set menggunakan templat.

  • Referensi dinamis untuk nilai aman, seperti ssm-secure dan secretsmanager, belum di-support di sumber daya kustom.

catatan

Jangan membuat referensi dinamis yang memiliki garis miring terbalik (\) sebagai nilai akhir. AWS CloudFormationtidak dapat menyelesaikan referensi tersebut, yang mengakibatkan kegagalan sumber daya.

Menentukan referensi dinamis dalam templat tumpukan

Referensi dinamis mematuhi pola berikut ini:

'{{resolve:service-name:reference-key}}' atau '{{resolve:ssm:[a-zA-Z0-9_.\-/]+(:\d+)?}}'.

nama layanan

Menentukan layanan di mana nilai disimpan dan dikelola.

Diperlukan.

Saat ini, nilai yang valid meliputi:

  • ssm: Parameter plaintext Systems Manager Parameter plaintext.

  • ssm-secure: Systems Manager Parameter string aman.

  • secretsmanager: rahasia Secrets Manager.

referensi-kunci

Kunci referensi. Tergantung pada jenis referensi dinamis, kunci referensi dapat terdiri dari beberapa segmen.

Diperlukan.

Parameter SSM

Gunakan referensissm dinamis untuk memasukkan nilai yang disimpan di Systems Manager Parameter Store dari jenisString atauStringList dalam templat Anda.

Pola referensi

Untuk SSM Parameters, segmen reference-key terdiri dari nama parameter dan nomor versi. Gunakan pola berikut:

'{{resolve:ssm:parameter-name:version}}'

Referensi Anda harus mematuhi pola ekspresi reguler berikut untuk nama parameter dan versi:

'{{resolve:ssm:[a-zA-Z0-9_.-/]+:\\d+}}'

parameter-nama

Nama parameter di Systems Manager Parameter Store. Perhatikan bahwa nama parameter bersifat peka huruf besar-kecil.

Diperlukan.

versi

Integer yang menentukan versi parameter untuk digunakan. Jika Anda tidak menentukan versi yang tepat, CloudFormation gunakan versi terbaru parameter setiap kali Anda membuat atau memperbarui tumpukan. Untuk informasi selengkapnya, lihat Bekerja dengan versi parameter di PanduanAWS Systems Manager Pengguna

Tidak wajib.

Contoh

Contoh berikut menggunakan referensissm dinamis untuk mengatur kontrol akses untuk bucket S3 ke nilai parameter yang disimpan di Systems Manager Parameter Store. Seperti yang ditentukan, CloudFormation akan menggunakan versi 2S3AccessControl parameter untuk tumpukan dan perubahan set operasi.

JSON

  "MyS3Bucket": {
    "Type": "AWS::S3::Bucket",
    "Properties": {
      "AccessControl": "{{resolve:ssm:S3AccessControl:2}}"
    }
  }

YAML

  MyS3Bucket:
    Type: 'AWS::S3::Bucket'
    Properties:
      AccessControl: '{{resolve:ssm:S3AccessControl:2}}'

Untuk menentukan parameter yang disimpan di Systems Manager Parameter Store, Anda harus memiliki aksesGetParameters untuk memanggil parameter yang ditentukan. Untuk informasi selengkapnya, lihat Mengontrol akses ke parameter Systems Manager di PanduanAWS Systems Manager Pengguna.

Pertimbangan tambahan yang perlu diperhatikan saat menggunakan ssm pola referensi dinamis:

  • Saat ini, CloudFormation tidak mendukung akses parameter SSM lintas akun.

  • Untuk sumber daya kustom, CloudFormation menyelesaikan referensissm dinamis sebelum mengirim permintaan ke sumber daya kustom. Untuk informasi selengkapnya, lihat Sumber daya khusus.

  • CloudFormation tidak mendukung penggunaan label parameter atau parameter publik dalam referensi dinamis.

    Label parameter adalah alias yang ditetapkan pengguna untuk membantu Anda mengelola versi parameter yang berbeda. Untuk informasi selengkapnya, lihat Parameter pelabelan di PanduanAWS Systems Manager Pengguna.

    Parameter publik adalah parameter yang disediakan oleh AWS layanan untuk digunakan dengan layanan tersebut, dan disimpan di AWS Systems Manager Penyimpanan Parameter. Untuk contoh parameter publik, lihat Mengambil metadata AMI yang dioptimalkan Amazon ECS di Panduan Developer Amazon Elastic Container Service.

  • CloudFormation Saat ini tidak mendukung deteksi drift pada referensi dinamis. Untuk referensissm dinamis yang versi parameternya belum Anda tentukan, kami merekomendasikan bahwa, kami merekomendasikan bahwa, kami merekomendasikan bahwa, kami juga melakukan operasi pembaruan tumpukan pada tumpukan yang mencakup referensissm dinamis, untuk mengambil versi parameter terbaru.

  • Untuk memverifikasi versi ssm referensi dinamis yang akan digunakan dalam operasi tumpukan, buat satu set perubahan untuk operasi tumpukan. Kemudian tinjau templat yang diproses pada tab Templat.

  • Parameter SSM tanpa versi tidak didukung di blok Parameter, gunakan tipe parameter SSM sebagai gantinya. Jika Anda menggunakan parameter SSM, Anda harus menentukan versi parameter Systems ManagerAWS CloudFormation untuk digunakan.

SSM parameter string aman

Gunakan pola referensissm-secure dinamis untuk menentukan parameterAWS Systems ManagerSecureString jenis dalam templat Anda. Untuk ssm-secure referensi dinamis, AWS CloudFormation jangan pernah menyimpan nilai parameter yang sebenarnya. AWS CloudFormation mengakses nilai parameter selama membuat dan memperbarui operasi untuk tumpukan dan perubahan set. Saat ini, parameter string aman hanya dapat digunakan untuk properti sumber daya yang men-support pola ssm-secure referensi dinamis.

parameter string aman adalah data sensitif yang perlu disimpan dan direferensikan dengan cara yang aman. Maksudnya adalah data tersebut tidak boleh digunakan atau diubah oleh pengguna dalam teks yang jelas, misalnya kata sandi atau kunci lisensi. Untuk informasi selengkapnya, lihat Gunakan parameter string aman di PanduanAWS Systems Manager Pengguna.

Nilai parameter string aman tidak disimpan di dalamnya CloudFormation, maupun tidak dikembalikan dalam hasil panggilan API.

Pola referensi

Untuk ssm-secure referensi dinamis, segmen reference-key terdiri dari nama parameter dan nomor versi. Gunakan pola berikut:

'{{resolve:ssm-secure:parameter-name:version}}'

Referensi Anda harus mematuhi pola ekspresi reguler berikut untuk nama parameter dan versi:

'{{resolve:ssm-secure:[a-zA-Z0-9_.-/]+:\\d+}}'

parameter-nama

Nama parameter di Systems Manager Parameter Store. Perhatikan bahwa nama parameter bersifat peka huruf besar-kecil.

Diperlukan.

versi

Integer yang menentukan versi parameter untuk digunakan. Jika Anda tidak menentukan versi yang tepat,AWS CloudFormation gunakan versi terbaru parameter setiap kali Anda membuat atau memperbarui tumpukan. Untuk informasi selengkapnya, lihat Bekerja dengan versi parameter di PanduanAWS Systems Manager Pengguna

Tidak wajib.

Contoh

Contoh berikut menggunakan referensissm-secure dinamis untuk mengatur kata sandi untuk pengguna IAM ke string aman yang disimpan di Systems Manager Parameter Store. Seperti yang ditentukan, CloudFormation akan menggunakan versi 10IAMUserPassword parameter untuk tumpukan dan perubahan set operasi.

JSON

  "MyIAMUser": {
    "Type": "AWS::IAM::User",
    "Properties": {
      "UserName": "MyUserName",
      "LoginProfile": {
        "Password": "{{resolve:ssm-secure:IAMUserPassword:10}}"
      }
    }
  }

YAML

  MyIAMUser:
    Type: AWS::IAM::User
    Properties:
      UserName: 'MyUserName'
      LoginProfile:
        Password: '{{resolve:ssm-secure:IAMUserPassword:10}}'

Pertimbangan tambahan yang perlu diperhatikan saat menggunakan ssm-secure pola referensi dinamis:

  • CloudFormation tidak mengembalikan nilai parameter aktual untuk string aman dalam setiap panggilan API, melainkan mengembalikan referensi dinamis literal.

  • CloudFormation tidak menyimpan referensi dinamis literal, yang berisi nama parameter teks biasa dari string aman.

  • Untuk set perubahan, CloudFormation membandingkan string referensi dinamis literal. Ini tidak menyelesaikan dan membandingkan nilai ssm-secure referensi sebenarnya.

  • Referensi dinamis untuk nilai aman, seperti ssm-secure dan secretsmanager, belum di-support di sumber daya kustom.

  • Ketika CloudFormation harus me-rollback pembaruan tumpukan, operasi rollback pembaruan akan gagal jika versi parameter string aman yang ditentukan sebelumnya tidak lagi tersedia. Dalam kasus tersebut, lakukan salah satu hal berikut:

    • Gunakan CONTINUE_UPDATE_ROLLBACK untuk melewati sumber daya.

    • Ciptakan ulang parameter string aman di Systems Manager Parameter Store, dan perbarui sampai versi parameter mencapai versi yang digunakan dalam templat. Kemudian gunakan CONTINUE_UPDATE_ROLLBACK tanpa melewatkan sumber daya.

  • Saat ini, AWS CloudFormation tidak men-support akses parameter SSM lintas akun.

  • CloudFormation tidak mendukung penggunaan label parameter atau parameter publik dalam referensi dinamis.

    Label parameter adalah alias yang ditetapkan pengguna untuk membantu Anda mengelola versi parameter yang berbeda. Untuk informasi selengkapnya, lihat Parameter pelabelan di PanduanAWS Systems Manager Pengguna.

    Parameter publik adalah parameter yang disediakan oleh AWS layanan untuk digunakan dengan layanan tersebut, dan disimpan di AWS Systems Manager Penyimpanan Parameter. Untuk contoh parameter publik, lihat Mengambil metadata AMI yang dioptimalkan Amazon ECS di Panduan Developer Amazon Elastic Container Service.

Rahasia Secrets Manager

Gunakan referensisecretsmanager dinamis untuk mengambil seluruh rahasia atau nilai rahasia yang disimpan di Secrets Manager untuk digunakan dalam templat Anda. Rahasia dapat berupa kredensyal basis data, kata sandi, kunci API pihak ke tiga, atau teks acak. Dengan menggunakan Secrets Manager, Anda dapat menyimpan dan mengontrol akses ke rahasia ini secara terpusat, sehingga Anda dapat mengganti mandat hardcode dalam kode Anda (termasuk kata sandi), dengan panggilan API ke Secrets Manager untuk mengambil rahasia secara terprogram. Untuk informasi lebih lanjut, lihat Apa AWS Secrets Manager? dalam AWS Secrets Manager Panduan Pengguna.

Pertimbangan penting saat menggunakan referensi dinamis untuk rahasia Secrets Manager

Anda harus mengambil pertimbangan keamanan penting berikut ini ke akun ketika menggunakan referensi dinamis untuk menentukan rahasia Secrets Manager dalam templat tumpukan Anda:

  • Referensi secretsmanager dinamis dapat digunakan di semua atribut sumber daya. Menggunakan referensisecretsmanager dinamis menunjukkan bahwa baik Secrets Manager maupun CloudFormation log harus bertahan setiap nilai rahasia yang diselesaikan. Namun, nilai rahasia mungkin muncul dalam layanan yang sumber dayanya sedang digunakan. Tinjau penggunaan Anda untuk menghindari kebocoran data rahasia.

  • Memperbarui rahasia di Secrets Manager tidak secara otomatis memperbarui rahasia di CloudFormation. CloudFormation Untuk memperbarui referensisecretsmanager dinamis, Anda harus melakukan pembaruan tumpukan yang memperbarui sumber daya yang berisi referensi dinamis, baik dengan memperbarui properti sumber daya yang berisi referensisecretsmanager dinamis, atau memperbarui properti sumber daya lainnya.

    Misalnya, Anda menentukan MasterPassword properti AWS::RDS::DBInstance sumber daya dalam templat untuk menjadi secretsmanager referensi dinamis, dan kemudian membuat tumpukan dari templat. Anda kemudian memperbarui nilai rahasia itu di Secrets Manager, tetapi tidak memperbaruiAWS::RDS::DBInstance sumber daya dalam templat Anda. Dalam kasus ini, bahkan jika Anda melakukan pembaruan tumpukan, nilai rahasia dalam MasterPassword properti tidak diperbarui, dan tetap nilai rahasia sebelumnya.

    Juga, pertimbangkan untuk menggunakan Secrets Manager untuk memutar rahasia secara otomatis untuk layanan atau basis data aman. Untuk informasi lebih lanjut, lihat AWS Secrets ManagerRotasikan rahasia.

  • Referensi dinamis untuk nilai aman, seperti secretsmanager, saat ini belum di-support di sumber daya kustom.

Izin diperlukan

Untuk menentukan rahasia yang disimpan di rahasia Secrets Manager, Anda harus memiliki aksesGetSecretValue untuk memanggil rahasia.

Pola referensi

Untuk rahasia Secrets Manager,reference-key segmen ini terdiri dari beberapa segmen, termasuk id rahasia, kunci nilai rahasia, tahap versi, dan versi id. Gunakan pola berikut:

{{resolve:secretsmanager:secret-id:secret-string:json-key:version-stage:version-id}}

rahasia-id

Nama atau ARN rahasia.

Untuk mengakses rahasia di AWS akun, Anda hanya perlu menentukan nama rahasia. Untuk mengakses rahasia pada AWS akun yang berbeda, tentukan ARN lengkap rahasia tersebut.

Diperlukan.

rahasia-string

Saat ini, satu-satunya nilai yang didukung adalah SecretString. Defaultnya adalah SecretString.

kunci json

Nama kunci dari pasangan nilai kunci yang ingin Anda ambil. Jika Anda tidak menentukanjson-key, CloudFormation mengambil seluruh teks rahasia.

Segmen ini mungkin tidak memasukkan karakter titik dua ( :).

versi-tahap

Label pementasan versi rahasia yang akan digunakan. Secrets Manager menggunakan label pementasan untuk melacak versi yang berbeda selama proses rotasi. Jika Anda menggunakan version-stage maka jangan tentukan version-id. Jika Anda tidak menentukan salah satuversion-stage atauversion-id, maka default adalahAWSCURRENT versi.

Segmen ini mungkin tidak memasukkan karakter titik dua ( :).

versi-id

Pengenal unik dari versi rahasia yang akan digunakan. Jika Anda menentukan version-id, jangan tentukan version-stage. Jika Anda tidak menentukan salah satuversion-stage atauversion-id, maka default adalahAWSCURRENT versi.

Segmen ini mungkin tidak memasukkan karakter titik dua ( :).

Contoh

Contoh berikut menggunakansecret-name danjson-key segmen untuk mengambil nilai nama pengguna dan kata sandi yang disimpan dalam rahasia MyRDSSecret. Secara default, versi rahasia yang diambil adalah versi dengan nilai tahap versi AWSCURRENT.

JSON

{
    "MyRDSInstance": {
        "Type": "AWS::RDS::DBInstance",
        "Properties": {
            "DBName": "MyRDSInstance",
            "AllocatedStorage": "20",
            "DBInstanceClass": "db.t2.micro",
            "Engine": "mysql",
            "MasterUsername": "{{resolve:secretsmanager:MyRDSSecret:SecretString:username}}",
            "MasterUserPassword": "{{resolve:secretsmanager:MyRDSSecret:SecretString:password}}"
        }
    }
}

YAML

  MyRDSInstance:
    Type: 'AWS::RDS::DBInstance'
    Properties:
      DBName: MyRDSInstance
      AllocatedStorage: '20'
      DBInstanceClass: db.t2.micro
      Engine: mysql
      MasterUsername: '{{resolve:secretsmanager:MyRDSSecret:SecretString:username}}'
      MasterUserPassword: '{{resolve:secretsmanager:MyRDSSecret:SecretString:password}}'
