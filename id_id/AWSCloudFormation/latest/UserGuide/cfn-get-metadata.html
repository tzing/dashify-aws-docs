cfn-get-metadata

Deskripsi

Anda dapat menggunakan blok cfn-get-metadata pembantu untuk mengambil blok metadata CloudFormation dan mencetaknya ke standar. Anda juga dapat mencetak subpohon dari blok metadata jika Anda menentukan kunci. Namun, hanya kunci tingkat atas yang didukung.

catatan

cfn-get-metadata tidak memerlukan kredensi, sehingga Anda tidak perlu menggunakan--access-key,,--secret-key--role, atau--credential-file pilihan. Namun, jika tidak ada kredensial yang ditentukan, CloudFormation memeriksa keanggotaan tumpukan dan membatasi cakupan panggilan ke tumpukan tempat instans berada.

Sintaksis

cfn-get-metadata --access-key access.key \
                 --secret-key secret.key \
                 --credential-file|f credential.file \
                 --key|k key \
                 --stack|-s stack.name.or.id \
                 --resource|-r logical.resource.id \
                 --role IAM.role.name \
                 --url|-u service.url \
                 --region region

Opsi

Nama Deskripsi Wajib

-k, --key

Untuk pasangan nilai kunci, mengembalikan nama kunci untuk nilai yang Anda tentukan.

Jenis: String

Contoh: Untuk { "Key1": "SampleKey1", "Key2": "SampleKey2" }, cfn-get-metadata -k Key2 mengembalikan SampleKey2.

Tidak

-s, --stack

Nama Tumpukan.

Jenis: String

Default: Tidak Ada

Contoh: -s { "Ref" : "AWS::StackName" },

Ya

-r, --resource

ID sumber daya logis dari sumber daya yang berisi metadata.

Jenis: String

Contoh: -r WebServerHost

Ya

--role (pemberian sinyal sumber daya saja)

Nama peran IAM yang terkait dengan instans.

Jenis: String

Syarat: Parameter file kredensial menggantikan parameter ini.

Tidak

--region

Wilayah yang akan memperoleh CloudFormation URL.

Jenis: String

Default: Tidak Ada

Contoh: --region ", { "Ref" : "AWS::Region" },

Tidak

--access-key

AWSKunci Akses untuk akun dengan izin untuk DescribeStackResource menelepon CloudFormation.

Jenis: String

Syarat: Parameter file kredensial menggantikan parameter ini.

Bersyarat

--secret-key

Secret Key AWS yang sesuai dengan Access Key AWS yang ditentukan.

Jenis: String

Syarat: Parameter file kredensial menggantikan parameter ini.

Bersyarat

-f, --credential-file

File yang berisi secret key dan access key.

Jenis: String

Syarat: Parameter file kredensial menggantikan parameter --access key dan --secret key.
