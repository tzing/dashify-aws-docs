Format templat AWS CloudFormation

Anda dapat menulis templat AWS CloudFormation dalam format JSON atau YAML. Kami mendukung semuaAWS CloudFormationfitur dan fungsi dan fungsi untuk kedua format, termasuk diAWS CloudFormationDesainer.

Ketika menentukan format yang akan digunakan, pilih format yang paling nyaman digunakan. Pertimbangkan juga bahwa YAML secara permanen menyediakan beberapa fitur, seperti komentar, yang tidak tersedia di JSON.

penting

Kami sarankan Anda tidak menambahkan#YAKL komentar untuk template Anda di Designer. Jika template YAKL Anda memiliki#komentar, Desainer tidak mempertahankan komentar tersebut saat mengonversi templat ke JSON. Selain itu, jika Anda mengubah templat Anda di Designer (misalnya, jika Anda memindahkan sumber daya di kanvas), komentar Anda akan hilang.

Anda dapat menambahkan komentar keAWS CloudFormationtemplate yang Anda buat di luar Designer. Contoh berikut menunjukkan templat YAML dengan komentar sejajar.


AWSTemplateFormatVersion: "2010-09-09"
Description: A sample template
Resources:
  MyEC2Instance: #An inline comment
    Type: "AWS::EC2::Instance"
    Properties: 
      ImageId: "ami-0ff8a91507f77f867" #Another comment -- This is a Linux AMI
      InstanceType: t2.micro
      KeyName: testkey
      BlockDeviceMappings:
        -
          DeviceName: /dev/sdm
          Ebs:
            VolumeType: io1
            Iops: 200
            DeleteOnTermination: false
            VolumeSize: 20

Untuk informasi lebih lanjut tentang sintaksis templat untuk setiap format, lihat Anatomi templat.

AWS CloudFormation mendukung spesifikasi JSON dan YAML berikut:

JSON

AWS CloudFormation mengikuti standar JSON ECMA-404. Untuk informasi tentang format JSON, lihat http://www.json.org.

YAML

AWS CloudFormation mendukung spesifikasi YAML Versi 1.1 dengan beberapa pengecualian. AWS CloudFormation tidak mendukung fitur berikut:

  • Tanda binary, omap, pairs, set, dan timestamp

  • Alias

  • Penggabungan hash

Untuk informasi lebih lanjut tentang YAML, lihat https://yaml.org.
