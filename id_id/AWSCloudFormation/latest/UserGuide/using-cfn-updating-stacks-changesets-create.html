Membuat set perubahan

Untuk membuat set perubahan untuk tumpukan yang sedang berjalan, kirimkan perubahan yang ingin Anda buat dengan menyediakan templat yang dimodifikasi, nilai parameter input baru, atau keduanya. CloudFormation menghasilkan perubahan yang ditetapkan dengan membandingkan tumpukan Anda dengan perubahan yang Anda kirimkan.

Anda dapat memodifikasi templat sebelum membuat set perubahan atau selama pembuatan set perubahan.

Create a change set for nested stacks (console)
Untuk membuat set perubahan untuk tumpukan nest (konsol)

  1. Di konsol AWS CloudFormation, di Tumpukan, pilih tumpukan berjalan yang ingin Anda buatkan set perubahannya.

  2. Di panel detail tumpukan, pilih Tindakan tumpukan, lalu pilih Buat set perubahan untuk tumpukan saat ini.

  3. Pada Buat set perubahan untuk halaman stack-name, lakukan salah satu dari berikut ini untuk memodifikasi nilai parameter input, menentukan lokasi templat yang diperbarui, atau memodifikasi templat:

    Tugas Action
    Untuk memodifikasi nilai parameter input Pilih Gunakan templat saat ini, dan kemudian pilih Selanjutnya untuk melanjutkan memasukkan atau memodifikasi nilai parameter input.
    Untuk menentukan lokasi templat yang diperbarui

    Jika Anda telah memodifikasi templat, pilih Ganti templat saat ini, dan kemudian lakukan salah satu hal berikut:

    • Untuk templat yang disimpan dalam bucket Amazon S3, pilih URL Amazon S3. Masukkan atau tempel URL untuk templat, lalu pilih Selanjutnya.

      Jika Anda memiliki templat dalam bucket dengan versioning yang diaktifkan, Anda dapat menentukan versi templat tertentu, seperti https://s3.amazonaws.com/templates/myTemplate.template?versionId=123ab1cdeKdOW5IH4GAcYbEngcpTJTDW. Untuk informasi selengkapnya, lihat Mengelola objek dalam bucket berkemampuan versi di Panduan Pengguna Amazon Simple Storage Service.

    • Untuk templat yang disimpan secara lokal di komputer Anda, pilihUnggah file templat. Pilih Pilih File untuk menavigasi ke file dan memilihnya, dan kemudian pilih Selanjutnya.
    Untuk memodifikasi templat Jika Anda belum memodifikasi templat, pilih Edit templat di perancang, dan kemudian pilih Lihat di perancang. Anda dialihkan keAWS CloudFormation Designer. Setelah Anda memodifikasi templat, pilih untuk kembali ke Buat set perubahan untuk halaman stack-name, dan kemudian pilih Selanjutnya.

  4. Jika templat anda berisi parameter, pada halaman Tentukan detail tumpukan, masukkan atau modifikasi nilai parameter input yang berlaku, dan kemudian pilih Selanjutnya.

    Jika Anda menggunakan kembali templat tumpukan, CloudFormation buat kembali parameter dengan nilai saat ini di tumpukan, dengan pengecualian parameter dinyatakan denganNoEcho atribut. Untuk menggunakan nilai yang ada untuk parameter tersebut, pilihGunakan nilai yang ada.

    Untuk informasi lebih lanjut tentang menggunakan NoEcho untuk menutupi informasi sensitif, serta menggunakan parameter dinamis untuk mengelola rahasia, lihat praktik terbaik Jangan menanamkan kredensial di templat anda.

  5. Pada halaman Konfigurasi opsi tumpukan, perbarui tanda tumpukan, peran layanan IAM, kebijakan tumpukan, konfigurasi rollback, topik notifikasi Amazon SNS (jika berlaku), atau set perubahan dan kemudian pilih Selanjutnya.

    catatan

    Set perubahan untuk tumpukan nest Diaktifkan secara default, yang akan membuat set perubahan untuk semua tumpukan nest yang ditentukan di templat Anda. Untuk informasi lebih lanjut tentang set perubahan untuk tumpukan nest, lihat Ubah set untuk tumpukan nest.

  6. Pada halaman Tinjau stack-name, tinjau perubahan untuk set perubahan ini.

    Jika template menyertakan sumber dayaAWS Identity and Access Management (IAM), pilih Saya mengakui bahwa CloudFormation mungkin membuat sumber daya IAM. Sumber daya IAM dapat mengubah izin diAWS akun Anda; tinjau sumber daya ini untuk memastikan bahwa Anda hanya mengizinkan tindakan yang Anda inginkan. Untuk informasi selengkapnya, lihat Mengontrol akses dengan AWS Identity and Access Management.

  7. Pilih Buat set perubahan. Tentukan nama untuk set perubahan dan secara opsional tentukan deskripsi set perubahan untuk mengidentifikasi tujuannya. Lalu, pilih Buat set perubahan.

    Anda dialihkan ke tab Perubahan di halaman detail set perubahan. Sementara CloudFormation menghasilkan set perubahan, status set perubahan adalah CREATE_IN_PROGRESS. Setelah membuat set perubahan, CloudFormation mengatur status ke CREATE_COMPLETE. Di bagian Perubahan, CloudFormation mencantumkan semua perubahan yang akan dibuat pada tumpukan Anda. Untuk informasi selengkapnya, lihat Melihat set perubahan.

    Jika CloudFormation gagal membuat set perubahan (melaporkanFAILED status), perbaiki kesalahan yang ditampilkan di bidang Status, dan kemudian buat kembali set perubahan.

Untuk membuat set perubahan (AWS CLI)

  • Jalankan perintah aws cloudformation create-change-set.

    Anda mengirimkan perubahan Anda sebagai opsi perintah. Anda dapat menentukan nilai parameter baru, templat yang dimodifikasi, atau keduanya. Misalnya, perintah berikut membuat set perubahan bernama SampleChangeSetuntuk tumpukan SampleStack. Set perubahan menggunakan templat tumpukan saat ini, tetapi dengan nilai yang berbeda untuk parameter Purpose:

    aws cloudformation create-change-set --stack-name arn:aws:cloudformation:us-east-1:123456789012:stack/SampleStack/1a2345b6-0000-00a0-a123-00abc0abc000
--change-set-name SampleChangeSet --use-previous-template --parameters ParameterKey="InstanceType",UsePreviousValue=true ParameterKey="KeyPairName",UsePreviousValue=true ParameterKey="Purpose",ParameterValue="production"
