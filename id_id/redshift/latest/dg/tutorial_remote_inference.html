Tutorial: Membangun model inferensi jarak jauh

Tutorial berikut membahas langkah-langkah cara membuat model Random Cut Forest yang sebelumnya telah dilatih dan diterapkan di AmazonSageMaker, di luar Amazon Redshift. Algoritma Random Cut Forest mendeteksi titik data anomali dalam dataset. Membuat model dengan inferensi jarak jauh memungkinkan Anda membawa SageMaker model Random Cut Forest ke Amazon Redshift. Kemudian, di Amazon Redshift, Anda menggunakan SQL untuk melakukan prediksi pada titik akhir jarak jauhSageMaker.

Anda dapat menggunakan perintah CREATE MODEL untuk mengimpor model machine learning dari SageMaker titik akhir Amazon dan menyiapkan fungsi prediksi Amazon Redshift. Saat menggunakan operasi CREATE MODEL, Anda memberikan nama titik akhir model pembelajaran SageMaker mesin.

Dalam tutorial ini, Anda membuat model pembelajaran mesin Amazon Redshift menggunakan titik akhir SageMaker model. Setelah model machine learning Anda siap, Anda dapat menggunakannya untuk melakukan prediksi di Amazon Redshift. Pertama, Anda melatih dan membuat titik akhir di AmazonSageMaker, dan kemudian Anda mendapatkan nama endpoint. Kemudian, Anda menggunakan perintah CREATE MODEL untuk membuat model dengan Amazon Redshift HTML. Akhirnya, Anda melakukan prediksi pada model menggunakan fungsi prediksi yang dihasilkan perintah CREATE MODEL.

Contoh kasus penggunaan

Anda dapat menggunakan model Random Cut Forest dan inferensi jarak jauh untuk deteksi anomali di dataset lain, seperti memprediksi peningkatan atau penurunan transaksi e-commerce yang cepat. Anda juga bisa memprediksi perubahan signifikan dalam cuaca atau aktivitas seismik.

Langkah 1: Menerapkan model Amazon SageMaker

  1. Untuk menerapkan model, buka SageMaker konsol Amazon, pilih Instans Notebook di bawah Notebook di panel navigasi.

  2. Pilih Open Jupyter untuk notebook Jupyter yang dibuat oleh template. CloudFormation

  3. Pilih bring-your-own-model-remote-inference.ipynb.

  4. Siapkan parameter untuk menyimpan input dan output latihan di Amazon S3 dengan mengganti baris berikut dengan bucket dan awalan Amazon S3 Anda.

    data_location=f"s3://{bucket}/{prefix}/",
output_path=f"s3://{bucket}/{prefix}/output",

  5. Pilih tombol fast-forward untuk menjalankan semua sel.

Langkah 2: Dapatkan SageMaker endpoint model

Di SageMaker konsol Amazon, di bawah Inferensi di panel navigasi, pilih Endpoint dan temukan nama model Anda. Anda harus menyalin nama endpoint model Anda saat membuat model inferensi jarak jauh di Amazon Redshift.

Langkah 3: Muat data dari Amazon S3 ke Amazon Redshift

Gunakan editor kueri Amazon Redshift v2 untuk menjalankan perintah SQL berikut di Amazon Redshift. Perintah ini menjatuhkan rcf_taxi_data tabel jika ada, membuat tabel dengan nama yang sama, dan memuat kumpulan data sampel ke dalam tabel.

DROP TABLE IF EXISTS public.rcf_taxi_data CASCADE;

CREATE TABLE public.rcf_taxi_data (ride_timestamp timestamp, nbr_passengers int);

COPY public.rcf_taxi_data
FROM
    's3://sagemaker-sample-files/datasets/tabular/anomaly_benchmark_taxi/NAB_nyc_taxi.csv'
    IAM_ROLE default
    IGNOREHEADER 1
    FORMAT AS CSV;

Langkah 4: Membuat model dengan Amazon Redshift ML~

Jalankan kueri berikut untuk membuat model di Amazon Redshift MLmenggunakan titik akhir SageMaker model yang Anda dapatkan di langkah sebelumnya. Ganti randomcutforest-xxxxxxxxxdengan nama SageMaker endpoint Anda sendiri.

CREATE MODEL public.remote_random_cut_forest
FUNCTION remote_fn_rcf(int)
RETURNS decimal(10, 6) SAGEMAKER '<randomcutforest-xxxxxxxxx>' IAM_ROLE default;

Langkah 5: Lakukan prediksi dengan model

Algoritma Amazon SageMaker Random Cut Forest dirancang untuk mendeteksi titik data anomali dalam kumpulan data. Dalam contoh ini, model Anda dirancang untuk mendeteksi lonjakan dalam perjalanan taksi karena peristiwa penting. Anda dapat menggunakan model untuk memprediksi kejadian anomali dengan menghasilkan skor anomali untuk setiap titik data.

Gunakan kueri berikut untuk menghitung skor anomali di seluruh dataset taksi. Perhatikan bahwa Anda referensi fungsi yang Anda gunakan dalam pernyataan CREATE MODEL Anda pada langkah sebelumnya.

SELECT
    ride_timestamp,
    nbr_passengers,
    public.remote_fn_rcf(nbr_passengers) AS score
FROM
    public.rcf_taxi_data;

Periksa anomali tinggi dan rendah (opsional)

Jalankan kueri berikut untuk menemukan titik data dengan skor lebih besar dari tiga deviasi standar dari skor rata-rata.

WITH score_cutoff AS (
    SELECT
        STDDEV(public.remote_fn_rcf(nbr_passengers)) AS std,
        AVG(public.remote_fn_rcf(nbr_passengers)) AS mean,
        (mean + 3 * std) AS score_cutoff_value
    FROM
        public.rcf_taxi_data
)
SELECT
    ride_timestamp,
    nbr_passengers,
    public.remote_fn_rcf(nbr_passengers) AS score
FROM
    public.rcf_taxi_data
WHERE
    score > (
        SELECT
            score_cutoff_value
        FROM
            score_cutoff
    )
ORDER BY
    2 DESC;

Jalankan kueri berikut untuk menemukan titik data dengan skor lebih besar dari tiga deviasi standar dari skor rata-rata.

WITH score_cutoff AS (
    SELECT
        STDDEV(public.remote_fn_rcf(nbr_passengers)) AS std,
        AVG(public.remote_fn_rcf(nbr_passengers)) AS mean,
        (mean - 3 * std) AS score_cutoff_value
    FROM
        public.rcf_taxi_data
)
SELECT
    ride_timestamp,
    nbr_passengers,
    public.remote_fn_rcf(nbr_passengers) AS score
FROM
    public.rcf_taxi_data
WHERE
    score < (
        SELECT
            score_cutoff_value
        FROM
            score_cutoff
    )
ORDER BY
    2 DESC;
