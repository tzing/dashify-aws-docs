Penyiapan klaster untuk menggunakan Amazon Redshift

Sebelum Anda bekerja dengan Amazon Redshift, selesaikan prasyarat berikut ini.

Sebagai administrator Amazon Redshift, lakukan pengaturan satu kali berikut.

Untuk melakukan penyiapan klaster satu kali untuk Amazon Redshift

  1. Buat cluster Redshift menggunakan AWS Management Console or the AWS Command Line Interface ()AWS CLI. Pastikan untuk melampirkan kebijakan AWS Identity and Access Management (IAM) saat membuat klaster. Untuk informasi selengkapnya tentang izin yang diperlukan untuk menggunakan Amazon Redshift MLdengan AmazonSageMaker, lihat Izin yang diperlukan untuk menggunakan Amazon Redshift machine learning (MS) dengan Amazon. SageMaker

  2. Buat peran IAM yang diperlukan untuk menggunakan Amazon Redshift MLdengan salah satu cara berikut:

    • Operasi sederhana adalah membuat peran IAM dengan AmazonS3FullAccess dan AmazonSageMakerFullAccess kebijakan untuk digunakan dengan Amazon Redshift HTML. Jika Anda berencana untuk juga membuat model Prakiraan, lampirkan AmazonForecastFullAccess kebijakan tersebut ke peran Anda juga.

    • Kami menyarankan Anda membuat peran IAM melalui konsol Amazon Redshift yang memiliki AmazonRedshiftAllCommandsFullAccess kebijakan dengan izin untuk menjalankan perintah SQL, seperti CREATE MODEL. Amazon Redshift menggunakan mekanisme berbasis API yang mulus untuk membuat peran IAM secara terprogram atas nama Anda. Akun AWS Amazon Redshift secara otomatis melampirkan kebijakan AWS terkelola yang ada ke peran IAM. Pendekatan ini berarti Anda dapat tetap berada di dalam konsol Amazon Redshift dan tidak perlu beralih ke konsol IAM untuk pembuatan peran. Untuk informasi selengkapnya, lihat Membuat peran IAM sebagai default untuk Amazon Redshift.

      Ketika peran IAM dibuat sebagai default untuk klaster Anda, sertakan redshift sebagai bagian dari nama sumber daya atau gunakan tag khusus RedShift untuk menandai sumber daya tersebut.

      Jika klaster Anda telah meningkatkan perutean Amazon VPC diaktifkan, Anda dapat menggunakan peran IAM yang dibuat melalui konsol Amazon Redshift. Peran IAM ini memiliki AmazonRedshiftAllCommandsFullAccess kebijakan yang dilampirkan dan menambahkan izin berikut ke kebijakan. Izin tambahan ini memungkinkan Amazon Redshift untuk membuat dan menghapus antarmuka jaringan elastis (ENI) di akun Anda dan melampirkannya ke tugas kompilasi yang berjalan di Amazon EC2 atau Amazon ECS. Melakukan hal ini memungkinkan objek di bucket Amazon S3 Anda diakses hanya dari dalam cloud pribadi virtual (VPC) dengan akses internet diblokir.

  3. Untuk mengizinkan Amazon Redshift dan SageMaker mengambil peran untuk berinteraksi dengan layanan lain, tambahkan kebijakan kepercayaan berikut ke peran IAM.

    {
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {
        "Service": [
          "redshift.amazonaws.com",
          "sagemaker.amazonaws.com",
          "forecast.amazonaws.com"
        ]
      },
      "Action": "sts:AssumeRole"
    }
  ]
}

  4. (Opsional) Buat bucket Amazon S3 dan AWS KMS kunci. Ini digunakan Amazon Redshift untuk menyimpan data pelatihan yang dikirim ke Amazon SageMaker dan menerima model terlatih dari AmazonSageMaker.

  5. (Opsional) Buat kombinasi peran IAM dan bucket Amazon S3 yang berbeda untuk mengontrol akses ke grup pengguna yang berbeda.

  6. (Opsional) Saat Anda mengaktifkan perutean VPC untuk klaster Redshift Anda, buat titik akhir Amazon S3 dan titik akhir untuk VPC tempat SageMaker klaster Redshift Anda berada. Melakukan hal ini memungkinkan lalu lintas untuk berjalan melalui VPC Anda antara layanan selama CREATE MODEL. Untuk informasi selengkapnya tentang perutean VPC, lihat Perutean VPC yang Ditingkatkan di Amazon Redshift.

Mengelola izin dan kepemilikan

Sama seperti objek database lainnya, seperti tabel atau fungsi, Amazon Redshift mengikat pembuatan dan penggunaan model MLuntuk mengakses mekanisme kontrol. Ada izin terpisah untuk membuat model yang menjalankan fungsi prediksi.

Contoh berikut menggunakan dua grup pengguna, retention_analyst_grp (pembuat model) dan marketing_analyst_grp (pengguna model) untuk menggambarkan cara Amazon Redshift mengelola kontrol akses. Analis retensi membuat model pembelajaran mesin yang dapat digunakan oleh pengguna lain melalui izin yang diperoleh.

Superuser dapat MEMBERIKAN USER atau GROUP izin untuk membuat model machine learning menggunakan pernyataan berikut.

GRANT CREATE MODEL TO GROUP retention_analyst_grp;

Pengguna atau grup dengan izin ini dapat membuat model dalam skema apa pun di cluster jika pengguna memiliki izin CREATE biasa pada SCHEMA. Model pembelajaran mesin adalah bagian dari hirarki skema dengan cara yang mirip dengan tabel, tampilan, prosedur, dan fungsi yang ditentukan pengguna.

Dengan asumsi skema demo_ml sudah ada, memberikan dua kelompok pengguna izin pada skema sebagai berikut.

GRANT CREATE, USAGE ON SCHEMA demo_ml TO GROUP retention_analyst_grp;
GRANT USAGE ON SCHEMA demo_ml TO GROUP marketing_analyst_grp;

Agar pengguna lain dapat menggunakan fungsi inferensi pembelajaran mesin Anda, berikan izin EXECUTE. Contoh berikut menggunakan izin EXECUTE untuk memberikan marketing_analyst_grp GROUP izin untuk menggunakan model.

GRANT EXECUTE ON MODEL demo_ml.customer_churn_auto_model TO GROUP marketing_analyst_grp;

Gunakan pernyataan REVOKE dengan CREATE MODEL dan EXECUTE untuk mencabut izin tersebut dari pengguna atau grup. Untuk informasi lebih lanjut tentang perintah kontrol izin, lihat HIBAH danMENCABUT.
