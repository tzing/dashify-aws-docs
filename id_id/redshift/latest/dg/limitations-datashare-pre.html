Batasan untuk berbagi data (pratinjau)

Ini adalah dokumentasi prarilis untuk gudang multi-data yang ditulis melalui fitur berbagi data untuk Amazon Redshift, yang tersedia dalam pratinjau publik di trek PREVIEW_2023. Dokumentasi dan fitur dapat berubah. Kami menyarankan Anda menggunakan fitur ini hanya dengan cluster pengujian, dan bukan di lingkungan produksi. Untuk syarat dan ketentuan pratinjau, lihat Partisipasi Layanan Beta dalam Ketentuan AWS Layanan.

Jika Anda belum membuat datashare di trek pratinjau, buka Berbagi data baca dan tulis dalam AWS akun atau di seluruh akun untuk memulai.

Berikut ini adalah batasan saat bekerja dengan datashares di Redshift:

  • Penulisan multi-gudang melalui berbagi data saat ini tidak tersedia di klaster ra3.xlplus. Untuk menggunakan fitur ini, buat cluster dengan tipe node berikut:

    • ra3.4xl

    • ra3.16xl

    Ini juga berfungsi dengan grup kerja Amazon Redshift Serverless dengan trek pratinjau.

  • Untuk berbagi data lintas akun dan lintas wilayah, baik cluster produsen dan konsumen serta ruang nama tanpa server harus dienkripsi. Ini untuk tujuan keamanan. Namun, mereka tidak perlu berbagi kunci enkripsi yang sama.

  • Anda hanya dapat berbagi SQL UDF melalui datashares. UDF Python dan Lambda tidak didukung.

  • Jika database produsen memiliki pemeriksaan khusus, gunakan pengaturan pemeriksaan yang sama untuk database konsumen.

  • Redshift tidak mendukung penambahan skema eksternal, tabel, atau tampilan pengikatan akhir pada tabel eksternal ke datashares.

  • Redshift tidak mendukung fungsi SQL yang ditentukan pengguna bersarang pada kluster produsen.

  • Redshift tidak mendukung berbagi tabel dengan tombol sortir interleaved dan tampilan yang merujuk ke tabel dengan tombol sortir interleaved.

  • Konsumen tidak dapat menambahkan objek datashare ke datashare lain. Selain itu, konsumen tidak dapat menambahkan tampilan yang mereferensikan objek datashare ke datashare lain.

  • Redshift tidak mendukung mengakses objek datashare yang memiliki DDL bersamaan terjadi antara Prepare dan Execute dari akses.

  • Redshift tidak mendukung berbagi prosedur tersimpan melalui datashares.
