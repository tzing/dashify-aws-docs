Tipe data interval

Anda dapat menggunakan tipe data interval untuk menentukan periode atau rentang waktu, seperti12 hours,6 weeks, atau1 month. Jenis data INTERVAL memungkinkan Anda menentukan periode atau rentang waktu, seperti12 hours,6 weeks, atau1 month. Anda dapat menggunakan tipe data interval dalam perhitungan, seperti menghitung perbedaan antara tanggal-waktu atau interval, jumlah interval, atau jumlah datetime dan interval.

Interval berikut didukung:

  • TAHUN KE BULAN

  • HARI KE DETIK

Sintaks

Untuk membuat interval tipe TAHUN KE BULAN:

INTERVAL <year_to_month_qualifier>

Untuk membuat interval literal tipe TAHUN KE BULAN:

INTERVAL <interval-string> <qualifier>

Gunakan sintaks berikut untuk membuat interval tipe DAY TO SECOND. fractional_precisionParameter hanya diperlukan jika interval Anda berisi SECOND.

INTERVAL <qualifier> [ <fractional_precision> ]

Gunakan sintaks berikut untuk membuat interval literal tipe DAY TO SECOND.

INTERVAL <interval-string> <qualifier> [ <fractional_precision> ]

Argumen

Interval-string

Menentukan nilai interval. Hanya diperlukan jika membuat konstanta interval

Kualifikasi

Menentukan rentang interval. Jika Anda menggunakan qualifier dan membuat interval dengan satuan waktu yang lebih kecil dari qualifier, Amazon Redshift memotong dan membuang bagian interval yang lebih kecil. Contoh berikut menunjukkan menggunakan kualifikasi MINUTE, dan Amazon Redshift membuang unit waktu yang lebih kecil dari MINUTE.

INTERVAL '1 day 1 hour 1 minute 1.123 seconds' MINUTE

akan terpotong untuk 

'1 day 01:01:00'
Fraksional_presisi

Parameter opsional yang menentukan jumlah digit fraksional yang diizinkan dalam interval. Misalnya, SECOND (3) menciptakan interval yang memungkinkan hanya tiga digit pecahan, seperti 1,234 detik. Hanya berlaku untuk interval HARI KE KEDUA yang berisi detik. Jumlah maksimum digit yang didukung adalah enam.

Tipe data YEAR TO MONTH mendukung kualifikasi berikut:

  • YEAR

  • MONTH

  • TAHUN KE BULAN

Tipe data DAY TO SECOND mendukung kualifikasi berikut:

  • DAY

  • HOUR

  • MINUTE

  • DETIK

  • HARI KE JAM

  • HARI KE MENIT

  • HARI KE DETIK

  • JAM KE MENIT

  • JAM KE DETIK

  • MENIT KE DETIK

Contoh-contoh

Contoh berikut menunjukkan cara membuat konstanta INTERVAL 1 tahun.

INTERVAL '1' YEAR

Jika Anda menentukan string interval yang melebihi kualifikasi, satuan waktu yang tersisa dipotong dari interval. Dalam contoh berikut, interval 13 bulan menjadi 1 tahun dan 1 bulan, tetapi sisanya 1 bulan ditinggalkan karena kualifikasi YEAR.

INTERVAL '13 months' YEAR
'1 year'

Jika Anda menggunakan qualifier yang lebih rendah dari string interval Anda, unit sisa disertakan.

INTERVAL '13 months' MONTH
'1 year 1 month'

Menentukan presisi dalam interval Anda memotong jumlah digit pecahan ke presisi yang ditentukan.

INTERVAL '1.234567' SECOND (3)
'00:00:01.234'

Jika Anda tidak menentukan presisi, Amazon Redshift menggunakan presisi maksimum 6.

INTERVAL '1.23456789' SECOND
'00:00:01.234567'

Contoh berikut menunjukkan cara membuat interval berkisar.

INTERVAL '2:2' MINUTE TO SECOND
'00:02:02'

Kualifikasi mendikte unit yang Anda tentukan. Misalnya, meskipun contoh berikut menggunakan string interval yang sama dengan '2:2' seperti contoh sebelumnya, Amazon Redshift mengenalinya menggunakan satuan waktu yang berbeda karena qualifier

INTERVAL '2:2' HOUR TO MINUTE
'02:02:00'

Singkatan dan bentuk jamak dari setiap unit juga didukung; misalnya:5s,5 second, dan 5 seconds merupakan interval yang setara. Unit yang didukung adalah tahun, bulan, jam, menit, dan detik.

INTERVAL '5s' SECOND
'00:00:05'

INTERVAL '5 second' SECOND
'00:00:05'
                     
INTERVAL '5 HOURS' HOUR
'05:00:00'
                     
INTERVAL '5 h' HOUR
'05:00:00

Contoh berikut menunjukkan bagaimana menggunakan tipe data INTERVAL dengan tabel

create table sample_intervals (y2m interval month, m2s interval hour to minute);
insert into sample_intervals values (interval '20' month, interval '2 days 1:1:1.123456' day to second);
select y2m::text, m2s::text from sample_intervals;

      y2m      |       m2s       
---------------+-----------------
 1 year 8 mons | 2 days 01:01:00

update sample_intervals set y2m = interval '2' year where y2m = interval '1-8' year to month;
select * from sample_intervals;

   y2m   |       m2s       
---------+-----------------
 2 years | 2 days 01:01:00

delete from sample_intervals where m2s = interval '2 1:1:0' day to second;
select * from sample_intervals;

 y2m | m2s 
-----+-----

Gaya interval

Anda dapat menggunakan SET perintah untuk mengubah format output string interval Anda. Pilihan yang tersedia adalah:postgres,sql_standard, danpostgres_verbose. Default-nya adalah postgres. Contoh dari setiap format output ada di bawah ini.

format keluaran sql_standard

Nilai interval tahun ke bulan diformat sebagai berikut. Menentukan negatif sebelum interval menunjukkan interval adalah nilai negatif dan berlaku untuk seluruh interval.

'[-]yy-mm'

Interval hari ke nilai kedua diformat sebagai berikut.

'[-]dd hh:mm:ss.ffffff'
INTERVAL '1-2' YEAR TO MONTH outputs as '1-2'
                     
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 2:03:04.5678'

format keluaran postgres

Berikut ini adalah format output untuk postgres. Setiap nilai numerik bisa negatif.

[@] '<numeric> <unit> [, <numeric> <unit> ...]'
INTERVAL '1-2' YEAR TO MONTH outputs as '1 year 2 mons'
                  
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 day 02:03:04.5678'

format keluaran postgres_verbose

sintaks postgres_verbose mirip dengan postgres, tetapi output postgres_verbose juga berisi satuan waktu.

'[@] <numeric> <unit> [, <numeric> <unit> ...] [direction]'
INTERVAL '1-2' YEAR TO MONTH outputs as '@ 1 year 2 mons'

INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '@ 1 day 2 hours 3 mins 4.5678 secs'

Perintah berikut menunjukkan cara mengatur gaya interval kesql_standard.

SET intervalstyle to 'sql_standard';

Batasan

Fitur INTERVAL prarilis ini memiliki batasan sebagai berikut:

  • Jika Anda menggunakan driver JDBC, Anda tidak dapat menggunakan interval tahun ke bulan dan hari ke kedua sebagai parameter untuk MEMPERSIAPKAN pernyataan.

  • Driver JDBC menafsirkan semua nilai interval sebagai string, sehingga Anda hanya bisa mendapatkan interval yang dikembalikan dari kueri SELECT dengan menggunakan metode ini. getString

  • Prosedur Tersimpan tidak menerima nilai interval sebagai parameter fungsi.

  • Anda tidak dapat melakukan cast numerik ke interval.
