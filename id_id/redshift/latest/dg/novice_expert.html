Pembelajaran mesin untuk pemula dan ahli

Amazon Redshift MLmemungkinkan Anda melatih model dengan satu perintah SQL CREATE MODEL. Perintah CREATE MODEL membuat model yang digunakan Amazon Redshift untuk menghasilkan prediksi berbasis model dengan konstruksi SQL yang sudah dikenal.

Amazon Redshift MLsangat berguna jika Anda tidak memiliki keahlian dalam pembelajaran mesin, alat, bahasa, algoritme, dan API. Dengan Amazon Redshift, Anda tidak perlu melakukan pengangkatan berat yang tidak berbeda yang diperlukan untuk mengintegrasikan dengan layanan machine learning eksternal. Amazon Redshift menghemat waktu Anda untuk memformat dan memindahkan data, mengelola kontrol izin, atau membangun integrasi kustom, alur kerja, dan skrip. Anda dapat dengan mudah menggunakan algoritma pembelajaran mesin populer dan menyederhanakan kebutuhan pelatihan yang memerlukan iterasi yang sering dari pelatihan ke prediksi. Amazon Redshift secara otomatis menemukan algoritme terbaik dan menyetel model terbaik untuk masalah Anda. Anda dapat membuat prediksi dari dalam klaster Amazon Redshift tanpa perlu memindahkan data keluar dari Amazon Redshift atau untuk berinteraksi dengan dan membayar layanan lain.

Amazon Redshift MLmendukung analis data dan ilmuwan data dalam menggunakan machine learning. Ini juga memungkinkan para ahli pembelajaran mesin untuk menggunakan pengetahuan mereka untuk memandu pernyataan CREATE MODEL untuk hanya menggunakan aspek yang mereka tentukan. Dengan demikian, Anda dapat mempercepat waktu yang dibutuhkan CREATE MODEL untuk menemukan kandidat terbaik, meningkatkan akurasi model, atau keduanya.

Pernyataan CREATE MODEL menawarkan fleksibilitas dalam bagaimana Anda dapat menentukan parameter untuk pekerjaan pelatihan. Dengan menggunakan fleksibilitas ini, baik pemula atau ahli pembelajaran mesin dapat memilih preprosesor, algoritme, jenis masalah, dan hiperparameter pilihan mereka. Misalnya, pengguna yang tertarik pada churn pelanggan mungkin menentukan untuk pernyataan CREATE MODEL bahwa jenis masalah adalah klasifikasi biner, yang bekerja dengan baik untuk churn pelanggan. Kemudian pernyataan CREATE MODEL mempersempit pencariannya untuk model terbaik ke dalam model klasifikasi biner. Bahkan dengan pilihan pengguna dari jenis masalah, masih ada banyak pilihan bahwa pernyataan CREATE MODEL dapat bekerja dengan. Misalnya, CREATE MODEL menemukan dan menerapkan transformasi preprocessing terbaik dan menemukan pengaturan hyperparameter terbaik.

Amazon Redshift MLmemudahkan latihan dengan secara otomatis menemukan model terbaik menggunakan Amazon SageMaker Autopilot. Di belakang layar, Amazon SageMaker Autopilot secara otomatis melatih dan menyetel model machine learning terbaik berdasarkan data yang Anda berikan. Amazon SageMaker Neo kemudian mengkompilasi model pelatihan dan membuatnya tersedia untuk prediksi di klaster Redshift Anda. Saat Anda menjalankan kueri inferensi machine learning menggunakan model terlatih, kueri dapat menggunakan kemampuan pemrosesan paralel secara besar-besaran dari Amazon Redshift. Pada saat yang sama, kueri dapat menggunakan prediksi berbasis machine learning.

  • Sebagai pemula pembelajaran mesin, dengan pengetahuan umum tentang berbagai aspek pembelajaran mesin seperti preprosesor, algoritma, dan hiperparameter, gunakan pernyataan CREATE MODEL hanya untuk aspek yang Anda tentukan. Kemudian Anda dapat mempersingkat waktu yang dibutuhkan CREATE MODEL untuk menemukan kandidat terbaik atau meningkatkan akurasi model. Selain itu, Anda dapat meningkatkan nilai bisnis prediksi dengan memperkenalkan pengetahuan domain tambahan seperti jenis masalah atau tujuan. Misalnya, dalam skenario churn pelanggan, jika hasil “pelanggan tidak aktif” jarang terjadi, maka tujuan F1 sering lebih disukai daripada tujuan Akurasi. Karena model Akurasi tinggi mungkin memprediksi “pelanggan aktif” sepanjang waktu, ini menghasilkan akurasi tinggi tetapi sedikit nilai bisnis. Untuk informasi tentang tujuan F1, lihat AutoKL JobObjective di Referensi SageMaker API Amazon.

    Untuk informasi lebih lanjut tentang pilihan dasar untuk pernyataan CREATE MODEL, lihatModel Buat Sederhana.

  • Sebagai praktisi tingkat lanjut pembelajaran mesin, Anda dapat menentukan jenis masalah dan preprosesor untuk fitur tertentu (tetapi tidak semua). Kemudian CREATE MODEL mengikuti saran Anda pada aspek yang ditentukan. Pada saat yang sama, CREATE MODEL masih menemukan preprocessors terbaik untuk fitur yang tersisa dan hyperparameter terbaik. Untuk informasi lebih lanjut tentang bagaimana Anda dapat membatasi satu atau lebih aspek dari pipeline pelatihan, lihatBUAT MODEL dengan panduan pengguna.

  • Sebagai ahli pembelajaran mesin, Anda dapat mengendalikan penuh pelatihan dan penyetelan hiperparameter. Kemudian pernyataan CREATE MODEL tidak mencoba untuk menemukan preprocessors optimal, algoritma, dan hyperparameters karena Anda membuat semua pilihan. Untuk informasi selengkapnya tentang cara menggunakan CREATE MODEL dengan AUTO OFF, lihatBUAT model XGBoost dengan AUTO OFF.

  • Sebagai teknisi data, Anda dapat membawa model XGBoost terlatih sebelumnya di Amazon SageMaker dan mengimpornya ke Amazon Redshift untuk inferensi lokal. Dengan membawa model Anda sendiri (BYOM), Anda dapat menggunakan model yang dilatih di luar Amazon Redshift dengan Amazon SageMaker untuk inferensi dalam database secara lokal di Amazon Redshift. Amazon Redshift MLmendukung penggunaan BYOM baik dalam inferensi lokal maupun jarak jauh.

    Untuk informasi selengkapnya tentang cara menggunakan pernyataan CREATE MODEL untuk inferensi lokal atau jarak jauh, lihatBawa model Anda sendiri (BYOM) - inferensi lokal.

Sebagai pengguna Amazon Redshift, Anda dapat memilih salah satu opsi berikut untuk melatih dan menerapkan model Anda:

  • Jenis masalah, lihatBUAT MODEL dengan panduan pengguna.

  • Tujuan, melihat BUAT MODEL dengan panduan pengguna atauBUAT model XGBoost dengan AUTO OFF.

  • Jenis model, lihatBUAT model XGBoost dengan AUTO OFF.

  • Preprocessors, lihatBUAT MODEL dengan panduan pengguna.

  • Hyperparameter, lihatBUAT model XGBoost dengan AUTO OFF.

  • Bawa model Anda sendiri (BYOM), lihat. Bawa model Anda sendiri (BYOM) - inferensi lokal
