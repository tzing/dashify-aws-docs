데이터 공유 제한 사항(미리 보기)

이 문서는 Amazon Redshift의 데이터 공유 기능을 통한 다중 데이터 웨어하우스 쓰기에 대한 사전 릴리스 설명서입니다. 이 기능은 PREVIEW_2023 트랙에서 공개 미리 보기로 제공됩니다. 설명서 및 기능은 모두 변경될 수 있습니다. 이 기능은 테스트 클러스터에만 사용하고 프로덕션 환경에서는 사용하지 않는 것이 좋습니다. 미리 보기 이용 약관은 AWS Service Terms의 Beta Service Participation을 참조하세요.

미리 보기 트랙에서 아직 데이터 공유를 생성하지 않은 경우 한 AWS 내에서 또는 여러 계정 간에 읽기 및 쓰기 데이터 공유(미리 보기)로 이동하여 시작하세요.

다음은 Redshift에서 데이터 공유 작업 시 제한 사항입니다.

  • 현재 ra3.xlplus 클러스터에서는 데이터 공유를 통한 다중 웨어하우스 쓰기를 사용할 수 없습니다. 이 기능을 사용하려면 다음 노드 유형으로 클러스터를 생성하세요.

    • ra3.4xl

    • ra3.16xl

    또한 프리뷰 트랙이 있는 Amazon Redshift Serverless 작업 그룹과도 호환됩니다.

  • 계정 간 또는 리전 간 데이터 공유의 경우 생산자 및 소비자 클러스터와 서버리스 네임스페이스를 모두 암호화해야 합니다. 이는 보안을 위한 것입니다. 하지만 동일한 암호화 키를 공유할 필요는 없습니다.

  • 데이터 공유를 통해서만 SQL UDF를 공유할 수 있습니다. Python 및 Lambda UDF는 지원되지 않습니다.

  • 생산자 데이터베이스에 특정 데이터 정렬이 있는 경우 소비자 데이터베이스에 동일한 데이터 정렬 설정을 사용합니다.

  • Redshift는 데이터 공유에 외부 스키마, 테이블, 지연 바인딩 뷰 또는 외부 테이블 추가를 지원하지 않습니다.

  • Redshift는 생산자 클러스터에서 중첩된 SQL 사용자 정의 함수를 지원하지 않습니다.

  • Redshift는 인터리브 정렬 키가 있는 테이블과 이러한 테이블을 참조하는 뷰 공유를 지원하지 않습니다.

  • 소비자는 데이터 공유 객체를 다른 데이터 공유에 추가할 수 없습니다. 또한 소비자는 데이터 공유 객체를 참조하는 뷰를 다른 데이터 공유에 추가할 수 없습니다.

  • Redshift는 액세스 준비와 실행 간에 동시 DDL이 발생한 데이터 공유 객체에 대한 액세스를 지원하지 않습니다.

  • Redshift는 데이터 공유를 통한 저장 프로시저 공유를 지원하지 않습니다.

데이터 공유 제한 사항에 대한 자세한 내용은 미리 보기 상태에서 데이터 공유 시 요구 사항 및 제한 사항 섹션을 참조하세요.
