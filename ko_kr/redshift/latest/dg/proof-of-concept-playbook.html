Amazon Redshift에 대한 개념 증명 수행

Amazon Redshift는 빠르고 확장 가능한 데이터 웨어하우스로, 간편하고 비용 효율적으로 모든 데이터를 기존 비즈니스 인텔리전스(BI) 도구와 표준 SQL을 사용하여 분석할 수 있게 해줍니다. Amazon Redshift는 저렴한 클라우드 데이터 웨어하우스에서 빠른 성능을 제공합니다. 정교한 쿼리 최적화, 가속화된 캐시, 고성능 로컬 디스크의 컬럼 방식 스토리지, 대량 병렬 쿼리 실행이 사용됩니다.

다음 섹션에서는 Amazon Redshift에서 개념 증명을 구축하기 위한 프레임워크를 확인할 수 있습니다. 이 프레임워크는 안정적이고 경제적인 고성능 데이터 웨어하우스를 설계하고 운영하기 위한 아키텍처 모범 사례를 사용하는 데 도움이 됩니다. 이 지침은 광범위한 업종과 사용 사례에 대한 수천 여 고객 아키텍처 설계를 검토하여 작성되었습니다. 데이터 웨어하우스 워크로드 평가 기준을 만드는 데 도움이 되도록 고객 경험을 한 곳에 모아 이 모범 사례를 개발했습니다.

프로세스 개요

개념 증명을 수행하는 것은 3단계 프로세스입니다.

  1. 개념 증명 목표 파악 - 비즈니스 요구 사항 및 성공 기준에서 거꾸로 작업하고 이를 기술 개념 증명 프로젝트 계획으로 변환할 수 있습니다.

  2. 개념 증명 환경 설정 - 대부분의 설정 프로세스는 리소스를 생성하기 위해 몇 개의 버튼을 클릭하는 것입니다. 몇 분 안에 데이터가 로드된 데이터 웨어하우스 환경을 준비할 수 있습니다.

  3. 목표가 충족되었는지 확인하기 위해 개념 증명 프로젝트 계획을 완료합니다.

다음 섹션에서 각 단계를 자세히 살펴봅니다.

비즈니스 목표 및 성공 기준 파악

개념 증명의 목표 식별은 평가 프로세스의 일부로 측정할 항목을 결정하는 데 중요한 역할을 합니다. 평가 기준에는 현재 조정 문제, 데이터 웨어하우스 고객 경험 개선을 위한 기능 향상, 현재 운영상 문제를 해결하는 방법이 포함되어야 합니다. 다음 질문을 사용하여 개념 증명 목표를 확인할 수 있습니다.

  • 데이터 웨어하우스를 조정하는 목적은 무엇입니까?

  • 조건을 개선하려는 특정 서비스 수준 계약은 무엇인가요?

  • 데이터 웨어하우스에 포함해야 하는 새 데이터 집합은 무엇인가요?

  • 테스트 및 측정이 필요한 비즈니스에 중요한 SQL 쿼리는 무엇인가요? 선택, 삽입, 업데이트, 삭제 등 다양한 쿼리를 비롯한 전체 SQL 복잡성을 포함해야 합니다.

  • 테스트할 일반 워크로드 유형은 무엇입니까? 예를 들면 추출, 변환, 로드(ETL) 워크로드, 보고 쿼리, 배치 추출 등이 있습니다.

위의 질문에 답을 했다면 개념 증명을 구축하기 위한 SMART 목표와 성공 기준을 설정할 수 있습니다. 목표 설정에 대한 자세한 내용은 Wikipedia의 SMART criteria 섹션을 참조하세요.

개념 증명 설정

온프레미스 데이터 웨어하우스에서 하드웨어 프로비저닝, 네트워킹 및 소프트웨어 설치를 제거했기 때문에 자체 데이터 집합으로 Amazon Redshift를 사용해 보는 것이 그 어느 때보다 쉬워졌습니다. 이전에는 필요했던 많은 크기 조정 결정과 추정이 이제는 클릭 한 번이면 됩니다. 클러스터 크기를 유연하게 조정하거나 스토리지와 컴퓨팅의 비율을 조정할 수 있습니다.

대체로 Amazon Redshift 개념 증명 환경을 설정하는 것은 2단계 프로세스입니다. 여기에는 데이터 웨어하우스를 시작한 다음 평가를 위해 스키마와 데이터 집합을 변환하는 작업이 포함됩니다.

시작 클러스터 크기 선택

Amazon Redshift 콘솔을 사용하여 노드 유형과 노드 수를 선택할 수 있습니다. 개념 증명 계획의 일부로 클러스터 크기 조정도 테스트하는 것이 좋습니다. 클러스터의 초기 크기 조정을 수행하려면 다음 단계를 따릅니다.

  1. AWS Management Console에 로그인한 후 https://console.aws.amazon.com/redshift/에서 Amazon Redshift 콘솔을 엽니다.

  2. 탐색 메뉴에서 클러스터 생성(Create cluster)을 클릭하여 구성 페이지를 엽니다.

  3. [클러스터 식별자(Cluster identifier)]에 클러스터 이름을 입력합니다.

  4. 다음 단계에서는 RA3 노드 유형을 지원하는 AWS 리전에서 실행되는 Amazon Redshift 콘솔에 대해 설명합니다. RA3 노드 유형을 지원하는 AWS 리전 목록은 Amazon Redshift 관리 가이드의 RA3 노드 유형 개요를 참조하세요.

    클러스터 크기를 얼마로 조정해야 할지 모르는 경우 선택 도움말(Help me choose)을 선택합니다. 이렇게 하면 데이터 웨어하우스에 저장하려는 데이터의 크기 및 쿼리 특성에 대한 질문을 하는 크기 조정 계산기가 시작됩니다. 클러스터의 필요한 크기(즉, 노드 유형 및 노드 수)를 알고 있는 경우 [직접 선택(I'll choose)]을 선택합니다. 그런 다음 개념 증명을 위해 클러스터 크기를 조정할 [노드 유형(Node type)]과 [노드(Nodes)] 수를 선택합니다.

    참고

    조직에 자격이 있으며 Amazon Redshift Serverless를 사용할 수 없는 AWS 리전에 클러스터가 생성되고 있는 경우, Amazon Redshift 무료 평가판 프로그램에서 클러스터를 생성하는 것이 가능할 수 있습니다. 이 클러스터를 어떤 용도로 사용할 계획입니까?라는 질문에 대해 프로덕션 또는 무료 평가판을 선택하여 답합니다. 무료 평가판을 선택하면 dc2.large 노드 유형으로 구성을 생성합니다. 무료 평가판 선택에 대한 자세한 내용은 Amazon Redshift 무료 평가판을 참조하세요. Amazon Redshift Serverless를 사용할 수 있는 AWS 리전 목록은 Amazon Web Services 일반 참조의 Redshift Serverless API에 나열된 엔드포인트를 참조하세요.

  5. 필요한 모든 클러스터 속성을 입력한 후 [클러스터 생성(Create cluster)]을 선택하여 데이터 웨어하우스를 시작합니다.

    Amazon Redshift 콘솔을 사용한 클러스터 생성에 대한 자세한 내용은 Amazon Redshift 관리 가이드의 클러스터 생성 섹션을 참조하세요.

스키마 변환 및 개념 증명을 위한 데이터 집합 설정

기존 데이터 웨어하우스가 없는 경우 이 섹션을 건너뛰고 Amazon Redshift 시작 안내서를 참조하세요. Amazon Redshift 시작 안내서에서는 클러스터 생성을 위한 튜토리얼과 Amazon Redshift에서 데이터를 설정하는 예를 제공합니다.

기존 데이터 웨어하우스에서 마이그레이션할 때 AWS Schema Conversion Tool 및 AWS Database Migration Service를 사용하여 스키마, 코드 및 데이터를 변환할 수 있습니다. 도구 선택은 데이터 원본과 지속적인 복제(옵션)에 따라 다릅니다. 자세한 내용은 AWS Schema Conversion Tool User Guide의 What Is the AWS Schema Conversion Tool?와 AWS Database Migration Service User Guide의 What Is AWS Database Migration Service? 섹션을 참조하세요. 다음은 Amazon Redshift에서 데이터를 설정하는 데 도움이 됩니다.

  • AWS Schema Conversion Tool를 사용하여 Amazon Redshift로 데이터 웨어하우스 마이그레이션 – 이 블로그에서는 AWS SCT 데이터 추출기를 사용하여 Amazon Redshift로 기존 데이터 웨어하우스를 마이그레이션하는 방법을 간략히 설명합니다. AWS SCT 도구는 많은 레거시 플랫폼(예: Oracle, Greenplum, Netezza, Teradata, Microsoft SQL Server 또는 Vertica)에서 데이터를 마이그레이션할 수 있습니다.

  • 소스에서 변경된 데이터의 지속적인 복제에 AWS Database Migration Service를 사용할 수도 있습니다. 자세한 내용은 AWS Database Migration Service User Guide의 Using an Amazon Redshift Database as a Target for AWS Database Migration Service 섹션을 참조하세요.

Amazon Redshift는 관계형 데이터베이스 관리 시스템(RDBMS)입니다. 따라서 스타 스키마, 눈송이 스키마, 데이터 볼트 모델, 단순, 플랫 또는 정규화된 테이블을 비롯한 다양한 유형의 데이터 모델을 실행할 수 있습니다. Amazon Redshift에서 스키마를 설정한 후 즉시 빠른 분석 쿼리를 위해 대규모 병렬 처리와 열 형식 데이터 스토리지를 활용할 수 있습니다. 스키마 유형에 대한 자세한 내용은 Wikipedia의 star schema, snowflake schema 및 data vault modeling을 참조하세요.

완전한 평가를 위한 체크리스트

완전한 평가가 모든 데이터 웨어하우스 요구 사항을 충족하는지 확인합니다. 성공 기준에 다음 항목을 포함하는 것이 좋습니다.

  • 데이터 로드 시간 – COPY 명령 사용은 데이터를 로드하는 데 걸리는 시간을 테스트하는 데 사용되는 일반적인 방법입니다. 자세한 내용은 데이터 로드에 대한 Amazon Redshift 모범 사례 섹션을 참조하세요.

  • 클러스터의 처리량 – 시간당 쿼리 측정은 처리량을 결정하는 일반적인 방법입니다. 이렇게 하려면 워크로드에 대한 일반 쿼리를 실행하도록 테스트를 설정합니다.

  • 데이터 보안 – Amazon Redshift에서 저장된 데이터와 전송 중인 데이터를 쉽게 암호화할 수 있습니다. 또한 키 관리를 위한 여러 옵션이 있습니다. Amazon Redshift는 Single Sign-On 통합도 지원합니다. Amazon Redshift 요금에는 기본 제공 보안, 데이터 압축, 백업 스토리지 및 데이터 전송이 포함됩니다.

  • 서드 파티 도구 통합 – JDBC 또는 ODBC 연결을 사용하여 비즈니스 인텔리스 및 기타 외부 도구를 통합할 수 있습니다.

  • 다른 AWS 서비스와의 상호 운용성 – Amazon Redshift는 Amazon EMR, Amazon QuickSight, AWS Glue, Amazon S3, Amazon Kinesis 등의 다른 AWS 서비스와 통합됩니다. 데이터 웨어하우스를 설정하고 관리할 때 이 통합을 사용할 수 있습니다.

  • 백업 및 스냅샷 - 백업 및 스냅샷이 자동으로 생성됩니다. 언제든지 또는 일정에 따라 특정 시점 스냅샷을 생성할 수도 있습니다. 스냅샷을 사용하여 평가의 일부로 두 번째 클러스터를 생성해 봅니다. 개발 및 테스트 조직에서 클러스터를 사용할 수 있는지 평가합니다.

  • 크기 조정 – 점점 더 많은 수 또는 유형의 Amazon Redshift 노드가 평가에 포함되어야 합니다. 크기 조정 전후의 워크로드 처리량이 워크로드 볼륨의 가변성을 충족하는지 평가합니다. 자세한 내용은 Amazon Redshift 관리 가이드의 Amazon Redshift에서 클러스터 크기 조정 섹션을 참조하세요.

  • 동시성 조정 - 이 기능은 데이터 웨어하우스에서 트래픽 볼륨의 가변성을 처리하는 데 도움이 됩니다. 동시성 조정을 사용하면 일관성 있게 빠른 쿼리 성능으로 동시 사용자 및 동시 쿼리를 사실상 무제한으로 지원할 수 있습니다. 자세한 내용은 동시성 확장 작업 섹션을 참조하세요.

  • 자동 워크로드 관리(WLM) - 자동 WLM을 사용하여 다른 쿼리보다 비즈니스 크리티컬 쿼리에 우선 순위를 매깁니다. 워크로드에 따라 대기열을 설정해 봅니다(예: ETL 대기열 및 보고 대기열). 그런 다음 자동 WLM을 사용하여 동시성 및 메모리 리소스를 동적으로 할당합니다. 자세한 내용은 자동 WLM 구현 섹션을 참조하세요.

  • Amazon Redshift Advisor – Advisor는 클러스터에 대한 워크로드 및 사용량 지표를 분석하여 성능을 높이고 비용을 최적화하기 위한 사용자 정의 권장 사항을 개발합니다. Amazon Redshift 콘솔에 로그인하여 Advisor 권장 사항을 확인합니다. 자세한 내용은 Amazon Redshift Advisor의 권장 사항 작업 섹션을 참조하세요.

  • 테이블 설계 – Amazon Redshift는 대부분의 워크로드에 대해 즉시 사용 가능한 뛰어난 성능을 제공합니다. 테이블을 생성할 때 기본 정렬 키와 배포 키는 AUTO입니다. 자세한 내용은 자동 테이블 최적화 작업 섹션을 참조하세요.

  • 지원 – 평가 중에 AWS Support를 평가하는 것이 좋습니다. 또한 계정 관리자에게 개념 증명에 대해 문의해야 합니다. 자격 요건을 충족하는 경우 AWS에서 개념 증명에 대한 기술 안내서 및 크레딧을 제공할 수 있습니다. 원하는 도움을 구하지 못하는 경우 Amazon Redshift 팀에 직접 문의할 수 있습니다. 도움이 필요하면 Amazon Redshift 개념 증명 지원 요청에서 양식을 제출하세요.

  • 레이크 하우스 통합 - 내장된 통합을 통해 기본 제공되는 Amazon Redshift Spectrum 기능을 사용해 보세요. Redshift Spectrum을 사용하면 데이터 웨어하우스를 데이터 레이크로 확장하고 기존 클러스터를 사용하여 Amazon S3에서 페타바이트 규모의 데이터에 대해 쿼리를 실행할 수 있습니다. 자세한 내용은 Amazon Redshift Spectrum을 사용하여 외부 데이터 쿼리 섹션을 참조하세요.

평가를 위한 프로젝트 계획 개발

쿼리 벤치마크 생성을 위한 다음 기술 중 일부는 Amazon Redshift 평가를 지원하는 데 도움이 될 수 있습니다.

  • 각 실행 시간 범주에 대한 쿼리 목록을 어셈블합니다. 충분한 수(예: 범주당 30개)를 확보하면 평가에서 실제 데이터 웨어하우스 구현을 반영하는 데 도움이 됩니다. 고유한 식별자를 추가하여 평가에 포함할 각 쿼리를 평가에 대해 설정한 범주 중 하나와 연결할 수 있습니다. 그런 다음 이 고유 식별자를 사용하여 시스템 테이블의 처리량을 결정할 수 있습니다.

    또한 쿼리 그룹을 생성하여 평가 쿼리를 구성할 수 있습니다. 예를 들어, 평가에 대한 "보고" 범주를 설정한 경우 코딩 시스템을 생성하여 "보고서"라는 단어로 평가 쿼리에 태그를 지정할 수 있습니다. 그런 다음 보고 내의 개별 쿼리를 R1, R2 등으로 식별할 수 있습니다. 다음 예에서는 이 방법을 보여 줍니다.

    
SELECT 'Reporting' AS query_category, 'R1' as query_id, * FROM customers;              

    
SELECT query, datediff(seconds, starttime, endtime)
  FROM stl_query 
  WHERE 
  querytxt LIKE '%Reporting%' 
  and starttime >= '2018-04-15 00:00' 
  and endtime < '2018-04-15 23:59';

    쿼리를 평가 범주와 연결한 경우 고유 식별자를 사용하여 시스템 테이블에서 각 범주에 대한 처리량을 결정할 수 있습니다.

  • 기존 데이터 웨어하우스에서 다양한 런타임을 가진 기존 사용자 또는 ETL 쿼리를 사용하여 처리량을 테스트합니다. 오픈 소스JMeter 또는 사용자 정의 유틸리티와 같은 로드 테스트 유틸리티를 사용할 수 있습니다. 이러한 경우 유틸리티에서 다음을 수행하는지 확인합니다.

    • 네트워크 전송 시간을 고려할 수 있습니다.

    • 내부 시스템 테이블의 처리량에 기반하여 실행 시간을 평가합니다. 이를 위한 자세한 방법은 쿼리 요약 분석 섹션을 참조하세요.

  • 평가 중에 테스트하려는 모든 순열을 식별합니다. 다음 목록에 일부 일반적인 변수가 나와 있습니다.

    • 클러스터 크기

    • 노드 유형

    • 로드 테스트 기간

    • 동시성 설정

  • 업무 외 시간과 주말에 클러스터를 일시 중지하여 개념 증명 비용을 줄입니다. 클러스터가 일시 중지되면 온디맨드 컴퓨팅 결제가 일시 중지됩니다. 클러스터에서 테스트를 실행하려면 초당 결제를 다시 시작합니다. 클러스터를 자동으로 일시 중지하고 다시 시작하는 일정을 생성할 수도 있습니다. 자세한 내용은 Amazon Redshift 관리 가이드의 클러스터 일시 중지 및 재개 섹션을 참조하세요.

이 단계에서 프로젝트 계획을 완료하고 결과를 평가할 준비가 되었습니다.

  • Amazon Redshift Spectrum 시작하기 – 이 튜토리얼에서는 Redshift Spectrum을 사용하여 Amazon S3의 파일에서 직접 데이터를 쿼리하는 방법에 대해 알아봅니다.

  • Amazon Redshift의 새로운 기능 - 새로운 기능과 개선 사항을 추적하는 데 도움이 되는 공지 사항입니다.
