CloudFormation에서 퍼블릭 익스텐션 사용

퍼블릭 익스텐션은 모든 CloudFormation 사용자가 사용할 수 있도록 레지스트리에 공개적으로 게시되는 리소스 유형 또는 모듈과 같은 CloudFormation 익스텐션입니다. 여기에는 AWS에서 게시한 모든 익스텐션이 포함됩니다. 타사도 퍼블릭 익스텐션을 게시할 수 있습니다.

퍼블릭 익스텐션에는 2가지 종류가 있습니다.

  • AWS 퍼블릭 익스텐션 – AWS가 게시한 익스텐션은 항상 퍼블릭이고 기본적으로 활성화되어 있으므로 계정에서 사용하기 전에 별도의 조치를 취하지 않아도 됩니다. 또한 AWS는 익스텐션의 버전 관리를 제어하므로 항상 사용 가능한 최신 버전을 사용하고 있습니다.

  • 타사 퍼블릭 익스텐션 - AWS 이외의 게시자가 일반적으로 사용할 수 있도록 만든 익스텐션입니다. 퍼블릭 익스텐션을 사용하려면 먼저 계정과 리전에서 활성화해야 합니다.

    일반 CloudFormation 사용자가 사용할 수 있도록 자체 타사 익스텐션을 게시할 수 있습니다. 자세한 내용을 알아보려면 CloudFormation Command Line Interface User Guide(CloudFormation 명령줄 인터페이스 사용 설명서)의 Publishing extensions(익스텐션 게시)를 참조하세요.

참고

프라이빗 익스텐션과 마찬가지로 타사 게시자의 퍼블릭 익스텐션은 리소스 생성, 읽기, 업데이트, 나열 및 삭제 스택 작업 중 실행되는 이벤트 핸들러를 구현할 수 있습니다. 이 때문에 CloudFormation 스택에서 이러한 익스텐션을 사용하면 계정에 요금이 부과될 수 있습니다. 이는 생성된 리소스에 대해 발생한 모든 비용에 추가적으로 적용됩니다. 자세한 내용은 AWS CloudFormation 요금을 참조하십시오.

AWS 계정에서 사용할 퍼블릭 익스텐션 활성화

템플릿에서 퍼블릭 타사 익스텐션을 사용하려면 먼저 해당 익스텐션을 사용하려는 계정과 리전에 대해 익스텐션을 활성화해야 합니다. 익스텐션이 활성화된 계정과 리전의 스택 작업에서 해당 익스텐션을 사용할 수 있습니다. 퍼블릭 타사 익스텐션을 활성화하면 CloudFormation은 활성화된 익스텐션에 대한 항목을 계정의 익스텐션 레지스트리에 프라이빗 익스텐션으로 생성합니다. 이를 통해 다음과 같은 방식으로 계정에서 활성화된 익스텐션을 사용자 지정할 수 있습니다.

  • 퍼블릭 타사 익스텐션 이름 대신 사용할 별칭을 지정합니다. 이렇게 하면 타사 익스텐션 간의 이름 충돌을 방지할 수 있습니다.

  • 새 마이너 버전이나 패치 버전을 사용할 수 있게 되면 익스텐션을 자동으로 업데이트할지 여부를 지정합니다.

  • 익스텐션에 대한 로깅을 구성하는 것 외에도 CloudFormation이 익스텐션을 활성화하는 데 사용하는 실행 역할을 지정합니다.

퍼블릭 타사 익스텐션을 활성화할 경우 익스텐션에 포함된 구성 속성을 설정할 수도 있습니다. 구성 속성은 지정된 계정과 리전에 대해 익스텐션이 구성되는 방법을 정의합니다. 구성에 대한 자세한 내용을 알아보려면 다음을 참조하세요.

  • 익스텐션의 계정 수준 구성 정의

  • 후크 구성 스키마

자동으로 새 버전의 익스텐션을 사용하도록 CloudFormation 설정

익스텐션을 활성화할 때 최신 마이너 버전을 사용하도록 익스텐션 유형을 지정할 수도 있습니다. 게시자가 활성화된 익스텐션의 새 버전을 출시할 때마다 익스텐션 유형이 마이너 버전을 업데이트합니다.

예를 들어, 다음에 해당 익스텐션이 포함된 템플릿을 사용하여 스택 생성 또는 업데이트와 같은 스택 작업을 수행할 때 CloudFormation은 새로운 마이너 버전을 사용합니다.

새 익스텐션 버전으로 자동 또는 수동 업데이트해도 스택에 이미 프로비저닝된 익스텐션 인스턴스에는 영향이 없습니다.

CloudFormation은 익스텐션의 메이저 버전 업데이트를 잠재적으로 주요 변경 사항을 포함하는 것으로 취급하므로 새로운 메이저 버전의 익스텐션으로 수동 업데이트해야 합니다.

AWS에 의해 게시된 익스텐션은 사용 가능한 모든 계정과 리전에 대해 기본적으로 활성화되며 항상 각 AWS 리전에서 사용 가능한 최신 버전을 사용합니다.

중요

익스텐션이 계정의 최신 버전으로 업데이트되는지 여부와 시기를 제어하므로 동일한 익스텐션의 다른 버전이 다른 계정 및 리전에 배포될 수 있습니다.

이로 인해 해당 계정과 리전에서 해당 익스텐션이 포함된 동일한 템플릿을 사용할 때 예기치 않은 결과가 발생할 수 있습니다.

익스텐션을 참조할 별칭 지정

지정된 계정과 리전에서 지정된 이름으로 여러 익스텐션을 활성화할 수 없습니다. 다른 게시자가 동일한 익스텐션 이름으로 퍼블릭 익스텐션을 제공할 수 있기 때문에 CloudFormation을 사용하면 활성화하는 모든 타사 퍼블릭 익스텐션에 대한 별칭을 지정할 수 있습니다.

익스텐션에 대한 별칭을 지정하면 CloudFormation은 별칭을 계정과 리전 내에서 익스텐션 유형 이름으로 처리합니다. 템플릿, API 호출 및 CloudFormation 콘솔에서 익스텐션을 참조하려면 별칭을 사용해야 합니다.

타사 별칭은 지정된 계정과 리전 내에서 고유해야 합니다. 서로 다른 유형 이름 별칭을 사용하여 동일한 계정과 리전에서 동일한 퍼블릭 리소스를 여러 번 활성화할 수 있습니다.

중요

익스텐션 별칭은 지정된 계정과 리전에서만 고유해야 하지만 사용자가 계정과 리전 전반에 걸쳐 다른 타사 퍼블릭 익스텐션에 동일한 별칭을 할당하지 않는 것이 좋습니다. 이렇게 하면 여러 계정 또는 리전에서 익스텐션 별칭이 포함된 템플릿을 사용할 때 예상치 못한 결과가 발생할 수 있습니다.

익스텐션 활성화 시 실행 역할 지정

리소스를 활성화할 때 계정 및 리전에서 해당 익스텐션을 호출할 때 수임할 CloudFormation의 IAM 실행 역할을 지정할 수 있습니다.

CloudFormation이 실행 역할을 수임하려면 역할에 CloudFormation으로 정의된 신뢰 정책이 있어야 합니다. 또한 IAM 정책을 생성하고 실행 역할에 연결하면 작업을 수행할 수 있는 권한이 부여됩니다. 필요한 권한은 익스텐션 스키마의 핸들러 섹션에 정의되어 있습니다.

다음은 리소스 유형 확장에 대한 IAM 역할 신뢰 정책의 예입니다.

{
    "Version": "2012-10-17",
    "Statement":[
        {
            "Effect": "Allow",
            "Principal":{
                "Service": "resources.cloudformation.amazonaws.com"
            },
            "Action":"sts:AssumeRole",
            "Condition":{
                "StringEquals":{
                    "aws:SourceAccount":"123456789012"
                },
                "StringLike":{
                    "aws:SourceArn":"arn:aws:cloudformation:us-east-1:123456789012:type/resource/Organization-Service-Resource/*"
                }
            }
        }
    ]
}

다음은 후크 확장에 대한 IAM 역할 신뢰 정책의 예입니다.

{
    "Version":"2012-10-17",
    "Statement":[
        {
            "Effect":"Allow",
            "Principal": {
                "Service": [
                    "resources.cloudformation.amazonaws.com",
                    "hooks.cloudformation.amazonaws.com"
                ]
            },
            "Action":"sts:AssumeRole",
            "Condition":{
                "StringEquals":{
                    "aws:SourceAccount":"123456789012"
                },
                "StringLike":{
                     "aws:SourceArn":"arn:aws:cloudformation:us-east-1:123456789012:type/hook/Organization-Service-Hook/*"
                }
            }
        }
    ]
}

자세한 내용을 알아보려면 AWS Identity and Access Management 사용 설명서의 역할 신뢰 정책 수정을 참조하세요.

레지스트리에서 퍼블릭 익스텐션 활성화

익스텐션을 사용할 수 있도록 AWS CloudFormation 레지스트리에서 활성화합니다.

CloudFormation 콘솔을 사용하여 계정에서 사용할 퍼블릭 익스텐션 활성화

  1. AWS Management Console에 로그인하여 https://console.aws.amazon.com/cloudformation에서 AWS CloudFormation 콘솔을 엽니다.

  2. CloudFormation 탐색 창의 CloudFormation registry(CloudFormation 레지스트리)에서 Public extensions(퍼블릭 익스텐션)를 선택합니다.

  3. Filter(필터)를 사용하여 익스텐션 유형을 선택하고 Third party(타사)를 선택합니다. AWS에서 게시하는 익스텐션은 기본적으로 활성화되어 있습니다.

  4. 익스텐션을 선택한 다음 Activate(활성화)를 선택합니다.

    참고

    여러 버전의 익스텐션을 사용할 수 있는 경우 Version(버전) 메뉴를 사용하여 활성화하려는 익스텐션 버전을 선택할 수 있습니다. 기본값은 가장 최신 버전입니다.

  5. Activate extension(익스텐션 활성화) 페이지의 옵션을 사용하여 이 계정에서 익스텐션이 활성화되는 방식을 사용자 지정합니다.

    • 익스텐션 유형 별칭을 사용하여 익스텐션 활성화

      1. Override default(기본값 재정의)를 선택합니다.

      2. 이 익스텐션에 사용할 익스텐션 유형 별칭을 입력합니다. 별칭은 익스텐션 유형에 대해 권장되는 형식을 따라야 합니다.

      자세한 내용을 알아보려면 익스텐션을 참조할 별칭 지정을 참조하세요.

    • 프라이빗 리소스를 활성화하는 경우 익스텐션을 호출할 때 CloudFormation이 수임할 IAM 역할을 지정할 수 있습니다. 자세한 내용을 알아보려면 익스텐션 활성화 시 실행 역할 지정을 참조하세요.

    • 원하는 경우 Logging config(로깅 구성)에서 익스텐션에 대한 로깅 구성 정보를 지정합니다. 예제:

      {
    "logRoleArn": "arn:aws:iam::account:role/rolename",
    "logGroupName": "log-group-name"
}

      로깅 구성 정보는 필수는 아니지만 디버깅 목적으로 권장됩니다. 후크와 함께 로깅 구성을 사용하려면 로그 역할이 로그 그룹에 로그를 쓸 수 있도록 지정된 실행 역할과 동일한 신뢰 정책을 추가합니다.

      logRoleArn 및 logGroupName 키 이름은 대/소문자를 구분합니다.

    • Versioning(버전 관리)에서 업데이트 수신 방법을 지정합니다.

      • On(켜기) - 최신 마이너 버전으로 자동 업데이트됩니다. 메이저 버전은 수동으로 업데이트됩니다.

      • Off(끄기) - 최신 마이너 버전으로 자동 업데이트되지 않습니다. 모든 버전이 수동으로 업데이트됩니다.

      자세한 내용을 알아보려면 자동으로 새 버전의 익스텐션을 사용하도록 CloudFormation 설정을 참조하세요.

  6. 구성 데이터를 지정합니다.

    참고

    이 단계는 후크에 필요합니다. TargetStack 속성에 ALL을 지정해야 합니다. 이 작업은 후크의 스키마 속성 섹션에 정의된 후크의 속성을 활성화합니다.

    익스텐션에 계정 및 리전 수준의 추가 구성이 필요한 경우 Activate extension(익스텐션 활성화) 페이지에 Configuration(구성) 섹션이 포함됩니다. 지금 또는 익스텐션이 활성화된 후 구성 데이터를 지정할 수 있는 옵션이 있습니다. 익스텐션에 구성이 필요하지 않은 경우 Configuration(구성) 섹션이 표시되지 않습니다.

    • 익스텐션을 활성화하는 동안 구성 데이터 지정

      1. Configure now(지금 구성)를 선택하고 Activate extension(익스텐션 활성화)을 선택합니다.

        Configure extension(익스텐션 구성) 페이지가 표시됩니다. 익스텐션에 대한 현재 구성 스키마를 보려면 View configuration schema(구성 스키마 보기)가 활성화되어 있는지 확인합니다.

      2. Configuration JSON(구성 JSON) 텍스트 상자에 이 익스텐션에 대해 지정하려는 구성 스키마를 나타내는 JSON 문자열을 입력합니다.

      3. Configure extension(익스텐션 구성)을 선택합니다.

    • 나중에 구성 데이터 지정

      • Configure later(나중에 구성)를 선택하고 Activate extension(익스텐션 활성화)을 선택합니다.

      이미 활성화된 익스텐션에 대한 구성 데이터를 지정하려면 활성화된 익스텐션의 세부 정보 페이지를 방문합니다.

    익스텐션이 성공적으로 활성화되면 해당 익스텐션에 대한 세부 정보 페이지가 표시됩니다.

CloudFormation 콘솔을 사용하여 계정에서 활성화된 퍼블릭 익스텐션 업데이트

익스텐션을 사용할 수 있도록 AWS CloudFormation 레지스트리에서 업데이트합니다.

계정과 리전에서 익스텐션을 활성화한 후 해당 익스텐션이 활성화되면 다음과 같은 속성을 업데이트할 수 있습니다.

  • 구성 스키마입니다.

  • 자동 버전 업데이트를 활성화할지 여부입니다.

  • 프라이빗 등록 익스텐션의 경우 사용할 실행 역할입니다.

  • 로깅 구성 정보입니다.

  1. AWS Management Console에 로그인하여 https://console.aws.amazon.com/cloudformation에서 AWS CloudFormation 콘솔을 엽니다.

  2. CloudFormation 탐색 창의 CloudFormation registry(CloudFormation 레지스트리)에서 Public extensions(퍼블릭 익스텐션)를 선택합니다.

  3. Actions(작업) 메뉴에서 Edit(편집)를 선택한 다음 적절한 편집 옵션을 선택합니다.

    • 구성 스키마를 업데이트하려면 CloudFormation 콘솔에서 익스텐션에 대한 구성 속성 지정을 참조하세요.

    • 자동 업데이트 활성화 또는 비활성화

      1. Edit automatic updates(자동 업데이트 편집)를 선택합니다.

      2. On(켜기) 또는 Off(끄기)를 선택한 다음 Save(저장)를 선택합니다.

    • 실행 역할 업데이트

      1. Edit execution role(실행 역할 편집)을 선택합니다.

      2. 이 익스텐션을 호출할 때 CloudFormation에서 사용할 IAM 역할의 ARN을 지정한 다음 Save(저장)를 선택합니다.

    • 로깅 구성 업데이트

      1. Edit logging config(로깅 구성 편집)를 선택합니다.

      2. 로깅 구성 JSON을 편집한 다음 Save(저장)를 선택합니다.

AWS CLI를 사용하여 계정에서 사용할 퍼블릭 익스텐션 활성화

AWS CLI를 통해 AWS CloudFormation 레지스트리에서 익스텐션을 활성화합니다.

후크 활성화

후크를 개발하고 등록한 후 레지스트리에 게시하여 AWS 계정에서 후크를 활성화할 수 있습니다.

  • 계정에서 후크를 활성화하려면 SetTypeConfiguration 작업을 사용합니다. 이 작업은 후크의 스키마 properties 섹션에 정의된 후크의 속성을 활성화합니다.

    참고

    계정에서 후크를 활성화하면 AWS 계정에서 정의된 권한을 사용하도록 후크에 권한이 부여됩니다. CloudFormation은 권한을 후크에 전달하기 전에 필요하지 않은 권한을 제거합니다. CloudFormation은 고객 또는 후크 사용자에게 후크 권한을 검토하고 계정에서 후크를 활성화하기 전에 후크가 허용되는 권한을 인지할 것을 권장합니다.

    동일한 계정과 AWS 리전에 등록된 후크 익스텐션에 대한 구성 데이터를 지정합니다.

    aws cloudformation --region us-west-2 set-type-configuration \
  --configuration "{"CloudFormationConfiguration":{"HookConfiguration":{"TargetStacks": "ALL", "FailureMode": "FAIL", "Properties":{}}}}" \
  --type-arn $HOOK_TYPE_ARN
    중요

    후크가 스택 구성을 사전에 검사하도록 하려면 HookConfiguration 섹션에서 TargetStacks를 ALL로 설정해야 합니다.

리소스 유형 및 모듈 활성화
AWS CLI를 사용하여 계정에서 사용할 퍼블릭 익스텐션 활성화

  • activate-type을 사용하여 익스텐션을 활성화하고 익스텐션의 새 마이너 버전이 게시될 때마다 익스텐션을 자동 업데이트할지 여부를 지정합니다.

    아래 예제에서는 이 계정에 대해 활성화할 퍼블릭 익스텐션의 퍼블릭 Amazon 리소스 이름(ARN)을 지정합니다. 또한 새 마이너 버전이 게시될 때마다 CloudFormation이 익스텐션을 업데이트하도록 지정합니다.

    aws cloudformation activate-type \
  --public-type-arn public_extension_ARN \
  --auto-update true

    이 명령은 이 계정 및 리전과 관련된 활성화된 익스텐션의 ARN을 반환합니다.

    {
    "Arn": "624af370-311a-11e8-b6b7-500cexample"
}
AWS CLI를 사용하여 퍼블릭 익스텐션의 버전 업데이트

activate-type을 사용하여 익스텐션을 다시 활성화합니다.

--version-bump 파라미터를 사용하여 익스텐션을 최신 MAJOR 버전, 최신 MINOR 버전으로 업데이트할지 여부를 지정합니다.

aws cloudformation activate-type \
    --region us-west-2 \
    --type RESOURCE \
    --type-name Example::Test::1234567890abcdef0 \
    --type-name-alias Example::Test::Alias \
    --version-bump MAJOR

계정에서 퍼블릭 익스텐션 비활성화

다음 섹션에서는 계정에서 퍼블릭 익스텐션을 비활성화하는 방법을 보여줍니다.

CloudFormation 콘솔을 사용하여 계정에서 퍼블릭 익스텐션 비활성화

  1. CloudFormation 레지스트리를 사용하여 익스텐션을 찾습니다.

  2. Actions(작업) 메뉴에서 Deactivate(비활성화)를 선택합니다.

  3. Deactivate(비활성화)를 선택합니다.

AWS CLI를 사용하여 계정에서 퍼블릭 익스텐션 비활성화

계정에서 퍼블릭 익스텐션을 비활성화하려면 deactivate-type을 지정합니다. 다음은 AWS CLI 비활성화 명령의 예제입니다.

aws cloudformation deactivate-type \
    --region us-west-2 \
    --type MODULE \
    --type-name Example::Test::Type::MODULE

AWS CLI를 사용하여 계정에서 후크 비활성화

후크를 비활성화하면 후크에서 AWS 계정이 실행되지 않습니다.

TargetStacks를 NONE으로 설정하면 계정에서 후크가 꺼지므로 스택 작업에는 적용되지 않습니다.

set-type-configuration 작업을 사용하여 TargetStacks를 NONE으로 지정하여 후크를 비활성화합니다.

다음 예제에서는 비활성화 중인 후크의 AWS 리전과 Amazon 리소스 이름(ARN)을 지정합니다.

aws cloudformation set-type-configuration \
  --region us-west-2 \
  --configuration "{"CloudFormationConfiguration":{"HookConfiguration":{"TargetStacks": "NONE", "FailureMode": "FAIL", "Properties":{}}}}" \
  --type-arn HOOK_TYPE_ARN
