Realizar uma prova de conceito para o Amazon Redshift

O Amazon Redshift é um data warehouse rápido e escalável que torna simples e econômico analisar todos os seus dados usando SQL padrão com suas ferramentas de business intelligence (BI) existentes. O Amazon Redshift oferece performance rápida em um data warehouse em nuvem de baixo custo. Ele usa otimização de consulta sofisticada, cache acelerado, armazenamento colunar em discos locais de alta performance e execução de consulta massivamente paralela.

Nas seções a seguir, você pode encontrar um framework para construir uma prova de conceito com o Amazon Redshift. O framework o ajuda a usar as práticas recomendadas de arquitetura para projetar e operar um data warehouse seguro, de alta performance e econômico. Essa orientação é baseada na revisão de projetos de milhares de arquiteturas de clientes em uma ampla variedade de tipos de negócios e casos de uso. Compilamos experiências dos clientes para desenvolver este conjunto de melhores práticas para ajudá-lo a desenvolver critérios para avaliar seu workload de data warehouse.

Visão geral do processo

Conduzir uma prova de conceito é um processo de três etapas:

  1. Identificar os objetivos da prova de conceito — Você pode trabalhar retrocedendo a partir de seus requisitos de negócios e critérios de sucesso, e traduzi-los em um plano de projeto de prova técnica de conceito.

  2. Configurar o ambiente de prova de conceito — A maior parte do processo de configuração é um clique de alguns botões para criar seus recursos. Em minutos, você pode ter um ambiente de data warehouse pronto com dados carregados.

  3. Execute o plano de projeto de prova de conceito para garantir que os objetivos sejam atingidos.

As seções a seguir exploram os detalhes de cada etapa.

Identificar as metas de negócios e os critérios de sucesso

Identificar os objetivos de prova de conceito tem uma função crucial em determinar o que você deseja considerar como parte do processo de avaliação. Os critérios de avaliação devem incluir os desafios de escalabilidade atuais, aprimoramentos para melhorar a experiência do cliente no data warehouse e métodos para abordar seus pontos de dificuldade operacionais atuais. É possível usar as seguintes perguntas para identificar os objetivos de prova de conceito:

  • Quais são os objetivos de escalonar o data warehouse do ?

  • Você tem acordos de nível de serviço específicos cujos termos deseja melhorar?

  • Quais novos conjuntos de dados você ou seus clientes precisam incluir no data warehouse?

  • Quais são as consultas SQL críticas para os negócios que você precisa testar e medir? Certifique-se de incluir toda a gama de complexidades SQL, como os diferentes tipos de consultas (por exemplo, selecionar, inserir, atualizar e excluir).

  • Quais são os tipos gerais de workloads que você planeja testar? Os exemplos podem incluir workloads de extração-transformação-carga (ETL), consultas de relatórios e extrações em lote.

Depois de responder a essas perguntas, você deverá ser capaz de estabelecer metas SMART e critérios de sucesso para construir sua prova de conceito. Para obter informações sobre a configuração de metas, consulte Critério SMART na Wikipedia.

Configurar a prova de conceito

Como eliminamos o provisionamento de hardware, a rede e a instalação de software de um data warehouse local, tentar o Amazon Redshift com seu próprio conjunto de dados nunca foi tão fácil. Muitas das decisões de dimensionamento e estimativas que costumavam ser necessárias agora estão apenas a um clique de distância. Você pode redimensionar seu cluster de forma flexível ou ajustar a proporção de armazenamento em relação à computação.

Em geral, a configuração do ambiente de prova de conceito do Amazon Redshift é um processo de duas etapas. Envolve o lançamento de um data warehouse e, em seguida, a conversão do esquema e conjuntos de dados para avaliação.

Escolher um tamanho de cluster inicial

Você pode escolher o tipo de nó e o número de nós usando o console do Amazon Redshift. Recomendamos que você também teste o redimensionamento do cluster como parte de seu plano de prova de conceito. Para obter o dimensionamento inicial do cluster, siga estas etapas:

  1. Faça login no AWS Management Console e abra o console do Amazon Redshift em https://console.aws.amazon.com/redshift/.

  2. No menu de navegação, selecione Create cluster (Criar cluster) para abrir a página de configuração.

  3. Para o Identificador do cluster, insira um nome para o seu cluster.

  4. A etapa a seguir descreve um console do Amazon Redshift que está sendo executado em uma região da AWS que suporta tipos de nó RA3. Para ver uma lista de regiões da AWS compatíveis com tipos de nó RA3, consulte “Visão geral dos tipos de nó RA3” no Guia de gerenciamento de clusters do Amazon Redshift.

    Se você não souber o tamanho do cluster, escolha Ajude-me a escolher. Isso inicia uma calculadora de dimensionamento que faz perguntas sobre o tamanho e as características de consulta dos dados que você planeja armazenar em seu data warehouse. Se você souber o tamanho necessário do cluster (ou seja, o tipo de nó e o número de nós), escolha Eu escolherei. Em seguida, escolha o Tipo de nó e número de Nós para dimensionar seu cluster para a prova de conceito.

    nota

    Se sua organização for elegível e seu cluster estiver sendo criado em uma Região da AWS em que o Amazon Redshift sem servidor não está indisponível, você poderá criar um cluster no programa de teste gratuito do Amazon Redshift. Escolha Produção ou Teste gratuito para responder à pergunta Para que você está planejando usar esse cluster? Ao escolher Teste gratuito, você crie uma configuração com o tipo de nó dc2.large. Para obter mais informações sobre a escolha de um teste gratuito, consulte Teste gratuito do Amazon Redshift. Para obter uma lista de Regiões da AWS nas quais o Amazon Redshift sem servidor está disponível, consulte os endpoints listados para a API do Redshift sem servidor na Referência geral da Amazon Web Services.

  5. Depois de inserir todas as propriedades de cluster necessárias, escolha Criar cluster para iniciar seu data warehouse.

    Para obter mais detalhes sobre a criação de clusters com o console do Amazon Redshift, consulte “Criar um cluster” no Guia de gerenciamento de clusters do Amazon Redshift.

Converter o esquema e configurar os conjuntos de dados para a prova de conceito

Se você não tiver um data warehouse existente, ignore esta seção e consulte o Guia de conceitos básicos do Amazon Redshift. O Guia de conceitos básicos do Amazon Redshift fornece um tutorial para criar um cluster e exemplos de configuração de dados no Amazon Redshift.

Ao migrar de seu data warehouse existente, você pode converter esquema, código e dados usando o AWS Schema Conversion Tool e o AWS Database Migration Service. A escolha de ferramentas depende da fonte dos dados e das replicações contínuas opcionais. Para obter mais informações, consulte O que é o AWS Schema Conversion Tool? no Manual do usuário do AWS Schema Conversion Tool e O que é AWS Database Migration Service? no Manual do usuário do AWS Database Migration Service. As seguintes informações podem ajudar você a configurar seus dados no Amazon Redshift:

  • Migrar seu data warehouse para o Amazon Redshift usando o AWS Schema Conversion Tool – Esta postagem do blog fornece uma visão geral sobre como você pode usar os extratores de dados do AWS SCT para migrar seu data warehouse existente para o Amazon Redshift. A ferramenta AWS SCT pode migrar seus dados de várias plataformas herdadas (como Oracle, Greenplum, Netezza, Teradata, Microsoft SQL Server ou Vertica).

  • Opcionalmente, você também pode usar o AWS Database Migration Service para replicações contínuas de dados alterados da fonte. Para obter mais informações, consulte Usar um banco de dados do Amazon Redshift como destino do AWS Database Migration Service no Manual do usuário do AWS Database Migration Service.

O Amazon Redshift é um sistema de banco de dados relacional (RDBMS). Como tal, ele pode executar muitos tipos de modelos de dados, incluindo esquemas em estrela, esquemas de floco de neve, modelos de cofre de dados e tabelas simples, planas ou normalizadas. Depois de configurar seus esquemas no Amazon Redshift, você pode aproveitar o processamento paralelo massivamente e o armazenamento de dados em coluna para consultas analíticas rápidas imediatamente. Para obter informações sobre tipos de esquemas, consulte Esquema em estrela, Esquema de flocos de neve, e Modelagem do data vault na Wikipedia.

Lista de verificação para uma avaliação completa

Certifique-se de que uma avaliação completa atenda a todas as suas necessidades de data warehouse. Considere incluir os seguintes itens em seus critérios de sucesso:

  • Tempo de carregamento de dados – Usar o comando COPY é uma maneira comum de testar quanto tempo leva para carregar os dados. Para obter mais informações, consulte Práticas recomendadas do Amazon Redshift para carregamento de dados.

  • Taxa de transferência do cluster – Medir consultas por hora é uma maneira comum de determinar a taxa de transferência. Para fazer isso, configure um teste para executar consultas típicas para o workload.

  • Segurança de dados – É possível criptografar dados em repouso e em trânsito facilmente com o Amazon Redshift. Você também tem várias opções para gerenciar chaves. O Amazon Redshift também é compatível com a integração de autenticação única. O custo do Amazon Redshift inclui segurança interna, compactação de dados, armazenamento de backup e transferência de dados.

  • Integração com ferramentas de terceiros – É possível usar uma conexão JDBC ou ODBC para integrar-se a ferramentas de business intelligence e a outras ferramentas externas.

  • Interoperabilidade com outros serviços da AWS — O Amazon Redshift se integra a outros serviços da AWS como Amazon EMR, Amazon QuickSight, AWS Glue, Amazon S3 e Amazon Kinesis. É possível usar essa integração ao configurar e gerenciar o data warehouse.

  • Backups e snapshots – Backups e snapshots são criados automaticamente. Você também pode criar um snapshot point-in-time a qualquer momento ou em uma programação. Tente usar um snapshot e criar um segundo cluster como parte de sua avaliação. Avalie se o desenvolvimento e organizações de teste podem usar o cluster.

  • Redimensionamento – A avaliação deve incluir o aumento do número ou dos tipos de nós do Amazon Redshift. Avalie se a taxa de transferência do workload antes e depois de um redimensionamento atende a qualquer variabilidade do volume de seu workload. Para obter mais informações, consulte “Redimensionar clusters no Amazon Redshift” no Guia de gerenciamento de clusters do Amazon Redshift.

  • Escalabilidade da simultaneidade — Esse recurso ajuda você a lidar com a variabilidade do volume de tráfego em seu data warehouse. Com a escalabilidade de simultaneidade, você pode oferecer suporte a usuários e consultas simultâneas virtualmente ilimitadas, com performance de consulta consistentemente rápida. Para obter mais informações, consulte Trabalhar com a escalabilidade de simultaneidade.

  • Gerenciamento de workload automático (WLM) — Prioriza suas consultas críticas de negócios em relação a outras consultas usando o WLM automático. Tente configurar filas com base em seus workloads (por exemplo, uma fila para ETL e uma fila para relatórios). Em seguida, habilite o WLM automático para alocar os recursos de simultaneidade e memória dinamicamente. Para obter mais informações, consulte Implementar o WLM automático.

  • Amazon Redshift Advisor — O Advisor desenvolve as recomendações personalizadas para aumentar a performance e otimizar os custos analisando métricas de workload e de uso para o cluster. Faça login no console do Amazon Redshift para visualizar as recomendações do Advisor. Para obter mais informações, consulte Trabalhar com recomendações do Amazon Redshift Advisor.

  • Design da tabela — O Amazon Redshift oferece excelente performance pronta para uso para a maioria dos workloads. Quando você cria uma tabela, a chave de classificação padrão e a chave de distribuição são AUTO. Para obter mais informações, consulte Trabalhar com otimização automática de tabelas.

  • Support – É altamente recomendável que você avalie o AWS Support como parte da avaliação. Além disso, não se esqueça de falar com seu gerente de conta sobre sua prova de conceito. A AWS pode ajudar com orientações técnicas e créditos para a prova de conceito, se você se qualificar. Se você não encontrar a ajuda que está procurando, fale diretamente com a equipe do Amazon Redshift. Para obter ajuda, envie o formulário em Solicitar suporte à prova de conceito do Amazon Redshift.

  • Integração de Lake House — com integração integrada, tente usar o recurso pronto para uso do Amazon Redshift Spectrum. Com o Redshift Spectrum, você pode estender o data warehouse ao seu data lake e executar consultas contra petabytes de dados no Amazon S3 usando seu cluster existente. Para obter mais informações, consulte Consultar dados externos usando o Amazon Redshift Spectrum.

Desenvolver um plano de projeto para sua avaliação

Algumas das seguintes técnicas para criar benchmarks de consulta podem ajudar a suportar sua avaliação do Amazon Redshift:

  • Monte uma lista de consultas para cada categoria de tempo de execução. Ter um número suficiente (por exemplo, 30 por categoria) ajuda a garantir que sua avaliação reflita uma implementação de data warehouse do mundo real. Adicione um identificador exclusivo para associar cada consulta incluída na avaliação com uma das categorias estabelecidas na avaliação. Você pode então usar esses identificadores exclusivos para determinar a taxa de transferência das tabelas do sistema.

    Também é possível criar um grupo de consulta para organizar as consultas da avaliação. Por exemplo, se você estabeleceu uma categoria “Relatórios” para a avaliação, pode criar um sistema de códigos para marcar as consultas da avaliação com uma tag com a palavra “Relatório“. Então, é possível identificar consultas individuais dentro de relatórios como R1, R2 e assim por diante. O exemplo a seguir demonstra essa abordagem.

    
SELECT 'Reporting' AS query_category, 'R1' as query_id, * FROM customers;              

    
SELECT query, datediff(seconds, starttime, endtime)
  FROM stl_query 
  WHERE 
  querytxt LIKE '%Reporting%' 
  and starttime >= '2018-04-15 00:00' 
  and endtime < '2018-04-15 23:59';

    Ao associar uma consulta a uma categoria de avaliação, você pode usar um identificador exclusivo para determinar a taxa de transferência das tabelas do sistema para cada categoria.

  • Teste a taxa de transferência com usuário histórico ou consultas ETL que têm uma variedade de tempos de execução em seu data warehouse existente. Você pode usar um utilitário de teste de carga, como o JMeter de código aberto ou um utilitário personalizado. Em caso afirmativo, verifique se o utilitário faz o seguinte:

    • Pode levar em conta o tempo de transmissão da rede.

    • Ele avalia o tempo de execução com base na taxa de transferência das tabelas internas do sistema. Para obter informações sobre como fazer isso, consulte Analisar o resumo da consulta.

  • Identifique todas as várias permutações que você planeja testar durante a avaliação. A lista a seguir fornece algumas variáveis comuns:

    • Tamanho do cluster

    • Tipo de nó

    • Duração do teste de carregamento

    • Configurações de simultaneidade

  • Reduz o custo de sua prova de conceito pausando seu cluster durante horas de folga e fins de semana. Quando um cluster é pausado, o faturamento da computação sob demanda é suspenso. Para executar testes no cluster, retome a cobrança por segundo. Você também pode criar uma programação para pausar e retomar o cluster automaticamente. Para obter mais informações, consulte “Pausar e retomar clusters” no Guia de gerenciamento de clusters do Amazon Redshift.

Nesta fase, você está pronto para executar seu plano de projeto e avaliar os resultados.
