Permissões em escopo

Use permissões em escopo para alterar permissões de um usuário ou uma função em um escopo de banco de dados ou esquema.

As permissões em escopo se aplicam a objetos no escopo selecionado quando você concede ou revoga a permissão, bem como a novos objetos criados após a concessão ou a revogação da permissão. Por exemplo, a concessão a uma função de permissão em escopo para selecionar tabelas em um banco de dados permite aos usuários com essa função selecionar tabelas que serão criadas no futuro, além de permitir que selecionem tabelas atualmente existentes no banco de dados.

Você pode usar permissões em escopo para definir as permissões padrão em um banco de dados compartilhado criado a partir de uma unidade de compartilhamento de dados. Um usuário da unidade de compartilhamento de dados no lado do consumidor que recebe permissões em escopo em um banco de dados compartilhado vai receber automaticamente essas permissões para qualquer novo objeto adicionado à unidade de compartilhamento de dados no lado do produtor.

Para obter mais informações sobre a aplicação de permissões em escopo definido, consulte GRANT e REVOKE.

Considerações sobre como usar permissões em escopo

Ao usar permissões em escopo, considere o seguinte:

  • Você pode usar permissões em escopo a fim de conceder permissões em um escopo de banco de dados ou esquema para uma função ou um usuário escolhido, ou revogá-las. Você não pode conceder permissões em escopo a grupos de usuários, logo, não pode revogá-las.

  • A concessão ou a revogação de permissões em escopo altera permissões de todos os objetos atuais e futuros no escopo.

  • As permissões em escopo se aplicam independentemente das permissões no nível do objeto. Elas são mutuamente excludentes. Por exemplo, considere um usuário que tenha permissão no nível de objeto para uma tabela e também permissão em escopo para o esquema no qual a tabela está. Em ambos os cenários, o usuário mantém acesso à tabela.

    • A permissão em escopo do usuário para o esquema no qual a tabela está é revogada, mas o usuário continua tendo acesso à tabela por meio da permissão no nível de objeto para a tabela.

    • A permissão no nível do objeto do usuário para a tabela é revogada, mas o usuário continua tendo acesso à tabela por meio da permissão em escopo para o esquema.

  • Para conceder ou revogar permissões em escopo, você deve atender a um dos seguintes critérios:

    • Superusuários.

    • Usuários com a opção de concessão dessa permissão. Para obter mais informações sobre opções de concessão, consulte o parâmetro WITH GRANT OPTION em GRANT.

  • As permissões em escopo só podem ser concedidas a ou revogadas de objetos do banco de dados conectado ou de bancos de dados importados de uma unidade de compartilhamento de dados.

  • Você pode conceder permissões em escopo que se aplicam a bancos de dados criados com base em unidades de compartilhamento de dados.
