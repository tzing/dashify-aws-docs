Tipos de dados de intervalo

É possível usar um tipo de dados de intervalo para especificar períodos ou intervalos de tempo, como 12 hours, 6 weeks ou 1 month. Tipos de dados INTERVALO permitem especificar períodos ou intervalos de tempo, como 12 hours, 6 weeks ou 1 month. É possível usar tipos de dados de intervalo em cálculos, por exemplo, para calcular diferenças entre uma data e hora e horários ou intervalos, somas de intervalos ou somas de uma data e hora e um intervalo.

Os seguintes intervalos são aceitos:

  • YEAR TO MONTH

  • DAY TO SECOND

Sintaxe

Para criar um intervalo do tipo YEAR TO MONTH:

INTERVAL <year_to_month_qualifier>

Para criar um literal de intervalo do tipo YEAR TO MONTH:

INTERVAL <interval-string> <qualifier>

Use a sintaxe a seguir para criar um intervalo do tipo DAY TO SECOND. O parâmetro fractional_precision só será necessário se o intervalo contiver SECOND.

INTERVAL <qualifier> [ <fractional_precision> ]

Use a sintaxe a seguir para criar um literal de intervalo do tipo DAY TO SECOND.

INTERVAL <interval-string> <qualifier> [ <fractional_precision> ]

Argumentos

Interval-string

Especifica o valor do intervalo. Necessário somente ao criar uma constante de intervalo

Qualificador

Especifica a faixa do intervalo. Se você usar um qualificador e criar um intervalo com unidades de tempo menores do que o qualificador, o Amazon Redshift truncará e descartará as partes menores do intervalo. O exemplo a seguir demonstra o uso de um qualificador MINUTE, e o Amazon Redshift descarta as unidades de tempo menores do que MINUTE.

INTERVAL '1 day 1 hour 1 minute 1.123 seconds' MINUTE

é truncado para 

'1 day 01:01:00'
Fractional_precision

Parâmetro opcional que especifica o número de dígitos fracionários permitidos no intervalo. Por exemplo, SECOND (3) cria um intervalo que permite somente três dígitos fracionários, como 1.234 segundos. Aplicável somente aos intervalos DAY TO SECOND que contêm segundos. O número máximo de dígitos aceito é seis.

O tipo de dados YEAR TO MONTH é compatível com os seguintes qualificadores:

  • YEAR

  • MONTH

  • YEAR TO MONTH

O tipo de dados DAY TO SECOND é compatível com os seguintes qualificadores:

  • DAY

  • HOUR

  • MINUTE

  • SECOND

  • DAY TO HOUR

  • DAY TO MINUTE

  • DAY TO SECOND

  • HOUR TO MINUTE

  • HOUR TO SECOND

  • MINUTE TO SECOND

Exemplos

O exemplo a seguir demonstra como criar uma constante INTERVAL de um ano.

INTERVAL '1' YEAR

Se você especificar uma string de intervalo que exceda o qualificador, as unidades de tempo restantes serão truncadas do intervalo. No exemplo a seguir, um intervalo de 13 meses se torna um ano e um mês, mas o mês restante é omitido devido ao qualificador YEAR.

INTERVAL '13 months' YEAR
'1 year'

Se você usar um qualificador inferior à string de intervalo, as unidades restantes serão incluídas.

INTERVAL '13 months' MONTH
'1 year 1 month'

Especificar uma precisão no intervalo trunca o número de dígitos fracionários até a precisão especificada.

INTERVAL '1.234567' SECOND (3)
'00:00:01.234'

Se você não especificar uma precisão, o Amazon Redshift usará a precisão máxima de seis.

INTERVAL '1.23456789' SECOND
'00:00:01.234567'

O exemplo a seguir demonstra como criar um intervalo em faixas.

INTERVAL '2:2' MINUTE TO SECOND
'00:02:02'

Os qualificadores ditam as unidades que você está especificando. Por exemplo, embora o exemplo a seguir use a mesma string de intervalo de “2:2” do exemplo anterior, o Amazon Redshift reconhece que ele usa unidades de tempo diferentes devido ao qualificador

INTERVAL '2:2' HOUR TO MINUTE
'02:02:00'

As abreviaturas e os plurais de cada unidade também são compatíveis; por exemplo: 5s, 5 second e 5 seconds são intervalos equivalentes. As unidades aceitas são anos, meses, horas, minutos e segundos.

INTERVAL '5s' SECOND
'00:00:05'

INTERVAL '5 second' SECOND
'00:00:05'
                     
INTERVAL '5 HOURS' HOUR
'05:00:00'
                     
INTERVAL '5 h' HOUR
'05:00:00

Os exemplos a seguir demonstram como usar tipos de dados INTERVALO com tabelas.

create table sample_intervals (y2m interval month, m2s interval hour to minute);
insert into sample_intervals values (interval '20' month, interval '2 days 1:1:1.123456' day to second);
select y2m::text, m2s::text from sample_intervals;

      y2m      |       m2s       
---------------+-----------------
 1 year 8 mons | 2 days 01:01:00

update sample_intervals set y2m = interval '2' year where y2m = interval '1-8' year to month;
select * from sample_intervals;

   y2m   |       m2s       
---------+-----------------
 2 years | 2 days 01:01:00

delete from sample_intervals where m2s = interval '2 1:1:0' day to second;
select * from sample_intervals;

 y2m | m2s 
-----+-----

Estilos de intervalo

É possível usar o comando SET para alterar o formato de saída das strings de intervalo. As opções disponíveis são postgres, sql_standard e postgres_verbose. O padrão é postgres. Veja a seguir exemplos de cada formato de saída.

formato de saída sql_standard

Os valores do intervalo de ano para mês são formatados da maneira a seguir. Especificar negativo antes do intervalo indica que o intervalo é um valor negativo e se aplica a todo o intervalo.

'[-]yy-mm'

Os valores do intervalo de dia para segundo são formatados da maneira a seguir.

'[-]dd hh:mm:ss.ffffff'
INTERVAL '1-2' YEAR TO MONTH outputs as '1-2'
                     
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 2:03:04.5678'

formato de saída postgres

O formato de saída a seguir é para postgres. Cada valor numérico pode ser negativo.

[@] '<numeric> <unit> [, <numeric> <unit> ...]'
INTERVAL '1-2' YEAR TO MONTH outputs as '1 year 2 mons'
                  
INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '1 day 02:03:04.5678'

formato de saída postgres_verbose

A sintaxe postgres_verbose é semelhante à do postgres, mas as saídas do postgres_verbose também contêm a unidade de tempo.

'[@] <numeric> <unit> [, <numeric> <unit> ...] [direction]'
INTERVAL '1-2' YEAR TO MONTH outputs as '@ 1 year 2 mons'

INTERVAL '1 2:3:4.5678' DAY TO SECOND outputs as '@ 1 day 2 hours 3 mins 4.5678 secs'

O comando a seguir demonstra como definir o estilo de intervalo como sql_standard.

SET intervalstyle to 'sql_standard';

Limitações

Esse recurso INTERVAL de pré-lançamento tem as seguintes limitações:

  • Se você estiver usando o driver JDBC, não poderá usar os intervalos de mês e dia para segundo como parâmetros para as declarações PREPARE.

  • O driver JDBC interpreta todos os valores de intervalo como strings, então você só poderá obter os intervalos exibidos de uma consulta SELECT usando o método getString.

  • Os procedimentos armazenados não aceitam valores de intervalo como parâmetros de função.

  • Não é possível converter um valor numérico em um intervalo.
