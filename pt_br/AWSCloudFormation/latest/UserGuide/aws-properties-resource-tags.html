Recurso de tag

Use a propriedade Tags de recurso para aplicar tags a recursos, que podem ajudá-lo a identificar e categorizar esses recursos. Atribua tags apenas aos recursos do AWS CloudFormation que permitem o uso de tags. Para informações sobre quais recursos você pode atribuir uma tag com o CloudFormation, consulte os recursos individuais em Referência de tipos de propriedades e recursos da AWS.

nota

As implementação com atribuição de tags pode variar de acordo com o recurso. Por exemplo, o AWS::AutoScaling::AutoScalingGroup fornece uma propriedade PropagateAtLaunch adicional, necessária como parte do esquema de marcação.

Além de quaisquer tags que você define, o CloudFormation cria automaticamente as seguintes tags em nível de pilha com o prefixo aws::

  • aws:cloudformation:logical-id

  • aws:cloudformation:stack-id

  • aws:cloudformation:stack-name

O prefixo aws: é reservado para uso da AWS. Esse prefixo não diferencia maiúsculas de minúsculas. Se você usar esse prefixo na propriedade Value ou Key, não será possível atualizar ou excluir a tag. As tags com esse prefixo não são contadas no número de tags por recurso.

A propagação de tags no nível da pilha para recursos, incluindo tags criadas automaticamente, que pode variar de acordo com o recurso. Por exemplo, as tags não são propagadas para volumes do Amazon EBS criados a partir do mapeamento de dispositivos de blocos.

Sintaxe

JSON

{
  "Key" : String,
  "Value" : String
}

YAML

Key: String
Value: String

Propriedades

Key

O nome da chave da tag. Especifique um valor com 1 a 128 caracteres Unicode e que não tenha o prefixo aws:. É possível usar qualquer um dos caracteres a seguir: o conjunto Unicode de letras, dígitos, espaço em branco, _, ., :, /, =, +, @, - e ".

Obrigatório: Sim

Tipo: string

Value

O valor da tag. É possível especificar um valor de 1 a 256 caracteres de comprimento. Use qualquer um dos seguintes caracteres: o conjunto de letras, dígitos, espaço em branco, _, ., /, =, + e -.

Obrigatório: Sim

Tipo: string

Exemplo

Este exemplo mostra uma propriedade Tags. Você especifica essa propriedade na seção Properties de um recurso compatível. Quando o recurso é criado, ele é marcado com as tags que você declara.

JSON

"Tags" : [
   {
      "Key" : "keyname1",
      "Value" : "value1"
   },
   {
      "Key" : "keyname2",
      "Value" : "value2"
   }
]

YAML

Tags: 
  - Key: "keyname1"
    Value: "value1"
  - Key: "keyname2"
    Value: "value2"
