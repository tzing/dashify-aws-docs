Gerenciar eventos do AWS CloudFormation usando o Amazon EventBridge

O Amazon EventBridge é um serviço com tecnologia sem servidor que usa eventos para conectar os componentes da aplicação, facilitando a criação de aplicações escaláveis orientadas por eventos. A arquitetura orientada por eventos é um estilo de criação de sistemas de software com acoplamento fraco que funcionam juntos emitindo e respondendo a eventos. Os eventos representam uma mudança em um recurso ou ambiente.

Como ocorre em muitos serviços da AWS, o AWS CloudFormation gera e envia eventos para o barramento de eventos padrão do EventBridge. (O barramento de eventos padrão é provisionado automaticamente em cada conta da AWS.) Um barramento de eventos é um roteador que recebe eventos e os entrega a zero ou mais destinos, ou alvos. As regras especificadas para o barramento de eventos avaliam os eventos à medida que eles chegam. Cada regra verifica se um evento corresponde ao padrão do evento. Se o evento corresponder, o barramento de eventos enviará o evento para os destinos especificados.

Eventos do CloudFormation

O AWS CloudFormation envia eventos para o EventBridge sempre que uma operação de criação, atualização, exclusão ou detecção de desvio é executada em uma pilha. O AWS CloudFormation também envia eventos para o EventBridge para alterações de status nos conjuntos de pilhas e nas instâncias de conjuntos de pilhas. Você pode usar regras do EventBridge para rotear eventos para outros destinos. O máximo esforço será feito para que esses eventos sejam entregues e eles poderão ser entregues fora de ordem.

Como os eventos do CloudFormation representam alterações em pilhas ou conjuntos de pilhas e seus recursos, você pode usá-los para iniciar fluxos de trabalho associados aos respectivos eventos. Por exemplo:

  • Crie tags específicas de pilhas ou de conjuntos de pilhas em todos os recursos provisionados por meio do AWS CloudFormation.

  • Estabeleça uma associação entre uma pilha ou um conjunto de pilhas do CloudFormation e um Amazon WorkSpaces Application Manager (Amazon WAM).

  • Especifique uma associação com um AppRegistry para a pilha ou o conjunto de pilhas criado.

Os eventos a seguir são gerados pelo CloudFormation e enviados para o barramento de eventos padrão no EventBridge. Para ter mais informações, consulte Referência detalhada de eventos do AWS CloudFormation.

Enviar eventos do AWS CloudFormation usando regras do EventBridge

Para que o barramento de eventos padrão do EventBridge envie eventos do AWS CloudFormation para um destino, é necessário criar uma regra que contenha um padrão de eventos que corresponda aos dados nos eventos do AWS CloudFormation desejados.

A criação de uma regra consiste nas seguintes etapas gerais:

  1. Criar um padrão de evento para a regra que especifica:

    • AWS CloudFormation é a fonte dos eventos avaliados pela regra.

    • (Opcional): qualquer outro dado de evento para comparar.

    Para ter mais informações, consulte Criar e testar padrões de eventos para eventos do AWS CloudFormation.

  2. (Opcional): criar um transformador de entrada que personaliza os dados do evento antes do EventBridge passar as informações para o destino da regra.

    Para obter mais informações, consulte Transformação da entrada no Guia do usuário do EventBridge.

  3. Especificar os destinos para os quais você deseja que o EventBridge entregue eventos correspondentes ao padrão do evento.

    Os destinos podem ser outros serviços da AWS, aplicações de software como serviço (SaaS), destinos de API ou outros endpoints personalizados. Para obter mais informações, consulte Targets (Alvos) no Guia do usuário do EventBridge.

Para obter instruções abrangentes de como criar regras de barramento de eventos, consulte Criar regras que reagem a eventos no Guia do usuário do EventBridge.

Criar e testar padrões de eventos para eventos do AWS CloudFormation

Quando o AWS CloudFormation entrega um evento ao barramento de eventos padrão, o EventBridge usa o padrão de evento definido para cada regra para determinar se o evento deve ser entregue aos destinos da regra. Um padrão de evento corresponde aos dados nos eventos do AWS CloudFormation desejados. Cada padrão de evento é um objeto JSON que contém:

  • Um atributo source que identifica o serviço que envia o evento. Para eventos do AWS CloudFormation, a origem é aws.cloudformation.

  • (Opcional): um atributo detail-type que contém uma matriz dos tipos de eventos a serem correlacionados.

  • (Opcional): um atributo detail que contém quaisquer outros dados relacionados aos eventos a serem correlacionados.

    Por exemplo, o ID da pilha, os recursos envolvidos, o status de vários recursos e outros dados relevantes para um tipo específico de eventos.

Por exemplo, o seguinte padrão de evento corresponde a todos os eventos de alteração do estado do recurso:

{
  "source": ["aws.cloudformation"],
  "detail-type": ["CloudFormation Resource Status Change"]
}

O seguinte padrão de eventos usa dados detalhados dos eventos a serem correlacionados somente com os eventos de alteração de estado do recurso em que o CloudFormation cria um novo recurso AWS::S3::Bucket ou AWS::SNS::Topic:

{
  "source": ["aws.cloudformation"],
  "detail-type": ["CloudFormation Resource Status Change"],
  "detail": {
    "status-details": {
      "status": ["CREATE_COMPLETE"]
    },
    "resource-type": ["AWS::S3::Bucket", "AWS::SNS::Topic"]
  }
}

Para obter mais informações sobre como escrever padrões de eventos, consulte Padrões de eventos no Guia do usuário do EventBridge.

Testar padrões de eventos para eventos do AWS CloudFormation no EventBridge

É possível usar o Sandbox no EventBridge para definir e testar um padrão de eventos com rapidez, sem a necessidade de concluir o processo mais amplo de criação ou de edição de uma regra. Ao usar o Sandbox, é possível definir um padrão de eventos e usar um evento de exemplo para confirmar se o padrão corresponde aos eventos desejados. O EventBridge disponibiliza a opção de criar uma nova regra usando esse padrão de eventos diretamente do sandbox.

Para obter mais informações, consulte Testar um padrão de evento usando a Sandbox do EventBridge no Guia do usuário do EventBridge.

Permissões Amazon EventBridge

O AWS CloudFormation não necessita de outras permissões para entregar eventos ao Amazon EventBridge. Os eventos contêm informações que já estão disponíveis por meio de operações de API do CloudFormation.

Talvez os destinos que você especificar precisem de determinadas permissões ou configurações. Para obter mais detalhes sobre o uso de serviços específicos para destinos, consulte Destinos do Amazon EventBridge no Amazon EventBridgeGuia do usuário do .
