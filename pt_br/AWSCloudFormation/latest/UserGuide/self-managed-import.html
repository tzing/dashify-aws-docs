Importação de pilhas autogerenciadas para o AWS CloudFormation StackSets

A operação importação de pilha do AWS CloudFormation pode importar pilhas existentes em conjuntos de pilhas novos ou existentes, para que você possa migrar pilhas existentes para um conjunto de pilhas em uma só operação. O StackSets amplia a funcionalidade das pilhas, permitindo que você crie, atualize ou exclua pilhas em várias contas e regiões com uma única operação.

Por exemplo, se você tiver uma pilha que especifica um perfil do AWS Identity and Access Management(IAM) de administrador em várias contas, poderá importar essa pilha para um conjunto de pilhas. Ao utilizar a importação de pilhas, você evita tempo de inatividade e interrupções por não precisar excluir e recriar esses recursos. Depois que a pilha for importada para um conjunto de pilhas, a original se tornará uma instância de pilha do conjunto de pilhas recém-gerado.

Requisitos autogerenciados para a importação de pilhas

Além da seção Requisitos para a importação de pilhas, importações de pilhas autogerenciadas requerem o seguinte.

  • A operação de importação de pilhas é compatível com a criação de um conjunto de pilhas com permissões autogerenciadas.

  • A operação de importação de pilhas requer uma conta de administrador na qual criar um conjunto de pilhas e uma conta de destino que contenha uma pilha.

  • A conta de destino deve ter permissão para utilizar a operação GetTemplate com a especificação do ID da pilha ou do ARN. Por isso, sua conta de administrador deve receber as permissões AWSCloudFormationStackSetsAdminstration ou AWSCloudFormationStackSetsExectionRole.

Importar uma pilha em um novo conjunto de pilhas

Importar uma pilha para um novo conjunto de pilhas com o AWS Management Console

Para importar uma pilha para um conjunto de pilhas, identifique uma pilha contendo o recurso que você quer importar.

  1. Faça login no AWS Management Console e abra o console AWS CloudFormation em https://console.aws.amazon.com/cloudformation.

  2. No painel de navegação, escolha StackSets.

  3. No topo da página StackSets, escolha Create StackSet (Criar StackSet).

  4. Na página Choose a template (Escolher um modelo), escolha um modelo com base em uma das seguintes opções e selecione Next (Próximo):

    • Escolha Amazon S3 URL e especifique o URL do modelo na caixa de texto.

    • Escolha Upload a template file (Fazer upload de um arquivo de modelo) e procure o arquivo de modelo.

    • Escolha From stack ID (Do ID da pilha) e insira seu ID da pilha.

  5. Na página Specify StackSet details (Especificar detalhes do StackSet), insira o nome de um conjunto de pilhas que você quer criar e selecione Next (Próximo).

    (Opcional) Insira uma descrição do conjunto de pilhas.

  6. Na página Configure stack options (Configurar opções da pilha), reveja suas escolhas e selecione Next (Próximo).

  7. Na página Set deployment options (Definir opções de implantação), selecione Import stacks to stack set (Importar pilhas para o conjunto de pilhas).

  8. Insira o ID da pilha a ser importada no campo Stacks to import (Pilhas para importar). Por exemplo, arn:123456789101:us-east-1:StackToImport.

    (Opcional) Selecione Add another stack ID (Adicionar outro ID de pilha) e insira o ID de outra pilha que você deseja importar. É possível adicionar até 10 pilhas por operação de importação.

  9. Revise as opções de implantação e selecione Next (Próximo).

  10. Na página Revisão, revise suas escolhas e as propriedades do seu conjunto de pilhas. Quando estiver pronto para importar sua pilha para o conjunto de pilhas, selecione Submit (Enviar).

Resultados: a pilha importada é agora uma instância de pilha do conjunto de pilhas especificado. Para saber mais sobre o status de importação de pilhas, consulte Códigos de status de conjuntos de pilhas e instâncias de pilha.

Importar uma pilha autogerenciada para um conjunto existente de pilhas

Importar uma pilha para um conjunto existente de pilhas com o AWS Management Console

Para importar uma pilha para um conjunto existente de pilhas, identifique uma pilha contendo o recurso que você quer importar.

  1. Faça login no AWS Management Console e abra o console AWS CloudFormation em https://console.aws.amazon.com/cloudformation.

  2. No painel de navegação, escolha StackSets.

  3. Na página StackSets, selecione o conjunto de pilhas para o qual importar uma pilha.

  4. Com o conjunto de pilhas selecionado, escolha Add stacks to StackSet (Adicionar pilhas ao StackSet) no menu Actions (Ações).

  5. Na página Set deployment options (Definir opções de implantação), selecione Import stacks to stack set (Importar pilhas para o conjunto de pilhas) e insira o ID da pilha a ser importada no campo Stacks to import (Pilhas para importar). Por exemplo, arn:123456789101:us-east-1:StackToImport.

    (Opcional) Selecione Add another stack ID (Adicionar outro ID de pilha) e insira o ID de outra pilha que você deseja importar. É possível adicionar até 10 pilhas por operação de importação.

  6. Escolha Next (Próximo).

  7. Na página Specify overrides (Especificar substituições), verifique suas opções e selecione Next (Próximo).

  8. Na página Revisão, revise suas escolhas e as propriedades do seu conjunto de pilhas. Quando estiver pronto para criar seu conjunto de pilhas, escolha Submit (Enviar).

Resultados: a pilha importada é agora uma instância de pilha do conjunto de pilhas especificado. Para saber mais sobre o status de importação de pilhas, consulte Códigos de status de conjuntos de pilhas e instâncias de pilha.

Importando uma pilha para um conjunto de pilhas (AWS CLI)

Importar uma pilha em um conjunto de pilhas

Para importar uma pilha para um conjunto existente de pilhas, identifique uma pilha contendo os recursos que você quer importar. Neste exemplo, a pilha para importar é arn:123456789101:us-east-1:StackToImport.

  1. Crie um conjunto de pilhas com base em um ID de pilha, especificando o ARN completo da pilha do CloudFormation a ser importada.

    aws cloudformation create-stack-set \
  --stack-id "arn:aws:cloudformation:us-east-1:123456789012:stack/StackToImport/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set-name "SingleStackSetName" \
  --permission-model "SELF_MANAGED" \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  2. Importe uma pilha especificada para o seu conjunto de pilhas.

    aws cloudformation import-stacks-to-stack-set \
  --stack-ids "arn:aws:cloudformation:us-east-1:123456789012:stack/StackToImport/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set "SingleStackSetName" \
  --permission-model SELF_MANAGED \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  3. Clone a pilha importada em outras regiões e contas.

    aws cloudformation create-stack-instances \
  --stack-set-name "StackSetToWhichStackimported" \
  --accounts "123556789101" \
  --regions "us-east-1"

Importar pilhas em um conjunto de pilhas

Para importar pilhas existentes para um conjunto de pilhas, identifique as pilhas contendo os recursos que você quer importar. Neste exemplo, as pilhas para importar são arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport1/f449b250-b969-11e0-a185-5081d0136786 e arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport2/f449b250-b969-11e0-a185-5081d0136786.

  1. Crie um conjunto de pilhas com base em um ID de pilha.

    aws cloudformation create-stack-set \
  --stack-id "arn:123456789101:us-east-1:StackToImport" \
  --stack-set-name "StackSetName" \
  --permission-model "SELF_MANAGED" \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  2. Importe pilhas especificadas para o seu conjunto de pilhas.

    aws cloudformation import-stacks-to-stackset 
  --stack-ids "arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport1/f449b250-b969-11e0-a185-5081d0136786, arn:aws:cloudformation:123456789101:us-east-1:stack/StackToImport2/f449b250-b969-11e0-a185-5081d0136786" \
  --stack-set "StackSetName" 
  --permission-model SELF_MANAGED \
  --administration-role-arn "arn:aws:iam::123456789012:role/AWSCloudFormationStackSetAdministrationRole" \
  --execution-role-name "AWSCloudFormationStackSetExecutionRole"

  3. Clone a pilha importada em outras regiões e contas.

    aws cloudformation create-stack-instances \
  --stack-set-name "StackSetName" \
  --accounts "123456789012" \
  --regions "us-east-1"
