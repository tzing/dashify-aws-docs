Atualizações de pilhas do AWS CloudFormation

Quando precisar alterar as configurações de uma pilha ou seus recursos, atualize essa pilha em vez de excluí-la e criar uma nova. Por exemplo, caso tenha uma pilha com uma instância EC2, você pode atualizar a pilha para alterar o ID de AMI da instância.

Ao atualizar uma pilha, você envia alterações, como novos valores de parâmetro de entrada ou um modelo atualizado. O AWS CloudFormation compara as alterações enviadas por você com o estado atual da pilha e atualiza apenas os recursos alterados. Para obter um resumo do fluxo de trabalho da atualização, consulte Como o AWS CloudFormation funciona?.

nota

Ao atualizar uma pilha, o AWS CloudFormation pode interromper os recursos ou substituir os recursos atualizados, dependendo de quais propriedades você atualiza. Para obter mais informações sobre os comportamentos de atualização de recursos, consulte Atualizar comportamentos de recursos da pilha.

Métodos de atualização

O AWS CloudFormation fornece dois métodos para atualizar pilhas: atualização direta ou criação e execução de conjuntos de alterações. Ao atualizar diretamente uma pilha, você envia alterações e o AWS CloudFormation as implanta imediatamente. Use atualizações diretas quando você quiser implantar rapidamente as atualizações.

Com conjuntos de alterações, você pode visualizar as alterações que o AWS CloudFormation fará na pilha e decidir se aplica ou não essas alterações. Os conjuntos de alterações são documentos formatados por JSON que resumem as alterações que o AWS CloudFormation fará em uma pilha. Use conjuntos de alterações para garantir que o AWS CloudFormation não faça alterações não intencionais ou quando você quiser considerar várias opções. Por exemplo, é possível usar um conjunto de alterações para verificar se o AWS CloudFormation não substituirá as instâncias de banco de dados da pilha durante uma atualização.
