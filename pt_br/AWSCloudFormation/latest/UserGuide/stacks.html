Como trabalhar com pilhas

Uma pilha é um conjunto de recursos da AWS que pode gerenciar como uma unidade. Em outras palavras, você pode criar, atualizar ou excluir um conjunto de recursos criando, atualizando ou excluindo pilhas. Todos os recursos em uma pilha são definidos pelo modelo AWS CloudFormation dela. Uma pilha, por exemplo, pode incluir todos os recursos necessários para executar um aplicativo web, como um servidor web, um banco de dados e as regras de rede. Se não precisar mais do aplicativo web, você pode simplesmente excluir a pilha e todos os recursos relacionados serão excluídos.

O AWS CloudFormation garante que todos os recursos da pilha sejam criados ou excluídos, conforme apropriado. Como o AWS CloudFormation trata os recursos da pilha como uma única unidade, eles devem ser criados ou excluídos com êxito para que a pilha seja criada ou excluída. Se um recurso não pode ser criado, o AWS CloudFormation reverte a pilha e exclui automaticamente todos os recursos que foram criados. Se um recurso não pode ser excluído, todos os recursos restantes são mantidos até que a pilha possa ser excluída com êxito.

É possível gerenciar pilhas utilizando o console do AWS CloudFormation, a API, a AWS CLI ou o Systems Manager Application Manager.

nota

Você será cobrado pelos recursos de pilha durante o tempo em que eles estiverem operando (mesmo que você tenha excluído a pilha imediatamente).
