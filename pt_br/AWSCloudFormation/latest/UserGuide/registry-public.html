Utilizar extensões públicas no CloudFormation

Extensões públicas são extensões do CloudFormation, como tipos de recursos ou módulos, que são publicadas publicamente no registro para uso por todos os usuários do CloudFormation. Isso inclui todas as extensões publicadas pela AWS. Terceiros também podem publicar extensões públicas.

Existem dois tipos de extensões públicas:

  • Extensões públicas da AWS: extensões publicadas pela AWS são sempre públicas e estão ativadas por padrão. Dessa forma, você não precisa tomar nenhuma medida antes de utilizá-los em sua conta. Além disso, a AWS controla o versionamento da extensão, para que você esteja sempre utilizando a versão mais recente disponível.

  • Extensões públicas de terceiros: são extensões disponibilizadas para uso geral por editores diferentes da AWS. Para utilizar uma extensão pública, você deve primeiro ativá-la na sua conta e região.

    Você pode publicar suas próprias extensões de terceiros para disponibilizá-las para usuários gerais do CloudFormation. Para mais informações, consulte Publicar extensões, no Guia do usuário da interface de linha de comando do CloudFormation.

nota

Como no caso de extensões privadas, as extensões públicas de editores terceiros podem implementar manipuladores de eventos que são executados durante operações de criação de recursos, leitura, atualização, listagem e exclusão de pilhas. Por causa disso, o uso dessas extensões nas suas pilhas do CloudFormation pode resultar em cobranças na sua conta. Esses encargos serão acrescentados além de todas as cobranças decorrentes dos recursos criados. Para obter mais informações, consulte Definição de preços do AWS CloudFormation.

Ativar extensões públicas para uso na sua Conta da AWS

Para utilizar uma extensão pública de terceiros no seu modelo, primeiro você deve ativar essa extensão para a conta e a região nas quais você deseja utilizá-lo. A ativação de uma extensão a torna utilizável em operações de pilha na conta e região em que ela está ativada. Quando você ativa uma extensão pública de terceiros, o CloudFormation cria uma entrada no registro de extensões da sua conta para a extensão ativada como uma extensão privada. Isso permite personalizar a extensão à medida que é ela ativada na sua conta, das seguintes maneiras:

  • Especifique um alias a ser utilizado em vez do nome da extensão pública de terceiros. Isso pode ajudar a evitar colisões de nomes entre extensões de terceiros.

  • Especifique se a extensão será atualizada automaticamente quando uma nova versão secundária ou uma versão de patch estiver disponível.

  • Especifique a função de execução utilizada pelo CloudFormation para ativar a extensão, além de configurar o registro em log para essa extensão.

Quando você ativa uma extensão pública de terceiros, também pode definir as propriedades de configuração incluídas nela. As propriedades de configuração definem como essa extensão é configurada para uma determinada conta e região. Consulte o seguinte para obter mais informações sobre configuração:

  • Definir a configuração no nível da conta de uma extensão

  • Esquema de configuração de hooks

Configurar o CloudFormation para utilizar novas versões de extensões automaticamente

Ao ativar uma extensão, você também pode especificar o tipo de extensão para utilizar a versão secundária mais recente. Seu tipo de extensão atualizará a versão secundária sempre que o editor lançar uma nova versão na sua extensão ativada.

Por exemplo, da próxima vez em que você executar uma operação de pilha, como criar ou atualizar uma pilha, utilizando um modelo que inclui essa extensão, o CloudFormation usará a nova versão secundária.

Atualizar para uma nova versão de extensão, automática ou manualmente, não afeta nenhuma instância de extensão já provisionada em pilhas.

O CloudFormation trata as atualizações de versões principais de extensões como contendo potencialmente alterações significativas e, portanto, exige que você atualize manualmente para uma nova versão principal de uma extensão.

Extensões publicadas pela AWS são ativadas por padrão para todas as contas e regiões em que estão disponíveis e sempre utilizam a versão mais recente disponível em cada Região da AWS.

Importante

Como você controla se e quando as extensões são atualizadas para a versão mais recente da sua conta, pode acabar com versões diferentes da mesma extensão implantadas em contas e regiões diferentes.

Isso pode gerar resultados inesperados ao utilizar o mesmo modelo, contendo essa extensão, entre essas contas e regiões.

Especificar aliases para fazer referência a extensões

Não é possível ativar mais de uma extensão com um determinado nome em uma determinada conta e região. Como diferentes editores podem oferecer extensões públicas com o mesmo nome, o CloudFormation permite que você especifique um alias para qualquer extensão pública de terceiros que você ativar.

Se você especificar um alias para a extensão, o CloudFormation tratará esse alias como o nome do tipo de extensão dentro da conta e da região. Você deve utilizar o alias para fazer referência à extensão nos seus modelos, chamadas de API e no console do CloudFormation.

Aliases de extensão devem ser exclusivos em uma conta e região especificadas. É possível ativar o mesmo recurso público várias vezes na mesma conta e região utilizando apelidos de nomes de tipos diferentes.

Importante

Embora aliases de extensões só precisem ser exclusivos em uma determinada conta e região, sugerimos que os usuários não atribuam o mesmo alias a diferentes extensões públicas de terceiros entre contas e regiões. Isso pode gerar resultados inesperados ao utilizar um modelo que contém o alias de extensão em várias contas ou regiões.

Especificar uma função de execução ao ativar extensões

Ao ativar um recurso, é possível especificar o perfil de execução do IAM para o CloudFormation assumir ao invocar essa extensão na sua conta e região.

Para que o CloudFormation assuma o perfil de execução, este deve ter uma política de confiança definida com o CloudFormation. Além disso, a permissão para executar uma operação é concedida por meio da criação de uma política do IAM e pela anexação ao perfil de execução. As permissões necessárias são definidas na seção de manipulador do esquema da extensão.

Veja a seguir um exemplo de política de confiança de perfil do IAM para a extensão de tipo de recurso:

{
    "Version": "2012-10-17",
    "Statement":[
        {
            "Effect": "Allow",
            "Principal":{
                "Service": "resources.cloudformation.amazonaws.com"
            },
            "Action":"sts:AssumeRole",
            "Condition":{
                "StringEquals":{
                    "aws:SourceAccount":"123456789012"
                },
                "StringLike":{
                    "aws:SourceArn":"arn:aws:cloudformation:us-east-1:123456789012:type/resource/Organization-Service-Resource/*"
                }
            }
        }
    ]
}

Veja a seguir um exemplo de política de confiança de perfil do IAM para a extensão de hook:

{
    "Version":"2012-10-17",
    "Statement":[
        {
            "Effect":"Allow",
            "Principal": {
                "Service": [
                    "resources.cloudformation.amazonaws.com",
                    "hooks.cloudformation.amazonaws.com"
                ]
            },
            "Action":"sts:AssumeRole",
            "Condition":{
                "StringEquals":{
                    "aws:SourceAccount":"123456789012"
                },
                "StringLike":{
                     "aws:SourceArn":"arn:aws:cloudformation:us-east-1:123456789012:type/hook/Organization-Service-Hook/*"
                }
            }
        }
    ]
}

Para obter mais informações, consulte Modificar uma política de confiança de função, no Guia do usuário do AWS Identity and Access Management.

Para ativar uma extensão pública no registro

Ative sua extensão no registro do AWS CloudFormation para que ele esteja disponível para uso.

Para ativar uma extensão pública para uso na sua conta utilizando o console do CloudFormation

  1. Faça login no AWS Management Console e abra o console AWS CloudFormation em https://console.aws.amazon.com/cloudformation.

  2. No painel de navegação do CloudFormation, em CloudFormation registry (Registro do CloudFormation), selecione Public extensions (Extensões públicas).

  3. Use Filter (Filtro) para selecionar o tipo de extensão e selecione Third party (Terceiros). (Extensões publicadas pela AWS são ativadas por padrão.)

  4. Selecione a extensão e depois Activate (Ativar).

    nota

    Se várias versões de uma extensão estiverem disponíveis, será possível utilizar o menu Version (Versão) para selecionar a versão da extensão que você deseja ativar. A versão padrão é a mais atual.

  5. Use as opções na página Activate extension (Ativar extensão) para personalizar como a extensão é ativada nessa conta.

    • Para ativar a extensão utilizando um alias de tipo de extensão:

      1. Selecione Override default (Substituir padrão)

      2. Insira o alias do tipo de extensão que você deseja utilizar com essa extensão. O alias deve seguir o formato recomendado para o tipo de extensão.

      Para saber mais, consulte Especificar aliases para fazer referência a extensões.

    • Se você estiver habilitando um recurso privado, poderá especificar o perfil do IAM para o CloudFormation assumir ao chamar a extensão. Para saber mais, consulte Especificar uma função de execução ao ativar extensões.

    • Em Logging config (Configuração do registro em log), especifique as informações de configuração de registro em log para uma extensão, se desejado. Por exemplo:

      {
    "logRoleArn": "arn:aws:iam::account:role/rolename",
    "logGroupName": "log-group-name"
}

      As informações de configuração de registro em log não são necessárias, mas são recomendadas para fins de depuração. Para utilizar a configuração de registro em log com hooks, adicione a mesma política de confiança que a função de execução especificada, para que a função de registro em log possa gravar logs no seu grupo de logs.

      Os nomes das chaves logRoleArn e logGroupName diferenciam maiúsculas de minúsculas.

    • Em Versioning (Versionamento), especifique como você recebe atualizações.

      • On (Ativado): atualiza automaticamente para a versão secundária mais recente. As versões principais são atualizadas manualmente.

      • Off (Desativado): nunca atualiza automaticamente para a versão mais recente. Todas as versões são atualizadas manualmente.

      Para saber mais, consulte Configurar o CloudFormation para utilizar automaticamente novas versões de extensões.

  6. Especifique os dados de configuração.

    nota

    Essa etapa é necessária para hooks. Você deve especificar ALL para a propriedade TargetStack. Essa operação ativa as propriedades do hook que são definidas na seção de propriedades do esquema de hooks.

    Se a extensão exigir configuração adicional em nível de conta e região, o CloudFormation incluirá a seção Configuration (Configuração) na página Activate extension (Ativar extensão). Você tem a opção de especificar os dados de configuração agora ou depois que a extensão tiver sido ativada. Se a extensão não exigir configuração, você não verá a seção Configuration (Configuração).

    • Para especificar os dados de configuração ao habilitar a extensão:

      1. Selecione Configure now (Configurar agora) e depois selecione Activate extension (Ativar extensão).

        O CloudFormation exibe a página Configure extension (Configurar extensão). Para visualizar o esquema de configuração atual da extensão, certifique-se de a opção View configuration schema (Visualizar esquema de configuração) esteja ativada.

      2. Na caixa de texto Configuration JSON (Configuração JSON), insira uma string JSON que represente o esquema de configuração que você deseja especificar para essa extensão.

      3. Selecione Configure extension (Configurar extensão).

    • Para especificar os dados de configuração mais tarde:

      • Selecione Configure later (Configurar mais tarde) e depois selecione Activate extension (Ativar extensão).

      Para especificar os dados de configuração de uma extensão que já esteja ativada, acesse a página de detalhes dessa extensão ativada.

    Depois que a extensão tiver sido ativada com êxito, o CloudFormation exibirá a página de detalhes dessa extensão.

Para atualizar uma extensão pública ativada na sua conta utilizando o console do CloudFormation

Atualize sua extensão no registro do AWS CloudFormation para que ele esteja disponível para uso.

Depois de ativar uma extensão na sua conta e região, será possível atualizar as seguintes propriedades dessa extensão à medida que ela é ativada, incluindo:

  • O esquema de configuração.

  • Se você deseja ativar as atualizações automáticas de versão.

  • Para extensões registradas privadas, a função da execução a ser usada.

  • As informações de configuração de registro em log.

  1. Faça login no AWS Management Console e abra o console AWS CloudFormation em https://console.aws.amazon.com/cloudformation.

  2. No painel de navegação do CloudFormation, em CloudFormation registry (Registro do CloudFormation), selecione Public extensions (Extensões públicas).

  3. No menu Actions (Ações), selecione Edit (Editar) e depois selecione a opção de edição apropriada:

    • Para atualizar o esquema de configuração, consulte Para especificar propriedades de configuração para uma extensão no console do CloudFormation

    • Para ativar ou desativar atualizações automáticas:

      1. Selecione Edit automatic updates (Editar atualizações automáticas).

      2. Selecione On (Ativado) ou Off (Desativado) e depois selecione Save (Salvar).

    • Para atualizar a função de execução:

      1. Selecione Edit execution role (Editar função de execução).

      2. Especifique o ARN do perfil do IAM que você deseja que o CloudFormation utilize ao invocar essa extensão e, em seguida, selecione Save (Salvar).

    • Para atualizar a configuração de registro em log:

      1. Selecione Edit logging config (Editar configuração de registro em log).

      2. Edite o JSON de configuração de registro em log e selecione Save (Salvar).

Para ativar uma extensão pública para uso na sua conta utilizando a AWS CLI

Ative sua extensão no registro do AWS CloudFormation por meio da AWS CLI.

Para ativar hooks

Depois de desenvolver e registrar seu hook, você pode ativá-lo na sua Conta da AWS publicando–o no registro.

  • Para ativar um hook na sua conta, use a operação SetTypeConfiguration. Essa operação ativa as propriedades do hook que são definidas na seção properties do esquema de hooks.

    nota

    Ao habilitar hooks na sua conta, você está autorizando um hook a utilizar permissões definidas da sua Conta da AWS. O CloudFormation remove permissões não obrigatórias antes de transmitir suas permissões ao hook. O CloudFormation recomenda que os clientes ou usuários do hook revisem as permissões desse hook e estejam cientes das permissões para as quais hooks têm permissão antes de os ativar na sua conta.

    Especifique os dados de configuração para sua extensão de hook registrada na mesma conta e Região da AWS.

    aws cloudformation --region us-west-2 set-type-configuration \
  --configuration "{"CloudFormationConfiguration":{"HookConfiguration":{"TargetStacks": "ALL", "FailureMode": "FAIL", "Properties":{}}}}" \
  --type-arn $HOOK_TYPE_ARN
    Importante

    Para permitir que seu hook inspecione proativamente a configuração da sua pilha, você deve definir TargetStacks como ALL na seção HookConfiguration.

Para ativar os tipos de recursos e módulos
Para ativar uma extensão pública para uso na sua conta utilizando a AWS CLI

  • Use activate-type para ativar a extensão e especificar se ela deve ser atualizada automaticamente sempre que uma nova versão secundária da extensão for publicada.

    O exemplo abaixo especifica o nome do recurso da Amazon (ARN) público de uma extensão pública a ser ativada para essa conta. Além disso, ele especifica que o CloudFormation deve atualizar a extensão sempre que uma nova versão secundária é publicada.

    aws cloudformation activate-type \
  --public-type-arn public_extension_ARN \
  --auto-update true

    Esse comando retorna um ARN da extensão ativada, específico dessa conta e região.

    {
    "Arn": "624af370-311a-11e8-b6b7-500cexample"
}
Para atualizar a versão de uma extensão pública utilizando a AWS CLI

Use activate-type para ativar a extensão novamente.

Use o parâmetro --version-bump para especificar se a extensão deve ser atualizada para a mais recente versão MAJOR, mais recente versão MINOR.

aws cloudformation activate-type \
    --region us-west-2 \
    --type RESOURCE \
    --type-name Example::Test::1234567890abcdef0 \
    --type-name-alias Example::Test::Alias \
    --version-bump MAJOR

Para desativar extensões públicas na sua conta

As seções a seguir mostram como desativar extensões públicas na sua conta.

Para desativar uma extensão pública na sua conta utilizando o console do CloudFormation

  1. Use o registro do CloudFormation para localizar a extensão.

  2. No menu Actions (Ações), selecione Deactivate (Desativar).

  3. Selecione Deactivate (Desativar).

Para desativar uma extensão pública na sua conta utilizando a AWS CLI

Para desativar uma extensão pública na sua conta, especifique deactivate-type. Veja a seguir um exemplo de um comando de desativação da AWS CLI.

aws cloudformation deactivate-type \
    --region us-west-2 \
    --type MODULE \
    --type-name Example::Test::Type::MODULE

Para desativar um hook na sua conta utilizando a AWS CLI

Desativar um hook impede que ele execute sua Conta da AWS.

TargetStacks definido como NONE desliga o hook na sua conta e, portanto, não se aplica a operações de pilha.

Use a operação set-type-configuration e especifique TargetStacks como NONE para desativar um hook.

O exemplo a seguir especifica o Região da AWS e o nome do recurso da Amazon (ARN) do hook que está sendo desativado.

aws cloudformation set-type-configuration \
  --region us-west-2 \
  --configuration "{"CloudFormationConfiguration":{"HookConfiguration":{"TargetStacks": "NONE", "FailureMode": "FAIL", "Properties":{}}}}" \
  --type-arn HOOK_TYPE_ARN
