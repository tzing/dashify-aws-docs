Especificar o nome e os parâmetros da pilha

Após selecionar um modelo de pilha, especifique o nome da pilha e os valores dos parâmetros que foram definidos no modelo.

Com parâmetros, você pode personalizar a pilha no momento da criação. Os valores dos parâmetros podem ser usados no modelo da pilha para modificar como os recursos são configurados. Dessa forma, você não precisa codificar valores em vários modelos para especificar diferentes configurações. Para obter mais informações sobre parâmetros em um modelo do AWS CloudFormation, consulte Parâmetros.

Como especificar valores de parâmetros e o nome da pilha

  1. Na página Specify stack details (Especificar detalhes da pilha), digite o nome de uma pilha na caixa Stack name (Nome da pilha).

    O nome da pilha é um identificador que ajuda a encontrar uma determinada pilha em uma lista de pilhas. Um nome de pilha pode conter apenas caracteres alfanuméricos (sensíveis a maiúsculas e minúsculas) e hifens. Ele deve começar com um caractere alfabético e não pode ter mais de 128 caracteres.

  2. Na seção Parâmetros, especifique os parâmetros que são definidos no modelo da pilha.

    Você pode usar ou alterar todos os parâmetros com valores padrão.

  3. Quando estiver satisfeito com os valores dos parâmetros, selecione Próximo para prosseguir com as opções de configuração da sua pilha.

Tipos de parâmetros específicos da AWS

Quando você cria pilhas que contêm tipos de parâmetros específicos da AWS, o console do CloudFormation fornece listas suspensas de valores válidos para esses parâmetros. Dependendo do tipo de parâmetro, você pode pesquisar valores por ID, nome ou valor da tag Name. Por exemplo, com o tipo de parâmetro AWS::EC2::VPC::Id, você pode pesquisar um ID de VPC específico, como vpc-b47658d1. Se a VPC estiver marcada com um nome, como Name:TestVPC, você também poderá pesquisar TestVPC. Atualmente, você pode pesquisar apenas valores de tag com a chave Name.

nota

O console não fornece uma lista suspensa nem permite que você procure valores com o tipo de parâmetro AWS::EC2::Image::Id. O CloudFormation somente verifica se os valores de entrada são IDs de imagem válidos do Amazon Elastic Compute Cloud.

Parâmetros de grupo e classificação

O console lista os parâmetros de entrada em ordem alfabética por seu ID lógico. Ao criar um modelo, você pode usar a chave de metadados AWS::CloudFormation::Interface para substituir a classificação padrão. Para obter mais informações e um exemplo da chave de metadados AWS::CloudFormation::Interface, consulte AWS::CloudFormation::Interface.
