Referência detalhada de eventos do AWS CloudFormation

Todos os eventos dos serviços da AWS têm um conjunto comum de campos contendo metadados sobre o evento, como o serviço da AWS que é a origem do evento, a hora em que o evento foi gerado, a conta e a região em que o evento ocorreu, e outros. Para obter as definições desses campos gerais, consulte Referência da estrutura de eventos no Guia do usuário do Amazon EventBridge.

Além disso, cada evento tem um campo de detail que contém dados específicos desse determinado evento. A referência abaixo define os campos de detalhes dos vários eventos do AWS CloudFormation.

Ao usar o EventBridge para selecionar e gerenciar eventos do AWS CloudFormation, é importante ter em mente o seguinte:

  • O campo de source para todos os eventos do AWS CloudFormation está definido como aws.cloudformation.

  • O campo do detail-type especifica o tipo de evento.

    Por exemplo, o CloudFormation Resource Status Change ou o CloudFormation Drift Detection Status Change.

  • O campo de detail contém os dados específicos desse determinado evento.

    Por exemplo, o ID da pilha, os recursos envolvidos, o status de vários recursos e outros dados relevantes para um tipo específico de eventos.

Para obter mais informações sobre a criação de padrões de eventos que habilitam regras para corresponder a eventos do AWS CloudFormation, consulte Padrões de eventos no Guia do usuário do Amazon EventBridge.

Para obter mais informações sobre eventos e como o EventBridge os processa, consulte Eventos do EventBridgeAmazon EventBridge no Guia do usuário do Amazon EventBridge.
