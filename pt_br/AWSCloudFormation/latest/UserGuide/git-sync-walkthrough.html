Demonstração do Git Sync do AWS CloudFormation

Aprenda a usar o Git Sync com a demonstração apresentada a seguir. Antes de começar, certifique-se de concluir os pré-requisitos do Git Sync.

Etapa 1: criar a pilha

  1. Acesse o console do CloudFormation.

  2. Selecione Criar pilha.

  3. No painel Criar pilha, na tela Pré-requisito: preparar modelo, confirme se a opção O modelo está pronto está selecionada.

  4. No painel Especificar modelo, escolha Sincronizar pelo Git.

  5. Escolha Próximo para especificar os detalhes da pilha.

Etapa 2: especificar os detalhes da pilha

  1. No painel Fornecer um nome para a pilha, insira um nome para a pilha. Os nomes para as pilhas podem incluir letras (A-Z e a-z), números (0 a 9) e travessões (-).

  2. O Arquivo de implantação da pilha tem duas opções:

    • Criar o arquivo usando os parâmetros apresentados a seguir e colocá-lo em meu repositório.

      Escolha esta opção se quiser que o Git Sync crie o arquivo de implantação da pilha e o coloque em seu repositório.

    • Estou fornecendo meu próprio arquivo em meu repositório.

      Escolha esta opção se você criou um arquivo de implantação da pilha e o adicionou ao seu repositório.

    Para obter mais informações, consulte Arquivo de implantação da pilha.

  3. O painel Repositório de definição de modelo tem duas opções:

    • Escolher um repositório Git vinculado

    • Vincular um repositório Git

    Se você já vinculou um repositório Git ao CloudFormation, selecione Escolher um repositório Git vinculado e prossiga para a próxima etapa. Caso contrário, execute as seguintes etapas intermediárias para conectar um repositório Git:

    1. Escolha Vincular um repositório Git.

    2. Escolha seu provedor de repositório Git dentre as seguintes opções:

      • GitHub

      • GitHub Enterprise

      • GitLab

      • Bitbucket

    3. Escolha uma Conexão na lista. Se nenhuma opção aparecer na lista Conexão, acesse o console do Connections para criar uma conexão para o seu repositório.

  4. Na lista Repositório, selecione o repositório Git que contém o arquivo de modelo da pilha.

  5. Na lista Ramificação, selecione a ramificação que você deseja que o Git Sync monitore.

    nota

    O Git Sync monitora somente a ramificação selecionada em busca de alterações no modelo do CloudFormation e nos arquivos de implantação da pilha. Quaisquer alterações que você gostaria de aplicar à sua pilha devem ser confirmadas para esta ramificação.

  6. Para o Caminho do arquivo de implantação, especifique o caminho completo, incluindo o nome do arquivo de implantação da pilha na raiz da ramificação do repositório.

    Se o CloudFormation estiver gerando o arquivo para você, é neste local que o arquivo será confirmado em seu repositório. Se você estiver fornecendo o arquivo, este será o local do arquivo em seu repositório.

  7. Adicione um Perfil do IAM. O perfil do IAM inclui as permissões necessárias para que o CloudFormation sincronize a pilha pelo seu repositório Git. É possível escolher Novo perfil do IAM para gerar um novo perfil ou escolher Perfil do IAM existente para selecionar um perfil existente em sua conta do CloudFormation. Se você optar por gerar um novo perfil, as permissões necessárias serão incluídas no perfil.

    nota

    Para obter informações importantes sobre as permissões necessárias e a política de confiança para o Git Sync, consulte Permissões obrigatórias .

  8. Para o Caminho do arquivo de modelo, especifique o caminho completo da raiz do seu repositório para o arquivo de modelo de pilha.

  9. Adicione parâmetros à pilha que você gostaria de definir usando o arquivo de implantação da pilha. Os parâmetros correspondem a pares chave/valor. Por exemplo, para adicionar o parâmetro Port ao arquivo de implantação da pilha:

    1. Selecione Add parameter (Adicionar parâmetro).

    2. Em Chave, digite port.

    3. Em Valor, insira 8080.

    4. Repita as etapas anteriores para todos os parâmetros da pilha que você deseja disponibilizar no arquivo de implantação da pilha.

  10. (Opcional) Adicione tags. As etiquetas correspondem a pares chave/valor que você pode usar para identificar e categorizar os recursos na pilha. Para adicionar as etiquetas à implantação da pilha, siga o mesmo processo usado para adicionar os parâmetros.

  11. Escolha Próximo a fim de prosseguir para Configurar opções da pilha.

Etapa 3: configurar opções da pilha

Para obter informações sobre como configurar as opções da pilha, consulte Configurar opções de pilha do AWS CloudFormation.

Ao concluir a configuração, escolha Próximo a fim de prosseguir para a etapa de análise e de implantação.

Etapa 4: analisar e implantar

  1. Analise as configurações da pilha e confirme o seguinte:

    • O modelo da pilha está configurado corretamente e definido como Sincronizar pelo Git.

    • O arquivo de implantação está configurado corretamente.

    • O repositório de definição de modelo está configurado corretamente e, em particular, o Repositório e o Nome da ramificação estão selecionados adequadamente.

    • A pré-visualização do arquivo de implantação está correta e contém os parâmetros e os valores esperados.

  2. Escolha Enviar para criar a pilha.

    Importante

    Uma solicitação pull é criada automaticamente em seu repositório Git quando você escolhe Enviar. Você deve mesclar essa solicitação pull em seu repositório Git para concluir o processo. Ao mesclar a solicitação pull, a pilha é criada e o Git Sync começa a monitorar o modelo do CloudFormation e o arquivo de implantação da pilha em busca de alterações para atualizar a pilha.

Modificações na pilha

Após concluir as etapas anteriores, uma nova pilha é criada e o CloudFormation começa a monitorar as alterações em seu repositório Git.

Para modificar a pilha, realize alterações diretamente no arquivo de implantação da pilha. Em seguida, confirme as alterações para o repositório e para a ramificação que você configurou para a pilha no Git Sync.

No painel do CloudFormation, é possível selecionar a pilha e escolher a guia Git Sync para visualizar informações sobre o status da pilha e sobre os eventos de sincronização. Para obter mais informações, consulte Painel de status do Git Sync do AWS CloudFormation.
